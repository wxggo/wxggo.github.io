Leetcode-226:翻转二叉树-递归后序遍历法（经典二叉树操作）

Created2025-11-15|Updated2025-11-15|Java学习笔记Leetcode Hot100
|Word Count:3.3k|Reading Time:11mins|Post Views:

1、题目

给你一棵二叉树的根节点 root，翻转这棵二叉树，并返回其根节点。

翻转二叉树的定义：将二叉树中每个节点的左子树和右子树互相交换，最终得到原二叉树的镜像结构。

示例

示例 1（经典满二叉树翻转）：

输入：root = [4,2,7,1,3,6,9]

输出：[4,7,2,9,6,3,1]

示例 2（普通二叉树翻转）：

输入：root = [2,1,3]
输出：[2,3,1]

示例 3（空树/单节点树）：

输入：root = [] / root = [1]
输出：[] / [1]

提示

  • 树中节点数目范围在 [0, 100] 内
  • -100 <= Node.val <= 100

问题核心

翻转二叉树的本质是对每个节点执行“左右子树交换”操作，由于需要先翻转当前节点的左右子树，再交换当前节点的左右孩子，符合后序遍历（左→右→根） “先处理子节点，再处理父节点”的特性，递归后序遍历是最简洁、最易理解的解法。

2、分析

2.1 翻转的核心逻辑

翻转整棵二叉树 ≠ 仅交换根节点的左右子树，而是从叶子节点到根节点，自底向上对每一个节点执行左右子树交换。如果仅交换根节点的左右子树，其子孙节点的结构并未翻转，最终无法得到镜像树。

举个反例：若仅交换示例1中根节点4的左右子树（2和7），但不翻转2、7的子树，结果会是：

这并非真正的镜像结构，核心问题就是未对每个节点执行翻转操作。

2.2 递归解法的核心思路（后序遍历）

递归解法的核心是“递推遍历+归并交换”，将翻转整棵树的大问题，拆解为“翻转左子树→翻转右子树→交换当前节点左右孩子”的小问题，符合分治思想，具体步骤为：

  1. 递归终止条件：遍历到空节点时，直接返回空节点（空树无需翻转，是递归的边界）；
  2. 递推遍历：递归翻转当前节点的左子树，再递归翻转当前节点的右子树，确保左右子树均已成为镜像结构；
  3. 归并交换：左右子树翻转完成后，交换当前节点的左、右孩子节点，完成当前节点的翻转；
  4. 返回节点：返回当前已完成翻转的节点，作为上层节点的子节点，完成结果回溯。

2.3 原代码细节说明

原代码采用标准的递归后序遍历逻辑，思路完全正确，仅存在一个**无意义的临时变量temp**（TreeNode temp = root;），该变量仅保存根节点引用，未参与任何逻辑操作，最终直接返回root即可，不影响算法正确性，仅为代码小瑕疵。

2.4 算法特性

  • 时间复杂度：O(n)，每个节点仅被访问一次，交换左右孩子的操作是O(1)，总操作数与节点数量成正比；
  • 空间复杂度：O(n)，空间开销来自递归调用栈：
    • 最优情况（平衡二叉树）：递归栈深度为O(logn)，树的高度为对数级别；
    • 最坏情况（二叉树退化为单链表）：递归栈深度为O(n)，需存储所有节点的递归状态；
  • 代码特性：递归解法代码极度简洁，逻辑直观，是翻转二叉树的面试首选解法，也是二叉树基础操作的经典案例。

3、程序

3.1 完整带详细注释代码（优化无意义变量）

3.2 代码逐行解释

  • 递归终止条件，是整个算法的边界保障：
    • 当遍历到叶子节点的左/右空孩子时，直接返回null，不再继续递归；
    • 空树直接返回null，符合题目要求。
  • 严格的后序遍历递推：先递归处理左子树，再递归处理右子树；
  • 这两步执行完成后，当前节点的左、右子树均已完成翻转，成为原左、右子树的镜像结构，为后续交换做准备。
  • 核心的左右孩子交换操作：
    • 必须用临时变量tempLeft保存原左孩子引用，否则直接执行root.left = root.right后，原左孩子的引用会被覆盖，无法再赋值给root.right；
    • 交换操作仅修改当前节点的左、右指针指向，是O(1)的常数时间操作。
  • 返回当前已完成翻转的节点，实现结果回溯：
    • 对于叶子节点，返回自身（已完成交换，实际无孩子可交换）；
    • 对于非叶子节点，返回已交换左右孩子的自身，作为其父节点的左/右子节点；
    • 最终递归结束时，返回的是根节点，即翻转后的整棵树的根。

3.3 原代码与优化代码对比

原代码中TreeNode temp = root;和最终return temp;的逻辑，与优化代码return root;完全等价，因为：

  • temp仅为根节点的引用拷贝，并未创建新节点，temp和root指向同一个对象；
  • 整个递归过程中仅修改节点的left/right指针，未修改根节点的引用，因此直接返回root更简洁，去除无意义变量让代码更整洁。

4、算法性能

4.1 复杂度分析

  • 时间复杂度：O(n)
    递归过程会遍历二叉树的每一个节点，每个节点仅被访问一次，交换左右孩子的操作是常数时间O(1)，无重复遍历和冗余操作，总时间复杂度为线性级别。

  • 空间复杂度：O(n)
    空间开销仅来自递归调用栈，无额外的数组、队列、哈希表等存储结构：

    • 平衡二叉树（如满二叉树）：树的高度h = log₂(n+1)，递归栈深度为O(logn)，空间复杂度接近O(1)；
    • 退化二叉树（如单链表形式的二叉树）：树的高度h = n，递归栈深度为O(n)，需存储所有节点的递归状态。

4.2 解法对比

解法 时间复杂度 空间复杂度 代码简洁性 理解难度 适用场景
递归后序法 O(n) O(n) 极高 低 面试首选、常规场景
迭代层序法 O(n) O(n) 中 中 避免递归栈溢出的场景
递归前序法 O(n) O(n) 极高 低 与后序法等价，仅顺序不同

注：翻转二叉树也可采用前序遍历（交换当前节点孩子→递归左→递归右），结果与后序法一致，仅遍历顺序不同；中序遍历不可用（会导致左子树被重复翻转）。

关键结论

6、拓展与总结

6.1 拓展1：迭代层序遍历法（非递归，避免栈溢出）

  • 迭代法无递归栈溢出风险，适合处理大规模二叉树。

6.3 核心避坑点

  1. 不可用中序遍历：若采用中序遍历（递归左→交换当前→递归右），会导致左子树被重复翻转，最终结果错误；
  2. 交换时必须用临时变量：交换左右孩子时，若不保存原左/右孩子引用，会导致节点引用丢失，无法完成交换；
  3. 递归终止条件不可少：若遗漏root == null的判断，会遍历到空节点的left/right，触发空指针异常。

6.4 整体总结

翻转二叉树是二叉树基础操作的入门经典题，递归后序/前序遍历法是面试中的首选解法，其核心价值在于：

  1. 体现分治思想：将翻转整棵树的大问题，拆解为翻转左右子树和交换当前节点的小问题，层层递推、归并结果；
  2. 强化遍历理解：清晰体现二叉树后序/前序遍历的应用场景，理解“先处理子节点还是先处理父节点”的遍历逻辑；
  3. 掌握节点交换：学会二叉树中节点指针的修改和引用保存，这是二叉树插入、删除、交换等操作的基础。

本题的递归解法代码仅数行，却包含了二叉树递归的核心要素：终止条件、递推遍历、结果回溯，掌握该解法，能为后续学习二叉树的复杂操作（如路径和、子树判断、最近公共祖先）打下坚实基础。

6.5 思维提升

翻转二叉树的解题思路，体现了“从局部到整体”的算法思维：要完成整个结构的修改，只需定义好单个节点的修改规则，再通过遍历（递归/迭代）将规则应用到所有节点，这种思维适用于所有二叉树的结构修改类问题，是二叉树学习的核心思维之一。
