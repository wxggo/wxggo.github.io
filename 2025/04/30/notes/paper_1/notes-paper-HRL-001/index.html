Notes-文献阅读:救灾环境下异构无人机集群任务规划与动态调度

发表于2025-04-30|更新于2025-04-30|科研笔记文献阅读
|字数总计:5.3k|阅读时长:16分钟|阅读量:

本文为个人科研方向文献阅读笔记，阅读对象为钟懿的硕士学位论文《救灾环境下异构无人机集群任务规划与动态调度》。

论文面向地震、洪水、山火等重大灾害后的无人机集群搜救任务，研究如何在复杂、动态、异构的救灾环境中，对多架无人机进行快速任务初始分配，并在执行过程中面对突发情况进行动态调度。

由于帖主目前关注方向包括多无人机任务分配、强化学习、任务规划、动态调度与无人系统协同决策等内容，因此本文重点关注论文中的方法设计，而不过多展开背景和实验细节。

一、论文基本信息

  • 论文题目：救灾环境下异构无人机集群任务规划与动态调度
  • 作者：钟懿
  • 学校：东华大学
  • 研究方向：复杂系统建模、控制与优化
  • 关键词：无人机集群；任务规划；动态调度；集中式决策；分布式决策

论文主要研究两个阶段的问题：

  1. 任务初始分配：灾害发生后，根据已知任务和无人机信息，快速生成可执行的初始任务序列；
  2. 动态调度：无人机执行任务过程中，若出现新任务、新无人机、任务取消、无人机召回等突发情况，需要快速重规划任务序列。

论文整体思路可以概括为：

初始分配阶段采用集中式决策，追求快速、高质量的全局规划；动态调度阶段采用分布式决策，追求面对突发情况时的鲁棒性和实时性。

二、研究问题：救灾环境下的异构无人机集群任务规划

救灾环境中的无人机任务规划和普通路径规划不同，主要体现在三个方面。

2.1 任务异构

论文将救灾任务划分为三类：

任务类型 含义 特点
定点巡查任务 无人机到达指定点进行拍照或探测 成本较低，到点即可完成
区域搜索任务 无人机对指定区域进行覆盖搜索 需要额外覆盖路径
救援物资运输任务 无人机从资源点运输物资到目标点 对无人机负载能力有要求

其中，救援物资运输任务具有明显的起点和终点，因此任务之间的飞行成本具有非对称性。

2.2 无人机异构

无人机之间也不是完全相同的。论文主要考虑无人机是否具备负载能力：

  • 普通无人机：可以执行定点巡查和区域搜索任务；
  • 负载无人机：可以执行定点巡查、区域搜索和救援物资运输任务。

这意味着任务分配不仅要考虑距离，还要考虑无人机能力是否匹配任务需求。

2.3 环境动态变化

在救灾过程中，任务和无人机状态可能不断变化，例如：

  1. 发现新任务；
  2. 新无人机加入；
  3. 原有任务被取消；
  4. 某架无人机被召回；
  5. 无人机电量不足；
  6. 无人机通信状态变化。

因此，论文不是只做一次静态任务分配，而是将问题拆分为初始分配和动态调度两个阶段。

三、总体决策框架：集中式 + 分布式

论文设计了一个两阶段决策结构：

阶段 决策结构 主要目标
初始分配阶段 集中式决策 快速生成高质量初始任务序列
动态调度阶段 分布式决策 面对突发情况快速重规划

3.1 初始分配为什么采用集中式？

初始分配发生在无人机出发前，此时任务信息和无人机信息相对完整，中央控制节点可以统一计算全局方案。

集中式的优点是：

  1. 全局信息完整；
  2. 分配质量较高；
  3. 便于统一考虑任务约束；
  4. 适合无人机出发前的快速规划。

3.2 动态调度为什么采用分布式？

无人机执行任务后，环境变化快，通信条件可能不稳定。如果所有信息都回传中央节点再重新规划，会带来较大时延。

分布式的优点是：

  1. 响应速度快；
  2. 容错能力强；
  3. 不完全依赖中央节点；
  4. 适合处理突发任务和无人机状态变化。

因此，论文采用了一个比较合理的结构：

出发前集中规划，执行中分布式协商。

四、问题建模：MADLC-VRP

论文将救灾环境下的异构无人机任务规划问题建模为：

具有负载特性要求的多行程非对称距离约束车辆路径问题
Multi-Trip Asymmetric Distance Constrained VRP with Load Characteristics Requirements, MADLC-VRP

该模型可以理解为 VRP 的扩展版本，主要增加了以下特性：

  1. 多架无人机；
  2. 多种任务类型；
  3. 任务飞行成本非对称；
  4. 无人机负载能力约束；
  5. 无人机最大飞行距离约束；
  6. 无人机可以回仓库换电后继续执行任务。

4.1 优化目标

论文的目标是最小化系统总飞行成本：

$$
\min Z=\sum_{k=1}^{m}\sum_{j=0}^{n}\sum_{i=0}^{n}x_{ij}^{k}e_{ij}
$$

其中：

  • $m$ 表示无人机数量；
  • $n$ 表示任务数量；
  • $x_{ij}^{k}$ 表示无人机 $k$ 是否按照 $i \rightarrow j$ 的顺序执行任务；
  • $e_{ij}$ 表示任务 $i$ 到任务 $j$ 的飞行成本；
  • $Z$ 表示无人机集群完成全部任务的总飞行成本。

4.2 主要约束

论文中的约束主要包括：

  1. 每个任务必须被执行；
  2. 每个任务只能被一架无人机执行；
  3. 每架无人机从仓库出发并最终回到仓库；
  4. 单次飞行距离不能超过最大飞行距离；
  5. 救援物资运输任务必须由具备负载能力的无人机执行；
  6. 防止任务路径中出现与仓库无关的子回路。

这个模型比较适合作为多无人机救灾任务分配的基础模型，因为它同时考虑了任务异构、无人机异构和续航约束。

五、初始分配方法：TA-PRM-CNN

论文第三章提出了用于任务初始分配的核心算法：

TA-PRM-CNN
Heterogeneous UAV Cluster Task Allocation Algorithm with CNN-based Priority Ranking Module

该方法可以理解为：

Transformer 编码任务和无人机信息，CNN 决定无人机选择任务的优先级，解码器逐步生成任务分配序列，并通过强化学习优化任务选择策略。

5.1 方法整体流程

TA-PRM-CNN 主要包含四个模块：

  1. 任务信息编码器
  2. 无人机信息编码器
  3. 基于 CNN 的无人机优先级排序模块
  4. 任务选择解码器

整体流程如下：

  1. 输入任务集合和无人机集合；
  2. 编码器分别提取任务特征和无人机特征；
  3. CNN 优先级排序模块决定当前轮次哪架无人机先选任务；
  4. 解码器根据当前无人机状态和任务状态输出任务选择概率；
  5. 叠加掩码策略过滤不可选任务；
  6. 无人机依次选择任务并更新状态；
  7. 循环直到所有任务被分配。

5.2 任务与无人机特征编码

论文为异构任务设计了统一特征表示：

$$
v=(x_1,y_1)\Vert(x_2,y_2)\Vert A\Vert I_{type}\Vert t
$$

其中：

  • $(x_1,y_1)$ 表示任务关键位置；
  • $(x_2,y_2)$ 表示任务结束位置；
  • $A$ 表示任务相关信息；
  • $I_{type}$ 表示任务类型 one-hot 编码；
  • $t$ 表示任务特性，例如是否需要负载能力。

不同任务对应的信息不同：

任务类型 关键位置 结束位置 任务相关信息
定点巡查 巡查点坐标 巡查点坐标 0
区域搜索 搜索区域圆心 搜索区域圆心 区域半径
救援物资运输 物资起点 物资终点 运输距离
回仓库 仓库坐标 仓库坐标 0

无人机也被拆分为静态特征和动态特征：

$$
u=(x_0,y_0)\Vert l
$$

$$
u’=(x_{now},y_{now})\Vert power_{remain}\Vert v_{first}\Vert v_{prev}
$$

其中：

  • $(x_0,y_0)$ 表示出发位置；
  • $l$ 表示无人机执行能力；
  • $(x_{now},y_{now})$ 表示当前位置；
  • $power_{remain}$ 表示剩余电量；
  • $v_{first}$ 表示第一步任务信息；
  • $v_{prev}$ 表示前一步任务信息。

这种设计的好处是，可以将异构任务和异构无人机统一输入神经网络。

六、优先级排序模块：PRM-CNN

论文中一个比较有意思的设计是：在每一轮任务选择中，并不是所有无人机同时选任务，而是先通过 CNN 得到无人机的选择优先级。

6.1 为什么需要优先级排序？

如果多架无人机同时选择任务，可能会出现多个无人机选择同一个任务的冲突。

因此，论文让无人机按照优先级依次选择任务：

  1. 优先级最高的无人机先选；
  2. 该任务被选后，叠加掩码会更新；
  3. 后续无人机不能再选择已被分配的任务；
  4. 当前轮次所有无人机完成选择后，进入下一轮。

这种机制能够减少任务冲突，也能让任务分配更加均衡。

6.2 CNN 的作用

PRM-CNN 将无人机的静态特征和动态特征拼接为当前状态，然后通过二维卷积提取特征，最后输出每架无人机的优先级概率。

其意义在于：

不同无人机在不同状态下，应该拥有不同的任务选择优先权。

例如，某架无人机剩余电量较少，可能更适合先选择靠近当前位置或靠近仓库的任务；具备负载能力的无人机则可能需要优先选择救援物资运输任务。

七、解码器与叠加掩码策略

7.1 解码器任务选择

解码器使用交叉注意力机制进行任务选择。对于当前无人机，Query 由以下信息拼接而成：

  1. 全局任务嵌入；
  2. 当前无人机嵌入；
  3. 当前无人机位置；
  4. 剩余电量；
  5. 前一步任务信息。

Key 和 Value 来自任务节点嵌入。

解码器输出当前无人机选择各个任务的概率：

$$
p_{v_i}=softmax(score_i)
$$

然后根据概率选择任务加入任务执行序列。

7.2 叠加掩码策略

为了保证生成的任务分配序列可行，论文设计了叠加掩码策略。

叠加掩码由三部分组成：

掩码 作用
$M_{complete}$ 过滤已经完成或已经被分配的任务
$M_{load}$ 过滤无人机能力不匹配的任务
$M_{power}$ 过滤电量不足以执行并返回仓库的任务

最终叠加掩码为：

$$
SM=M_{complete}\cup M_{load}\cup M_{power}
$$

如果某个任务在任一约束下不可选，则最终掩码为 1，该任务会被屏蔽；只有同时满足所有约束时，掩码为 0，任务才可以被选择。

这一部分对于强化学习求解组合优化非常关键，因为它能显著减少无效动作，提高训练效率，并保证输出结果满足基本约束。

八、强化学习训练方法

由于任务分配问题缺少标准标签，论文没有使用监督学习，而是采用强化学习训练任务选择策略。

模型策略表示为：

$$
\pi_{\theta}
$$

任务分配序列的生成概率可以写为：

$$
p_{\theta}(\Omega|V)=
\prod_{t=1}^{curstep}
p_{\theta}(\Omega_t|V,\Omega_{1:t-1})
$$

其中：

  • $V$ 表示任务节点集合；
  • $\Omega$ 表示生成的任务执行序列；
  • $\Omega_t$ 表示第 $t$ 步选择的任务；
  • $\theta$ 表示神经网络参数。

论文采用带滑动基线的强化学习方法进行优化。其核心思想是：

  1. 模型生成任务分配方案；
  2. 根据飞行成本、整体完成时间、不均衡性等指标评价方案；
  3. 使用基线降低策略梯度方差；
  4. 不断更新网络参数，使任务分配序列质量提升。

这类方法与神经组合优化中的 Attention Model、Pointer Network 思路较接近。

九、动态调度方法：ACBBA

论文第四章提出动态调度算法：

ACBBA
Adjustment-factor-based Consensus-Based Bundle Algorithm

即：结合调整因子的一致性捆绑包算法。

9.1 为什么需要动态调度？

初始分配方案在实际执行过程中可能被突发事件打破，例如：

  1. 出现新任务；
  2. 出现新无人机；
  3. 取消某一任务；
  4. 召回某一无人机。

这些情况都会导致原任务序列不再最优，甚至不可执行。

9.2 ACBBA 的核心思想

ACBBA 分为两个部分：

  1. 根据动态情况和调整因子释放部分原有任务；
  2. 使用 CBBA 的任务包构建和一致性冲突消解机制重新分配任务。

其中，调整因子记为：

$$
\varepsilon
$$

它控制释放多少原任务：

  • $\varepsilon=0$：不释放原有任务，只处理新增或变化部分；
  • $\varepsilon=1$：释放全部未执行任务，相当于完全重规划；
  • $0<\varepsilon<1$：释放部分收益较低的任务，进行局部重规划。

9.3 调整因子的意义

调整因子提供了一个“速度—质量”的权衡：

调整因子 特点
较小 调整范围小，计算快，适合频繁动态变化
较大 调整范围大，方案质量更高，但计算时间更长
0 不释放原任务，最快但可能收益有限
1 全部释放重规划，质量可能更高但耗时更长

论文实验表明，调整因子在 $0.5$ 到 $0.75$ 之间时，通常能获得较高整体收益；而调整因子在 $0$ 到 $0.25$ 之间时，计算时间更短。

9.4 ACBBA 动态调度流程

ACBBA 的流程可以概括为：

  1. 检测动态事件；
  2. 根据事件类型更新任务集或无人机集；
  3. 根据调整因子释放部分未执行任务；
  4. 各无人机基于自身状态构建任务包；
  5. 通过一致性冲突消解解决任务归属冲突；
  6. 得到新的无冲突任务调度方案；
  7. 无人机按新任务序列继续执行。

这个方法适合动态救灾环境，因为它不依赖单一中央节点，且可以通过调整因子控制重规划范围。

十、实验结果简要理解

论文实验主要验证两个方面：

  1. 初始分配算法 TA-PRM-CNN 的有效性；
  2. 动态调度算法 ACBBA 的有效性。

10.1 初始分配实验

TA-PRM-CNN 与 TA-FPRM、OR-tools 进行对比。

结果显示：

  • OR-tools 在飞行成本上更优；
  • TA-PRM-CNN 在任务均衡性和整体完成时间上更有优势；
  • TA-PRM-CNN 的求解速度明显优于传统 OR-tools；
  • 小规模样本训练后的模型可以泛化到更大规模任务。

这一点很适合救灾场景，因为救灾任务更关注快速响应和整体完成时间，而不是单纯追求最短总路径。

10.2 动态调度实验

论文分别测试了以下动态情况：

  1. 出现新任务；
  2. 出现新无人机；
  3. 取消某一任务；
  4. 召回某一无人机。

实验结果表明，ACBBA 能够在这些情况下重新生成可执行任务序列，并保证所有任务继续被完成。

十一、个人思考：对多无人机任务分配研究的启发

11.1 初始规划和动态调度应分开设计

这篇论文一个重要启发是：任务规划不应该只看静态初始分配，而应同时考虑执行过程中的动态变化。

对于多无人机系统来说，完整任务规划应包括：

  1. 出发前的初始分配；
  2. 执行过程中的动态调度；
  3. 突发事件下的任务重规划；
  4. 无人机故障或新增情况下的资源再分配。

这比单纯研究静态 VRP 或 MTSP 更接近真实应用。

11.2 Transformer 适合任务序列生成

任务分配本质上是一个序列生成问题：每一步选择一个任务，并逐步构造完整任务执行序列。

因此，Seq2Seq + Transformer 的思路非常适合多无人机任务分配。后续可以进一步结合：

  1. Pointer Network；
  2. Graph Attention Network；
  3. Transformer Decoder；
  4. 多智能体注意力机制；
  5. 强化学习策略梯度。

11.3 掩码机制非常关键

论文中的叠加掩码策略值得重点学习。很多强化学习任务分配算法训练困难，并不是因为网络不够复杂，而是动作空间中存在大量非法动作。

掩码机制可以直接过滤不可行动作，例如：

  1. 已完成任务；
  2. 能力不匹配任务；
  3. 电量不足任务；
  4. 时间窗不满足任务；
  5. 通信范围外任务；
  6. 优先级不满足任务。

这能显著提高训练效率和解的可行性。

11.4 动态调度不一定必须完全重规划

ACBBA 中调整因子的设计很有意义。面对突发情况时，如果完全重规划，解质量可能较好，但计算开销大；如果只局部调整，速度快但收益可能有限。

因此，后续研究可以考虑自适应调整因子：

根据突发事件严重程度、剩余任务数量、通信状态和任务紧急程度，动态决定释放多少任务。

这可能比固定调整因子更适合真实救灾环境。

十二、后续可拓展方向

结合本人研究方向，后续可以从以下方向扩展：

  1. 强化学习动态调度
    用多智能体强化学习替代或辅助 ACBBA，使无人机学习更灵活的动态调度策略。

  2. 图神经网络建模
    将无人机、任务、仓库构造成异构图，用 GNN 编码任务关系和无人机能力关系。

  3. 多目标优化
    同时考虑总飞行成本、完成时间、任务优先级、能耗、风险和通信质量。

  4. 自适应调整因子
    让调整因子不再人工设定，而是由策略网络根据当前系统状态自动输出。

  5. 真实仿真平台验证
    结合 ROS、Gazebo、PX4 对任务分配结果进行飞行仿真，进一步验证算法可执行性。

十三、总结

本文阅读的《救灾环境下异构无人机集群任务规划与动态调度》针对应急救灾环境下的异构无人机集群任务规划问题，提出了一个较完整的解决框架。

论文主要贡献可以概括为：

  1. 将任务规划划分为初始分配和动态调度两个阶段；
  2. 设计集中式初始分配与分布式动态调度相结合的决策结构；
  3. 建立考虑任务异构、无人机异构、非对称飞行成本和负载约束的 MADLC-VRP 模型；
  4. 提出基于 Transformer、CNN 优先级排序和强化学习的 TA-PRM-CNN 初始分配算法；
  5. 提出叠加掩码策略保证任务选择满足约束；
  6. 提出结合调整因子的 ACBBA 动态调度算法，应对新任务、新无人机、任务取消、无人机召回等突发情况。

从个人研究角度看，这篇论文最大的价值在于提供了一个比较完整的多无人机任务规划范式：

异构任务建模 → 初始集中式分配 → Transformer 强化学习求解 → 约束掩码过滤 → 动态事件触发 → 分布式 ACBBA 重调度 → ROS/Gazebo 仿真验证。

该思路对于后续研究多无人机任务分配、强化学习、动态调度和无人系统协同决策都具有较强参考价值。
