Leetcode-160:相交链表-双指针法

Created2025-10-15|Updated2025-10-15|Java学习笔记Leetcode Hot100
|Word Count:1.6k|Reading Time:5mins|Post Views:

1、题目

编写一个程序，找到两个单链表相交的起始节点。如果两个链表没有交点，返回 null。

示例

示例 1（有相交节点）：

输入：

  • 链表A：4 → 1 → 8 → 4 → 5
  • 链表B：5 → 0 → 1 → 8 → 4 → 5
  • 相交节点：值为8的节点

输出：8（实际返回该节点对象，示例简化为值）

示例 2（无相交节点）：

输入：

  • 链表A：2 → 6 → 4
  • 链表B：1 → 5

输出：null

提示

  • 链表中节点数目范围是 [0, 10⁵]
  • -10⁹ <= Node.val <= 10⁹
  • 如果两个链表没有交点，返回 null
  • 要求算法时间复杂度 O(n)，空间复杂度 O(1)

2、分析

2.1 问题核心

寻找两个单链表的相交节点，常规思路是：

  1. 计算两个链表的长度，得到长度差 diff；
  2. 让长链表先遍历 diff 个节点，再让两个链表同步遍历，遇到第一个相同节点即为相交节点。

但该思路需要两次遍历计算长度，代码冗余。双指针交叉遍历法 无需计算长度，仅通过一次遍历即可实现，且满足空间复杂度 O(1) 的要求，是更优解。

2.2 核心疑问（高频痛点）

很多人会疑惑：若两个链表无相交节点且长度相同，是否会陷入死循环？
结论：不会。核心原因是「两个指针的总遍历长度相等」，最终会同时指向 null，触发循环终止条件。

2.3 双指针法核心思想

让两个指针 tempA（遍历链表A）、tempB（遍历链表B）遵循以下规则：

  1. 若指针未遍历完当前链表，继续向后走；
  2. 若指针遍历完当前链表（指向 null），则跳转到另一个链表的头节点继续遍历；
  3. 当 tempA == tempB 时终止循环：
    • 若有相交节点：tempA/tempB 会在相交节点相遇；
    • 若无相交节点：tempA/tempB 会同时指向 null。

2.4 逻辑拆解（无死循环验证）

前提：总遍历长度相等

无论链表是否相交、长度是否相同，两个指针的总遍历长度均为 len(A) + len(B)：

  • tempA 遍历路径：A链表全节点 → B链表全节点（总长度 len(A)+len(B)）；
  • tempB 遍历路径：B链表全节点 → A链表全节点（总长度 len(B)+len(A)）。

场景1：无相交 + 长度相同

假设：

  • 链表A：A1 → A2 → null（长度2）；
  • 链表B：B1 → B2 → null（长度2）。

遍历步骤：

循环次数 tempA位置 tempB位置 更新逻辑
初始 A1 B1 进入循环（tempA≠tempB）
1 A2 B2 正常后移
2 null null 正常后移
3 B1 A1 跳转到对方链表头
4 B2 A2 正常后移
5 null null 正常后移
6 - - tempA==tempB（均为null），循环终止

场景2：无相交 + 长度不同

假设：

  • 链表A：A1 → A2 → null（长度2）；
  • 链表B：B1 → B2 → B3 → null（长度3）。

遍历路径：

  • tempA：A1→A2→null→B1→B2→B3→null（总长度5）；
  • tempB：B1→B2→B3→null→A1→A2→null（总长度5）；
    最终同时指向 null，循环终止。

3、程序

3.1 完整代码

3.2 代码逐行解释

  • 初始化两个临时指针，分别指向链表A、B的头节点，避免修改原链表头节点。
  • 循环核心条件：只要两个指针指向的节点不同，就继续遍历；
  • 终止条件：要么指向同一个相交节点，要么同时指向 null（无相交）。
  • 三元运算符简化逻辑：
    • 若指针未遍历完当前链表（tempA != null），则向后移动一位（tempA.next）；
    • 若指针遍历完当前链表（tempA == null），则跳转到另一个链表的头节点（headB）。
  • 循环终止时，tempA 和 tempB 指向同一节点：
    • 有相交：指向相交节点；
    • 无相交：均为 null，返回 null 符合题目要求。

4、算法性能

  • 时间复杂度：O(n)
    • 最坏情况下（无相交），两个指针各遍历 len(A)+len(B) 个节点，总操作数为 2*(len(A)+len(B))，仍为线性时间。
  • 空间复杂度：O(1)
    • 仅使用两个临时指针，无额外空间开销，满足题目要求。

优势对比

解法 时间复杂度 空间复杂度 代码冗余度
长度差法 O(n) O(1) 高（需计算长度）
哈希表法 O(n) O(n) 低（但空间不满足）
双指针交叉法 O(n) O(1) 极低（最优解）

5、测试示例

示例1：有相交节点

输入：

  • 链表A：4 → 1 → 8 → 4 → 5（len=5）；
  • 链表B：5 → 0 → 1 → 8 → 4 → 5（len=6）。

遍历过程：

  • tempA：4→1→8→4→5→null→5→0→1→8（相遇，终止）；
  • tempB：5→0→1→8→4→5→null→4→1→8（相遇，终止）；
    返回值：值为8的节点（符合预期）。

示例2：无相交节点（长度相同）

输入：

  • 链表A：2 → 6 → 4（len=3）；
  • 链表B：1 → 5 → 7（len=3）。

遍历过程：

  • tempA：2→6→4→null→1→5→7→null；
  • tempB：1→5→7→null→2→6→4→null；
    最终 tempA == tempB == null，返回 null（符合预期）。

6、总结

双指针交叉遍历法的精妙之处在于：

  1. 无需计算长度：通过“交叉遍历对方链表”的方式，让两个指针的总遍历长度强制相等；
  2. 无死循环风险：无论是否相交、长度是否相同，最终都会因 tempA == tempB 终止循环；
  3. 极致空间效率：仅使用两个指针，满足 O(1) 空间复杂度要求。

该思路是链表类问题中“双指针”技巧的经典应用，核心是通过「路径补偿」让两个指针的遍历节奏对齐。
