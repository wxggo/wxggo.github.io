Leetcode-001:两数之和-哈希表一次遍历最优解

Created2025-02-25|Updated2025-02-25|Java学习笔记Leetcode Hot100
|Word Count:979|Reading Time:3mins|Post Views:

1、题目

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，且数组中同一个元素不能使用两遍，你可以按任意顺序返回答案。

示例

输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：nums[0] + nums[1] = 2 + 7 = 9，返回对应下标。

提示

  • 2 <= nums.length <= 10⁴
  • -10⁹ <= nums[i] <= 10⁹
  • -10⁹ <= target <= 10⁹
  • 仅存在一个有效答案

2、分析

2.1 问题核心

两数之和的核心是寻找数组中两个数的互补关系：对于数nums[i]，只需找到是否存在target - nums[i]，并记录两者下标。

  • 暴力解法：双层循环遍历，逐个匹配和为target的两个数，时间复杂度O(n²)，效率较低；
  • 最优解法：哈希表一次遍历，利用HashMap存储「数值-下标」映射，边遍历边查询互补数，时间复杂度降至O(n)，为题目最优解。

2.2 核心思路

  1. 用HashMap存储已遍历过的数值及其下标，键为数值，值为对应数组下标；
  2. 一次遍历数组，对当前元素nums[i]，计算互补数complement = target - nums[i]；
  3. 若HashMap中包含互补数，说明已找到答案，直接返回互补数下标和当前下标i；
  4. 若不包含，将当前nums[i]和下标i存入HashMap，继续遍历。

该思路无需二次遍历，且利用哈希表O(1)的查询特性，实现线性时间复杂度。

3、程序

3.1 完整代码

3.2 代码简洁解释

  • 初始化结果数组ans和HashMap，分别用于存答案、存「数值-下标」；
  • 遍历过程中先查互补数，再存当前数，避免使用同一元素；
  • 找到互补数后立即记录下标并break，无需遍历剩余元素，优化执行效率；
  • 题目保证有唯一解，故无需处理无结果的情况。

4、算法性能

  • 时间复杂度：O(n)。仅一次遍历数组，哈希表的containsKey和put操作均为O(1)；
  • 空间复杂度：O(n)。最坏情况下需存储数组中n-1个元素的哈希表（答案在最后两个位置）；
  • 核心优势：线性时间复杂度，代码极简，执行效率远高于暴力解法，适配题目数据规模。

5、测试示例

以nums = [2,7,11,15], target = 9为例，执行过程如下：

  1. 初始化ans=[0,0]，hashMap为空；
  2. i=0，nums[0]=2，complement=7，hashMap无7 → 存入(2,0)，hashMap={2:0}；
  3. i=1，nums[1]=7，complement=2，hashMap包含2 → ans[0]=0，ans[1]=1，break；
  4. 返回ans=[0,1]，与示例一致。

6、扩展思考

  1. 暴力解法对比：双层循环暴力解法代码简单但效率低，数据量稍大即超时，实际开发中不推荐；
  2. 注意点：必须先查询互补数再存入当前数，若先存后查，会出现自己和自己匹配的情况（如nums=[3,3], target=6）；
  3. 扩展场景：若题目允许多个答案，只需移除break，用List存储所有符合条件的下标数组即可。
