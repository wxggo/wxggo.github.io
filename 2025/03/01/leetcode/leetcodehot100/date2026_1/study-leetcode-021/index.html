Leetcode-021:合并两个有序链表-迭代法极简实现

Created2025-03-01|Updated2025-03-01|Java学习笔记Leetcode Hot100
|Word Count:1.5k|Reading Time:5mins|Post Views:

1、题目

将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的，要求原地操作，不创建新的节点（仅改变节点指针）。

示例

示例 1：
输入：list1 = [1,2,4], list2 = [1,3,4]
输出：[1,1,2,3,4,4]

示例 2：
输入：list1 = [], list2 = []
输出：[]

示例 3：
输入：list1 = [], list2 = [0]
输出：[0]

提示

  • 两个链表的节点数目范围是 [0, 50]
  • -100 <= Node.val <= 100
  • list1 和 list2 均按 非递减顺序 排列

2、分析

2.1 问题核心

合并两个升序链表的核心是利用双指针逐个比较节点值，按升序拼接指针，关键难点是处理头节点为空的边界情况（如其中一个链表为空、两个链表都为空）。

迭代法是该题的最优解之一：通过虚拟头节点避免繁琐的头节点判断，双指针分别遍历两个链表，一次遍历即可完成合并，时间复杂度为线性级别，实现简单且无递归栈开销。

2.2 核心思路

  1. 虚拟头节点：创建一个空的虚拟头节点（new ListNode()），作为新链表的起始，避免处理头节点为空的特殊情况；
  2. 遍历指针：定义cur指针指向虚拟头节点，用于拼接新链表的节点；
  3. 双指针迭代：用两个原链表头节点list1、list2作为遍历指针，逐个比较节点值：
    • 将值较小的节点拼接到cur的下一个位置，并将该链表的遍历指针后移；
    • 同步后移cur指针，保持新链表的连续；
  4. 拼接剩余节点：当其中一个链表遍历完毕，直接将另一个链表的剩余部分拼接到新链表末尾（剩余部分本身已是升序）；
  5. 返回结果：虚拟头节点的下一个节点（newList.next）即为合并后新链表的真实头节点。

3.2 代码简洁解释

  • **虚拟头节点newList**：统一处理所有边界情况，无需单独判断list1/list2是否为空；
  • **遍历指针cur**：始终指向新链表的最后一个节点，负责后续节点的拼接；
  • while循环：仅在两个链表都有节点时执行，每次选择值更小的节点拼接，保证新链表升序；
  • 剩余节点拼接：利用三目运算符快速判断，直接拼接非空链表的剩余部分，无需额外循环；
  • **返回newList.next**：虚拟头节点无实际值，其下一个节点才是合并后链表的真正起始。

4、算法性能

  • 时间复杂度：O(m + n)（m、n分别为两个链表的节点数）。两个链表各被遍历一次，每次仅做指针操作，无重复计算；
  • 空间复杂度：O(1)。仅使用了虚拟头节点和几个遍历指针，属于原地操作，无额外空间开销；
  • 核心优势：
    1. 迭代实现无递归栈开销，适合链表节点数较多的场景；
    2. 代码极简，逻辑清晰，易理解易书写；
    3. 原地操作，仅改变节点指针，不创建新节点，空间利用率拉满。

5、测试示例

以list1 = [1,2,4]，list2 = [1,3,4]为例，执行过程如下：

  1. 初始化newList（虚拟头）、cur = newList，list1→1，list2→1；
  2. 第一次循环：1<=1 → cur.next=list1，list1→2，cur→1；
  3. 第二次循环：2>1 → cur.next=list2，list2→3，cur→1；
  4. 第三次循环：2<3 → cur.next=list1，list1→4，cur→2；
  5. 第四次循环：4>3 → cur.next=list2，list2→4，cur→3；
  6. 第五次循环：4<=4 → cur.next=list1，list1→null，cur→4；
  7. 循环结束，list1为空，cur.next=list2（→4）；
  8. 返回newList.next，最终链表：1→1→2→3→4→4，与示例一致。

6、扩展思考

6.1 递归解法对比

递归解法代码更简洁，但存在O(m + n)的空间复杂度（递归栈深度），适合理解递归思想，实际开发中迭代法更优：

6.2 关键边界情况处理

代码天然适配所有边界场景，无需额外判断：

  • 两个链表都为空：newList.next直接返回null；
  • 其中一个链表为空：循环不执行，直接拼接另一个链表，返回非空链表；
  • 链表节点值全相同：按<=判断拼接，保证顺序不变。

6.3 拓展应用

该迭代思路可扩展至合并k个有序链表（Leetcode23），只需在此基础上结合优先队列（小顶堆）筛选最小值节点，核心的虚拟头节点+指针拼接逻辑保持不变。
