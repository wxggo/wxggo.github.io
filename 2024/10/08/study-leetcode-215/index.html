Java实现寻找第k大元素：快速选择法（随机基准+三分组优化）

Created2024-10-08|Updated2024-10-08|算法学习笔记
|Post Views:

1、题目

给定一个整数数组 nums 和一个整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素，要求时间复杂度尽可能优于 O(n log n)。

示例

示例 1：
输入：nums = [3,2,1,5,6,4], k = 2
输出：5

示例 2：
输入：nums = [3,2,3,1,2,4,5,5,6], k = 4
输出：4

提示

  • 1 <= k <= nums.length <= 10⁵
  • -10⁴ <= nums[i] <= 10⁴

2、分析

2.1 问题核心

寻找第 k 大元素的常规思路是对数组排序后直接取对应位置元素（时间复杂度 O(n log n)），但对于大数据量（如题目中 n=10⁵），该方法效率不足。因此选择快速选择法（QuickSelect），基于快速排序的分治思想，仅递归处理目标元素所在的子区间，平均时间复杂度可降至 O(n)。

2.2 核心优化点

（1）随机基准选择

快速排序/选择的性能依赖基准值的选择：若每次选固定位置（如区间首/尾），在数组有序/逆序时会退化为 O(n²) 时间复杂度。随机选择基准索引 可避免最坏情况，使算法在概率上保持线性时间。

（2）三分组（荷兰国旗问题）优化

传统快速排序仅将数组分为“小于基准”和“大于基准”两组，当数组存在大量重复元素时，会导致递归区间仍较大。三分组策略 把数组划分为三个部分：

  • 小于基准的元素（small组）
  • 等于基准的元素（equal组）
  • 大于基准的元素（big组）

若目标位置落在equal组，可直接返回基准值，无需继续递归；仅当目标在small/big组时，才递归处理对应区间，大幅减少递归次数。

2.3 算法流程

  1. 目标位置转换：第 k 大元素等价于数组“从小到大排序”后索引为 n - k 的元素（记为 targetPos）。
  2. 分治划分：
    • 随机选择基准值，通过三指针（lessIdx/greaterIdx/cur）在原数组上划分出small/equal/big三组。
    • 对比 targetPos 与三组区间的边界：
      • 若 targetPos 在small组（targetPos < lessIdx）：递归处理 [start, lessIdx-1] 区间。
      • 若 targetPos 在big组（targetPos > greaterIdx）：递归处理 [greaterIdx+1, end] 区间。
      • 若 targetPos 在equal组：直接返回基准值（equal组所有元素均等于基准）。
  3. 递归终止：当区间仅含一个元素时，该元素即为目标。

2.4 三指针逻辑（荷兰国旗划分）

指针 初始值 作用
lessIdx start small组的右边界（[start, lessIdx-1] 为已划分的small组）
greaterIdx end big组的左边界（[greaterIdx+1, end] 为已划分的big组）
cur start 当前遍历指针，逐个检查数组元素

遍历规则：

  • 若 nums[cur] < 基准：交换 nums[cur] 和 nums[lessIdx]，lessIdx++、cur++（扩展small组，当前元素已处理）。
  • 若 nums[cur] > 基准：交换 nums[cur] 和 nums[greaterIdx]，greaterIdx--（扩展big组，交换后的新元素需重新检查，cur 不递增）。
  • 若 nums[cur] == 基准：cur++（元素归入equal组，无需交换）。

3.2 代码逐行解释

（1）成员变量与入口函数

  • 初始化随机数生成器，避免多次创建对象，提升效率。
  • 入口函数：将“第k大”转换为“从小到大排序后索引为 n-k”的目标位置，调用核心函数 quickSelect。

（2）快速选择核心函数

  • 递归终止条件：当处理区间 [start, end] 仅含一个元素时，该元素即为目标。
  • 随机生成基准索引：random.nextInt(end - start + 1) 生成 [0, end-start] 的随机数，加上 start 后范围为 [start, end]。
  • 获取基准值 pivot，后续基于该值划分数组。
  • 初始化三指针，分别对应small组右边界、big组左边界、当前遍历位置。
  • 核心遍历逻辑：完成数组的三分组划分，注释已清晰说明每一步操作的意义，需注意 nums[cur] > pivot 时 cur 不递增（交换后的新元素未检查）。
  • 根据目标位置 targetPos 选择递归区间：
    • 若在small组：递归处理 [start, lessIdx-1]。
    • 若在big组：递归处理 [greaterIdx+1, end]。
    • 若在equal组：直接返回基准值（无需递归）。

4、算法性能

  • 时间复杂度：平均 O(n)，最坏情况（极端随机）O(n²)（但随机基准可将最坏情况概率降至极低）。
  • 空间复杂度：O(log n)（递归调用栈的深度，平均为对数级别；若改为非递归实现，可降至 O(1)）。
  • 优势：在原数组上操作，无需额外空间，且针对重复元素的三分组优化大幅提升实际运行效率，适合处理大数据量的第k大/小元素问题。

5、测试示例

以 nums = [3,2,1,5,6,4], k = 2 为例：

  1. n=6，targetPos = 6-2 = 4（从小到大排序后索引4的元素为5）。
  2. 随机选择基准（假设选索引3，值为5）。
  3. 三分组划分后：
    • small组（<5）：[3,2,1,4] → 索引[0,3]
    • equal组（=5）：[5] → 索引[4]
    • big组（>5）：[6] → 索引[5]
  4. targetPos=4 落在equal组，直接返回基准值5，与预期结果一致。
