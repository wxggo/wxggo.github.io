Leetcode-026&027：双指针法搞定数组原地修改

Created2024-09-30|Updated2024-10-01|学习笔记Leetcode自由刷
|Word Count:1.9k|Reading Time:6mins|Post Views:

在LeetCode的数组类题目中，“原地修改数组”是一类经典考点，核心要求是不使用额外空间（O(1) 空间复杂度）完成数组元素的筛选与重构。其中，双指针法是解决这类问题的核心技巧。本文将通过两道经典题目——「删除有序数组中的重复项（LeetCode 26）」和「移除元素（LeetCode 27）」，拆解双指针法的应用逻辑，并补充Python/Java双语言实现，帮助大家掌握这一高频解题思路。

一、核心背景：为什么选择双指针？

数组的“原地修改”要求我们不能新建数组存储结果，只能在原数组上覆盖操作。双指针法通过两个指针分别承担“扫描遍历”和“记录赋值”的角色：

  • 一个指针（扫描/快指针）负责遍历数组，寻找符合条件的元素；
  • 一个指针（记录/慢指针）负责标记需要保留的元素位置，接收符合条件的元素。

这种思路既能满足O(1)空间复杂度要求，又能保证O(n)时间复杂度（仅遍历数组一次），是“原地修改数组”类问题的最优解。

二、实战解析：LeetCode 26 - 删除有序数组中的重复项

1. 题目描述

给你一个升序排列的数组 nums，请你原地删除重复出现的元素，使每个元素只出现一次，返回删除后数组的新长度。要求：

  • 元素的相对顺序保持一致；
  • 结果需存放在数组 nums 的前 k 个位置（k 为新长度）；
  • 仅使用 O(1) 额外空间。

示例：

  • 输入：nums = [1,1,2]，输出：2，且 nums 前两位变为 [1,2]；
  • 输入：nums = [0,0,1,1,1,2,2,3,3,4]，输出：5，且 nums 前五位变为 [0,1,2,3,4]。

2. 解题思路：双指针的“扫描+记录”逻辑

由于数组是升序排列的，重复元素必然连续出现，因此我们可以定义两个指针：

  • 慢指针（slow/记录指针）：标记下一个“非重复元素”要填入的位置，初始值为0（通用命名，更易理解）；
  • 快指针（fast/扫描指针）：遍历数组，检查当前位置的元素是否为重复项，初始值为0。

注：原思路中指针初始值为1，本质与“0起始”逻辑一致，仅判断条件略有调整，两种写法均可，本文补充更通用的“0起始”版本。

核心逻辑：

  1. 快指针逐个遍历数组，若当前元素与慢指针指向的元素不重复；
  2. 慢指针右移一位，将快指针指向的元素赋值到慢指针位置；
  3. 无论是否重复，快指针始终右移（持续遍历）；
  4. 遍历结束后，慢指针的数值+1即为新数组的长度（因为慢指针从0开始计数）。

3. 代码实现

4. 代码解析

  • 边界处理：优先判断空数组，避免后续遍历出错；
  • 指针命名优化：使用slow/fast（慢/快指针），是行业通用命名，更易理解；
  • 核心逻辑：通过快慢指针的“对比-赋值”，实现原地去重；
  • 复杂度：时间O(n)（仅遍历一次），空间O(1)（仅使用两个指针变量）。

三、实战解析：LeetCode 27 - 移除元素

1. 题目描述

给你一个数组 nums 和一个值 val，请你原地移除所有数值等于 val 的元素，返回移除后数组的新长度。要求：

  • 仅使用 O(1) 额外空间；
  • 元素顺序可改变，无需考虑超出新长度的元素。

示例：

  • 输入：nums = [3,2,2,3], val = 3，输出：2，且 nums 前两位变为 [2,2]；
  • 输入：nums = [0,1,2,2,3,0,4,2], val = 2，输出：5，且 nums 前五位为 [0,1,4,0,3]（顺序可调整）。

2. 解题思路：双指针的“筛选”逻辑

核心思路与LeetCode 26一致，调整判断条件为“元素是否等于目标值”：

  • 慢指针（slow/记录指针）：标记下一个“非val元素”要填入的位置，初始值为0；
  • 快指针（fast/扫描指针）：遍历数组，检查当前元素是否等于val，初始值为0。

核心逻辑：

  1. 快指针遍历数组，若当前元素不等于val，说明该元素需要保留；
  2. 将快指针指向的元素赋值到慢指针位置，慢指针右移；
  3. 若当前元素等于val，快指针右移，慢指针不动；
  4. 遍历结束后，慢指针的数值即为新数组长度（慢指针从0开始，每保留一个元素右移一次）。

3. 代码实现

4. 代码解析

  • 命名统一：延续slow/fast命名，保持双指针逻辑的一致性；
  • 逻辑简化：无需额外的边界判断（空数组时，循环不执行，直接返回0，符合预期）；
  • 跨语言对齐：Python和Java版本逻辑完全一致，仅语法差异，便于对比学习；
  • 复杂度：时间O(n)，空间O(1)，满足题目要求。

四、同类题目对比与总结

1. 两道题的核心对比

维度 LeetCode 26（删除重复项） LeetCode 27（移除元素）
指针初始值 slow=0、fast=0 slow=0、fast=0
判断条件 nums[fast] != nums[slow] nums[fast] != val
数组特性 升序、重复元素连续 无顺序要求、目标值离散
新长度计算 slow + 1（0起始计数） slow（直接计数）
核心逻辑 去重（保留唯一元素） 筛选（保留非目标元素）

2. 双指针法的核心思想（跨语言通用）

双指针法的核心是将“遍历”和“赋值”解耦：

  • 快指针（fast）：负责“找”——遍历数组，筛选符合条件的元素；
  • 慢指针（slow）：负责“存”——标记保留元素的位置，接收符合条件的元素。

这种思路的优势：

  • 空间效率：无需额外数组，仅在原数组操作，满足O(1)空间要求；
  • 时间效率：仅遍历数组一次，时间复杂度为O(n)；
  • 跨语言通用：逻辑不依赖编程语言，Python/Java/C++等均可复用；
  • 逻辑清晰：两个指针各司其职，代码易读、易维护。

3. 适用场景拓展

双指针法不仅适用于数组的原地修改，还可拓展到：

  • 有序数组的两数之和（左右指针法）；
  • 链表的遍历、反转、找中间节点；
  • 字符串的反转、去重、回文判断；
  • 数组的快慢指针（如寻找重复数、环形数组检测）。
