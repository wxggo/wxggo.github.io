Leetcode学习第1课

Created2024-09-30|Updated2024-10-01|学习笔记
|Word Count:2k|Reading Time:6mins|Post Views:

在LeetCode的数组类题目中，“原地修改数组”是一类经典考点，核心要求是不使用额外空间（O(1) 空间复杂度）完成数组元素的筛选与重构。其中，双指针法是解决这类问题的核心技巧。本文将通过两道经典题目——「删除有序数组中的重复项（LeetCode 26）」和「移除元素（LeetCode 27）」，拆解双指针法的应用逻辑，帮助大家掌握这一高频解题思路。

一、核心背景：为什么选择双指针？

数组的“原地修改”要求我们不能新建数组存储结果，只能在原数组上覆盖操作。双指针法通过两个指针分别承担“扫描遍历”和“记录赋值”的角色，一个指针负责遍历数组寻找符合条件的元素，另一个指针负责记录需要保留的元素位置，既能满足空间复杂度要求，又能保证时间复杂度为O(n)（仅遍历数组一次），是这类问题的最优解。

二、实战解析：LeetCode 26 - 删除有序数组中的重复项

1. 题目描述

给你一个升序排列的数组 nums，请你原地删除重复出现的元素，使每个元素只出现一次，返回删除后数组的新长度。要求：

  • 元素的相对顺序保持一致；
  • 结果需存放在数组 nums 的前 k 个位置（k 为新长度）；
  • 仅使用 O(1) 额外空间。

示例：

  • 输入：nums = [1,1,2]，输出：2，且 nums 前两位变为 [1,2]；
  • 输入：nums = [0,0,1,1,1,2,2,3,3,4]，输出：5，且 nums 前五位变为 [0,1,2,3,4]。

2. 解题思路：双指针的“扫描+记录”逻辑

由于数组是升序排列的，重复元素必然连续出现，因此我们可以定义两个指针：

  • 扫描指针（sm）：遍历数组，检查当前位置的元素是否为重复项，初始值为1（因为数组至少有1个元素时，第一个元素必然保留）；
  • 记录指针（jl）：标记下一个“非重复元素”要填入的位置，初始值也为1。

核心逻辑：

  1. 扫描指针逐个遍历数组，判断当前元素是否与前一个元素重复（nums[sm] != nums[sm-1]）；
  2. 若不重复，说明该元素需要保留，将其赋值到记录指针的位置，然后记录指针右移；
  3. 无论是否重复，扫描指针始终右移（持续遍历）；
  4. 遍历结束后，记录指针的数值即为新数组的长度。

3. 代码实现（Python）

4. 代码解析

  • 边界处理：空数组直接返回0，避免后续遍历出错；
  • 双指针初始化：从下标1开始，因为第一个元素无需判断，必然保留；
  • 循环遍历：扫描指针覆盖整个数组，仅在找到非重复元素时更新记录指针；
  • 空间复杂度：仅使用了几个变量，满足O(1)要求；
  • 时间复杂度：仅遍历数组一次，时间复杂度为O(n)。

三、实战解析：LeetCode 27 - 移除元素

1. 题目描述

给你一个数组 nums 和一个值 val，请你原地移除所有数值等于 val 的元素，返回移除后数组的新长度。要求：

  • 仅使用 O(1) 额外空间；
  • 元素顺序可改变，无需考虑超出新长度的元素。

示例：

  • 输入：nums = [3,2,2,3], val = 3，输出：2，且 nums 前两位变为 [2,2]；
  • 输入：nums = [0,1,2,2,3,0,4,2], val = 2，输出：5，且 nums 前五位为 [0,1,4,0,3]（顺序可调整）。

2. 解题思路：双指针的“筛选”逻辑

与LeetCode 26的核心思路一致，仍使用“扫描+记录”双指针，但判断条件调整为“元素是否等于目标值”：

  • 扫描指针（sm）：遍历数组，检查当前元素是否等于 val，初始值为0；
  • 记录指针（jl）：标记下一个“非val元素”要填入的位置，初始值也为0。

核心逻辑：

  1. 扫描指针遍历数组，若当前元素不等于 val，说明该元素需要保留；
  2. 将扫描指针指向的元素赋值到记录指针位置，记录指针右移；
  3. 若当前元素等于 val，扫描指针右移，记录指针不动；
  4. 遍历结束后，记录指针的数值即为新数组长度。

3. 代码实现（Python）

4. 代码解析

  • 双指针初始化：从下标0开始，因为需要从第一个元素开始筛选；
  • 循环优化：使用for循环替代while，简化扫描指针的遍历逻辑；
  • 核心判断：仅保留不等于val的元素，直接覆盖到记录指针位置；
  • 复杂度：时间O(n)，空间O(1)，满足题目要求。

四、同类题目对比与总结

1. 两道题的异同

维度 LeetCode 26（删除重复项） LeetCode 27（移除元素）
指针初始值 均为1（第一个元素保留） 均为0（从第一个元素筛选）
判断条件 nums[sm] != nums[sm-1] nums[sm] != val
数组特性 升序、重复元素连续 无顺序要求、目标值离散
核心逻辑 去重（保留唯一元素） 筛选（保留非目标元素）

2. 双指针法的核心思想

无论是“去重”还是“筛选”，双指针法的核心都是将“遍历”和“赋值”解耦：

  • 扫描指针：负责“找”——遍历数组，找到符合条件的元素；
  • 记录指针：负责“存”——标记需要保留的元素位置，接收符合条件的元素。

这种思路的优势在于：

  • 空间效率：无需额外数组，仅在原数组操作，满足O(1)空间要求；
  • 时间效率：仅遍历数组一次，时间复杂度为O(n)；
  • 逻辑清晰：两个指针各司其职，代码易读、易维护。

3. 适用场景拓展

双指针法不仅适用于数组的原地修改，还可拓展到：

  • 数组的两数之和（有序数组）；
  • 链表的遍历与反转；
  • 字符串的反转、去重等场景。

五、总结

通过两道经典的LeetCode数组题目，详细拆解了双指针法在“原地修改数组”场景下的应用。核心是理解“扫描指针”和“记录指针”的分工，根据题目要求调整判断条件和指针初始值，即可高效解决这类问题。

刷题的核心不是“背代码”，而是理解思路的本质。建议大家尝试修改代码（比如用while循环实现LeetCode 27，或用for循环实现LeetCode 26），加深对双指针逻辑的理解。掌握这一技巧后，面对同类的数组、链表题目，就能快速找到解题思路。
