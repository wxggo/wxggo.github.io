Notes-课程实验复盘:基于SCARA结构的小型写字机器人设计与控制

发表于2026-04-30|更新于2026-04-30|项目回顾
|字数总计:4.9k|阅读时长:16分钟|阅读量:

本文是对之前完成的小型写字机器人实验的一次整理复盘。项目目标是设计并制作一台能够在 A4 纸上完成写字、画圆和画矩形的小型机器人。整体结构参考 SCARA 机械臂形式，使用两个转动关节完成平面定位，再通过一个竖直方向的移动机构实现抬笔和落笔。本人在项目中主要负责软件控制部分，包括 Python 上位机 UI、机械臂逆运动学计算、坐标点到舵机 PWM 的转换、串口通信以及 STM32 下位机控制逻辑调试。

一、前言

这次实验的任务看起来很直接：让机器人在纸上写字、画圆、画矩形。
但真正开始做之后才发现，它并不是简单地“让舵机转几下”就能完成。写字机器人本质上是一个小型机械臂系统，需要同时解决机械结构、运动学建模、轨迹规划、上下位机通信、舵机控制和误差修正等问题。
相比三轴直角坐标平台，我们最后选择了类似 SCARA 的机械臂结构。它的优点是结构紧凑、成本较低，也更符合机器人课程中对机械臂运动学的训练要求；缺点是控制上比直角坐标机构复杂，需要进行逆运动学解算，把平面坐标转换成两个舵机的转角。

整个项目中，我主要关注的问题是：

  1. 如何把用户在上位机中绘制的轨迹转换为机器人末端轨迹；
  2. 如何根据末端坐标计算两个机械臂舵机的目标角度；
  3. 如何把角度进一步转换成 PWM 占空比；
  4. 如何通过串口把轨迹点稳定发送给 STM32；
  5. 如何让下位机控制舵机平稳运动，而不是生硬跳变。

二、实验目标与系统方案

2.1 实验任务

本次写字机器人需要完成以下功能：

  1. 在普通 A4 纸上绘制 100mm × 100mm 的正方形；
  2. 在普通 A4 纸上绘制直径 100mm 的圆；
  3. 在普通 A4 纸上书写“不忘初心”；
  4. 能够完成基本的抬笔、落笔动作；
  5. 机器人整体尺寸和结构不限，但需要具备基本的机械运动能力。

任务虽然简单，但它覆盖了一个小型机器人系统的完整闭环：

三、机械结构方案简述

3.1 为什么选择SCARA结构

写字任务主要发生在二维平面内，同时还需要一个竖直方向的抬笔动作。因此机器人至少需要三个自由度：

  1. X-Y 平面内定位；
  2. Z 方向抬笔和落笔。

常见方案有两种：

方案 特点
三轴直角坐标机构 控制简单，结构较大
两转动 + 一升降机构 结构紧凑，但控制复杂

我们最终选择第二种方案，即简化版 SCARA 机械臂：

  • 两个转动关节负责平面定位；
  • 一个竖直方向移动副负责抬笔落笔；
  • 末端夹持签字笔、铅笔或毛笔。

这种结构和人的手臂比较类似，机械上更有挑战性，软件上也需要做逆运动学计算。

3.2 结构设计中的一些问题

机械结构主要由 3D 打印件、舵机、连杆、滑块和笔夹组成。为了降低质量、提高刚度，连杆部分参考了工字梁结构。

实际制作中遇到的问题主要有：

  1. 3D打印件精度有限
    舵机安装孔、螺纹孔和滑块配合位置都会出现偏差，因此后续结构中需要给安装位置留出一定调整余量。

  2. 机械臂末端下垂明显
    由于机械臂较长，且整体由 PLA 打印件组成，当机械臂伸直时末端下垂会更加明显。

  3. 抬笔落笔机构阻力较大
    初始设计中滑动结构摩擦较大，后来改用铝制滑块，减小阻力后效果明显改善。

  4. 毛笔和铅笔效果差异很大
    毛笔笔尖较软，对机械误差容忍度更高；铅笔较硬，对末端高度和轨迹误差更敏感。

7.2 机械臂刚度不足

由于机械臂主要由 3D 打印件组成，末端下垂比较明显。尤其是在机械臂伸直时，笔尖高度会比收回时更低，导致有些区域笔迹过重，有些区域笔尖离纸。

我们尝试过加支撑杆和万向球，但发现支撑杆会带来摩擦和拖拽，反而影响轨迹精度，最后放弃了这个方案。

最终采用的处理方式比较工程化：

  1. 适当垫高纸面；
  2. 让纸面形成一定倾角；
  3. 对特定轨迹加入坐标补偿；
  4. 调整落笔 PWM，使笔尖压力适中。

7.3 矩形边缘倾斜修正

在绘制正方形时，我们发现某一条横线会出现明显向左下方倾斜的问题。原因包括：

  1. 机械臂末端下垂；
  2. 连杆刚度不足；
  3. 舵机间隙；
  4. 笔尖与纸面摩擦；
  5. 坐标映射误差。

八、最终实现效果

最终写字机器人实现了以下功能：

  1. 能够在 A4 纸范围内绘制正方形；
  2. 能够绘制指定直径的圆；
  3. 能够通过自由绘制功能书写简单文字；
  4. Python 上位机可以采集轨迹点并发送给 STM32；
  5. STM32 可以解析数据并控制三个舵机；
  6. 末端机构可以完成基本抬笔和落笔；
  7. 通过补偿修正后，图形偏斜问题有所改善。

当然，最终效果并不是工业级写字机的水平。由于舵机间隙、机械臂刚度、3D 打印精度和笔尖摩擦等问题，笔迹仍然存在一定误差。但作为课程实验，它已经完整跑通了从建模、控制到实物调试的流程。

九、个人收获

这次实验对我最大的意义，是让我真正理解了“软件控制不是只写程序”。

在这个项目里，Python 程序、STM32 程序、舵机、机械结构、纸面高度和笔尖摩擦全部会影响最终效果。很多时候，程序逻辑没有错，但写出来的字就是歪的；这时不能只盯着代码，还要回到机械结构和实际运动过程里找原因。

我的主要收获有以下几点。

1. 逆运动学必须结合实际安装修正

公式推导出来的角度只是理论值。舵机安装方向、零点位置、连杆长度误差、基座位置偏移都会影响最终结果。因此程序中必须留出修正量。

2. 上位机可以承担更多计算任务

STM32F103 算力有限，不适合处理太多浮点计算。把逆运动学、PWM转换、轨迹生成放到 Python 上位机中，可以让下位机逻辑更简单、更稳定。

3. 数据格式越简单越可靠

一开始如果用字符串传输，看似直观，但解析复杂，也容易出错。后来使用固定 6 字节数据包，每个舵机两个字节，反而更稳定。

4. 机械误差需要软件补偿

机械臂下垂、舵机虚位、笔尖摩擦都无法完全避免。通过坐标补偿、抬笔缓冲、速度插补等方式，可以在软件层面改善最终效果。

5. 调试效率很重要

写字机器人每次运行都需要等待机械臂实际动作。如果调试流程复杂，效率会很低。因此上位机 UI、预设圆形/矩形轨迹、串口自动发送等功能都非常必要。

十、总结

这次小型写字机器人实验，是一次从零开始完成小型机器人系统的过程。

从机械结构上，我们选择了简化 SCARA 结构，用两个转动关节完成平面定位，用一个升降机构实现抬笔落笔；从软件控制上，我们使用 Python 上位机完成轨迹采集、逆运动学计算和 PWM 数据生成，再通过串口发送给 STM32，下位机负责解析数据并控制舵机运动。

如果用一句话总结这个项目：

写字机器人看起来是在纸上画线，实际上是在把机械结构、运动学、通信协议和嵌入式控制全部串起来。

后续如果继续优化，我认为可以从以下方向入手：

  1. 使用精度更高、间隙更小的舵机或步进电机；
  2. 对机械臂末端下垂建立更准确的补偿模型；
  3. 将轨迹点做平滑处理，减少舵机突变；
  4. 加入更完整的轨迹规划，而不是简单点到点运动；
  5. 优化抬笔落笔机构，提高笔尖压力稳定性；
  6. 增加字体矢量解析，实现真正的文字路径生成。

虽然最终作品还有不少粗糙的地方，但整个过程让我对机器人控制系统有了更完整的认识。一个能写字的小机器人，背后其实藏着很多工程细节。
