6、扩展思考

这道题看起来简单，但有几个细节很容易出错。

第一，不能忘记处理最后的进位。

例如：

如果循环条件没有加上 carry != 0，最后的最高位 1 就会丢失。

第二，两个字符串长度可能不同。

所以当某个指针已经越界时，应该把这一位当作 0 来处理，而不是继续访问字符串。

第三，结果是反向构造的。

如果从右往左计算，并且每次都 append 到 StringBuilder 后面，那么最后一定要 reverse()。

这道题也可以扩展到类似问题，比如：

  • 字符串相乘；
  • 二进制字符串相加；
  • 链表表示的两数相加；
  • 大数加法模板。

我对这道题的理解是：它重点不是算法有多难，而是训练我们把日常计算规则准确翻译成代码。面试中这类题很常见，写的时候一定要注意边界、进位和字符串下标。

总结

Leetcode 415「字符串相加」的核心思路是：

从右往左逐位相加，用 carry 记录进位，最后反转结果字符串。

这道题可以作为字符串模拟题的基础模板。只要理解了“逐位计算 + 进位处理”，后续处理大数加法、二进制加法、链表加法都会更顺手。
