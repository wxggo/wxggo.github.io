Leetcode-1143:最长公共子序列-二维动态规划

发表于2026-06-07|更新于2026-06-07|Java学习笔记Leetcode自由刷
|字数总计:3k|阅读时长:10分钟|阅读量:

最近复习动态规划时，我发现「最长公共子序列」是一道非常适合作为 DP 入门和面试复盘的题。它的难点不在代码很长，而在于能不能清楚定义状态、理解下标偏移，并把“当前字符相等/不相等”两种情况讲明白。

1、题目

题目链接：LeetCode 1143. 最长公共子序列

给定两个字符串 text1 和 text2，返回这两个字符串的最长公共子序列的长度。

子序列是指：在不改变原字符串字符相对顺序的情况下，删除某些字符（也可以不删除）后得到的新字符串。

需要注意：

  • 子序列不要求连续；
  • 子串要求连续；
  • 本题求的是两个字符串共同拥有的最长子序列长度。

2、分析

这道题是经典的二维动态规划问题，也就是 Longest Common Subsequence，简称 LCS。

如果直接暴力枚举两个字符串的所有子序列，再判断是否相同，复杂度会非常高，因为一个长度为 $n$ 的字符串最多有 $2^n$ 个子序列。因此，本题更适合用动态规划来避免重复计算。

2.1 状态定义

设：

$$
dp[i][j]
$$

表示：text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列长度。

这里的“前 i 个字符”不是下标 i，而是长度为 i 的前缀：

  • i = 0 表示 text1 的空字符串；
  • i = 1 表示 text1 的第一个字符，也就是 text1.charAt(0)；
  • j 同理。

这也是本题最容易写错的地方之一。

2.2 初始化

当其中一个字符串为空时，最长公共子序列长度一定为 0：

$$
dp[0][j] = 0
$$

$$
dp[i][0] = 0
$$

在 Java 中，int 类型二维数组默认初始化为 0，因此不需要手动赋值。

2.3 状态转移

遍历到 text1 的第 i 个字符和 text2 的第 j 个字符时，有两种情况。

情况一：当前字符相等

如果：

说明这两个字符可以共同加入公共子序列，那么答案就来自两个字符串都去掉当前字符之前的状态，再加上当前这个匹配字符：

$$
dp[i][j] = dp[i - 1][j - 1] + 1
$$

例如 abcde 和 ace 在比较到字符 e 时，两个字符相等，就可以在之前 abc d 与 ac 的 LCS 基础上再加 1。

情况二：当前字符不相等

如果当前字符不相等，那么这两个字符不能同时作为公共子序列的最后一个字符。此时需要做选择：

  1. 不使用 text1 当前字符，看 dp[i - 1][j]；
  2. 不使用 text2 当前字符，看 dp[i][j - 1]。

取两者最大值：

$$
dp[i][j] = \max(dp[i - 1][j], dp[i][j - 1])
$$

这一步的本质是：当前两个字符匹配不上，那就分别尝试丢掉其中一个字符，看哪种选择能保留更长的公共子序列。

2.4 为什么最后返回 dp[m][n]

因为 dp[i][j] 表示两个前缀的最长公共子序列长度，所以当 i = m、j = n 时，刚好表示完整的 text1 和完整的 text2 的答案。

3、程序

3.1 关键代码解释

1）为什么数组大小是 m + 1 和 n + 1

因为我们需要额外保留一行和一列来表示空字符串状态。

如果不加这一行一列，就需要对 i = 0 或 j = 0 的情况写很多边界判断。使用 m + 1 和 n + 1 后，状态转移会更统一。

2）为什么比较 charAt(i - 1) 和 charAt(j - 1)

因为 dp[i][j] 中的 i 和 j 表示长度，不是字符串下标。

3）为什么字符相等时来自 dp[i - 1][j - 1]

因为当前两个字符已经匹配上了，就相当于从两个字符串中都拿掉当前字符，看前面的最长公共子序列长度，再加上当前这一个公共字符。

4）为什么字符不相等时不考虑 dp[i - 1][j - 1]

当当前字符不相等时，不能同时把它们加入同一个公共子序列。此时最优解一定来自：

  • 去掉 text1 当前字符；
  • 或者去掉 text2 当前字符。

因此只需要比较 dp[i - 1][j] 和 dp[i][j - 1]。

4、算法性能

设 m = text1.length()，n = text2.length()。

指标 复杂度 说明
时间复杂度 $O(mn)$ 需要填完整个二维 DP 表
空间复杂度 $O(mn)$ 使用了大小为 (m + 1) * (n + 1) 的二维数组

这份代码的优点是逻辑清晰，非常适合第一次学习和面试时讲解。

如果进一步优化空间，可以使用一维滚动数组，将空间复杂度优化到 $O(n)$。不过一维写法对遍历顺序和临时变量要求更高，第一次学习时我更推荐先掌握二维版本。

6、扩展思考

6.1 和最长公共子串的区别

最长公共子序列不要求字符连续，因此当字符不相等时，可以继承前面状态的最大值：

但如果是最长公共子串，字符必须连续。一旦当前字符不相等，当前连续长度就会断掉，通常需要置为 0。

所以这两类题的 DP 转移差别很大，面试时一定要先确认题目要求的是“子序列”还是“子串”。

6.2 空间优化思路

二维 DP 中，dp[i][j] 只依赖：

  • dp[i - 1][j]：上一行同一列；
  • dp[i][j - 1]：当前行前一列；
  • dp[i - 1][j - 1]：上一行前一列。

因此可以使用一维数组保存上一行状态，再用一个变量记录左上角的值。

6.3 常见易错点

易错点 说明
把子序列误认为子串 子序列不要求连续，子串要求连续
dp 下标和字符串下标混淆 dp[i][j] 的 i/j 是长度，字符串访问要 i - 1/j - 1
忘记空字符串状态 使用 m + 1 和 n + 1 可以统一处理边界
字符不相等时转移写错 应该取 dp[i - 1][j] 和 dp[i][j - 1] 的最大值
只背代码，不会解释状态含义 面试时重点是讲清楚为什么这样转移

6.4 面试口述版

如果面试中让我讲这道题，我会这样回答：

这道题可以用二维动态规划。定义 dp[i][j] 表示 text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列长度。初始化时，如果任意一个字符串为空，结果就是 0，所以第一行和第一列都是 0。然后遍历两个字符串，如果当前字符相等，就说明这个字符可以加入公共子序列，状态转移为 dp[i][j] = dp[i - 1][j - 1] + 1；如果当前字符不相等，就分别尝试丢掉其中一个当前字符，取 dp[i - 1][j] 和 dp[i][j - 1] 的最大值。最后返回 dp[m][n]。时间复杂度是 $O(mn)$，空间复杂度是 $O(mn)$。

6.5 相关题目

这道题是很多字符串 DP 问题的基础，后续可以继续复习：

  • 最长公共子串；
  • 编辑距离；
  • 不同的子序列；
  • 两个字符串的删除操作；
  • 判断一个字符串是否是另一个字符串的子序列。

这些题都和“前缀状态”“字符是否匹配”“删除或保留字符”有关，可以放在一起做专题复盘。

总结

这道题给我最大的收获是：动态规划题目并不是一上来就写转移方程，而是要先把状态定义想清楚。

对于最长公共子序列：

  1. dp[i][j] 表示两个前缀的 LCS 长度；
  2. 多开一行一列表示空字符串，方便处理边界；
  3. 当前字符相等时，来自左上角加 1；
  4. 当前字符不相等时，来自上方或左方最大值；
  5. 最终返回 dp[m][n]。

这道题非常适合用来训练二维 DP 的基本套路。只要能把状态定义、初始化、转移逻辑和复杂度讲清楚，面试中遇到同类字符串 DP 问题时就会更有底气。
