6.3 常见易错点

易错点 说明
忘记使用 dummy 当 left = 1 时，头节点会变化，不用 dummy 容易写出很多特殊判断
p0 定位错误 p0 应该指向反转区间的前一个节点，而不是 left 节点本身
反转次数写错 区间长度是 right - left + 1，不是 right - left
忘记保存 next 修改 cur.next 前必须保存后继节点，否则会断链
接链顺序理解不清 反转后 p0.next 仍然是原区间头节点，也就是反转后的尾节点

6.4 面试口述版

如果面试中让我讲这道题，我会这样回答：

这道题可以用一次遍历加局部链表反转来做。首先创建一个虚拟头节点 dummy，这样可以统一处理 left = 1 的情况。然后找到反转区间前一个节点 p0。接着从 p0.next 开始，按照普通反转链表的方式反转 right - left + 1 个节点。反转完成后，pre 指向反转区间的新头节点，cur 指向反转区间后面的第一个节点，而 p0.next 仍然指向原区间头节点，也就是反转后的尾节点。最后先让 p0.next.next = cur 接上后半段，再让 p0.next = pre 接上反转后的区间头节点。最终返回 dummy.next。时间复杂度是 $O(n)$，空间复杂度是 $O(1)$。

总结

这道题给我最大的启发是：链表题的难点通常不在算法思想很复杂，而在于指针含义要足够清楚。

对于「反转链表 II」，我需要重点记住：

  1. 使用 dummy 统一处理头节点变化；
  2. p0 表示反转区间前一个节点；
  3. 从 p0.next 开始反转固定长度的节点；
  4. 反转后 pre 是新区间头，cur 是后半段头；
  5. p0.next 是原区间头，也就是反转后的尾节点；
  6. 最后通过两句代码完成重新连接。

这道题非常适合用来复习链表反转模板，也能很好地训练面试中对指针过程的表达能力。
