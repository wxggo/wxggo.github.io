Leetcode-142:环形链表II-快慢指针寻找入环节点

发表于2026-06-05|更新于2026-06-05|Java学习笔记Leetcode自由刷
|字数总计:1.9k|阅读时长:6分钟|阅读量:

本文用于链表双指针题型复习。题目是 LeetCode 142「环形链表 II」，核心方法是 Floyd 快慢指针算法：先判断链表是否存在环，再利用数学关系找到环的入口节点。

这道题非常适合面试复习，因为它不是单纯会写 slow、fast 就结束了，更重要的是能解释清楚：为什么相遇以后，把一个指针放回头节点，两个指针同速走，再次相遇的位置就是入环点。

1、题目

给定一个链表的头节点 head，判断链表中是否有环。如果链表中存在环，则返回链表开始入环的第一个节点；如果链表无环，则返回 null。

注意：

  • 不允许修改链表；
  • 需要返回的是入环节点，不是判断是否有环；
  • 最好使用 O(1) 额外空间。

示例理解：

这个链表存在环，入环节点是值为 2 的节点。

2、分析

2.1 问题核心

本题拆解为两个核心子问题：

  1. 判断链表是否有环
  2. 找到环的入口节点
  • 判环：快慢指针法，slow 每次走1步，fast 每次走2步；有环则必相遇，无环则fast先走到null。
  • 找入口：快慢指针首次相遇点≠入环点，需二次同步移动指针求解。

2.3 核心推导（极简版）

定义：

  • $a$：head 到入环点的步数
  • $n$：快指针比慢指针多绕的环数
  • $s$：慢指针总路程，$f$：快指针总路程
  1. 速度关系：快指针速度是慢指针2倍 → $\boldsymbol{fast=2slow}$
  2. 路程关系：相遇时快指针多走 $n$ 圈 → $\boldsymbol{fast = slow + nb}$
  3. 联立公式：推出 $\boldsymbol{slow = nb}$（慢指针总路程为 $n$ 倍环长）
  4. 关键结论：从 head 走到入环点，需要走 $\boldsymbol{a + nb}$ 步
  5. 最终逻辑：慢指针已走 $nb$ 步，再走 $a$ 步就到达入环点；
    让指针从 head 出发，和慢指针同步走1步，相遇时刚好走了 $a$ 步，即为入环点。

4、算法性能

  • 时间复杂度：$O(n)$
    快慢指针最多遍历链表有限次数。即使存在环，指针也只是在环内走若干步后相遇，整体仍然是线性复杂度。

  • 空间复杂度：$O(1)$
    只使用了 slow 和 fast 两个指针，没有使用哈希表等额外数据结构。

核心优势：

  1. 不需要修改链表结构；
  2. 不需要额外集合存储访问过的节点；
  3. 可以同时判断是否有环并找到入环点；
  4. 面试中能体现对链表指针和数学关系的理解。

6、扩展思考

6.1 和 LeetCode 141 的区别

LeetCode 141「环形链表」只需要判断链表是否有环，返回 true 或 false。
LeetCode 142「环形链表 II」不仅要判断有环，还要返回入环节点。

题目 目标 是否需要第二阶段
LeetCode 141 判断是否有环 不需要
LeetCode 142 找到入环节点 需要

所以 142 的关键不是快慢指针相遇，而是相遇后的入口定位。

6.2 能不能用哈希表

可以。

思路是遍历链表，把访问过的节点放入 HashSet。如果某个节点第二次出现，它就是入环节点。

这种方法更直观，但空间复杂度是 $O(n)$。
快慢指针方法空间复杂度是 $O(1)$，所以更符合本题的进阶要求。

6.3 面试时怎么讲

可以这样回答：

我先用快慢指针判断链表是否有环，慢指针每次走一步，快指针每次走两步。如果存在环，快慢指针一定会在环内相遇。相遇后，将一个指针重新放回头节点，另一个指针留在相遇点，两个指针每次都走一步。根据距离关系，头节点到入环点的距离等于相遇点到入环点的距离加若干圈环长，所以它们再次相遇的位置就是入环节点。

这个回答既说明了实现流程，也解释了为什么算法正确。

7、总结

这道题的核心是 Floyd 快慢指针算法。

解题流程可以记成：

我觉得这道题最值得复习的点有两个：

  1. 边界判断要稳：while (fast != null && fast.next != null) 防止空指针；
  2. 原理要讲清楚：通过 $a=(n-1)(b+c)+c$ 解释为什么第二次相遇就是入环点。

如果只是背代码，面试时容易卡在“为什么这样能找到入口”。把距离关系推一遍后，这道题就从模板题变成了真正理解的链表题。
