Leetcode-088:合并两个有序数组-从后往前双指针原地合并

1、题目

给你两个按非递减顺序排列的整数数组 nums1 和 nums2，另有两个整数 m 和 n，分别表示 nums1 和 nums2 中的有效元素个数。

请你将 nums2 合并到 nums1 中，使合并后的数组同样按非递减顺序排列。

注意：

  • 最终合并后的数组不应由函数返回，而是存储在数组 nums1 中；
  • 为了容纳 nums2 中的元素，nums1 的长度为 m + n；
  • nums1 中前 m 个元素表示需要参与合并的有效元素；
  • nums1 后 n 个元素为 0，只是占位，不参与合并；
  • nums2 的长度为 n。

2、分析

2.1 问题核心

这道题的核心并不是简单地“把两个数组合并后排序”，因为题目要求我们直接在 nums1 原数组中完成合并。

如果使用最直接的做法，可以先把 nums2 放到 nums1 后面，然后整体排序：

这种方法虽然简单，但是没有利用两个数组本身已经有序的特点，时间复杂度也会变成排序的复杂度。

更好的方法是使用双指针。

由于 nums1 和 nums2 本身都是有序数组，我们只需要不断比较两个数组当前最大的元素，然后把较大的元素放到 nums1 的末尾即可。

2.2 为什么要从后往前合并？

这道题最容易想到的是从前往后合并，但这样会遇到一个问题：

如果从前往后把较小元素写入 nums1，很可能会覆盖掉 nums1 中还没有处理的有效元素。

而 nums1 的后半部分本来就是预留空间，所以我们可以反过来思考：

从两个数组的有效末尾开始比较，把更大的数放到 nums1 的最后面。

这样就不会覆盖还没处理的元素。

2.3 核心思路

具体步骤如下：

  1. 定义三个指针：

    • p1 = m - 1，指向 nums1 中有效元素的最后一个位置；
    • p2 = n - 1，指向 nums2 的最后一个位置；
    • p = m + n - 1，指向 nums1 最终合并数组的最后一个位置。

  2. 当 p2 >= 0 时持续合并：

    • 如果 p1 >= 0 且 nums1[p1] > nums2[p2]，说明 nums1[p1] 更大，应放到 nums1[p]；
    • 否则，将 nums2[p2] 放到 nums1[p]。

  3. 每放入一个元素，就将对应指针和写入指针左移。

  4. 当 p2 < 0 时，说明 nums2 已经全部合并完成，算法结束。

这里循环条件只需要判断 p2 >= 0，原因是：

  • 如果 nums2 合并完了，剩下的 nums1 元素本来就在正确位置；
  • 如果 nums1 有效元素先合并完了，也就是 p1 < 0，那就继续把 nums2 剩余元素依次放入 nums1 前面。

4、算法性能

  • 时间复杂度：O(m + n)
    每个元素最多被比较和移动一次，因此整体时间复杂度为两个数组长度之和。

  • 空间复杂度：O(1)
    只使用了三个额外指针变量，没有申请新的数组空间，属于原地合并。

  • 核心优势：

    1. 利用两个数组已有序的特点，不需要重新排序；
    2. 利用 nums1 尾部预留空间，避免覆盖有效元素；
    3. 原地修改 nums1，满足题目要求；
    4. 代码短，边界情况处理自然。

7、总结

这道题是数组双指针中非常经典的一题，难点不在于合并两个有序数组本身，而在于如何在 nums1 原数组中完成合并，并且不覆盖还没处理的有效元素。

如果用一句话总结解法：

从后往前比较两个数组当前最大值，把更大的元素放到 nums1 的最后一个空位。

这道题给我的启发是：
当题目给了一个数组尾部的预留空间时，往往可以考虑从后往前处理。这样既能避免元素覆盖，又能做到原地操作。

以后遇到类似“原地合并”“原地移动”“数组后面有空位”的题目时，可以优先思考：

  1. 是否可以从后往前遍历；
  2. 是否可以用多个指针分别指向有效区间和写入位置；
  3. 是否可以避免额外数组空间；
  4. 剩余元素是否真的需要继续处理。

这类思路在数组题中非常常见，掌握以后可以解决很多原地操作问题。
