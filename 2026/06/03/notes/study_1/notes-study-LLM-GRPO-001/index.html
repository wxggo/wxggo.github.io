Notes-大模型学习:PPO、GRPO 与 DQN 的区别-从控制任务到大模型强化学习

发表于2026-06-03|更新于2026-06-03|学习笔记AI大模型
|字数总计:7.2k|阅读时长:25分钟|阅读量:

本文用于强化学习算法与大模型对齐训练复习。我主要整理 PPO、GRPO、DPO、DQN 四类方法的区别，并回答几个容易混淆的问题：大模型最常用的强化学习算法是什么？为什么不是 DQN？DPO 算不算强化学习？GRPO 相比 PPO 改了什么？控制领域的任务分配、路径规划强化学习，和 LLM 领域的强化学习到底差在哪里？

这篇笔记会尽量把公式、流程和直觉放在一起。因为只背算法名字很容易混乱，真正需要抓住的是：状态是什么、动作是什么、奖励怎么来、策略怎么更新、训练时是否需要在线采样。

一、先给结论：四个算法分别适合什么

如果先不展开公式，可以这样粗略理解：

算法 类型 核心思想 在 LLM 中的典型用途 是否需要在线采样
DQN Value-based RL 学习 $Q(s,a)$，选择价值最高的动作 一般不直接用于 LLM 对齐 需要与环境交互
PPO Policy-based / Actor-Critic 限制策略更新幅度，稳定优化策略 经典 RLHF 主流算法 需要模型采样 response
GRPO PPO 变体 去掉 critic，用同一 prompt 的多组输出做相对优势估计 推理模型、数学/代码强化学习常见 需要模型采样 response
DPO Preference optimization 用偏好数据直接优化策略，不显式训练 reward model 和 RL loop 轻量级偏好对齐常用 通常不需要在线采样

一句话版本：

DQN 适合动作空间相对明确的控制问题；PPO 是经典 RLHF 中最常见的策略优化算法；GRPO 是更适合大模型推理强化学习的一类 PPO 变体；DPO 则是把偏好对齐转成监督式分类损失的“非在线 RL”方法。

二、强化学习的基本框架

强化学习一般可以抽象成一个智能体与环境交互的过程：

核心变量包括：

  • 状态 $s$ ：当前环境信息；
  • 动作 $a$ ：智能体能做出的选择；
  • 奖励 $r$ ：动作好坏的反馈；
  • 策略 $\pi(a|s)$ ：在状态 $s$ 下选择动作 $a$ 的概率；
  • 价值函数 $V(s)$ 或 $Q(s,a)$ ：估计未来累计回报；
  • 折扣因子 $\gamma$ ：控制未来奖励的重要程度。

目标一般是最大化期望累计回报:
$$ J(\theta)=\mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T}\gamma^t r_t\right] $$
对于控制领域，这个框架非常直观：机器人移动一步、无人机选择一个任务、车辆选择一段路径，环境就会反馈代价或奖励。
对于 LLM，框架会变得抽象一些：prompt 是状态，生成 token 或完整 response 是动作，奖励往往来自人类偏好模型、规则验证器或 AI 反馈。

三、DQN：从价值函数出发的深度强化学习

DQN，全称 Deep Q-Network，是 DeepMind 提出的经典深度强化学习算法，最著名的应用是 Atari 游戏。

3.1 DQN 的核心思想

DQN 要学习的是动作价值函数：

$$
Q(s,a)
$$

它表示在状态 $s$ 下执行动作 $a$，之后继续按照某种策略行动，最终能获得多少累计回报。

DQN 的动作选择通常是：

$$
a=\arg\max_a Q(s,a)
$$

也就是选择当前估计价值最高的动作。

3.2 Bellman 更新

DQN 的训练目标来自 Bellman 方程：

$$
y=r+\gamma \max_{a’}Q_{\text{target}}(s’,a’)
$$

损失函数为：

$$
L=(Q_{\theta}(s,a)-y)^2
$$

其中：

  • $Q_{\theta}$ 是当前 Q 网络；
  • $Q_{\text{target}}$ 是目标网络；
  • $r$ 是当前奖励；
  • $s’$ 是下一个状态；
  • $\gamma$ 是折扣因子。

Target Network 用一个较慢更新的目标网络计算 target，提升训练稳定性

3.4 为什么 DQN 不适合直接做 LLM 对齐

LLM 的动作空间太大了。
在 DQN 中，动作通常是有限离散动作，例如 Atari 游戏中的“上、下、左、右、开火”。但 LLM 生成时，动作至少是词表中的下一个 token，词表可能有几万到几十万个 token；如果把完整回答看成动作，动作空间更是指数级。

主要困难包括：

  1. 动作空间巨大：对每个状态枚举所有动作并估计 $Q(s,a)$ 很难；
  2. 序列长度长：一个回答由很多 token 组成，信用分配困难；
  3. 奖励稀疏：通常整个 response 结束后才有偏好奖励；
  4. 环境不明确：LLM 的“环境”不像游戏或机器人那样有稳定物理规则；
  5. 语言质量需要约束：不能只追求 reward，还要避免偏离原模型语言分布。

所以，LLM 对齐更常用直接优化策略的 policy gradient 方法，例如 PPO、GRPO，或者使用偏好数据直接训练的 DPO。

四、PPO：经典 RLHF 最常见的策略优化算法

PPO，全称 Proximal Policy Optimization，是 OpenAI 提出的经典策略优化算法。它的核心目标是：

让策略变好，但每次更新不要变得太猛。

4.1 为什么策略更新不能太猛

在强化学习中，如果新策略和旧策略差异太大，训练很容易不稳定。
比如模型一开始还能正常回答，优化几步后突然学会钻 reward model 的空子，输出奇怪但高分的内容，这就是 RLHF 中常见的风险。

PPO 通过 clipped objective 限制策略更新幅度：

$$
r_t(\theta)=\frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}
$$

$$
L^{CLIP}(\theta)=\mathbb{E}_t[\min(r_t(\theta)A_t, clip(r_t(\theta),1-\epsilon,1+\epsilon)A_t)]
$$

其中：

  • $r_t(\theta)$ 表示新旧策略概率比；
  • $A_t$ 是优势函数，表示这个动作比平均水平好多少；
  • $\epsilon$ 控制更新幅度；
  • clip 会防止新策略相对旧策略变化过大。

4.3 LLM 中的 PPO 奖励通常长什么样

LLM 对齐时的奖励不只是 reward model 分数，还会加上 KL 惩罚：

$$
R(x,y)=RM(x,y)-\beta KL(\pi_\theta(y|x)||\pi_{\text{ref}}(y|x))
$$

其中：

  • $RM(x,y)$ 表示奖励模型对回答的打分；
  • $\pi_\theta$ 是当前训练策略；
  • $\pi_{\text{ref}}$ 是参考模型，通常是 SFT 模型；
  • KL 惩罚用于约束当前模型不要偏离参考模型太远；
  • $\beta$ 控制约束强度。

这个 KL 项非常关键。
如果没有它，模型可能为了 reward model 分数而牺牲语言质量，甚至出现 reward hacking。

4.4 为什么 PPO 成为经典 LLM 强化学习算法

PPO 在 LLM 中常见，主要因为它有几个优点：

  1. 稳定性较好：相比普通 policy gradient，PPO 的 clipped objective 更稳；
  2. 适合大动作空间策略优化：不需要像 DQN 一样枚举所有动作；
  3. 可以自然加入 KL 约束：适合防止模型偏离 SFT 分布；
  4. 工程经验成熟：RLHF 早期成功案例很多基于 PPO；
  5. 能优化非可微奖励：人类偏好模型、规则打分、环境反馈都可以作为 reward。

但 PPO 的缺点也明显：

  1. 要训练 reward model；
  2. 要维护 actor、critic、reference model，有时还有 reward model；
  3. 显存和算力开销大；
  4. 训练不稳定，需要调 KL、学习率、采样长度等超参数；
  5. reward model 可能被策略模型利用。

五、GRPO：更适合大模型推理训练的 PPO 变体

GRPO，全称 Group Relative Policy Optimization，是 DeepSeekMath 中提出并推广的一类 PPO 变体。它可以理解为：

用同一个 prompt 下的一组回答做相对比较，从而估计 advantage，减少对 critic/value model 的依赖。

5.1 GRPO 为什么出现

PPO 在 LLM 中的问题之一是需要 critic。
critic 要估计 $V(s)$，也就是当前状态的未来价值。对于 LLM 来说，这意味着还要训练或维护一个价值模型，成本很高。

GRPO 的动机是：

能不能不用单独的 critic，而是让同一个问题生成多个回答，通过组内相对分数来判断哪个回答更好？

5.2 GRPO 的基本流程

对于同一个 prompt $q$，模型采样 $G$ 个回答：

$$
{o_1,o_2,\dots,o_G}
$$

然后用 reward model 或规则验证器给每个回答打分：

$$
{r_1,r_2,\dots,r_G}
$$

再计算组内相对 advantage：

$$
A_i=\frac{r_i-mean(r_1,\dots,r_G)}{std(r_1,\dots,r_G)}
$$

直观理解：

  • 如果某个回答比同组平均水平好，$A_i>0$；
  • 如果某个回答比同组平均水平差，$A_i<0$；
  • 模型会提高好回答的概率，降低差回答的概率。

5.3 GRPO 和 PPO 的区别

对比点 PPO GRPO
advantage 来源 通常依赖 critic/value model 依赖同一 prompt 的组内相对奖励
额外模型 actor、critic、reference、reward model 通常省去 critic
训练成本 较高 相对更低
适用场景 通用 RLHF 数学、代码、推理等可验证任务更常见
稳定性 成熟但调参复杂 减少 critic 开销，但依赖 group 采样质量

5.4 GRPO 为什么适合推理任务

数学、代码、逻辑推理任务有一个特点：
很多时候可以用规则验证器判断结果对不对，比如答案是否匹配、代码是否通过测试、数学结果是否正确。

这时 reward 不一定完全依赖人类偏好，而可以来自自动验证：

这非常适合训练 reasoning model。
相比传统 RLHF 的人类偏好奖励，推理任务奖励更明确，也更容易大规模自动化。

六、DPO：偏好优化，不是传统在线强化学习

DPO，全称 Direct Preference Optimization。它的思想很漂亮：

不显式训练 reward model，也不跑 PPO，而是直接用偏好对训练模型。

6.1 DPO 的数据形式

DPO 需要的数据通常是偏好对：

其中：

  • $y_w$ 是更好的回答；
  • $y_l$ 是更差的回答。

6.2 DPO 的损失函数

DPO 的目标可以写成：

$$
L_{DPO}=-\mathbb{E}[\log \sigma(\beta(\log \pi_\theta(y_w|x)-\log \pi_{ref}(y_w|x))-\beta(\log \pi_\theta(y_l|x)-\log \pi_{ref}(y_l|x)))]
$$

这个公式看起来长，但直觉很简单：

相对于参考模型，当前模型应该更倾向 chosen response，而不是 rejected response。

其中：

  • $\pi_\theta$ 是当前训练模型；
  • $\pi_{ref}$ 是参考模型；
  • $\beta$ 控制偏离参考模型的程度；
  • $\sigma$ 是 sigmoid 函数。

6.3 DPO 和 PPO-RLHF 的区别

对比点 PPO-RLHF DPO
是否训练 reward model 是 否
是否在线采样 是 通常否
是否需要 critic 通常需要 不需要
训练复杂度 高 低
稳定性 需要较多调参 相对稳定
本质 强化学习策略优化 偏好分类式优化

DPO 的优势是工程简单、稳定、成本较低，所以在很多开源模型偏好对齐中非常常见。

但要注意：

DPO 不等同于传统强化学习。它来自 RLHF 目标的重新参数化，可以看作偏好优化方法，而不是需要环境交互的在线 RL 算法。

6.4 DPO 的局限

DPO 也不是万能的：

  1. 依赖偏好数据质量；
  2. 不能像在线 RL 那样主动探索新输出；
  3. 对复杂长链推理任务，单纯 chosen/rejected 可能不够细；
  4. 如果偏好数据覆盖不足，模型提升会受限；
  5. 不适合需要实时环境反馈的控制任务。

七、大模型最常用的强化学习算法是什么

这个问题要分场景回答。

7.1 如果严格说“经典 RLHF 强化学习算法”：PPO 最经典、最常见

从 InstructGPT、ChatGPT 早期 RLHF 路线看，PPO 是最经典、影响最大的 LLM 强化学习算法。

原因是：

  1. 它直接优化策略模型；
  2. 能处理大规模随机策略；
  3. 可以结合 reward model；
  4. 可以用 KL 约束保持语言模型能力；
  5. 相比更早的 policy gradient 方法更稳定。

所以面试里如果问：

大模型 RLHF 最常用的强化学习算法是什么？

比较稳妥的回答是：

经典 RLHF 中最常用、最有代表性的算法是 PPO。它通过 clipped objective 限制策略更新幅度，并结合 reward model 与 KL 惩罚，使模型在优化人类偏好奖励的同时尽量不偏离原始语言模型。

7.2 如果说“当前开源偏好对齐训练”：DPO 也非常常见

很多开源模型并不一定跑完整 PPO-RLHF，而是使用 DPO 或其变体。原因是 DPO 更容易实现：

  1. 不需要训练独立 reward model；
  2. 不需要在线 rollout；
  3. 不需要 critic；
  4. 显存和调参成本低；
  5. 可以直接使用偏好数据。

但严格来说，DPO 更像“偏好优化”，不是传统意义上的在线强化学习。

7.3 如果说“推理模型强化学习”：GRPO 近几年很常见

在数学、代码、推理模型中，GRPO 这类方法越来越常见。原因是：

  1. 可以利用规则奖励或 verifier；
  2. 去掉 critic 后训练成本更低；
  3. 同一 prompt 多采样适合推理任务；
  4. 组内相对奖励可以减少 reward scale 问题；
  5. 更适合长 CoT、多答案探索的训练场景。

九、控制领域强化学习与 LLM 强化学习的区别

控制领域包括任务分配、路径规划、机器人控制、无人机调度等。它们和 LLM 的强化学习都叫 RL，但问题形态差异很大。

9.1 状态不同

领域 状态
控制领域 位置、速度、电量、地图、障碍物、任务队列、资源状态
LLM 领域 prompt、已生成 token、上下文、对话历史

控制领域的状态通常来自物理环境或仿真器，更接近连续空间或图结构。
LLM 的状态是文本上下文，本质上是离散 token 序列。

9.2 动作不同

领域 动作
路径规划 上下左右、速度、航向角、下一个路点
任务分配 给某个智能体分配某个任务
LLM 生成下一个 token，或生成完整 response

控制领域动作可以是低维连续动作，也可以是有限离散动作。
LLM 的 token 动作空间巨大，而且一个完整 response 是长序列动作。

9.3 奖励不同

领域 奖励来源
控制领域 距离、时间、能耗、碰撞、任务完成率、约束违背惩罚
LLM 领域 人类偏好、AI 偏好、reward model、规则验证器、安全约束

控制领域奖励通常更容易设计成明确数学指标。
LLM 的奖励更主观，特别是“有帮助、真实、安全、符合人类意图”这些标准很难完全公式化。

9.4 环境交互不同

控制领域一般有环境：

LLM 对齐里的“环境”很多时候不是物理环境，而是：

也就是说，LLM 的强化学习常常是在文本分布和偏好模型上优化，不一定有真实环境动态。

9.5 训练目标不同

控制领域更关注：

  1. 完成任务；
  2. 最短路径；
  3. 最小能耗；
  4. 避障安全；
  5. 多智能体协同；
  6. 实时决策。

LLM 更关注：

  1. 指令跟随；
  2. 回答有帮助；
  3. 减少有害输出；
  4. 提升推理正确率；
  5. 保持语言自然；
  6. 不偏离预训练能力。

9.6 约束方式不同

控制领域常见硬约束：

  • 不能撞障碍物；
  • 电量不能低于阈值；
  • 无人机不能超出航程；
  • 每个任务只能被执行一次；
  • 时间窗必须满足。

LLM 领域常见软约束：

  • 不要生成有害内容；
  • 不要胡编事实；
  • 不要偏离用户指令；
  • 不要为了 reward 生成重复模板；
  • 不要损失通用语言能力。

因此 LLM 中 KL penalty 很重要，它相当于给策略加上“别偏离原模型太远”的约束。

十一、PPO、GRPO、DPO、DQN 横向对比

11.1 从算法范式看

算法 学什么 优化对象 典型公式
DQN $Q(s,a)$ 动作价值函数 Bellman MSE
PPO $\pi(a s)$ + $V(s)$ 策略与价值函数
GRPO $\pi(o q)$ 策略
DPO $\pi(y x)$ 偏好概率

11.2 从 LLM 适配性看

算法 LLM 适配性 原因
DQN 低 动作空间太大，序列太长，Q 值难估计
PPO 高 直接优化生成策略，能结合 reward model 和 KL
GRPO 高 适合多采样推理任务，省去 critic
DPO 高 训练简单，适合偏好数据对齐

11.3 从工程成本看

算法 工程复杂度 主要成本
DQN 中 环境交互、replay buffer、target network
PPO 高 rollout、reward model、critic、KL、调参
GRPO 中高 多采样、reward/verifier、KL、批量生成
DPO 低 偏好数据质量、reference logprob

十二、常见误区整理

12.1 误区一：DPO 是 PPO 的简化版

不准确。
DPO 不是把 PPO 少写几步，而是从 RLHF 目标推导出一个可以直接用偏好对优化的损失。它绕过了显式 reward model 和在线 RL 采样。

12.2 误区二：GRPO 等于不用奖励模型

也不准确。
GRPO 不用 critic，但仍然需要 reward 来源。这个 reward 可以来自：

  1. reward model；
  2. 规则验证器；
  3. 单元测试；
  4. 数学答案匹配；
  5. AI judge。

12.3 误区三：PPO 一定比 DPO 更好

不一定。
PPO 有在线探索能力，可以直接优化 reward，但成本高、调参难。DPO 稳定简单，在很多偏好对齐任务上效果很好。选择哪个方法取决于数据、算力、奖励质量和任务目标。

12.4 误区四：控制领域 RL 可以直接搬到 LLM

不能简单照搬。
控制领域强调环境动态和动作反馈，LLM 强调语言分布、偏好对齐和长序列生成。虽然数学框架都叫 MDP 或策略优化，但工程实现和失败模式完全不同。

十三、面试可背版本

13.1 PPO、GRPO、DPO、DQN 的区别

DQN 是 value-based 方法，学习 $Q(s,a)$，适合动作空间较明确的控制或游戏任务，但不适合直接处理 LLM 的巨大词表动作空间。PPO 是 policy optimization 方法，通过 clipped objective 限制策略更新幅度，是经典 RLHF 中最常用的算法。GRPO 是 PPO 的变体，它用同一 prompt 下多条输出的组内相对奖励估计 advantage，减少 critic 开销，适合数学、代码等推理训练。DPO 则是基于偏好数据的直接优化方法，不显式训练 reward model，也不进行在线 RL rollout，工程上更简单稳定。

13.2 大模型最常用的强化学习算法是什么

如果严格说经典 RLHF，最有代表性、最常用的是 PPO，因为它能稳定优化策略模型，并通过 KL 惩罚防止模型偏离 SFT 模型。近年来，DPO 在开源偏好对齐中也非常常见，因为它更简单、成本更低；GRPO 则在推理模型强化学习中越来越常见，因为它省去 critic，并适合基于 verifier 的多采样训练。

13.3 LLM 强化学习怎么实现

通常先预训练基础模型，再做 SFT，让模型具备指令跟随能力。之后收集人类或 AI 偏好数据训练 reward model，或者直接构造偏好对。PPO-RLHF 会让模型生成回答，用 reward model 打分并加入 KL 惩罚，再用 PPO 更新策略；DPO 则直接最大化 chosen response 相对 rejected response 的概率优势；GRPO 会对同一 prompt 采样多个回答，用组内相对奖励更新策略。

13.4 控制领域 RL 与 LLM RL 的区别

控制领域的状态通常是物理环境状态，动作是移动、控制量或任务选择，奖励来自距离、能耗、碰撞、任务完成率等明确指标；LLM 的状态是文本上下文，动作是 token 或完整回答，奖励来自人类偏好、reward model 或 verifier。控制领域有明确环境动态和实时交互，LLM 则更关注语言质量、偏好对齐、KL 约束和长序列信用分配。

十四、个人理解与学习收获

这几个算法放在一起看，最容易混乱的是“它们都和强化学习有关”，但其实出发点很不同。

DQN 的世界里，智能体面对的是一个相对明确的环境，它需要学会每个动作的长期价值。
PPO 的世界里，智能体直接优化策略，但每次更新要克制，不能一步跨太大。
GRPO 的世界里，同一个问题的多个回答互相比较，谁更好谁就得到更高 advantage。
DPO 的世界里，强化学习目标被改写成偏好分类目标，训练过程更像监督学习。

我觉得学习 LLM 强化学习时，最重要的是不要只问“用了哪个算法”，而要继续追问：

  1. 奖励来自哪里？
  2. 有没有在线采样？
  3. 有没有 reward model？
  4. 有没有 critic？
  5. 有没有 KL 约束？
  6. 优化的是 token 级别，还是 response 级别？
  7. 任务是否有可验证答案？

这些问题比单纯背 PPO、DPO、GRPO 的名字更有用。

十五、总结

本文可以总结为：

  1. DQN 是 value-based 深度强化学习代表方法，适合游戏和控制任务，但不适合直接做 LLM 对齐；
  2. PPO 是经典 RLHF 中最有代表性的强化学习算法，优点是稳定、成熟、能结合 reward model 和 KL 约束；
  3. GRPO 是 PPO 的大模型推理训练变体，用组内相对奖励替代 critic，降低训练成本；
  4. DPO 是偏好优化方法，不跑传统在线 RL，适合用 chosen/rejected 数据进行高效对齐；
  5. LLM 强化学习与控制 RL 最大区别在于状态、动作、奖励和环境都不同：LLM 面对的是文本序列和偏好奖励，控制领域面对的是物理或仿真环境和明确任务指标；
  6. 大模型最常用算法要分语境回答：经典 RLHF 说 PPO，开源偏好对齐常说 DPO，推理强化学习常见 GRPO。

最终我自己的理解是：

两者共享强化学习的数学语言，但面对的问题、数据来源、训练稳定性和工程难点都有明显差别。
