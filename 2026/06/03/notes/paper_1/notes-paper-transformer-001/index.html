Notes-文献阅读:Transformer架构与大模型理论基础

发表于2026-06-03|更新于2026-06-03|学习笔记AI大模型
|字数总计:5.7k|阅读时长:19分钟|阅读量:

本文用于Transformer 与大语言模型基础复习。文章以 Vaswani 等人在 2017 年提出的论文《Attention Is All You Need》为主线，结合我自己的理解，整理 Transformer 为什么能替代传统 RNN/CNN 序列建模方式、Self-Attention 的核心机制、QKV 到底在做什么，以及 token、位置、距离、上下文窗口这些读大模型时绕不开的概念。

这篇笔记不追求把论文每个实验细节都翻译一遍，而是希望回答一个更实用的问题：当我们说“大模型基于 Transformer”时，它到底基于了什么？

一、论文基本信息

  • 论文标题：Attention Is All You Need
  • 作者：Ashish Vaswani, Noam Shazeer, Niki Parmar, Jakob Uszkoreit, Llion Jones, Aidan N. Gomez, Lukasz Kaiser, Illia Polosukhin
  • 发表时间：2017 年
  • 会议：NeurIPS 2017
  • 核心贡献：提出 Transformer 架构，完全基于注意力机制进行序列建模，不再依赖循环结构和卷积结构。
  • 主要应用起点：机器翻译任务
  • 后续影响：BERT、GPT、T5、LLaMA 等大语言模型都可以看作 Transformer 思想的不同变体或扩展。

论文里最关键的一句话可以概括为：

Transformer 用注意力机制替代了传统序列模型中的 recurrence 和 convolution。

这句话背后其实有两层意思：

  1. 模型不再像 RNN 那样一个 token 一个 token 顺序处理；
  2. 模型也不再像 CNN 那样依赖局部卷积核逐层扩大感受野；
  3. 每个 token 可以通过注意力机制直接和序列中的其他 token 建立联系。

二、为什么需要 Transformer：RNN 和 CNN 的局限

在 Transformer 出现之前，处理文本序列最常见的思路主要有两类：RNN 系列和 CNN 系列。

2.1 RNN 的问题：顺序依赖太强

RNN 的基本思路是：

也就是说，模型读到第 $t$ 个 token 时，需要依赖前一个隐藏状态 $h_{t-1}$。这种结构很符合人类“从左到右读句子”的直觉，但在深度学习训练里会带来几个明显问题。

问题 具体表现 对大模型训练的影响
无法充分并行 第 $t$ 步必须等第 $t-1$ 步算完 GPU 利用率不高，训练长序列很慢
长距离依赖困难 前面的信息要经过很多步才能传到后面 容易遗忘早期 token 的信息
梯度传播路径长 远距离 token 之间的计算路径很长 容易出现梯度消失或梯度爆炸
状态压缩压力大 所有历史信息被压进隐藏状态 信息瓶颈明显

LSTM、GRU 通过门控机制缓解了长距离依赖问题，但它们没有从根本上解决顺序计算瓶颈。

我的理解是：
RNN 像一个人逐字读书并不断更新脑中的摘要，摘要写得再好，也很难避免前面信息被压缩、覆盖或遗忘。

2.2 CNN 的问题：局部感受野需要逐层扩展

CNN 用卷积核处理序列时，天然擅长捕捉局部模式。例如在文本里，局部 n-gram 特征可以对应短语、固定搭配、局部语法结构。

如果卷积核大小为 3，那么一层卷积大致只能看到相邻的 3 个位置。想让第 1 个 token 和第 100 个 token 建立联系，就需要堆叠很多层，或者使用膨胀卷积等技巧。

问题 具体表现
长距离建模路径仍然较长 两个距离很远的 token 需要经过多层卷积才能交互
局部归纳偏置较强 更擅长局部模式，但全局依赖需要额外设计
卷积核大小固定 每一层的感受野扩展方式相对固定

CNN 比 RNN 更容易并行，但对文本中“任意位置之间的语义关系”来说，它仍然不是最直接的表达方式。

2.3 Transformer 想解决的核心矛盾

Transformer 要解决的不是单个小问题，而是序列建模中的一个核心矛盾：

模型既要能看见长距离依赖，又要能并行训练，还要能让不同位置之间直接交互。

Self-Attention 给出的答案是：

每个 token 在计算自己的表示时，可以直接关注序列中所有其他 token。

这就让两个相距很远的 token 不需要通过很多中间状态传递信息，而是可以在一层注意力中直接建立联系。

三、Transformer 的整体架构

原始 Transformer 是一个典型的 Encoder-Decoder 架构，主要用于机器翻译。

每个 Encoder 层主要包含：

  1. Multi-Head Self-Attention；
  2. Add & LayerNorm；
  3. Feed Forward Network；
  4. Add & LayerNorm。

每个 Decoder 层主要包含：

  1. Masked Multi-Head Self-Attention；
  2. Encoder-Decoder Attention；
  3. Feed Forward Network；
  4. 残差连接与 LayerNorm。

3.1 Encoder 的作用

Encoder 负责理解输入序列，把每个 token 转成包含上下文信息的向量表示。

例如句子：

其中 it 指代什么，不是只看 it 本身能决定的，它需要结合前文的 animal、street、tired 等信息。Encoder 的 Self-Attention 就可以让 it 这个位置去关注相关 token。

3.2 Decoder 的作用

Decoder 负责生成输出序列。生成时有一个重要限制：

当前 token 不能偷看未来 token。

所以 Decoder 里的 Self-Attention 需要使用 mask。比如生成第 5 个 token 时，只能看到第 1 到第 4 个 token，不能看到第 6 个 token 之后的答案。

这就是 GPT 类模型中的核心训练方式：
给定前文，预测下一个 token。

四、Self-Attention：Transformer 的核心机制

Self-Attention 的核心问题是：

对于当前 token，我应该从序列中的哪些 token 获取信息？各自获取多少？

假设输入序列有 $n$ 个 token，每个 token 都有一个向量表示。Self-Attention 会让每个 token 与其他所有 token 计算相关性，然后根据相关性加权汇总信息。

4.1 一个直观例子

句子：

当模型处理 它 这个 token 时，需要判断 它 指代什么。此时模型可能会对不同 token 分配不同注意力：

token 可能的注意力权重 原因
我 低 主语，不太可能是“会坏”的对象
苹果 高 很可能是“它”的指代对象
冰箱 中 与“放进”有关，但不是“会坏”的主体
坏 高 帮助判断语义关系

注意力机制不是手写规则，而是在大量数据训练中学出来的。

4.2 Scaled Dot-Product Attention 公式

论文中的注意力计算可以写成：

$$
Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V
$$

其中：

  • $Q$ 是 Query；
  • $K$ 是 Key；
  • $V$ 是 Value；
  • $d_k$ 是 Key 向量的维度；
  • $\sqrt{d_k}$ 用于缩放，避免点积值过大导致 softmax 梯度过小。

这一行公式非常重要，基本可以看作 Transformer 的心脏。

4.3 为什么要除以 $\sqrt{d_k}$

如果向量维度很高，$QK^T$ 的点积值可能变得很大。softmax 在输入差距很大时会变得非常尖锐，也就是某个位置接近 1，其他位置接近 0。

这会带来两个问题：

  1. 注意力分布过早变得极端；
  2. 梯度变小，训练不稳定。

所以论文使用 $\sqrt{d_k}$ 做缩放，使注意力分数更平稳。

五、QKV 的核心作用

QKV 是很多初学者第一次看 Transformer 时最容易卡住的地方。我的理解是：

QKV 不是三个不同的 token，而是同一个 token 在注意力计算中扮演的三种角色。

每个输入 token 的 embedding 会分别经过三个线性变换，得到：

$$
Q=XW_Q
$$

$$
K=XW_K
$$

$$
V=XW_V
$$

5.1 Query：我现在想找什么信息

Query 可以理解为“当前 token 提出的问题”。

例如处理 它 时，Query 可能在问：

当前这个代词应该和前文哪个实体关联？

5.2 Key：我能被什么问题匹配

Key 可以理解为“每个 token 对外展示的索引或标签”。

例如 苹果 的 Key 可能包含“名词、实体、可腐坏对象”等语义线索，因此更容易和 它 的 Query 匹配。

5.3 Value：真正要被取走的信息

Value 是匹配成功后要汇总的信息内容。

也就是说：

  1. Query 和 Key 负责计算“关注谁”；
  2. Value 负责提供“拿走什么信息”。

可以用一个检索系统来类比：

组件 类比 作用
Query 搜索词 表达当前需要什么
Key 文档索引 判断哪些内容相关
Value 文档内容 真正返回给模型的信息

5.4 为什么不直接用同一个向量

如果不用 QKV，而是所有用途都使用同一个表示，模型表达能力会弱很多。QKV 允许同一个 token 在不同角色下拥有不同投影：

  • 作为 Query 时，它表达“我要找什么”；
  • 作为 Key 时，它表达“我适合被谁找到”；
  • 作为 Value 时，它表达“我能贡献什么内容”。

这相当于给 token 分配了更灵活的交互方式。

六、Multi-Head Attention：为什么需要多个头

单个注意力头只能在一个表示空间里计算相关性。可是自然语言中的关系非常复杂：

  • 代词指代；
  • 主谓宾结构；
  • 修饰关系；
  • 时间关系；
  • 因果关系；
  • 代码中的变量引用；
  • 数学表达式中的符号依赖。

Multi-Head Attention 的做法是：
把 Q、K、V 投影到多个子空间，每个 head 分别计算注意力，最后再拼接起来。

公式可以概括为：

$$
head_i=Attention(QW_i^Q,KW_i^K,VW_i^V)
$$

$$
MultiHead(Q,K,V)=Concat(head_1,\dots,head_h)W^O
$$

直观理解：

多头注意力让模型可以从不同角度同时理解一句话。

比如一个 head 关注语法结构，一个 head 关注指代关系，一个 head 关注局部短语搭配。实际训练出来的 head 不一定能被这么整齐地人工解释，但这种多视角建模能力是 Multi-Head Attention 的核心价值。

七、Token：大模型真正处理的基本单位

大模型不是直接处理“句子”或“文字”，而是先把文本切成 token。

7.1 token 不一定等于一个词

token 可以是：

  1. 一个英文单词，例如 hello；
  2. 一个英文子词，例如 un, believ, able；
  3. 一个中文字符或词片段；
  4. 一个标点符号；
  5. 一段空格；
  6. 代码中的符号片段。

7.2 为什么要用 subword token

如果完全按词切分，会遇到未登录词问题：训练时没见过的词，模型不知道怎么处理。
如果完全按字符切分，序列会变得很长，计算成本又会升高。

Subword tokenization 是折中方案：

  • 常见词可以作为较完整 token；
  • 低频词可以拆成多个子词；
  • 模型更容易处理生词、拼写变化和不同语言。

Sennrich 等人在《Neural Machine Translation of Rare Words with Subword Units》中使用 BPE 思路处理稀有词和未登录词，这也是后来很多大模型 tokenizer 的基础思想之一。

八、token 距离：为什么 Transformer 更擅长长距离依赖

理解 Transformer 时，“距离”是一个很关键的概念。

8.1 序列距离

序列距离指两个 token 在文本位置上的间隔。

token_1 和 token_100 的序列距离是 99。

在 RNN 中，token_1 的信息传到 token_100，理论上要经过 99 次状态更新。
在 CNN 中，如果卷积核很小，也需要多层卷积才能让两者相互影响。
在 Self-Attention 中，token_100 可以直接对 token_1 分配注意力权重。

8.2 计算路径长度

原始论文里对比了不同层结构中长距离依赖的路径长度。可以简单理解为：

结构 两个远距离 token 交互所需路径 并行能力
RNN $O(n)$ 差
CNN 与卷积核和层数有关 较好
Self-Attention $O(1)$ 好

这里的 $O(1)$ 不是说注意力计算成本是常数，而是说任意两个 token 在一层 Self-Attention 中就可以直接交互。

8.3 注意力距离不等于真实语义距离

两个 token 在序列上距离很远，但语义上可能很近。

例如：

它 和 电脑 的序列距离不算特别近，但语义关系很强。Self-Attention 的优势就在于，它可以学习这种“语义上的近”。

8.4 上下文窗口限制

需要注意的是，Transformer 也不是无限记忆。模型一次能处理的 token 数量由 context window 决定。

例如一个模型上下文窗口是 128K token，意思是它一次推理时最多把约 128K 个 token 放进上下文里。超过窗口的内容，如果没有检索、压缩或外部记忆机制，模型本身是看不到的。

所以：

Self-Attention 解决的是窗口内部 token 之间的直接交互问题，不等于模型拥有无限上下文。

九、位置编码：既然没有 RNN，模型怎么知道顺序

RNN 天然有顺序，因为它按时间步处理序列。
CNN 也有局部位置结构，因为卷积核在相邻区域滑动。
Transformer 的 Self-Attention 本身对顺序不敏感，如果不加位置信息，模型只知道有一堆 token，却不知道它们的先后。

因此原始 Transformer 使用 Positional Encoding，把位置信息加入 embedding。

9.1 原始论文中的正弦位置编码

论文使用 sin 和 cos 函数构造位置编码：

$$
PE_{(pos,2i)}=\sin(pos/10000^{2i/d_{model}})
$$

$$
PE_{(pos,2i+1)}=\cos(pos/10000^{2i/d_{model}})
$$

其中：

  • $pos$ 表示 token 的位置；
  • $i$ 表示向量维度索引；
  • $d_{model}$ 表示模型隐藏维度。

这种设计的直观意义是：

  1. 不同位置会得到不同的位置向量；
  2. 不同维度对应不同频率；
  3. 模型可以学习相对位置关系；
  4. 理论上可以泛化到比训练长度更长的位置。

9.2 位置编码解决什么问题

如果没有位置编码，下面两句话对模型来说可能很难区分：

这两句话 token 集合相同，但顺序不同，语义完全不同。
位置编码让模型知道每个 token 出现在什么位置，从而理解语序。

9.3 后续大模型的位置编码

原始 Transformer 使用绝对位置编码。后来大模型中还出现了很多位置建模方式，例如：

  • learned positional embedding；
  • relative position embedding；
  • RoPE；
  • ALiBi。

它们的共同目标都是：
让模型更好地理解 token 的顺序、距离和相对位置。

十、Feed Forward、残差连接与 LayerNorm

Transformer 不只有注意力。每一层里还有前馈网络、残差连接和归一化。

10.1 Feed Forward Network

每个位置都会经过同一个前馈网络：

$$
FFN(x)=max(0,xW_1+b_1)W_2+b_2
$$

注意力层负责 token 之间的信息交互，FFN 更像是对每个 token 的表示做非线性变换和特征加工。

可以粗略理解为：

  • Attention：混合不同 token 的信息；
  • FFN：加工当前 token 的信息。

10.2 残差连接

残差连接可以写成：

它的作用是让深层网络更容易训练，避免信息在层层变换中丢失。

10.3 LayerNorm

LayerNorm 用于稳定训练，使每层的激活分布更可控。
Transformer 层数变深后，如果没有归一化和残差连接，训练会困难很多。

十一、Encoder-only、Decoder-only 与 Encoder-Decoder

原始 Transformer 是 Encoder-Decoder，但后来的大模型根据任务目标发展出三类常见结构。

类型 代表模型 适合任务 注意力特点
Encoder-only BERT 理解、分类、检索、抽取 双向注意力
Decoder-only GPT 系列 文本生成、对话、代码生成 因果 Masked Attention
Encoder-Decoder T5、原始 Transformer 翻译、摘要、输入到输出转换 Encoder 理解输入，Decoder 生成输出

11.1 为什么 GPT 使用 Decoder-only

GPT 的训练目标是预测下一个 token：

$$
P(x_t|x_1,x_2,\dots,x_{t-1})
$$

这天然需要因果 mask，不能看到未来 token。因此 Decoder-only 架构非常适合生成式语言模型。

11.2 为什么 BERT 使用 Encoder-only

BERT 更关注理解任务，它可以同时看左右上下文，所以使用双向注意力。
它的典型训练方式是 Masked Language Modeling，即遮住一部分 token，让模型根据上下文预测被遮住的内容。

十二、Transformer 与大模型的关系

大语言模型可以粗略理解为：

使用海量文本训练的大规模 Transformer 语言模型。

当然，现代大模型不只是把原始 Transformer 放大，还包含许多工程和算法改进，例如：

  1. 更大的数据规模；
  2. 更深或更宽的网络；
  3. 更高效的位置编码；
  4. 更稳定的优化方法；
  5. 更复杂的 tokenizer；
  6. 指令微调；
  7. 人类反馈对齐；
  8. 长上下文优化；
  9. MoE 等稀疏结构。

但最基础的骨架仍然离不开：

  • token embedding；
  • self-attention；
  • QKV；
  • multi-head attention；
  • positional encoding；
  • FFN；
  • residual connection；
  • normalization；
  • next-token prediction。

如果这些概念能串起来，读大模型相关论文会轻松很多。

十四、面试和复习时可以这样回答

14.1 Transformer 解决了 RNN 和 CNN 的哪些问题？

可以这样回答：

Transformer 主要解决了传统 RNN 难以并行、长距离依赖路径长的问题，也缓解了 CNN 依赖局部卷积、需要多层才能建模远距离关系的问题。它通过 Self-Attention 让任意两个 token 在一层中直接交互，从而提升了并行训练能力和全局建模能力。

14.2 Self-Attention 的核心是什么？

可以这样回答：

Self-Attention 的核心是让每个 token 根据与其他 token 的相关性，动态加权汇总上下文信息。相关性由 Query 和 Key 的点积计算，真正被汇总的信息来自 Value。

14.3 QKV 分别是什么？

可以这样回答：

Query 表示当前 token 想查询什么信息，Key 表示每个 token 可供匹配的索引特征，Value 表示匹配后真正传递的内容。Q 和 K 决定注意力权重，V 决定输出信息。

14.4 为什么 Transformer 需要位置编码？

可以这样回答：

因为 Self-Attention 本身不包含顺序结构，如果不加入位置信息，模型无法区分相同 token 集合的不同排列。位置编码用于告诉模型每个 token 在序列中的位置，并帮助模型学习相对距离关系。

十五、个人理解与学习收获

我觉得理解 Transformer 时，不应该只背公式，而要抓住一个主线：

Transformer 把“序列逐步传递信息”改造成了“序列内部全局检索信息”。

RNN 的信息流像链条，前后 token 需要一步一步传。
CNN 的信息流像局部窗口，需要逐层扩大范围。
Self-Attention 的信息流更像全局检索，每个 token 都可以主动查询其他 token。

QKV 机制则进一步把“查询、匹配、取值”拆开，让模型学会更灵活的信息交互。
这也是为什么 Transformer 能成为大语言模型的基础：语言理解和生成中大量问题，本质上都是在上下文里找到相关信息、组合信息、再预测下一个合理 token。

不过 Transformer 也不是没有代价。Self-Attention 的计算复杂度通常与序列长度平方相关：

$$
O(n^2)
$$

这意味着上下文越长，注意力计算成本越高。后续很多长上下文模型、高效注意力机制、稀疏注意力方法，本质上都是在解决这个问题。

十六、总结

Transformer 的核心价值可以概括为以下几点：

  1. 去掉顺序递归结构：相比 RNN，更容易并行训练；
  2. 直接建模全局关系：相比局部 CNN，更容易捕捉远距离依赖；
  3. Self-Attention 是核心：每个 token 可以根据相关性汇总其他 token 的信息；
  4. QKV 是注意力的角色分工：Q 负责查询，K 负责匹配，V 负责提供内容；
  5. Multi-Head 提供多视角理解：不同 head 可以学习不同类型的关系；
  6. 位置编码补充顺序信息：让无递归结构的模型仍然知道 token 的排列；
  7. token 是大模型的基本处理单位：文本需要先切成 token，再映射成向量；
  8. 大模型是在 Transformer 基础上的规模化结果：现代 LLM 加入了大量训练、数据、对齐和工程优化，但基础机制仍与 Transformer 密切相关。
