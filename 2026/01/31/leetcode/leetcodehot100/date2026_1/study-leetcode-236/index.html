Leetcode-236:二叉树的最近公共祖先-递归后序遍历法

Created2026-01-31|Updated2026-01-31|Java学习笔记Leetcode Hot100
|Word Count:3k|Reading Time:10mins|Post Views:

1、题目

给定一个普通二叉树，找到该树中两个指定节点的最近公共祖先（Lowest Common Ancestor，LCA）。

百度百科中最近公共祖先的定义为：“对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

示例

示例 1（p、q分属根节点左右子树）：

输入：根节点root = [3,5,1,6,2,0,8,null,null,7,4]，p = 5，q = 1
输出：3
解释：节点5和节点1的最近公共祖先是根节点3。

示例 2（p是q的祖先）：

输入：根节点root = [3,5,1,6,2,0,8,null,null,7,4]，p = 5，q = 4
输出：5
解释：节点4在节点5的子树中，节点5自身是其与4的最近公共祖先。

提示

  • 树中节点数目在范围 [2, 10⁵] 内
  • -10⁹ <= Node.val <= 10⁹
  • 所有节点的值都是唯一的
  • p != q
  • p 和 q 均存在于给定的二叉树中

关键说明

本题为普通二叉树求解LCA，无二叉搜索树（BST）的“左小右大”排序特性，无法通过节点值比较快速定位，递归后序遍历法是通用最优解。

2、分析

2.1 问题核心

普通二叉树无排序规律，无法通过值判断节点位置，求解LCA的核心是从叶子节点向上回溯，找到第一个能同时包含p、q两个节点的祖先节点。

由于需要先查找左右子树的结果，再根据结果判断当前节点是否为LCA，符合后序遍历（左→右→根） 的“先处理子节点，再处理父节点”的特性，因此递归后序遍历是该问题的天然解法。

2.2 核心思路（递归回溯）

整个算法基于递归查找+结果回溯，分为三个核心步骤，本质是后序遍历的递归实现：

  1. 递归终止条件：遍历到空节点，或当前节点是p/q其一，直接返回当前节点（空节点表示未找到，p/q表示找到目标节点）；
  2. 后序遍历查找：递归遍历当前节点的左子树和右子树，分别获取左、右子树的查找结果（left/right）；
  3. 结果判断与回溯：根据左、右子树的返回结果，分三种情况判断当前节点是否为LCA，并将结果向上回溯：
    • 左右子树均找到目标（left≠null && right≠null）：当前节点是p、q的最近公共祖先，返回当前节点；
    • 仅左子树找到目标（left≠null && right=null）：将左子树的结果向上回溯，返回left；
    • 仅右子树找到目标/均未找到（left=null）：将右子树的结果向上回溯，返回right（包含right≠null和right=null两种情况）。

2.3 核心原理拆解

  1. 回溯的本质：递归遍历会从根节点深入到叶子节点，再从叶子节点向上返回结果，这个“向上返回”的过程就是回溯，能让父节点基于子节点的结果做判断；
  2. 节点是自身祖先：当p是q的祖先（或反之）时，遍历到p节点会直接返回p，后续遍历其子孙节点均会返回null，最终回溯到p节点时，直接返回p作为LCA，符合题目定义；
  3. 最近的保证：当左右子树均返回非空结果时，说明p、q分别在当前节点的左右子树中，当前节点是第一个能同时包含两者的祖先节点，深度最大，即为最近公共祖先。

2.4 算法特性

  • 该方法是普通二叉树LCA的通用解法，不依赖任何树的特殊性质，适用于所有二叉树；
  • 遍历过程中每个节点仅被访问一次，无重复操作；
  • 利用递归调用栈实现回溯，无需额外存储节点路径，空间复杂度由递归栈深度决定。

3.1 完整带详细注释代码

3.2 代码逐行解释

  • 递归终止条件，是整个算法的基础：
    • root == null：表示当前遍历的分支无目标节点，返回null供上层判断；
    • root == p || root == q：表示找到其中一个目标节点，立即返回该节点，开始向上回溯，让父节点基于此结果做判断；
    • 该条件天然支持“节点是自身祖先”的场景，无需额外处理。
  • 严格的后序遍历操作：先递归处理左子树，再递归处理右子树，保证父节点能获取到左右子树的完整查找结果；
  • left/right 存储的是左右子树中找到的目标节点或null，是后续判断的核心依据。
  • 最核心的判断条件：左右子树各找到一个目标节点，说明p、q分别在当前节点的左右子树中；
  • 当前节点是第一个能同时包含p、q的祖先节点，深度最大，即为最近公共祖先，直接返回。
  • 包含3种子情况：
    1. left != null && right == null, 仅左子树找到目标节点，说明p、q都在左子树中，将左子树的结果向上回溯，让更上层的节点继续判断。
    2. left == null && right != null, 仅右子树找到目标节点：p、q都在右子树中，回溯右子树结果；
    3. left == null && right == null, 左右子树均未找到：当前分支无目标节点，返回null；
  • 合并为一个分支，简化代码，不影响逻辑。

4、算法性能

4.1 复杂度分析

  • 时间复杂度：O(n)
    递归过程中会遍历二叉树的每个节点，每个节点仅被访问一次，无重复遍历，操作数与节点数量成正比。
  • 空间复杂度：O(n)
    空间开销来自递归调用栈：
    • 最优情况（平衡二叉树）：递归栈深度为O(log n)，树的高度为对数级别；
    • 最坏情况（二叉树退化为单链表）：递归栈深度为O(n)，需要存储所有节点的递归状态。

4.2 优势对比

解法 时间复杂度 空间复杂度 适用场景 核心优势
递归后序法 O(n) O(n) 所有二叉树 通用、代码简洁、无需额外空间
路径记录法 O(n) O(n) 所有二叉树 直观、易理解
BST专属法 O(log n) O(1) 仅二叉搜索树 时间效率更高

注：路径记录法需先遍历记录p、q的路径，再找路径最后一个公共节点，虽直观但需要额外的集合存储路径，空间复杂度同为O(n)但代码更冗余。

5、测试示例（分步递归演示）

以示例2（root=[3,5,1,6,2,0,8,null,null,7,4]，p=5，q=4）为例，分步演示递归回溯过程，直观理解LCA的查找逻辑：

节点结构简化

递归回溯步骤

  1. 从根3开始，先递归左子树5，再递归右子树1；
  2. 遍历节点5：触发终止条件root==p，直接返回5（无需遍历其子孙节点，后续子孙节点遍历仅为验证）；
  3. 验证节点5的左子树6：遍历至叶子，左右均为null，返回null；
  4. 验证节点5的右子树2：继续递归其左右子树7和4；
  5. 遍历节点4：触发终止条件root==q，返回4；遍历节点7：返回null；
  6. 节点2的判断：left=7（null），right=4（非null），返回4；
  7. 节点5的判断：left=6（null），right=2（返回4），但因步骤2已直接返回5，此判断无需执行；
  8. 根节点3的判断：left=5（非null），right=1（遍历后返回null），返回5；
  9. 最终结果：5，与示例预期一致。

关键结论：当p是q的祖先时，遍历到p会直接返回，后续子孙节点的结果不会覆盖该返回值，最终回溯返回p作为LCA。

以示例1（p=5，q=1）为例，核心步骤简化：

  1. 根3递归左子树5，返回5；递归右子树1，返回1；
  2. 根3的判断：left=5（非null），right=1（非null），直接返回3；
  3. 最终结果：3，符合预期。

6.1 拓展1：二叉搜索树（BST）的LCA解法

6.2 拓展2：递归改迭代（普通二叉树）

若想避免递归栈的空间开销，可将递归后序遍历改为迭代后序遍历，通过栈模拟递归调用，同时记录节点的访问状态，实现非递归回溯，核心逻辑与递归法一致，代码稍显繁琐，适合对空间复杂度要求极高的场景。

6.3 核心总结

  1. 普通二叉树LCA的核心：利用后序遍历+回溯，先获取子节点的查找结果，再判断父节点是否为LCA，这是二叉树“自底向上”解决问题的经典思路；
  2. 递归终止条件的设计：直接返回null或目标节点，简洁且天然支持“节点是自身祖先”的场景，是整个算法的关键；
  3. 三种判断场景：左右均非空→返回根，左非空→返回左，否则返回右，覆盖了所有可能的查找情况，无遗漏；
  4. 算法的通用性：该递归后序遍历法不依赖二叉树的任何特殊性质，是解决普通二叉树LCA问题的工业级通用解法，也是Leetcode该题的标准最优解。

6.4 思维提升

该题的核心价值不仅在于解决LCA问题，更在于培养二叉树的回溯思维：当二叉树问题需要“从叶子节点向上传递结果，父节点基于子节点结果做判断”时，后序遍历（递归/迭代）是最优选择，这种思维可迁移至二叉树的路径和、节点计数、子树判断等诸多问题中。
