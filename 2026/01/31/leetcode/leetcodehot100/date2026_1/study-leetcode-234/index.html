Leetcode-234:回文链表-快慢指针+链表反转

Created2026-01-31|Updated2026-01-31|Java学习笔记Leetcode Hot100
|Word Count:3.7k|Reading Time:12mins|Post Views:

1、题目

给你一个单链表的头节点 head，请你判断该链表是否为回文链表。如果是，返回 true；否则，返回 false。

示例

示例 1（偶数节点回文）：

输入：head = [1,2,2,1]
输出：true

示例 2（奇数节点回文）：

输入：head = [1,2,3,2,1]
输出：true

示例 3（非回文）：

输入：head = [1,2]
输出：false

提示

  • 链表中节点数目在范围 [1, 10⁵] 内
  • 0 <= Node.val <= 9

进阶要求

请用 O(n) 时间复杂度和 O(1) 空间复杂度解决此题（常规转数组法空间复杂度为O(n)，无法满足进阶要求）。

问题核心难点

单链表无法反向遍历、无法随机访问，且进阶要求原地操作（O(1)空间），不能借助数组、栈等额外存储结构，需通过快慢指针和链表反转将链表转化为可前后对比的结构。

2、分析

2.1 常规解法的痛点

面对回文判断问题，新手易想到两种常规解法，但均无法满足进阶的空间要求：

  1. 数组辅助法：遍历链表将节点值存入数组，再用双指针从数组首尾向中间对比。时间O(n)，但空间O(n)，额外存储数组破坏原地要求；
  2. 递归回溯法：利用递归栈实现链表反向遍历，与正向遍历的指针对比节点值。时间O(n)，但递归调用栈的空间复杂度为O(n)，仍不满足O(1)要求。

2.2 最优解核心思路（三步法）

本题的最优解结合快慢指针和链表反转两大链表核心技巧，通过“找中间节点 → 反转后半部分链表 → 前后半部分对比” 三步实现原地回文判断，完美满足O(n)时间、O(1)空间要求，核心逻辑如下：

  1. 找中间节点：用快慢指针（快指针走两步、慢指针走一步）遍历链表，快指针遍历结束时，慢指针恰好指向链表中间节点（偶数节点指向后半段首节点，奇数节点指向正中间节点）；
  2. 反转后半部分：将慢指针指向的中间节点开始的后半段链表反转，使后半段链表变为反向顺序，可与前半段正向顺序直接对比；
  3. 前后对比：用两个指针分别指向原链表头（前半段首节点）和反转后的后半段头，依次对比节点值，全部相等则为回文，否则不是。

2.3 关键细节拆解

（1）快慢指针找中间的原理

  • 快指针fast：每次走2步，fast = fast.next.next；
  • 慢指针slow：每次走1步，slow = slow.next；
  • 循环条件：fast != null && fast.next != null（避免空指针异常）；
  • 结果：遍历结束后，偶数节点链表中slow指向后半段首节点，奇数节点链表中slow指向正中间节点（中间节点无需参与对比，反转后半段后其为反转链表的尾节点，不影响对比流程）。

（2）反转后半段的意义

单链表无法反向遍历，将后半段反转后，后半段的遍历方向与前半段一致，只需两个指针依次后移即可完成前后对比，这是解决链表回文判断的核心技巧。

（3）奇偶节点的兼容处理

奇数节点链表中，中间节点是回文的“对称轴”，无需与任何节点对比。反转后半段后，反转后的后半段长度比前半段少1，对比时循环至反转后的后半段为null即可，代码无需额外处理，天然兼容奇偶场景。

2.4 算法特性

  • 时间复杂度：O(n)（快慢指针遍历O(n) + 反转后半段O(n/2) + 前后对比O(n/2)，总操作数为线性级别）；
  • 空间复杂度：O(1)（仅使用若干指针变量，无额外存储结构，纯原地操作）；
  • 兼容性：天然兼容奇数、偶数节点的链表，无需额外分支判断；
  • 可恢复性：若题目要求保留原链表结构，可在对比完成后再次反转后半段并拼接，不影响核心逻辑。

3.2 代码逐行解释

主方法：isPalindrome

  • 调用辅助方法完成找中间和反转后半段，将原链表拆分为“前半段正向”和“后半段反向”两个可直接对比的部分；
  • 原链表前半段保持不变，后半段被反转，无额外空间开销。
  • 核心对比逻辑：以反转后的后半段长度为基准（奇数节点时更短），依次对比前后段节点值；使用 while (head2 != null) 而非 while (head != null) 是为了确保只比较前半部分和反转后的后半部分，一旦后半部分遍历完毕就停止，避免在偶数长度链表中因 head 继续往后遍历而访问到空的 head2。
    补充：如果想用 while (head != null)，需要在 findMiddle 后手动切断前半部分和后半部分的连接（将前半部分的尾节点 next 设为 null），但那样代码会更复杂。使用 head2 != null 是最简洁安全的做法。
  • 只要有一个节点值不相等，立即返回false，提前终止遍历，优化性能；
  • 两个指针同步后移，保证对比的顺序性。
  • 所有节点对比完成且全部相等，说明是回文链表，返回true。

辅助方法1：findMiddle（快慢指针找中间）

  • 快慢指针均从原链表头节点开始遍历。
  • 循环条件是避免空指针的关键：
    • 偶数节点：fast最终指向null，循环终止；
    • 奇数节点：fast最终指向最后一个节点（fast.next=null），循环终止；
    • 若仅写fast.next != null，奇数节点时会执行fast.next.next导致空指针。
  • 快慢指针按“1步、2步”的节奏遍历，保证快指针遍历完时，慢指针恰好到中间。

辅助方法2：reverse（迭代反转链表）

  • 复用经典的迭代链表反转逻辑，仅反转从中间节点开始的后半段，与完整链表反转无区别；
  • 核心是先备份、再反转、后推进，防止链表断链，保证反转的完整性。

4、算法性能

4.1 复杂度分析

  • 时间复杂度：O(n)
    整个流程包含三次线性级遍历，且无嵌套：

    1. 快慢指针遍历：O(n)，遍历整个链表一次；
    2. 反转后半段：O(n/2)，仅遍历链表的一半节点；
    3. 前后对比：O(n/2)，最多遍历链表的一半节点；
      总操作数为n + n/2 + n/2 = 2n，按算法复杂度规则，仍为O(n)。

  • 空间复杂度：O(1)
    仅使用fast、slow、pre、cur、nxt等有限个指针变量，无数组、哈希表、递归栈等额外存储结构，完全满足原地算法要求。

4.2 解法对比

解法 时间复杂度 空间复杂度 满足进阶要求 代码复杂度 性能优化
数组辅助法 O(n) O(n) 否 低 一般
递归回溯法 O(n) O(n) 否 中 一般
快慢指针+反转 O(n) O(1) 是 中 优（可提前终止）

5、测试示例（分步详细演示）

分别以偶数节点回文、奇数节点回文、非回文三个典型场景为例，分步演示算法执行过程，直观理解核心逻辑。

示例1：偶数节点回文 head = [1,2,2,1]

步骤1：快慢指针找中间

  • 初始：fast=1，slow=1
  • 第1次循环：slow=2，fast=2（fast.next.next=1.next.next=2）
  • 第2次循环：slow=2（后半段首节点），fast=null（fast.next.next=2.next.next=null）
  • 循环终止，返回midNode=2（第二个2）。

步骤2：反转后半段

  • 待反转链表：2 → 1
  • 反转后：1 → 2，返回reverseRight=1。

步骤3：前后对比

  • 初始：head=1（前半段），reverseRight=1（反转后半段）
    1. 1=1 → 后移，head=2，reverseRight=2
    2. 2=2 → 后移，head=2，reverseRight=null
  • 循环终止，返回true。

示例2：奇数节点回文 head = [1,2,3,2,1]

步骤1：快慢指针找中间

  • 遍历结束后，slow=3（正中间节点），返回midNode=3。

步骤2：反转后半段

  • 待反转链表：3 → 2 → 1
  • 反转后：1 → 2 → 3，返回reverseRight=1。

步骤3：前后对比

  • 初始：head=1，reverseRight=1
    1. 1=1 → 后移，head=2，reverseRight=2
    2. 2=2 → 后移，head=3，reverseRight=3
    3. 3=3 → 后移，head=2，reverseRight=null
  • 循环终止，返回true。

示例3：非回文 head = [1,2]

步骤1：快慢指针找中间

  • 遍历结束后，slow=2，返回midNode=2。

步骤2：反转后半段

  • 待反转链表：2
  • 反转后：2，返回reverseRight=2。

步骤3：前后对比

  • 初始：head=1，reverseRight=2
  • 1≠2 → 立即返回false，提前终止遍历。

6.2 拓展2：边界场景测试

本题代码天然兼容所有边界场景，无需额外处理：

  1. 单节点链表：head=[1] → 找中间返回1，反转后还是1，对比循环不执行，返回true；
  2. 空链表：按题目提示节点数≥1，无需处理；
  3. 全相同节点：head=[2,2,2] → 对比全部相等，返回true。

6.3 核心避坑点

  1. 快慢指针的循环条件：必须写fast != null && fast.next != null，缺一不可，否则会出现空指针异常；
  2. 反转的起始节点：必须从慢指针指向的中间节点开始，不能从慢指针的下一个节点开始，否则会遗漏节点；
  3. 对比的终止条件：必须以反转后的后半段为基准（reverseRight != null），而非前半段，否则奇数节点时会出现空指针。

6.4 整体总结

本次讲解的快慢指针+链表反转法是回文链表的工业级最优解，完美契合题目进阶要求的O(n)时间和O(1)空间，其核心价值在于：

  1. 技巧融合：将“快慢指针找中间”和“迭代链表反转”两个链表核心技巧结合，解决了单链表无法反向遍历、无法随机访问的痛点；
  2. 原地操作：全程仅使用指针变量，无额外存储，最大化节省内存，适合处理大规模链表（如节点数10⁵）；
  3. 逻辑通用：三步核心逻辑（找中间→反转→对比）可迁移至其他链表分割、对比类问题，如链表的对称判断、半段反转等。

该方法也是Leetcode本题的标准解法，掌握后不仅能解决回文链表问题，更能加深对链表双指针技巧和原地操作思想的理解，是链表学习中必须掌握的经典案例。

6.5 思维提升

本题的解题思路体现了“转化问题”的算法核心思维：将“单链表回文判断”这个无法直接解决的问题，转化为“找中间→反转→前后对比”三个可解决的子问题，通过拆解和转化，让复杂问题变得简单。这种思维在数据结构和算法解题中至关重要，适用于树、图、链表等各类复杂问题的求解。
