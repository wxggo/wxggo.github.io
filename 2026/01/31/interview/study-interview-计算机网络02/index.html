Interview-计算机网络02:键入网址到网页显示，期间发生了什么？

Created2026-01-31|Updated2026-01-31|学习笔记面试学习
|Word Count:3k|Reading Time:9mins|Post Views:

1、前言

“输入URL后，网页是如何显示出来的？”——这是计算机网络面试中最高频的经典问题，考察对DNS、TCP/IP、HTTP、网络分层等核心知识点的综合理解。

本文基于小林coding的详细讲解，梳理从URL输入到页面渲染的完整流程，拆解每个环节的核心逻辑、协议作用和面试考点。

2、核心流程总览

输入URL到网页显示的全过程，可概括为7个核心步骤，本质是“数据封装→传输→解封装”的过程，涉及多层网络协议的协同工作：

  1. URL解析：浏览器解析URL，生成HTTP请求信息；
  2. DNS解析：查询目标服务器域名对应的IP地址；
  3. 协议栈封装：HTTP数据依次封装TCP头部、IP头部、MAC头部；
  4. 网络传输：通过网卡、交换机、路由器将数据包送达目标服务器；
  5. 服务器解封装与响应：服务器逐层解封装数据包，处理请求并生成响应；
  6. 响应数据回传：响应数据按原路径反向传输回客户端；
  7. 浏览器渲染：客户端解封装响应数据，浏览器渲染网页。

3、分步拆解（含面试考点）

3.1 第一步：URL解析 + HTTP请求生成

核心逻辑

浏览器首先对URL进行解析，提取协议、服务器域名、资源路径等关键信息，再生成HTTP请求报文。

  • URL结构：协议://服务器域名/资源路径?参数#锚点（如http://www.server.com/dir1/file1.html）；
  • 解析结果：确定使用HTTP协议，目标服务器为www.server.com，请求资源为/dir1/file1.html；
  • 默认资源：若URL无资源路径，默认请求根目录下的index.html或default.html；
  • HTTP请求报文结构：
    • 请求行：方法（GET/POST） + 资源路径 + 协议版本（如GET / HTTP/1.1）；
    • 首部字段：Host、Accept、Cookie等（如Host: www.server.com）；
    • 消息体：POST请求携带数据，GET请求为空。

面试考点

  • 问：URL各部分的作用是什么？缺失资源路径时会请求什么文件？
  • 答：协议指定数据传输规则，服务器域名定位目标服务器，资源路径指定请求文件；缺失路径时请求默认首页（如index.html）。

3.2 第二步：DNS解析（域名→IP）

核心逻辑

HTTP请求需要目标服务器的IP地址才能发送，DNS（域名系统）的作用是“域名→IP”的映射查询，类似“网络通讯录”。

  • 域名层级：域名采用“从右到左层级递增”结构（如www.server.com.，最后的.为根域），层级关系：根域→顶级域（.com）→二级域（server.com）→主机名（www）；
  • 解析流程（递归+迭代查询）：
    1. 客户端先查浏览器缓存→操作系统缓存→hosts文件，未命中则请求本地DNS服务器；
    2. 本地DNS服务器查询自身缓存，未命中则向根DNS服务器请求；
    3. 根DNS服务器返回顶级域（.com）DNS服务器地址；
    4. 本地DNS向顶级域DNS服务器请求，获取权威DNS服务器地址；
    5. 本地DNS向权威DNS服务器请求，获取目标IP；
    6. 本地DNS缓存IP并返回给客户端。

面试考点

  • 问：DNS解析的完整流程是什么？为什么需要本地DNS服务器？
  • 答：流程为“客户端缓存→本地DNS→根DNS→顶级域DNS→权威DNS”；本地DNS作为代理，避免客户端直接与各级DNS通信，减少网络开销，同时提供缓存加速。
  • 问：DNS缓存有哪些层级？作用是什么？
  • 答：分为浏览器缓存、操作系统缓存、本地DNS缓存；作用是减少重复解析，提高查询效率。

3.3 第三步：协议栈封装（HTTP→TCP→IP→MAC）

核心逻辑

浏览器通过Socket库委托操作系统的协议栈，对HTTP请求数据进行分层封装，每一层添加对应的头部信息，确保数据能正确传输和解析。

  • 封装顺序：应用层（HTTP）→传输层（TCP）→网络层（IP）→链路层（MAC）；
  • 各层封装作用：
（1）传输层：TCP封装（可靠传输）
  • 核心作用：提供可靠的字节流传输，解决丢包、乱序、重传问题；
  • TCP头部关键字段：
    • 源端口（客户端随机端口）、目标端口（HTTP默认80，HTTPS默认443）；
    • 序号（解决乱序）、确认号（解决丢包）；
    • 状态位（SYN=建立连接，ACK=确认，FIN=关闭连接）；
    • 窗口大小（流量控制）；
  • 面试考点：TCP三次握手的过程和目的？
    • 过程：①客户端发SYN→②服务器回SYN+ACK→③客户端发ACK；
    • 目的：确保双方都具备发送和接收能力，初始化序号。
（2）网络层：IP封装（远程定位）
  • 核心作用：定义IP地址，实现跨网络的路由转发；
  • IP头部关键字段：
    • 源IP（客户端IP）、目标IP（DNS解析得到的服务器IP）；
    • 协议号（TCP=6，UDP=17）；
    • TTL（生存时间，防止数据包循环）；
  • 路由选择：客户端根据路由表选择出口网卡（多网卡时），数据包通过网关转发。
（3）链路层：MAC封装（两点传输）
  • 核心作用：定义MAC地址，实现局域网内的点对点传输；
  • MAC头部关键字段：
    • 源MAC（客户端网卡MAC地址）、目标MAC（网关/目标设备MAC地址）；
    • 协议类型（IP协议=0800，ARP协议=0806）；
  • MAC地址获取：通过ARP协议广播查询（同一子网），结果存入ARP缓存（有效期几分钟）。

面试考点

  • 问：TCP、IP、MAC的核心作用分别是什么？
  • 答：TCP负责可靠传输，IP负责远程定位，MAC负责局域网内点对点传输。
  • 问：封装过程中各层头部的添加顺序是什么？
  • 答：从应用层到链路层逐层添加（HTTP→TCP→IP→MAC），解封装时反向逐层删除。

3.4 第四步：网络传输（网卡→交换机→路由器）

核心逻辑

封装完成的数据包（二进制数据）通过网卡转换为电信号，经交换机、路由器转发，最终抵达目标服务器所在网络。

（1）网卡：数据发送
  • 核心作用：将内存中的二进制数据包转换为电信号，通过网线传输；
  • 附加信息：在数据包首尾添加起始帧分界符（标记包起始）和FCS（帧校验序列，检测传输错误）。
（2）交换机：局域网转发
  • 工作层级：数据链路层（二层设备），基于MAC地址转发；
  • 核心逻辑：
    • 维护MAC地址表（MAC地址→端口映射）；
    • 接收数据包后，根据目标MAC地址查询表，转发到对应端口；
    • 表中无目标MAC时，广播到所有端口（目标设备响应后更新表）。
（3）路由器：跨网络转发
  • 工作层级：网络层（三层设备），基于IP地址转发；
  • 核心逻辑：
    • 维护路由表（目标网络→出口接口/网关）；
    • 接收数据包后，剥离MAC头部（MAC仅作用于局域网），根据目标IP查询路由表；
    • 确定出口后，通过ARP获取下一跳设备MAC地址，重新封装MAC头部并转发；
  • 关键特性：转发过程中，源IP和目标IP不变，仅源MAC和目标MAC动态变化。

面试考点

  • 问：交换机和路由器的区别是什么？（高频考点）

    对比维度 交换机 路由器
    工作层级 数据链路层（二层） 网络层（三层）
    转发依据 MAC地址 IP地址
    端口特性 无IP和MAC地址 有IP和MAC地址
    核心作用 局域网内数据交换 跨网络路由转发
    地址表 MAC地址表 路由表

  • 问：路由器转发时会修改哪些地址？为什么？

  • 答：修改源MAC和目标MAC（适配下一跳链路），源IP和目标IP不变（确保最终定位）。

3.5 第五步：服务器处理 + 响应生成

核心逻辑

数据包抵达服务器后，逐层解封装（MAC→IP→TCP→HTTP），服务器处理请求并生成响应报文，再按原封装流程反向封装。

  • 解封装过程：服务器网卡接收电信号→转换为数据包→逐层剥离头部（MAC→IP→TCP）→交给HTTP进程；
  • 处理请求：HTTP服务器（如Nginx）根据请求路径读取资源（如HTML文件）；
  • 生成响应：HTTP响应报文包含状态行（如HTTP/1.1 200 OK）、首部字段（如Content-Type: text/html）、消息体（HTML数据）；
  • 响应封装：响应数据按“HTTP→TCP→IP→MAC”顺序封装，源IP/端口为服务器地址/端口，目标IP/端口为客户端地址/端口。

3.6 第六步：响应回传 + 浏览器渲染

核心逻辑

响应数据包按原传输路径反向回传（服务器→路由器→交换机→客户端），客户端逐层解封装后，浏览器渲染HTML页面。

  • 回传路径：服务器→本地交换机→路由器→互联网→客户端路由器→客户端交换机→客户端；
  • 客户端解封装：逐层剥离MAC→IP→TCP头部，获取HTTP响应数据；
  • 浏览器渲染：解析HTML、CSS、JavaScript，构建DOM树、CSSOM树，合成渲染树，最终显示网页；
  • 连接关闭：页面渲染完成后，TCP通过四次挥手断开连接。

面试考点

  • 问：TCP四次挥手的目的是什么？
  • 答：确保双方都已完成数据传输，优雅关闭连接（避免数据丢失）。
  • 问：浏览器渲染的大致流程是什么？
  • 答：解析HTML生成DOM树→解析CSS生成CSSOM树→DOM+CSSOM合成渲染树→布局→绘制→显示。

4、核心考点总结（面试必背）

4.1 高频问答

  1. 问：输入URL到网页显示的完整流程？（核心必背）
    答：URL解析→DNS解析→TCP三次握手→HTTP请求→数据传输→服务器响应→TCP四次挥手→浏览器渲染。

  2. 问：DNS解析的流程和缓存层级？
    答：流程：客户端缓存→本地DNS→根DNS→顶级域DNS→权威DNS；缓存层级：浏览器缓存→操作系统缓存→本地DNS缓存。

  3. 问：TCP三次握手和四次挥手的目的？
    答：三次握手：建立连接，确保双方收发能力；四次挥手：关闭连接，确保数据传输完成。

  4. 问：交换机和路由器的核心区别？
    答：交换机二层设备，基于MAC转发；路由器三层设备，基于IP转发。

  5. 问：传输过程中IP和MAC地址的变化情况？
    答：IP地址（源/目标）不变，MAC地址（源/目标）随每一跳链路动态变化。

4.2 易错点提醒

  • DNS解析是“递归+迭代”结合，本地DNS是客户端的代理，并非直接查询根DNS；
  • TCP三次握手的第二次是“SYN+ACK”，第三次是“ACK”，不可省略；
  • MAC地址仅作用于局域网，跨网络传输时需路由器转发，MAC头部会重新封装；
  • HTTP是基于TCP的应用层协议，无连接可靠性，依赖TCP保证数据传输。

5、总结

输入URL到网页显示的过程，是多层网络协议协同工作的典型案例，核心是“封装→传输→解封装”的循环。面试中考察该问题，本质是检验对DNS、TCP/IP、网络分层等基础知识点的理解和串联能力。

记住核心流程和高频考点，结合本文的分步拆解，能帮助你在面试中清晰、有条理地回答，同时加深对计算机网络整体架构的理解。建议结合实际场景（如HTTPS、跨域）拓展学习，巩固知识点。
