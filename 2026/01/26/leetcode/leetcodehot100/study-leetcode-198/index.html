Leetcode-198:实现打家劫舍-动态规划法

Created2026-01-26|Updated2026-01-26|Java学习笔记Leetcode Hot100
|Word Count:1.6k|Reading Time:6mins|Post Views:

1、题目

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组 nums，请计算你 不触动警报装置的情况下，一夜之内能够偷窃到的最高金额。

示例

示例 1：
输入：nums = [1,2,3,1]
输出：4
解释：偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。偷窃到的最高金额 = 1 + 3 = 4。

示例 2：
输入：nums = [2,7,9,3,1]
输出：12
解释：偷窃 2 号房屋 (金额 = 7)、4 号房屋？不，正确路径是偷窃 2（金额2）、9（金额9）、1（金额1）？不，正确计算：2 + 9 + 1 = 12 或 7 + 3 = 10，最终最高是12。

提示

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 400

2、分析

2.1 问题核心

打家劫舍问题的核心是在“不能选取相邻元素”的约束下，找到数组的最大子序列和。常规暴力枚举所有可能的偷窃组合（时间复杂度 O(2ⁿ)）效率极低，而动态规划（DP）通过“状态定义+状态转移”将时间复杂度降至 O(n)，是该问题的最优解法。

2.2 核心思路（动态规划）

（1）状态定义

定义 dp[i] 表示“偷窃到第 i 间房屋时，能获得的最高金额”（i 从0开始，对应数组第 i 个元素）。

（2）状态转移方程

对于第 i 间房屋，有两种选择：

  • 不偷第 i 间：此时最高金额等于偷窃到第 i-1 间的最高金额，即 dp[i] = dp[i-1]。
  • 偷第 i 间：则第 i-1 间不能偷，此时最高金额等于偷窃到第 i-2 间的最高金额 + 第 i 间的金额，即 dp[i] = dp[i-2] + nums[i]。

综上，状态转移方程为：
dp[i] = max(dp[i-1], dp[i-2] + nums[i])

（3）初始条件

  • 当只有1间房屋（i=0）：dp[0] = nums[0]（只能偷这一间）。
  • 当有2间房屋（i=1）：dp[1] = max(nums[0], nums[1])（选金额更高的那一间）。

2.3 算法流程

  1. 边界处理：若数组为空，返回0；若数组长度为1，直接返回该元素。
  2. DP数组初始化：创建长度与nums相同的dp数组，初始化前两个元素。
  3. 状态转移遍历：从第3间房屋（i=2）开始，按状态转移方程依次计算每个 dp[i]。
  4. 结果返回：dp[n-1]（n 为数组长度）即为偷窃到的最高金额。

3.2 代码逐行解释

  • 边界条件处理：若输入数组为空（null或长度0），没有房屋可偷，返回0。
  • 边界条件处理：若只有1间房屋，只能偷这一间，直接返回该房屋的金额。
  • 初始化DP数组：数组长度与输入nums一致，用于存储每一步的最优解。
  • dp[0] = nums[0]：第0间房屋的最优解就是偷它本身。
  • dp[1] = Math.max(dp[0], nums[1])：前两间房屋选金额更高的那一间，保证不触发警报。
  • 核心状态转移循环：从第2间房屋开始遍历，每一步都选择“不偷当前房屋（取前一间的最优解）”或“偷当前房屋（取前前间的最优解+当前金额）”中的最大值。
  • 返回最终结果：遍历完成后，DP数组最后一个元素即为偷完所有房屋能获得的最高金额。

4、算法性能

  • 时间复杂度：O(n)。仅需一次遍历数组（从索引2到末尾），每个元素的计算为常数时间 O(1)，整体时间复杂度与数组长度线性相关。
  • 空间复杂度：O(n)。需要额外创建长度为 n 的DP数组存储中间状态。

5、测试示例

以 nums = [2,7,9,3,1] 为例，分步计算DP数组：

  1. 初始化：dp[0] = 2，dp[1] = max(2,7) = 7。
  2. i=2（第3间房，金额9）：dp[2] = max(dp[1]=7, dp[0]+9=2+9=11) → 11。
  3. i=3（第4间房，金额3）：dp[3] = max(dp[2]=11, dp[1]+3=7+3=10) → 11。
  4. i=4（第5间房，金额1）：dp[4] = max(dp[3]=11, dp[2]+1=11+1=12) → 12。
  5. 最终返回 dp[4] = 12，与示例结果一致。

以 nums = [1,2,3,1] 为例：

  1. 初始化：dp[0] = 1，dp[1] = max(1,2) = 2。
  2. i=2（金额3）：dp[2] = max(2, 1+3=4) → 4。
  3. i=3（金额1）：dp[3] = max(4, 2+1=3) → 4。
  4. 最终返回4，与示例结果一致。
