Leetcode-215:实现寻找第k大元素-快速选择法

Created2026-01-30|Updated2026-01-30|Java学习笔记Leetcode Hot100
|Word Count:2.2k|Reading Time:8mins|Post Views:

1、题目

给定一个整数数组 nums 和一个整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素（非第 k 个不同元素），且要求时间复杂度尽可能优于 O(n log n)。

示例

示例 1：
输入：nums = [3,2,1,5,6,4], k = 2
输出：5
解释：数组排序后为 [1,2,3,4,5,6]，第2大的元素是5。

示例 2：
输入：nums = [3,2,3,1,2,4,5,5,6], k = 4
输出：4
解释：数组排序后为 [1,2,2,3,3,4,5,5,6]，第4大的元素是4。

提示

  • 1 <= k <= nums.length <= 10⁵
  • -10⁴ <= nums[i] <= 10⁴

2、分析

2.1 问题核心

寻找第 k 大元素的常规思路是「排序后取对应位置」（时间复杂度 O(n log n)），但面对 n=10⁵ 的大数据量时效率不足。

快速选择法（QuickSelect） 是最优解：基于快速排序的分治思想，仅递归处理「目标元素所在的子区间」，无需排序整个数组，平均时间复杂度可降至 O(n)。

2.2 核心优化点

（1）随机基准选择

快速选择的性能依赖基准值：若固定选区间首尾为基准，数组有序/逆序时会退化为 O(n²)。
随机选择基准索引 可避免最坏情况，使算法在概率上保持线性时间。

（2）三分组（荷兰国旗问题）优化

传统快速排序仅将数组分为「小于基准」「大于基准」两组，重复元素多会导致递归区间过大。
三分组策略 把数组划分为三部分：

  • 小于基准的元素（small组）
  • 等于基准的元素（equal组）
  • 大于基准的元素（big组）

若目标位置落在equal组，可直接返回基准值（无需递归）；仅目标在small/big组时递归，大幅减少递归次数。

2.3 算法流程

  1. 目标位置转换：第 k 大元素 = 数组「从小到大排序」后索引为 n - k 的元素（记为 targetPos）。
  2. 分治划分：
    • 随机选基准值，通过三指针在原数组划分出small/equal/big三组；
    • 对比 targetPos 与三组区间边界：
      • 若在small组（targetPos < less）：递归处理 [start, less-1]；
      • 若在big组（targetPos > bigger）：递归处理 [bigger+1, end]；
      • 若在equal组：直接返回基准值。
  3. 递归终止：区间仅含一个元素时，该元素即为目标。

2.4 三指针逻辑（荷兰国旗划分）

指针 初始值 作用
less start small组右边界（[start, less) 为已划分的small组）
bigger end big组左边界（(bigger, end] 为已划分的big组）
cur start 当前遍历指针，逐个检查数组元素

遍历规则：

  • nums[cur] < 基准：交换 nums[cur] 和 nums[less]，less++、cur++（扩展small组，当前元素已处理）；
  • nums[cur] > 基准：交换 nums[cur] 和 nums[bigger]，bigger--（扩展big组，新元素需重新检查，cur 不递增）；
  • nums[cur] == 基准：cur++（归入equal组，无需交换）。

3.2 代码逐行解释

（1）成员变量与入口函数

  • 初始化随机数生成器（成员变量）：避免在递归中重复创建对象，提升执行效率。
  • 入口函数核心逻辑：将「第k大」转换为「从小到大排序后的索引 nums.length - k」，调用核心方法 quickselect。

（2）快速选择核心方法

  • 递归终止条件：当处理区间 [start, end] 仅含一个元素时，该元素即为目标。
  • 随机生成基准索引：random.nextInt(end - start + 1) 生成 [0, end-start] 的随机数，加上 start 后范围为 [start, end]；
  • 获取基准值 pivot，后续基于该值划分数组。
  • 初始化三指针：分别对应small组右边界、big组左边界、当前遍历位置。
  • 三分组遍历逻辑：
    • 元素小于基准：交换后扩展small组，less 和 cur 均右移；
    • 元素大于基准：交换后扩展big组，仅 bigger 左移（新元素需重新检查）；
    • 元素等于基准：直接跳过，归入equal组。
  • 递归区间选择：
    • 目标在small组 → 递归左区间 [start, less-1]；
    • 目标在big组 → 递归右区间 [bigger+1, end]；
    • 目标在equal组 → 直接返回基准值（无需递归）。

（3）工具方法：swap

  • 提取交换逻辑为独立方法：简化主逻辑代码，提升可读性和可维护性。

4、算法性能

  • 时间复杂度：平均 O(n)，最坏情况（极端随机）O(n²)（随机基准可将最坏情况概率降至极低）；
  • 空间复杂度：O(log n)（递归调用栈深度，平均为对数级别；非递归实现可降至 O(1)）；
  • 核心优势：
    1. 原地操作，无需额外空间；
    2. 三分组优化对重复元素友好，大幅减少递归次数；
    3. 线性时间复杂度适配大数据量（如 n=10⁵）。

5、测试示例

以 nums = [3,2,1,5,6,4], k = 2 为例：

  1. 转换目标位置：n=6 → targetPos = 6-2 = 4；
  2. 随机选基准（假设选索引3，值为5）；
  3. 三分组划分后：
    • small组（<5）：[3,2,1,4] → 区间 [0,4)；
    • equal组（=5）：[5] → 区间 [4,4]；
    • big组（>5）：[6] → 区间 (4,5]；
  4. targetPos=4 落在equal组，直接返回基准值5，与预期一致。

以 nums = [3,2,3,1,2,4,5,5,6], k = 4 为例：

  1. 转换目标位置：n=9 → targetPos = 9-4 = 5；
  2. 随机选基准（假设选索引6，值为5）；
  3. 三分组划分后，targetPos=5 落在small组，递归处理左区间；
  4. 最终递归到equal组返回4，与预期一致。

6、扩展思考

  • 非递归实现：将递归替换为循环，手动维护待处理的区间（start/end），可将空间复杂度降至 O(1)；
  • 其他解法对比：
    • 堆排序：维护大小为k的小顶堆，时间复杂度 O(n log k)，适合「海量数据无法一次性加载」的场景；
    • 普通快排：时间复杂度 O(n log n)，效率低于快速选择，但可得到完整排序数组；
  • 适用场景：快速选择法适合「只需找第k大/小元素，无需完整排序」的场景，是TopK问题的经典解法。
