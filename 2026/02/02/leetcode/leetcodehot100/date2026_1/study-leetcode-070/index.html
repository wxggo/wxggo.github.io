Leetcode-070:爬楼梯-动态规划入门经典实现（附空间优化版）

Created2026-02-02|Updated2026-02-02|Java学习笔记Leetcode Hot100
|Word Count:2.8k|Reading Time:10mins|Post Views:

前言

爬楼梯问题是动态规划（DP）的入门经典题，其递推关系简单、状态定义直观，是理解动态规划「自底向上」解题思想的最佳切入点。本题的核心是找到状态转移的递推公式，基础动态规划实现仅需数行代码，而通过分析状态依赖的局部性，还能将空间复杂度从O(n)优化至O(1)（面试高频优化点）。本文以基础DP实现为核心，详细推导递推公式、定义DP数组含义，同时讲解空间优化的核心思路，夯实动态规划入门基础。

1、题目

假设你正在爬楼梯。需要爬 n 阶你才能到达楼顶。
每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

示例

示例 1：
输入：n = 2
输出：2
解释：有两种方法爬到楼顶：1. 1阶 + 1阶；2. 2阶。

示例 2：
输入：n = 3
输出：3
解释：有三种方法爬到楼顶：1. 1+1+1；2. 1+2；3. 2+1。

示例 3：
输入：n = 1
输出：1
解释：仅一种方法：1阶。

提示

  • 1 <= n <= 45

2、分析

2.1 动态规划核心解题思路

动态规划的核心是将复杂问题拆分为重叠子问题，通过求解子问题的最优解推导出原问题的解，并通过DP数组存储子问题的结果，避免重复计算。解决爬楼梯问题的动态规划思路分为四步核心步骤，这也是所有简单DP问题的通用解题模板。

2.2 爬楼梯DP四步核心推导（面试必背）

步骤1：定义DP数组的含义

设dp[i]表示爬到第i阶楼梯的不同方法数，这是整个DP解法的基础，明确状态定义后才能推导后续公式。

步骤2：推导状态转移递推公式（核心）

这是本题的关键，分析到达第i阶的所有可能路径：
要爬到第i阶，最后一步只有两种选择：

  1. 从第i-1阶爬1个台阶到达i阶，此时方法数等于爬到i-1阶的方法数，即dp[i-1]；
  2. 从第i-2阶爬2个台阶到达i阶，此时方法数等于爬到i-2阶的方法数，即dp[i-2]。

因此，爬到第i阶的总方法数为两种路径的方法数之和，递推公式为：
dp[i] = dp[i-1] + dp[i-2]

步骤3：确定DP数组的初始化条件

递推公式dp[i] = dp[i-1] + dp[i-2]需要dp[0]和dp[1]作为初始值，否则无法从i=2开始推导，初始化条件为：

  • dp[0] = 1：爬到第0阶（地面）的方法数为1，这是为了递推合理性的边界定义（无实际物理意义，仅保证i=2时dp[2] = dp[1]+dp[0] = 2，符合实际结果）；
  • dp[1] = 1：爬到第1阶只有1种方法（直接爬1阶），符合实际情况。

步骤4：确定遍历顺序

由于dp[i]的计算依赖前两个状态dp[i-1]和dp[i-2]，因此需要从左到右正序遍历（从i=2到i=n），依次计算每个dp[i]的值，最终dp[n]即为答案。

2.3 核心结论

爬楼梯问题的DP解法完全遵循通用模板，其递推公式与斐波那契数列一致，dp[i]等价于斐波那契数列的第i+1项（斐波那契数列定义：F(0)=0, F(1)=1, F(2)=1, F(3)=2…）。

3、程序：爬楼梯基础DP实现（O(n)空间）

3.1 完整代码

3.2 代码核心点逐行解释

  1. DP数组初始化int[] dp = new int[n + 1] ：数组长度为n+1，因为需要存储dp[0]到dp[n]的所有状态，索引与楼梯阶数一一对应，避免索引越界；
  2. 边界初始化dp[0] = dp[1] = 1 ：为递推公式提供初始值，是整个DP计算的「起点」；
  3. 正序遍历循环for (int i = 2; i <= n; i++) ：保证计算dp[i]时，dp[i-1]和dp[i-2]已经被计算完成，符合状态依赖关系；
  4. 状态转移dp[i] = dp[i-1] + dp[i-2] ：直接套用推导的递推公式，这是本题的核心代码，仅一行即可实现状态转移；
  5. 返回dp[n] ：DP数组的最后一个元素即为原问题的解，对应爬到第n阶的不同方法数。

4、面试高频拓展：空间优化版（O(1)空间，最优解）

4.1 优化核心思路

分析基础DP实现的状态依赖：计算dp[i]时，仅需要前两个状态dp[i-1]和dp[i-2]，后续的计算不会再用到dp[0]、dp[1]、…、dp[i-3]的值。因此，无需用长度为n+1的数组存储所有状态，仅需两个变量分别保存dp[i-1]和dp[i-2]，每次迭代更新这两个变量即可，将空间复杂度从O(n)降至**O(1)**（常数级空间），这是面试中面试官大概率会追问的优化点。

4.2 空间优化版完整代码

4.3 优化版代码解释

  1. 特殊边界判断if (n <=1) return 1 ：避免n=1时循环不执行，直接返回正确结果，简化逻辑；
  2. 双变量初始化prev2=1, prev1=1 ：分别对应dp[0]和dp[1]，替代DP数组的基础初始化；
  3. 变量更新逻辑 ：
    • cur = prev1 + prev2：计算当前阶数的方法数，等价于基础版的dp[i]；
    • prev2 = prev1：将「前一个值」变为新的「前前值」，为下一次迭代做准备；
    • prev1 = cur：将「当前值」变为新的「前一个值」，为下一次迭代做准备；
  4. 返回prev1 ：遍历结束后，prev1存储的是最后一次计算的cur，即dp[n]的值。

4.4 优化版执行示例（n=3）

直观理解变量更新过程：

  1. 初始化：prev2=1（dp[0]）、prev1=1（dp[1]）；
  2. i=2：cur=1+1=2 → prev2=1 → prev1=2（dp[2]=2）；
  3. i=3：cur=2+1=3 → prev2=2 → prev1=3（dp[3]=3）；
  4. 返回prev1=3，与示例结果一致。

5、算法性能分析

5.1 基础DP版

  • 时间复杂度：O(n)。仅需一次从2到n的正序遍历，循环执行n-1次，每次循环为常数级计算；
  • 空间复杂度：O(n)。开辟了长度为n+1的DP数组，空间开销与n成正比。

5.2 空间优化版

  • 时间复杂度：O(n)。遍历次数与基础版一致，无额外计算；
  • 空间复杂度：O(1)。仅使用了prev2、prev1、cur三个变量，空间开销为常数级，与n无关，是最优空间复杂度。

两种解法的时间复杂度一致，优化版在空间上更优，是面试中解决本题的首选解法。

6、边界情况天然适配

本文的基础DP版和空间优化版代码，均无需额外处理边界情况，天然适配题目所有输入范围：

  1. n=1：基础版直接返回dp[1]=1，优化版触发if (n<=1)返回1；
  2. n=2：基础版dp[2]=dp[1]+dp[0]=2，优化版i=2时cur=2，返回2；
  3. n=45：题目最大输入，两种解法均能在O(n)时间内计算完成，无性能问题。

7、拓展思考

7.1 进阶问题：扩展爬台阶数

若题目修改为每次可以爬1、2、3个台阶，递推公式如何修改？

  • 状态定义不变：dp[i]表示爬到第i阶的方法数；
  • 递推公式更新：dp[i] = dp[i-1] + dp[i-2] + dp[i-3]（最后一步可从i-1、i-2、i-3阶爬上来）；
  • 初始化条件更新：dp[0]=1, dp[1]=1, dp[2]=2。

该思路可推广至每次爬k个台阶的情况，递推公式为dp[i] = sum(dp[i-j])（j从1到k，且i-j>=0）。

7.2 与斐波那契数列的关联

爬楼梯的DP数组序列为：dp[0]=1, dp[1]=1, dp[2]=2, dp[3]=3, dp[4]=5, dp[5]=8...，与斐波那契数列（F(0)=0, F(1)=1, F(2)=1, F(3)=2, F(4)=3, F(5)=5…）的关系为：
dp[i] = F(i+1)
掌握这一关联，可直接用斐波那契数列的解法解决爬楼梯问题，思路互通。

7.3 为什么不用递归？

递归解法（自顶向下）虽能实现，但存在大量重复计算，时间复杂度为O(2^n)，当n=45时会严重超时；即使通过记忆化搜索（缓存子问题结果）将时间复杂度降至O(n)，空间复杂度仍为O(n)（递归栈+缓存），不如空间优化版的DP解法高效。因此，面试中不推荐使用纯递归解法。

总结

爬楼梯作为动态规划的入门经典题，其核心价值在于让初学者理解DP的通用解题模板和状态依赖的分析思路，本文核心总结如下：

  1. 解决简单DP问题的四步核心：定义DP数组含义 → 推导递推公式 → 初始化边界 → 确定遍历顺序，这是所有DP问题的通用解题思路；
  2. 爬楼梯的递推公式为dp[i] = dp[i-1] + dp[i-2]，其本质是斐波那契数列的变形；
  3. 基础DP实现空间复杂度为O(n)，通过分析状态依赖的局部性，可优化为O(1)空间的双变量解法，这是面试高频优化点；
  4. 两种解法的时间复杂度均为O(n)，空间优化版是本题的最优解，面试中优先书写。

掌握爬楼梯的DP解法，不仅能解决本题，更能为后续解决「打家劫舍」「最小花费爬楼梯」等简单DP问题打下基础。复习建议：手动推导n=5时的DP数组值和优化版变量更新过程，深刻理解递推公式和状态依赖关系，做到面试中能快速书写基础版和优化版两种代码。
