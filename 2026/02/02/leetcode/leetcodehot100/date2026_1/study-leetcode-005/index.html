Leetcode-005:最长回文子串-中心扩展法极简实现

Created2026-02-02|Updated2026-02-02|Java学习笔记Leetcode Hot100
|Word Count:2.9k|Reading Time:11mins|Post Views:

1、题目

给你一个字符串 s，找到 s 中最长的回文子串。若存在多个长度相同的最长回文子串，返回任意一个即可。

回文定义：一个字符串正读和反读完全相同，如"aba"、"bb"、"a"。
子串定义：字符串中连续的字符序列（区别于子序列，子序列可不连续）。

示例

示例 1：
输入：s = “babad”
输出：”bab” 或 “aba”
解释：两个均为最长回文子串，长度为3。

示例 2：
输入：s = “cbbd”
输出：”bb”
解释：最长回文子串为”bb”，长度为2。

示例 3：
输入：s = “a”
输出：”a”
解释：单个字符本身就是回文。

示例 4：
输入：s = “ac”
输出：”a” 或 “c”
解释：无长度大于1的回文子串，返回任意单个字符。

提示

  • 1 <= s.length <= 1000
  • s 仅由数字和英文字母组成

2、分析

2.1 问题核心

最长回文子串的核心特性是中心对称：无论回文长度是奇数还是偶数，都可以找到一个「中心」，从中心向左右两边扩展，字符依次相等则为回文。
这是中心扩展法的设计依据，也是该题最适合面试的解法——实现简单、空间开销极小，无需复杂数据结构，仅通过两次遍历+双指针扩展即可求解。

2.2 核心难点：为什么要分「奇回文」和「偶回文」？

回文子串的中心类型由长度奇偶性决定，不存在统一的通用中心，必须分两种情况处理，这是中心扩展法的关键：

  • 奇回文：长度为奇数，中心是单个字符，如"aba"的中心是中间的"b"，"abcba"的中心是中间的"c"；
  • 偶回文：长度为偶数，中心是两个相邻的相同字符，如"bb"的中心是两个"b"之间的间隙，"abba"的中心是两个"b"之间的间隙。

若只处理一种中心，会遗漏另一种类型的回文子串，导致无法找到真正的最长回文。

2.3 核心思路

  1. 初始化边界：用ansLeft和ansRight记录最长回文子串的左右边界（适配Javasubstring左闭右开的特性）；
  2. 字符数组转换：将字符串转为字符数组，减少遍历中charAt()的调用开销，提升效率；
  3. 遍历所有可能的中心：
    • 奇回文遍历：以每个字符为中心（l=r=i），向左右扩展；
    • 偶回文遍历：以每两个相邻字符为中心（l=i, r=i+1），向左右扩展；
  4. 双指针扩展：对每个中心，左指针l左移、右指针r右移，直到越界或字符不相等，停止扩展；
  5. 更新最长回文边界：扩展停止后，[l+1, r-1]为当前中心的最长回文子串区间，计算其长度，若大于已记录的最长长度，则更新ansLeft和ansRight；
  6. 返回结果：根据最终的ansLeft和ansRight，通过substring截取最长回文子串。

2.4 算法流程

  1. 处理边界：若字符串长度为1，直接返回原字符串；
  2. 初始化ansLeft=0、ansRight=0，初始最长回文长度为1（单个字符）；
  3. 奇回文扩展：遍历每个字符为中心，双指针扩展并更新最长边界；
  4. 偶回文扩展：遍历每两个相邻字符为中心，双指针扩展并更新最长边界；
  5. 截取并返回s.substring(ansLeft, ansRight)。

3.2 代码核心点逐行解释

（1）基础初始化

  • 转字符数组：避免多次调用s.charAt(i)（每次调用会做边界检查），提升遍历效率；
  • 边界变量：ansLeft/ansRight初始为0，对应初始最长回文为第一个字符，长度ansRight-ansLeft=1。

（2）奇回文核心逻辑

  • 中心定义：l=r=i，以单个字符为中心；
  • 扩展条件：l>=0（左不越界）、r<length（右不越界）、chars[l]==chars[r]（字符相等），三者同时满足才继续扩展。

（3）回文区间与边界更新

  • 实际回文区间：最后一次扩展不满足条件，故l多左移一位、r多右移一位，真实回文是[l+1, r-1]；
  • 长度计算：r-l-1是当前回文长度（如l=0, r=2，长度为2-0-1=1，对应[1,1]）；
  • 适配substring：Javasubstring(a,b)是左闭右开，故直接将ansRight设为r，无需r-1，截取时自动取到r-1。

（4）偶回文核心逻辑

  • 中心定义：l=i, r=i+1，以两个相邻字符为中心，其余扩展、更新逻辑与奇回文完全一致，实现复用性。

（5）结果返回

  • 直接根据记录的边界截取，利用左闭右开特性，完美匹配扩展得到的回文区间[ansLeft, ansRight-1]。

4、算法性能

  • 时间复杂度：O(n²)（n为字符串长度）。需遍历2n个中心（n个奇回文中心+n个偶回文中心），每个中心最多向左右扩展n/2次，整体为二次方复杂度；
  • 空间复杂度：O(n)（仅为字符数组的存储空间，若直接使用s.charAt(i)，可降至O(1)原地操作）；
  • 核心优势（面试重点）：
    1. 实现极简，无复杂数据结构，面试中可快速书写，不易出错；
    2. 空间开销极小，远超动态规划解法（动态规划需O(n²)的二维数组）；
    3. 逻辑直观，基于回文中心对称的特性，易理解、易调试；
    4. 适配题目数据规模（n<=1000），执行效率足够。

5、测试示例

以示例2：s=”cbbd”（偶回文典型） 为例，完整执行过程：

  1. 初始化：length=4，chars=['c','b','b','d']，ansLeft=0，ansRight=0（初始长度1）；
  2. 奇回文扩展：
    • i=0（c）：l=0,r=0，扩展后l=-1,r=1，长度1-(-1)-1=1，不更新边界；
    • i=1（b）：l=1,r=1，扩展后l=0,r=2（chars[0]=’c’≠chars[2]=’b’），长度2-0-1=1，不更新边界；
    • i=2（b）：l=2,r=2，扩展后l=1,r=3（chars[1]=’b’≠chars[3]=’d’），长度3-1-1=1，不更新边界；
    • i=3（d）：l=3,r=3，扩展后l=2,r=4，长度4-2-1=1，不更新边界；
  3. 偶回文扩展：
    • i=0（c&b）：chars[0]≠chars[1]，无扩展，长度0，不更新；
    • i=1（b&b）：l=1,r=2，chars[1]==chars[2]，扩展后l=0,r=3（chars[0]≠chars[3]），长度3-0-1=2，大于当前1 → 更新ansLeft=1，ansRight=3；
    • i=2（b&d）：chars[2]≠chars[3]，无扩展，不更新；
    • i=3（d&空）：r=4越界，无扩展，不更新；
  4. 截取结果：s.substring(1,3) → "bb"，与示例一致。

以示例1：s=”babad”（奇回文典型） 为例，核心步骤：

  • 奇回文i=1（a）：扩展后l=-1,r=3，长度3 → 更新ansLeft=0,ansRight=3，对应子串"bab"；
  • 奇回文i=2（b）：扩展后l=0,r=4，长度3 → 与当前最长一致，不更新（保留任意一个即可）；
  • 最终返回"bab"或"aba"，符合题目要求。

6、扩展思考

6.1 与其他解法的对比（面试高频问答）

解法 时间复杂度 空间复杂度 优点 缺点 面试推荐度
中心扩展法 O(n²) O(1)/O(n) 实现极简、空间开销小 二次方时间复杂度 ⭐⭐⭐⭐⭐
动态规划 O(n²) O(n²) 思路通用，适配多类回文题 空间开销大、代码繁琐 ⭐⭐⭐
Manacher算法 O(n) O(n) 线性时间复杂度，最优解 实现极复杂，易出错 ⭐⭐

结论：面试中优先选择中心扩展法，兼顾实现难度和效率，是面试官最认可的解法。

6.2 边界情况天然适配

代码无需额外处理边界，天然支持所有场景：

  • 字符串长度为1：直接返回原字符串（两次遍历无有效扩展，边界保持0,0，substring(0,0)？不，初始长度1，ansRight-ansLeft=0，第一次奇回文i=0扩展后长度1，更新ansRight=1，返回s.substring(0,1)即原字符）；
  • 无长度>1的回文（如”ac”）：最终边界为0,1，返回第一个字符；
  • 全相同字符（如”aaaaa”）：奇回文和偶回文扩展都会取到最长长度，最终返回原字符串。

总结

中心扩展法是解决最长回文子串问题的面试最优解，其核心在于抓住回文「中心对称」的本质，通过奇偶回文分治解决中心不统一的问题，再利用双指针向左右扩展，记录最长回文的边界。

本文重点拆解了「为什么分奇偶」「扩展后边界如何计算」「如何适配substring左闭右开」等核心问题，同时提供了代码优化版本。掌握该解法后，不仅能解决本题，还能快速迁移到「回文子串数量」「最短回文串」等同类回文问题，是字符串模块的必备知识点。
