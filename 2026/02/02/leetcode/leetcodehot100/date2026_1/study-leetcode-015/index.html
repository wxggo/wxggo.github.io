Leetcode-015:三数之和-排序+双指针最优解（含去重+剪枝）

Created2026-02-02|Updated2026-02-02|Java学习笔记Leetcode Hot100
|Word Count:3.4k|Reading Time:12mins|Post Views:

1、题目

给你一个整数数组 nums，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j != k 且 nums[i] + nums[j] + nums[k] = 0。请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组，顺序不限。

示例

示例 1：
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：两个不重复的三元组和均为0。

示例 2：
输入：nums = [0,1,1]
输出：[]
解释：无满足条件的三元组。

示例 3：
输入：nums = [0,0,0,0]
输出：[[0,0,0]]
解释：仅一个不重复的三元组。

提示

  • 3 <= nums.length <= 3000
  • -10^5 <= nums[i] <= 10^5

2、分析

2.1 问题核心

三数之和的核心是将三元组问题转化为有序数组的双指针两数之和问题：固定一个数nums[i]，寻找另外两个数使其和为-nums[i]，即满足nums[i] + nums[left] + nums[right] = 0。
暴力解法需三层循环枚举所有三元组，时间复杂度O(n³)，面对n=3000时计算量达2.7×10¹⁰，必然超时；排序+双指针是最优解，排序将时间复杂度降为O(nlogn)，双指针遍历将内层两层循环降为一层，整体时间复杂度为O(n²)，适配题目数据规模。

本题的两大关键难点（面试必考）：

  1. 去重：避免答案中出现重复的三元组，需对固定数、左指针、右指针分别去重；
  2. 剪枝：提前跳过无效循环，减少计算量，提升程序执行效率。

2.2 排序的三大核心作用

排序是整个解法的基础，没有排序则双指针无法移动、去重和剪枝无从谈起，核心作用有三：

  1. 双指针移动依据：有序数组中，两数之和过小则左指针右移，过大则右指针左移，可线性寻找目标和；
  2. 方便去重：相同元素相邻排列，只需判断当前元素与前/后元素是否相等，即可快速跳过重复值；
  3. 支持剪枝优化：有序数组中，可通过简单的和判断，提前终止或跳过无效的固定数遍历。

2.3 核心难点1：三层去重逻辑（为什么要去重？怎么去重？）

去重的本质是跳过重复的起始点，避免生成重复的三元组，需在三个关键位置执行去重，缺一不可：

  1. 固定数i的去重：若nums[i] == nums[i-1]（i>0），说明当前数与上一个数相同，继续遍历会生成重复三元组，直接continue；
    • 注意：必须判断i>0，否则i-1会数组越界；
  2. 左指针left的去重：找到有效三元组后，若nums[left] == nums[left+1]，说明后续左指针值重复，需left++跳过；
  3. 右指针right的去重：找到有效三元组后，若nums[right] == nums[right-1]，说明后续右指针值重复，需right--跳过；
    • 注意：左右指针去重需在找到有效三元组后执行，且需保证left < right，避免越界。

2.4 核心难点2：两步剪枝优化（为什么能剪枝？）

剪枝是对程序的效率优化，利用有序数组的特性，提前跳过无效循环，减少不必要的计算：

  1. 提前终止循环：若nums[i] + nums[i+1] + nums[i+2] > 0，说明当前固定数nums[i]已过大，后续数均≥nums[i]，三数之和必然>0，直接break终止整个固定数遍历；
  2. 跳过无效固定数：若nums[i] + nums[length-2] + nums[length-1] < 0，说明当前固定数nums[i]过小，即使与数组最后两个最大数相加仍<0，无满足条件的三元组，直接continue跳过当前i。

2.5 核心思路

  1. 数组排序：对nums进行升序排序，为双指针、去重、剪枝打下基础；
  2. **遍历固定数i**：遍历数组，固定第一个数nums[i]，遍历边界为length-2（需留两个数给左右指针）；
  3. 固定数去重+剪枝：执行i的去重逻辑，再执行两步剪枝优化，跳过无效情况；
  4. 双指针初始化：左指针left = i+1（固定数下一个数），右指针right = length-1（数组末尾）；
  5. 双指针遍历：当left < right时，计算nums[left] + nums[right]与目标值-nums[i]比较：
    • 等于目标值：收集三元组，执行左右指针去重，再同时移动左右指针寻找新组合；
    • 小于目标值：左指针右移（增大两数之和）；
    • 大于目标值：右指针左移（减小两数之和）；
  6. 返回结果：遍历完成后，返回收集到的所有不重复三元组。

3.2 代码核心点逐行解释

（1）数组排序

  • 整个解法的基础，升序排序后所有后续逻辑才有依据，无排序则本题无法高效求解；
  • Java内置排序为快速排序优化版，时间复杂度O(nlogn)。

（2）固定数去重

  • 必须加i>0，否则i=0时i-1=-1，数组越界；
  • 跳过重复的固定数，从源头上避免生成重复三元组，是最关键的去重步骤。

（3）两步剪枝优化

  • 剪枝1用break：有序数组中后续数更大，三数之和只会更大，直接终止整个循环；
  • 剪枝2用continue：仅当前i过小，后续i增大后可能有有效解，仅跳过当前i。

（4）双指针与目标值定义

  • left=i+1：保证left在i之后，避免重复使用同一个元素（i!=left）；
  • target=-nums[i]：将三数之和为0转化为两数之和为target，简化问题。

（5）找到有效三元组后的去重与指针移动

  • 去重需加left < right，避免指针越界；
  • 先去重再移动指针，确保下一次遍历的指针值为新值，无重复；
  • 同时移动左右指针：有序数组中，单个指针移动后的两数之和必然不等于目标值，需同时移动。

（6）三元组收集

  • Arrays.asList快速生成固定大小的列表，满足题目返回要求；
  • 因排序后数组升序，收集的三元组也为升序，天然避免因顺序不同导致的重复（如[-1,0,1]和[0,-1,1]）。

4、算法性能

  • 时间复杂度：O(n²)（整体由主导步骤决定）；
    • 排序：O(nlogn)；
    • 固定数遍历：O(n)，每层遍历中双指针遍历：O(n)，合计O(n²)；
    • 去重和剪枝为常数级操作，不影响整体时间复杂度。
  • 空间复杂度：O(logn)（排序的递归栈空间，若忽略排序的额外空间，可认为是O(1)原地操作）；
    • 结果集ans的存储空间为题目要求的输出空间，不计入算法额外空间。
  • 核心优势（面试重点）：
    1. 时间复杂度O(n²)为本题最优，完美适配n=3000的数规模；
    2. 去重+剪枝双重优化，实际执行效率远高于基础版排序+双指针；
    3. 代码逻辑清晰，无复杂数据结构，面试中可快速书写；
    4. 思路可迁移至四数之和「Leetcode18」等多数之和问题。

5、测试示例

以经典示例：nums = [-1,0,1,2,-1,-4] 为例，完整模拟执行过程，直观理解核心逻辑：

  1. 排序后：nums = [-4,-1,-1,0,1,2]，length=6，ans=[]；
  2. i=0（nums[i]=-4）：
    • 无去重，剪枝1：-4+(-1)+(-1)=-6<0，剪枝2：-4+1+2=-1<0 → continue；
  3. i=1（nums[i]=-1）：
    • 无去重（i=1，nums[1]≠nums[0]），剪枝1：-1+(-1)+0=-2<0，剪枝2：-1+1+2=2>0 → 进入双指针；
    • left=2（-1），right=5（2），target=1；
    • sum=-1+2=1 == target → 收集[-1,-1,2]，ans=[[-1,-1,2]]；
    • 左指针去重：nums[2]==nums[3]？-1≠0 → 不执行；右指针去重：nums[5]≠nums[4] → 不执行；
    • left=3，right=4；
    • sum=0+1=1 == target → 收集[-1,0,1]，ans=[[-1,-1,2],[-1,0,1]]；
    • 左右指针去重均不执行，left=4，right=3 → 双指针循环终止；
  4. i=2（nums[i]=-1）：
    • 去重：i>0且nums[2]==nums[1] → continue；
  5. i=3（nums[i]=0）：
    • 剪枝1：0+1+2=3>0 → break，整个固定数循环终止；
  6. 返回结果：[[-1,-1,2],[-1,0,1]]，与示例一致。

6、扩展思考

6.1 暴力解法的局限性

暴力解法通过三层循环枚举所有i<j<k的组合，判断和为0并收集，最后去重，代码如下：

  • 时间复杂度O(n³)，n=3000时完全超时；
  • 需借助HashSet去重，且每次收集后需排序，额外空间和时间开销大；
  • 无任何剪枝，存在大量无效计算，工程中无实用价值。

6.2 去重的注意事项（面试易出错点）

  1. 固定数去重不能放在剪枝之后：若先剪枝再去重，会导致重复的i未被跳过，生成重复三元组；
  2. 左右指针去重必须在找到有效三元组后：若提前去重，会跳过可能的有效组合；
  3. 所有去重步骤都需加边界判断（i>0/left<right），避免数组越界。

6.3 多数之和的通用思路

三数之和的排序+双指针思路可直接迁移至四数之和「Leetcode18」、五数之和等问题，通用模板为：

  1. 对数组进行升序排序；
  2. 遍历固定前k-2个数（三数之和固定1个，四数之和固定2个），每层都执行去重和剪枝；
  3. 最后两个数用双指针遍历，寻找和为目标值的组合；
  4. 收集所有不重复的组合并返回。

该模板将k数之和的时间复杂度从O(n^k)降至O(n^(k-1))，是解决多数之和问题的最优思路。

6.4 为什么不能用哈希表解三数之和？

哈希表可解两数之和，但解三数之和时，去重逻辑会变得异常复杂，且需要额外的哈希表存储已遍历的数，空间复杂度升至O(n)，同时无法进行剪枝优化，实际执行效率远低于排序+双指针解法，因此面试中不推荐使用。

总结

三数之和的排序+双指针解法是数组问题中的经典最优解，其核心在于将三元组问题转化为有序数组的双指针两数之和问题，并通过三层去重和两步剪枝解决本题的两大关键难点。
