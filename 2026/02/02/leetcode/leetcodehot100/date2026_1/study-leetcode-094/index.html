Leetcode-094:二叉树的中序遍历-递归极简实现（附先序/后序/迭代写法）

Created2026-02-02|Updated2026-02-02|Java学习笔记Leetcode Hot100
|Word Count:2.5k|Reading Time:8mins|Post Views:

前言

二叉树的中序、先序、后序遍历是二叉树模块的入门基础，也是Leetcode和面试的高频考点，其中中序遍历是最常用的遍历方式（如二叉搜索树的中序遍历为升序序列）。递归遍历是三种遍历的最简实现方式，仅需调整节点处理顺序，即可实现中序、先序、后序的切换，代码量极少且易理解；迭代遍历则基于栈模拟递归调用过程，是面试中递归解法的常见追问点。本文以Leetcode94二叉树的中序遍历为核心，详解递归实现，顺带梳理先序/后序的递归写法，简要讲解迭代遍历的核心思路，夯实二叉树遍历基础。

1、题目

给你一棵二叉树的根节点 root，返回其节点值的中序遍历结果。

中序遍历定义

二叉树的中序遍历遵循左子树 → 根节点 → 右子树的顺序，即先遍历当前节点的左子树，再访问当前节点，最后遍历右子树（深度优先遍历DFS）。

示例

示例 1：
输入：root = [1,null,2,3]
输出：[1,3,2]
解释：先遍历1的左子树（空），访问1；再遍历1的右子树2，先遍历2的左子树3（访问3），再访问2，最终顺序1→3→2。

示例 2：
输入：root = []
输出：[]

示例 3：
输入：root = [1]
输出：[1]

提示

  • 树中节点数目在范围 [0, 100] 内
  • -100 <= Node.val <= 100

2、分析

2.1 问题核心

中序遍历的核心是严格遵循左根右的遍历顺序，递归实现的底层是分治思想和递归栈：将原问题拆分为「遍历左子树」「访问根节点」「遍历右子树」三个子问题，每个子问题继续递归拆分，直到遇到空节点（递归终止条件）。

2.2 递归遍历的核心优势

递归是实现二叉树遍历的首选方式，尤其适合面试快速书写，核心优势：

  1. 代码极简，逻辑直观，仅需数行代码即可实现，不易出错；
  2. 完全贴合遍历的定义，无需额外数据结构辅助；
  3. 先序、后序遍历仅需调整根节点的处理时机，即可快速切换，复用性极强。

2.3 递归核心思路

  1. 递归终止条件：当当前节点为null时，直接返回，无需处理（空树无节点可遍历）；
  2. 分治处理：严格遵循左根右顺序，依次执行「递归遍历左子树」→「将当前节点值加入结果集」→「递归遍历右子树」；
  3. 结果集存储：用全局/成员变量列表存储遍历结果，避免递归过程中频繁创建列表，提升效率。

5、拓展2：中序遍历的迭代实现（栈模拟递归，面试追问点）

递归的底层是JVM的方法调用栈，迭代遍历则通过显式创建栈模拟递归栈的执行过程，手动控制节点的入栈、出栈顺序，实现非递归的遍历。迭代写法是面试中递归解法的常见追问，核心思路是先遍历到左子树最底层，依次入栈所有左节点；出栈时处理根节点，再处理右子树。

5.2 迭代核心思路简要解释

  1. 栈的作用：存储待处理的根节点，保证出栈顺序符合「左根右」；
  2. 双层循环：外层循环控制遍历终止（指针为空且栈空），内层循环负责将左子树所有节点入栈；
  3. 节点处理顺序：出栈节点即为「当前根节点」，处理后将指针指向其右子树，重复左入栈、出栈、处理右的过程；
  4. 先序/后序迭代：仅需调整节点入栈/出栈的时机和根节点的处理时机，核心都是利用栈模拟递归，其中后序遍历的迭代写法稍复杂（可借助两个栈或标记法）。

6、算法性能分析

无论是递归遍历还是迭代遍历，二叉树的中序/先序/后序遍历性能一致，因为每个节点都会被访问且仅访问一次，栈/递归栈的空间开销与树的高度相关。

6.1 时间复杂度

O(n) ，n为二叉树的节点数。每个节点入栈/出栈各一次，或递归访问一次，无重复计算，时间复杂度为线性级别。

6.2 空间复杂度

O(h) ，h为二叉树的高度。

  • 递归遍历：空间开销为递归调用栈，栈的深度等于树的高度；
  • 迭代遍历：空间开销为显式栈，栈中存储的节点数最多为树的高度。
  • 最坏情况：二叉树退化为单链表（斜树），h=n，空间复杂度为**O(n)**；
  • 最好情况：满二叉树/完全二叉树，h=logn，空间复杂度为**O(logn)**。

7、关键边界情况处理

本文的递归和迭代代码均天然适配所有边界情况，无需额外判断：

  1. 空树（root=null）：递归直接返回，迭代栈为空，结果集为空，返回[]；
  2. 单节点树：仅根节点，递归遍历左（空）→ 处理根 → 遍历右（空），结果集为[val]；
  3. 斜树（全左/全右节点）：严格遵循左根右顺序，遍历结果符合预期。

总结

本文以Leetcode94二叉树中序遍历为核心，详解了递归极简实现，并梳理了二叉树遍历的核心知识点，关键总结如下：

  1. 中序遍历的核心是左根右，递归实现的关键是递归终止条件+分治处理，代码极简，面试首选；
  2. 先序/后序遍历的递归写法仅需调整根节点的处理时机，先序「根左右」、后序「左右根」，可快速切换；
  3. 迭代遍历的核心是用显式栈模拟递归栈，中序迭代遵循「左入栈→出栈处理根→处理右」的逻辑，是面试递归解法的常见追问点；
  4. 三种遍历的时间复杂度均为O(n)，空间复杂度均为O(h)，与树的高度相关。
