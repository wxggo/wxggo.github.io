Leetcode-283:移动零-双指针两种极简实现

Created2026-02-06|Updated2026-02-06|Java学习笔记Leetcode Hot100
|Word Count:1.8k|Reading Time:6mins|Post Views:

1、题目

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

说明：必须在原数组上操作，不能拷贝额外的数组，尽量减少操作次数。

示例

示例 1：
输入：nums = [0,1,0,3,12]
输出：[1,3,12,0,0]

示例 2：
输入：nums = [0]
输出：[0]

提示

  • 1 <= nums.length <= 10^4
  • -2^31 <= nums[i] <= 2^31 - 1

2、分析

2.1 问题核心

移动零的核心是在原地操作的前提下，将非零元素按原顺序前移，零元素集中移到末尾。常规思路（新建数组存储非零元素再补零）虽简单，但违反“不能拷贝额外数组”的要求；暴力法（遇到零则逐个后移）时间复杂度高（O(n²)），非最优解。

双指针法是该题的最优解：通过两个指针分工协作，在一次/两次遍历内完成操作，时间复杂度O(n)、空间复杂度O(1)，且保证非零元素相对顺序不变。本文提供两种双指针实现思路：覆盖补零法、交换法。

2.2 核心思路

方法1：覆盖补零法（两次遍历）

核心是“先收集所有非零元素到数组前部，再统一补零”：

  1. 初始化指针：定义index指针（记录非零元素应放置的位置），初始值为0；
  2. 收集非零元素：遍历数组，将所有非零元素依次覆盖到index位置，每覆盖一个非零元素，index自增1；
  3. 统一补零：遍历结束后，index及之后的位置全部置为0（这些位置原本的元素已被覆盖，需填充零）。

方法2：交换法（一次遍历）

核心是“遇到非零元素就与零指针位置交换，零指针后移”：

  1. 初始化指针：定义i0指针（记录下一个非零元素应交换到的位置），初始值为0；
  2. 遍历交换：遍历数组，遇到非零元素时，将其与i0位置的元素交换，交换后i0自增1；
  3. 结果达成：遍历结束后，所有非零元素已按原顺序前移至数组前部，零元素自然集中在末尾。

3.2 代码简洁解释

方法1：覆盖补零法

  • index指针：本质是“非零元素的尾指针”，始终指向当前最后一个非零元素的下一个位置；
  • 第一次遍历：仅做覆盖操作，不改变非零元素的相对顺序，效率高；
  • 第二次遍历：仅处理需要补零的位置，操作次数等于零元素的数量，无冗余计算。

方法2：交换法

  • i0指针：本质是“零元素的头指针”，始终指向当前第一个未被交换的零元素位置；
  • 交换逻辑：非零元素与零元素交换，既完成非零元素前移，又完成零元素后移，一次遍历即可完成；
  • 无额外补零：交换操作天然将零元素“挤”到数组末尾，无需二次遍历。

4、算法性能

解法 时间复杂度 空间复杂度 遍历次数 核心优势
覆盖补零法 O(n) O(1) 2次 逻辑简单，易理解，无交换开销
交换法 O(n) O(1) 1次 遍历次数更少，操作更精简

通用性能分析

  • 时间复杂度：两种方法均为O(n)（n为数组长度）。覆盖补零法两次遍历总操作数仍为n；交换法一次遍历，每个元素仅判断+最多一次交换；
  • 空间复杂度：均为O(1)。仅使用少量临时变量（index/i0/temp），无额外数组、哈希表等空间开销，严格满足“原地操作”要求；
  • 非零顺序保证：两种方法均通过“按顺序遍历+按顺序前移/交换”，完全保证非零元素的相对顺序不变。

5、测试示例

以nums = [0,1,0,3,12]为例，演示两种方法的执行过程：

方法1：覆盖补零法

  1. 初始化：index=0；
  2. 遍历i=0：nums[0]=0 → 跳过；
  3. 遍历i=1：nums[1]=1≠0 → nums[0]=1，index=1；
  4. 遍历i=2：nums[2]=0 → 跳过；
  5. 遍历i=3：nums[3]=3≠0 → nums[1]=3，index=2；
  6. 遍历i=4：nums[4]=12≠0 → nums[2]=12，index=3；
  7. 补零阶段：i从3到4，nums[3]=0，nums[4]=0；
  8. 最终数组：[1,3,12,0,0]。

方法2：交换法

  1. 初始化：i0=0；
  2. 遍历i=0：nums[0]=0 → 跳过；
  3. 遍历i=1：nums[1]=1≠0 → 交换nums[1]与nums[0]，数组变为[1,0,0,3,12]，i0=1；
  4. 遍历i=2：nums[2]=0 → 跳过；
  5. 遍历i=3：nums[3]=3≠0 → 交换nums[3]与nums[1]，数组变为[1,3,0,0,12]，i0=2；
  6. 遍历i=4：nums[4]=12≠0 → 交换nums[4]与nums[2]，数组变为[1,3,12,0,0]，i0=3；
  7. 遍历结束，最终数组：[1,3,12,0,0]。

6、扩展思考

6.1 两种解法对比

  • 覆盖补零法：更适合新手理解，逻辑拆分清晰（收集非零+补零），无交换操作，对数值类型无特殊要求；
  • 交换法：更适合追求极致效率的场景，一次遍历完成所有操作，交换开销远小于二次遍历的补零开销（尤其当零元素占比高时）。

6.2 边界情况处理

两种解法天然适配所有边界场景，无需额外判断：

  • 数组全为零：覆盖补零法index=0，二次遍历全置零；交换法无交换操作，数组保持全零；
  • 数组无零：覆盖补零法二次遍历无操作；交换法每个元素与自身交换（无实际变化），结果不变；
  • 数组长度为1：无论是否为零，两种方法均能正确处理。

6.3 拓展应用

双指针思路可扩展至同类“数组元素分类移动”问题：

  • 移动指定值元素到数组末尾（如将所有2移到末尾，保持其他元素顺序）；
  • 数组奇偶分离（奇数在前、偶数在后，保持各自相对顺序）；
  • 核心思路不变：用一个指针记录目标位置，另一个指针遍历数组，通过覆盖/交换完成元素重排。
