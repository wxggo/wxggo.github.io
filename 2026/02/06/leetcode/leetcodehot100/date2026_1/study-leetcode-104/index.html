Leetcode-104:二叉树的最大深度-递归法极简实现

Created2026-02-06|Updated2026-02-06|Java学习笔记Leetcode Hot100
|Word Count:2.4k|Reading Time:8mins|Post Views:

1、题目

给定一个二叉树，找出其最大深度。二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

示例

示例 1：

输入：root = [3,9,20,null,null,15,7] 输出：3

示例 2：
输入：root = [1,null,2]
输出：2

提示

  • 树中节点的数量范围是 [0, 10000]
  • -100 <= Node.val <= 100

2、分析

2.1 问题核心

求二叉树最大深度的核心是遍历二叉树的所有路径，找到根节点到叶子节点的最长路径长度，关键在于利用二叉树的递归结构，将“整棵树的最大深度”拆解为子问题求解。

本题有两种经典的递归实现思路，均属于深度优先搜索（DFS）范畴：

  • 自底向上（分治递归）：将整棵树的最大深度拆解为“左子树最大深度”和“右子树最大深度”，取最大值加当前节点的贡献（+1）；
  • 自顶向下（DFS递归）：从根节点开始逐层遍历，记录当前路径的深度，实时更新全局最大深度。

两种思路均能高效解决问题，核心区别在于“深度计算的方向”：自底向上是从叶子节点回溯到根节点，自顶向下是从根节点遍历到叶子节点。

2.2 核心思路

思路1：自底向上（分治递归）

  1. 边界处理：若节点为空，说明空节点无深度贡献，返回0；
  2. 递归分解：将“求当前节点子树的最大深度”拆解为：
    • 递归计算左子树的最大深度，结果加1（包含当前节点）；
    • 递归计算右子树的最大深度，结果加1（包含当前节点）；
  3. 结果合并：取左、右子树深度的最大值，即为当前节点子树的最大深度；
  4. 递归终止：遍历到叶子节点的空子节点时返回0，逐层回溯计算。

思路2：自顶向下（DFS递归）

  1. 全局变量：定义全局变量记录遍历过程中遇到的最大深度；
  2. 递归参数：传入当前节点和当前路径的深度；
  3. 边界处理：若节点为空，说明路径已到尽头，直接返回；
  4. 深度更新：当前节点非空时，深度加1（计入当前节点），并更新全局最大深度；
  5. 递归遍历：分别递归遍历左、右子树，传递当前深度；
  6. 结果返回：遍历完成后，全局变量即为整棵树的最大深度。

3.2 代码简洁解释

解法1（自底向上）

  • 空节点判空：空树/空节点无深度，直接返回0，是递归的终止条件；
  • 左/右子树深度计算：maxDepth(root.left)得到左子树的最大深度，加1是因为当前节点是左子树的父节点，需计入深度；
  • Math.max(left, right) ：合并子问题结果，取左右子树深度的最大值，即为当前节点子树的最大深度；
  • 执行逻辑：先遍历到叶子节点（左右子节点均为空），再从叶子节点回溯到根节点，逐层计算深度。

解法2（自顶向下）

  • 全局变量ans：用于实时记录遍历过程中遇到的最大深度，初始值为0；
  • dfs函数：核心遍历函数，参数为“当前节点”和“当前路径深度”；
  • depth++ ：当前节点非空时，深度加1（当前节点计入路径）；
  • ans更新：每次遍历到节点时，比较当前深度与全局最大深度，保留较大值；
  • 执行逻辑：从根节点开始，逐层向下遍历每个节点，记录每条路径的深度，最终全局变量ans即为最长路径的深度。

4、算法性能

4.1 时间复杂度

两种解法的时间复杂度均为O(n)（n为二叉树的节点数）：

  • 每个节点仅被递归遍历一次，无重复计算；
  • 自底向上解法中，每个节点的计算仅包含两次子递归调用和一次最大值比较，均为常数级操作；
  • 自顶向下解法中，每个节点仅执行深度更新和两次子递归调用，无额外开销。

4.2 空间复杂度

两种解法的空间复杂度均为O(height)（height为二叉树的高度）：

  • 空间开销来自递归调用栈的深度，栈深度等于二叉树的高度；
  • 最优情况（平衡二叉树）：height = logn，空间复杂度为O(logn)；
  • 最坏情况（二叉树退化为链表）：height = n，空间复杂度为O(n)；
  • 注意：解法2的全局变量仅占用常数级空间，不影响整体空间复杂度。

4.3 核心优势对比

解法 核心优势 适用场景
自底向上 无全局变量，纯函数式递归 追求代码简洁、无副作用
自顶向下 直观体现“路径遍历”思想 需记录遍历过程中的中间值

5、测试示例

以root = [3,9,20,null,null,15,7]为例，分别分析两种解法的执行过程：

5.1 解法1（自底向上）执行过程

  1. 根节点3非空，计算左子树（节点9）深度：
    • 节点9非空，左子树（null）深度=0，右子树（null）深度=0；
    • 节点9的深度=Math.max(0+1, 0+1)=1；
  2. 计算根节点3的右子树（节点20）深度：
    • 节点20非空，左子树（节点15）深度=1（节点15的左右子树均为null）；
    • 节点20的右子树（节点7）深度=1（节点7的左右子树均为null）；
    • 节点20的深度=Math.max(1+1, 1+1)=2；
  3. 根节点3的深度=Math.max(1, 2)=3，最终返回3。

5.2 解法2（自顶向下）执行过程

  1. 初始化ans=0，调用dfs(3, 0)：
    • 节点3非空，depth=1，ans更新为1；
    • 调用dfs(9, 1)：
      • 节点9非空，depth=2，ans更新为2；
      • 调用dfs(null, 2)：直接返回；
      • 调用dfs(null, 2)：直接返回；
    • 调用dfs(20, 1)：
      • 节点20非空，depth=2，ans仍为2；
      • 调用dfs(15, 2)：
        • 节点15非空，depth=3，ans更新为3；
        • 调用dfs(null, 3)：返回；
        • 调用dfs(null, 3)：返回；
      • 调用dfs(7, 2)：
        • 节点7非空，depth=3，ans仍为3；
        • 调用dfs(null, 3)：返回；
        • 调用dfs(null, 3)：返回；
  2. 遍历完成，ans=3，最终返回3。

6、扩展思考

6.1 两种递归解法对比

维度 自底向上（分治） 自顶向下（DFS）
计算方向 叶子节点 → 根节点 根节点 → 叶子节点
变量依赖 无全局变量，纯返回值传递 依赖全局变量记录中间结果
思维方式 分治思想（分解-合并） 遍历思想（逐路径记录）
代码简洁度 更简洁，一行核心逻辑 稍繁琐，需额外辅助函数

6.3 关键边界情况处理

两种递归解法均天然适配所有边界场景：

  • 空树：根节点判空直接返回0（解法1）/dfs直接返回，ans保持0（解法2）；
  • 单节点树：解法1返回Math.max(0+1, 0+1)=1，解法2遍历到节点后depth=1，ans=1；
  • 链表型二叉树（如[1,2,3,4,null,null,5]）：两种解法均能遍历到最深节点，正确返回最大深度。

6.4 拓展应用

两种递归思路可扩展至多个二叉树深度相关问题：

  1. 求二叉树的最小深度（Leetcode111）：
    • 自底向上：需判断节点是否为叶子节点（避免将“单侧空节点”误判为最小深度）；
    • 自顶向下：遍历到叶子节点时立即更新最小深度，提前终止无效遍历；
  2. 判断平衡二叉树（Leetcode110）：基于自底向上思路，计算深度的同时判断左右子树深度差是否超过1；
  3. 求二叉树的直径（Leetcode543）：基于自底向上思路，记录“左深度+右深度”的最大值，即为直径。
