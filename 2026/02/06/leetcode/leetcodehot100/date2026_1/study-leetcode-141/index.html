Leetcode-141:环形链表-快慢指针法极简实现

Created2026-02-06|Updated2026-02-06|Java学习笔记Leetcode Hot100
|Word Count:1.5k|Reading Time:5mins|Post Views:

1、题目

给你一个链表的头节点 head ，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

示例

示例 1：

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。

示例 2：
输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。

示例 3：
输入：head = [1], pos = -1
输出：false
解释：链表中没有环。

提示

  • 链表中节点的数目范围是 [0, 10^4]
  • -10^5 <= Node.val <= 10^5
  • pos 为 -1 或者链表中的一个有效索引

2、分析

2.1 问题核心

判断链表是否存在环的核心是检测节点是否被重复访问，关键难点是在不使用额外存储空间（或仅使用常数级空间）的前提下高效完成检测。

快慢指针法（弗洛伊德龟兔赛跑算法） 是该题的最优解：通过两个速度不同的指针遍历链表，若链表有环，快指针终将追上慢指针；若链表无环，快指针会率先到达链表末尾（指向null）。该方法无需额外空间，时间复杂度为线性级别，实现简单且效率极高。

2.2 核心思路

  1. 初始化指针：定义慢指针slow和快指针fast，初始均指向链表头节点head；
  2. 快慢遍历：慢指针每次向后移动1步，快指针每次向后移动2步：
    • 若链表无环：快指针会先遇到null（或fast.next为null），此时可判定无环；
    • 若链表有环：快慢指针会进入环内循环，由于快指针速度更快，最终一定会追上慢指针（slow == fast），此时可判定有环；
  3. 终止条件：快指针遍历到链表末尾（fast == null 或 fast.next == null）时终止循环，返回false；若循环中快慢指针相遇，直接返回true。

3.2 代码简洁解释

  • 快慢指针初始化：slow和fast均从链表头节点开始，保证遍历起点一致；
  • while循环条件：fast != null && fast.next != null 避免快指针访问null的next属性导致空指针异常，同时覆盖“链表节点数为0/1/2”等边界情况；
  • 指针移动规则：慢指针每次走1步，快指针每次走2步，利用速度差实现环的检测；
  • 相遇判定：若快慢指针指向同一个节点，说明链表存在环（环内循环使快指针追上慢指针）；
  • 无环返回：快指针率先遍历到链表末尾，说明链表无环，返回false。

4、算法性能

  • 时间复杂度：O(n)（n为链表节点数）。无论链表有无环，快慢指针最多遍历n次：
    • 无环场景：快指针遍历n/2次到达末尾，循环终止；
    • 有环场景：快慢指针在环内相遇前的遍历次数不超过n；
  • 空间复杂度：O(1)。仅使用了快慢两个指针，无额外空间开销，属于原地操作；
  • 核心优势：
    1. 相比哈希表法（O(n)空间复杂度），快慢指针法更节省内存；
    2. 代码极简，逻辑清晰，无复杂数据结构操作；
    3. 时间效率最优，线性遍历无重复计算。

5、测试示例

示例1：有环场景（head = [3,2,0,-4], pos = 1）

  1. 初始化：slow→3，fast→3；
  2. 第一次循环：slow→2，fast→0；（未相遇）
  3. 第二次循环：slow→0，fast→2；（未相遇）
  4. 第三次循环：slow→-4，fast→0；（未相遇）
  5. 第四次循环：slow→2，fast→2；（快慢指针相遇，返回true）

示例3：无环场景（head = [1], pos = -1）

  1. 初始化：slow→1，fast→1；
  2. 循环条件检查：fast.next == null，循环不执行；
  3. 返回false，符合预期。

6、扩展思考

6.1 哈希表解法对比

哈希表法通过记录已访问的节点实现环检测，代码易懂但空间复杂度较高：

  • 时间复杂度：O(n)，与快慢指针法一致；
  • 空间复杂度：O(n)，最坏情况下需存储所有节点。

6.2 进阶问题：找环的入口节点

若需进一步找到环的入口节点，可在快慢指针相遇后，将其中一个指针重置为头节点，然后两个指针以相同速度（每次1步）遍历，再次相遇的节点即为环的入口（Leetcode142题考点）。

6.3 边界情况处理

代码天然适配所有边界场景，无需额外判断：

  • 链表为空（head = null）：循环条件不满足，直接返回false；
  • 链表仅有一个节点（无环）：fast.next == null，返回false；
  • 链表仅有两个节点且成环：快慢指针第二次循环即可相遇，返回true。
