Leetcode-004:寻找两个有序数组的中位数-二分查找（时间O(log(min(m,n)))）

Created2026-02-06|Updated2026-02-06|Java学习笔记Leetcode Hot100
|Word Count:3.2k|Reading Time:12mins|Post Views:

前言

寻找两个有序数组的中位数是Leetcode数组模块的经典难题，也是面试中考察二分查找灵活运用的高频压轴题。本题的核心要求是设计时间复杂度为O(log(m+n))的算法，暴力合并数组的解法虽直观但时间复杂度为O(m+n)，无法满足要求。我们以二分查找分割点为核心解法，详细推导分割点的计算逻辑、边界处理规则，拆解二分过程的每一步思路，最终实现时间复杂度优化至O(log(min(m,n)))的最优解。

1、题目

给定两个大小分别为 m 和 n 的有序数组 nums1 和 nums2。请你找出并返回这两个有序数组的中位数。

核心定义回顾

中位数的作用是将一个集合划分为两个长度相等的子集，其中一个子集中的元素总是大于另一个子集的元素：

  • 若总长度为奇数，中位数是排序后数组的中间元素；
  • 若总长度为偶数，中位数是排序后数组的中间两个元素的平均值。

示例

示例 1：
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组为 [1,2,3]，中位数是 2。

示例 2：
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组为 [1,2,3,4]，中位数是 (2+3)/2 = 2.5。

提示

  • nums1.length == m
  • nums2.length == n
  • 0 <= m, n <= 1000
  • 1 <= m + n <= 2000
  • -10^6 <= nums1[i], nums2[i] <= 10^6

2、分析

2.1 问题核心：中位数的分割本质

两个有序数组合并后的中位数，本质是找到一个分割方式，将 nums1 和 nums2 分别划分为左、右两部分：

满足两个核心条件：

  1. 长度平衡：左半部分总长度 = 右半部分总长度（或左半部分比右半部分多1，适配奇数总长度）；
  2. 数值有序：左半部分的最大值 ≤ 右半部分的最小值。

此时：

  • 若总长度为奇数，中位数 = 左半部分的最大值；
  • 若总长度为偶数，中位数 = (左半部分最大值 + 右半部分最小值) / 2。

2.2 核心难点：如何高效找到分割点？

暴力解法是合并两个有序数组（时间O(m+n)），再取中位数，但不符合O(log(m+n))的时间要求。

二分查找分割点是最优思路：利用数组有序的特性，在较短的数组上进行二分查找（减少二分次数），通过调整分割点i（nums1的分割点）和j（nums2的分割点），满足上述两个核心条件，将时间复杂度优化至O(log(min(m,n)))。

2.3 核心公式推导（分割点的约束）

为满足「长度平衡」，推导分割点的核心公式：
设总长度为total = m + n，左半部分总长度totalLeft = (total + 1) / 2（+1是为了让奇数总长度时左半部分多1，简化中位数计算）。
则 i + j = totalLeft，即 j = totalLeft - i。

其中：

  • i 的取值范围：0 ≤ i ≤ m（i=0表示nums1左半部分为空，i=m表示nums1右半部分为空）；
  • j 的取值范围：0 ≤ j ≤ n（同理j=0/j=n对应nums2的空分割）。

2.4 核心判断条件（数值有序）

为满足「左半部分最大值 ≤ 右半部分最小值」，只需保证：
nums1[i-1] ≤ nums2[j] 且 nums2[j-1] ≤ nums1[i]（边界情况需特殊处理，如i=0/i=m/j=0/j=n）。

若 nums1[i-1] > nums2[j]，说明nums1左半部分偏大，需将i左移（减小i）；
若 nums2[j-1] > nums1[i]，说明nums1左半部分偏小，需将i右移（增大i）。

3.2 代码核心点逐行解释

（1）保证短数组二分（效率优化）

  • 核心逻辑：二分查找的次数由数组长度决定，短数组的二分次数更少（如m=100, n=1000时，二分次数为log2(100)≈7次，远少于log2(1000)≈10次）；
  • 作用：将时间复杂度从O(log(max(m,n)))优化至O(log(min(m,n)))，进一步降低时间开销。

（2）二分查找初始化

  • left/right：定义nums1分割点i的二分范围（0~m）；
  • totalLeft：左半部分总长度，+1是关键技巧：无论总长度奇偶，都能保证奇数时左半部分多1，统一中位数计算逻辑。

（3）二分查找核心循环

  • 分割点i的计算：left + (right - left + 1)/2 是「上中位数」取法，避免left=right-1时的死循环（如left=2, right=3时，i=3而非2，能跳出循环）；
  • 条件判断：nums1[i-1] > nums2[j] 说明nums1左半部分偏大，需将i左移（right=i-1）；否则保留当前i并尝试右移（left=i），直到找到最优分割点。

（4）边界值处理（关键容错）

  • 核心逻辑：当分割点在数组两端（如i=0表示nums1左半部分为空），用Integer.MIN_VALUE/Integer.MAX_VALUE替代，保证后续Math.max/Math.min的比较逻辑成立；
  • 示例：若i=0（nums1左半为空），则nums1Left=MIN_VALUE，Math.max(nums1Left, nums2Left) 等价于直接取nums2Left，符合实际逻辑。

（5）中位数计算

  • 奇数总长度：左半部分多1个元素，最大值即为中位数；
  • 偶数总长度：左半最大值 + 右半最小值的平均值，需除以2.0（保证浮点型结果）。

4、算法性能分析

本解法是寻找两个有序数组中位数的最优解，时间复杂度达到题目要求的对数级别，空间复杂度为常数级。

4.1 时间复杂度：O(log(min(m,n)))

  • 二分查找仅在较短的数组上进行，二分次数为log2(min(m,n))；
  • 每次二分迭代的操作均为O(1)（分割点计算、条件判断）；
  • 总时间复杂度为O(log(min(m,n)))，远优于暴力合并的O(m+n)，适配题目输入规模。

4.2 空间复杂度：O(1)

  • 仅使用了有限个变量（left、right、i、j等），无额外数组或数据结构；
  • 空间开销为常数级，是最优的空间复杂度。

4.3 核心优势

  1. 时间最优：对数级时间复杂度，满足题目核心要求；
  2. 逻辑统一：通过totalLeft = (m+n+1)/2统一奇偶长度的分割逻辑，减少代码分支；
  3. 边界鲁棒：极值替代法处理分割点边界，兼容空数组、单元素数组等所有场景；
  4. 面试友好：代码简洁，核心逻辑清晰，易拆解、易讲解。

5、经典示例手动模拟（直观理解）

以示例2：nums1 = [1,2], nums2 = [3,4]（m=2, n=2） 为例，手动模拟二分过程：

5.1 初始化

  • 保证nums1更短（已满足），totalLeft = (2+2+1)/2 = 2；
  • 二分范围：left=0，right=2。

5.2 第一次二分（left=0, right=2）

  • i = 0 + (2-0+1)/2 = 1，j = 2 - 1 = 1；
  • 检查条件：nums1[0] (1) ≤ nums2[1] (4)，满足，left = 1。

5.3 第二次二分（left=1, right=2）

  • i = 1 + (2-1+1)/2 = 2，j = 2 - 2 = 0；
  • 检查条件：nums1[1] (2) ≤ nums2[0] (3)，满足，left = 2。

5.4 二分结束（left=right=2）

  • 最终分割点：i=2，j=0；
  • 边界值处理：
    • nums1Left = 2==2 ? MIN_VALUE : nums1[1] → MIN_VALUE；
    • nums1Right = 2==2 ? MAX_VALUE : nums1[2] → MAX_VALUE；
    • nums2Left = 0==0 ? MIN_VALUE : nums2[-1] → MIN_VALUE；
    • nums2Right = 0==2 ? MAX_VALUE : nums2[0] → 3；
  • 中位数计算（总长度偶数）：
    • 左半最大值 = max(MIN_VALUE, MIN_VALUE) = MIN_VALUE？→ 修正：实际i=2时nums1左半为[1,2]，j=0时nums2左半为空，因此nums1Left应为nums1[1]=2，nums2Left=MIN_VALUE，左半最大值=2；
    • 右半最小值 = min(MAX_VALUE, 3) = 3；
    • 中位数 = (2+3)/2 = 2.5，与示例结果一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 其中一个数组为空：如nums1=[], nums2=[1]，i=0，j=1，左半最大值=1，中位数=1；
  2. 单元素数组：如nums1=[2], nums2=[], i=1，j=0，左半最大值=2，中位数=2；
  3. 总长度为1：所有逻辑正常生效，无数组越界风险。

6.2 为什么取上中位数（i = left + (right-left+1)/2）？

若取「下中位数」（i = left + (right-left)/2），当left=right-1时会陷入死循环：

  • 例：left=1, right=2，下中位数i=1，若满足条件则left=1，循环无法结束；
  • 上中位数能保证每次迭代都能缩小范围，最终跳出循环。

6.3 与暴力解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
暴力合并数组 O(m+n) O(m+n) 逻辑直观 效率低、占空间 小数据量测试
二分查找 O(log(min(m,n))) O(1) 时间空间最优 逻辑稍抽象 面试最优书写

总结

寻找两个有序数组中位数的二分查找解法是本题的最优解，也是理解二分查找灵活运用的关键，本文核心总结如下：

  1. 中位数的本质是「平衡分割」：左半部分总长度=右半部分（或多1），且左半最大值≤右半最小值；
  2. 核心公式：分割点j = (m+n+1)/2 - i，保证长度平衡；
  3. 效率优化：在短数组上二分，将时间复杂度降至O(log(min(m,n)))；
  4. 边界处理：用极值替代空分割的情况，保证比较逻辑成立；
  5. 奇偶统一：通过totalLeft = (m+n+1)/2统一奇偶长度的分割逻辑，简化中位数计算。

掌握该解法的核心是理解「分割点的二分调整逻辑」，这也是面试中考察的核心，需熟练推导分割点的约束条件和边界处理规则。
