Leetcode-042:接雨水-前后缀最大值预处理经典解法（双数组版）

Created2026-02-03|Updated2026-02-03|Java学习笔记Leetcode Hot100
|Word Count:3.6k|Reading Time:12mins|Post Views:

前言

接雨水问题是Leetcode数组模块的经典难题，也是面试高频考察的压轴题之一，其核心考察对数组预处理和木桶短板效应的理解。本题的解法有多种，其中前后缀最大值双数组预处理是最直观、最易理解的基础最优解，该解法通过一次遍历构建左右两侧的最大高度数组，将每个位置的储水量计算从暴力的重复查询优化为O(1)查询，整体时间复杂度降至线性级别。本文以该解法为核心，详细推导储水量计算逻辑、拆解前后缀数组的构建过程，为后续理解更优的双指针空间优化版打下坚实基础。

1、题目

给定 n 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

核心图示理解

柱子高度构成的轮廓为[0,1,0,2,1,0,1,3,2,1,2,1]时，下雨后能接6个单位的雨水（灰色部分为雨水），是本题的经典示例。

示例

示例 1：
输入：height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出：6
解释：上面是由数组表示的高度图，接雨水后共能存储6个单位的雨水。

示例 2：
输入：height = [4,2,0,3,2,5]
输出：9
解释：按高度图排列，雨水存储量为9个单位。

示例 3：
输入：height = [2,0,2]
输出：2
解释：两侧柱子高2，中间柱子高0，可接2个单位雨水。

提示

  • n == height.length
  • 1 <= n <= 2 * 10^4
  • 0 <= height[i] <= 10^5

2、分析

2.1 问题核心：木桶短板效应的延伸

接雨水问题的核心遵循木桶短板效应，且是双端短板：每个位置能存储的雨水量，由该位置左侧所有柱子的最大高度和右侧所有柱子的最大高度中的较小值决定。

这是因为雨水是被左右两侧的高柱子「兜住」的，左侧最高柱和右侧最高柱形成一个「木桶」，雨水的高度无法超过这个木桶的短板，而当前柱子的高度会占据一部分空间，因此单个位置的储水量 = 左右最大高度的较小值 - 当前柱子的高度（结果非负，若为负则表示无法储水）。

2.2 核心难点：如何快速获取每个位置的左右最大高度？

暴力解法的思路是，对每个位置i，分别向左遍历找到左侧最大高度、向右遍历找到右侧最大高度，再计算储水量。但该方法的时间复杂度为O(n²)，面对2*10^4的数组长度会超时。

前后缀最大值预处理是解决该难点的最优思路：通过一次遍历构建两个辅助数组——preMax（前缀最大高度数组）和sufMax（后缀最大高度数组），提前存储每个位置的左右最大高度，后续计算储水量时可直接O(1)查询，将整体时间复杂度优化至O(n)。

2.3 核心概念定义（解法基础）

为后续推导清晰，先明确两个核心数组的严格定义，这是构建数组和计算储水量的前提：

  1. 前缀最大高度数组preMax：preMax[i]表示从数组起始位置到第i个位置（含i）的柱子最大高度（即i左侧+自身的最大高度）；
  2. 后缀最大高度数组sufMax：sufMax[i]表示从第i个位置到数组末尾（含i）的柱子最大高度（即i右侧+自身的最大高度）。

2.4 核心解题思路

  1. 初始化辅助数组：创建与height等长的preMax和sufMax数组，为递推构建做准备；
  2. 递推构建前后缀数组：从左到右遍历构建preMax，从右到左遍历构建sufMax（可在一次循环中完成，优化代码简洁性）；
  3. 计算总储水量：遍历每个位置i，根据核心公式计算单个位置的储水量，累加所有位置的储水量得到总结果；
  4. 返回总结果：累加完成后，总储水量即为答案。

2.5 核心储水量公式（面试必背）

基于木桶短板效应和前后缀数组的定义，推导出单个位置的储水量计算公式（核心中的核心）：
v = Math.min(preMax[i], sufMax[i]) - height[i]

  • 公式合理性：Math.min(preMax[i], sufMax[i])是左右最大高度的短板，即该位置能容纳的最大雨水高度；
  • 非负性保证：由于preMax[i]和sufMax[i]均包含height[i]本身，因此Math.min(preMax[i], sufMax[i]) >= height[i]，v恒≥0，无需额外判断负数情况。

3.2 代码核心点逐行解释

（1）边界优化判断

  • 核心逻辑：接雨水需要至少3根柱子（两侧高、中间低），少于3根时无法形成储水区域，直接返回0；
  • 作用：提前过滤无效输入，减少后续无意义计算，提升程序效率。

（2）前后缀数组初始化

  • preMax[0]：数组第一个位置的前缀最大值就是自身高度，无左侧元素；
  • sufMax[length-1]：数组最后一个位置的后缀最大值就是自身高度，无右侧元素；
  • 作用：为后续的递推构建提供初始边界值，是数组构建的「起点」。

（3）一次循环构建双数组（代码优化亮点）

  • 前缀数组递推：preMax[i]依赖前一个位置的preMax[i-1]，左到右遍历保证递推的有效性；
  • 后缀数组索引映射：sufIndex = length-1-i实现右到左遍历，如i=1时对应倒数第二个位置，i=2时对应倒数第三个位置，直至遍历到数组起始位置；
  • 后缀数组递推：sufMax[sufIndex]依赖后一个位置的sufMax[sufIndex+1]，右到左遍历保证递推的有效性；
  • 优化价值：将两次独立的遍历合并为一次，代码更简洁，遍历次数仍为O(n)，不影响时间复杂度。

（4）核心储水量计算与累加

  • 直接套用推导的核心公式，O(1)完成单个位置储水量计算；
  • 无需判断v是否为负：因preMax[i]和sufMax[i]均包含height[i]，Math.min(preMax[i], sufMax[i]) >= height[i]，v恒为非负。

4、算法性能分析

本解法的时间和空间复杂度均为线性级别，是接雨水问题的基础最优解，兼顾效率和理解性。

4.1 时间复杂度：O(n)

n为数组height的长度，整个解法包含三次线性遍历（实际为两次，边界判断为常数级），且每次遍历的操作均为O(1)：

  1. 前后缀数组构建：一次循环遍历n-1次，O(n)；
  2. 总储水量计算：一次循环遍历n次，O(n)；
  • 总时间复杂度为O(n) + O(n) = O(n)，完美适配题目2*10^4的输入规模。

4.2 空间复杂度：O(n)

空间开销主要来自两个辅助数组preMax和sufMax，二者均与height等长，总空间开销为O(n) + O(n) = O(n)。

  • 该空间复杂度是基础解法的必然开销，后续可通过双指针法将空间复杂度优化至O(1)（面试高频追问点，见拓展思考）。

4.3 核心优势

  1. 逻辑直观：完全贴合接雨水的核心原理，易理解、易推导，适合面试入门书写；
  2. 效率最优：线性时间复杂度，无超时风险；
  3. 代码简洁：一次循环构建双数组，减少代码冗余；
  4. 扩展性强：是理解双指针优化版的基础，掌握后可快速迁移至更优解法。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 数组长度<3：边界判断直接返回0，正确；
  2. 数组为严格递增/递减：如[1,2,3,4]或[4,3,2,1]，每个位置的preMax[i]=sufMax[i]=height[i]，v=0，总储水量0，正确；
  3. 数组全为相同高度：如[2,2,2]，v=0，总储水量0，正确；
  4. 存在0高度柱子：如示例中的i=2、i=5，公式正常计算，正确。

6.2 与暴力解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
暴力解法 O(n²) O(1) 无需额外空间 效率极低 小数据量测试
双数组预处理 O(n) O(n) 效率高、直观 需额外空间 面试基础书写
双指针法 O(n) O(1) 时间空间最优 逻辑稍抽象 面试最优书写

6.3 面试高频追问：空间优化（双指针O(1)版）

双数组解法的空间复杂度为O(n)，面试中面试官大概率会追问如何将空间优化至O(1)，核心思路是用两个指针替代两个数组，用变量保存当前的左右最大高度，边遍历边计算储水量：

核心思路

  1. 定义左指针left=0、右指针right=length-1，保存左最大高度leftMax=0、右最大高度rightMax=0，总储水量ans=0；
  2. 若height[left] < height[right]，则当前左指针位置的储水高度由leftMax决定（短板在左），更新leftMax并计算储水量，左指针右移；
  3. 否则，当前右指针位置的储水高度由rightMax决定（短板在右），更新rightMax并计算储水量，右指针左移；
  4. 遍历至left >= right，返回ans。

简洁代码实现

6.4 进阶问题：三维接雨水

本题是二维接雨水（一维数组），面试中可能会拓展至三维接雨水（二维矩阵），核心思路仍基于木桶短板效应，需计算每个位置上下左右四个方向的最大高度，取最小值减去当前高度得到储水量，可通过BFS或动态规划预处理四个方向的最大高度。

总结

接雨水的前后缀最大值双数组预处理解法是本题的基础最优解，也是理解接雨水核心原理的关键，本文核心总结如下：

  1. 接雨水的核心原理是双端木桶短板效应，单个位置的储水量由左右最大高度的较小值决定；
  2. 双数组解法的核心是通过预处理将每个位置的左右最大高度存储为前后缀数组，将时间复杂度从暴力的O(n²)优化至O(n)；
  3. 核心公式需牢记：单个储水量 = min(前缀最大高度, 后缀最大高度) - 当前柱子高度，且结果天然非负；
  4. 双数组解法的时间复杂度O(n)、空间复杂度O(n)，兼顾效率和理解性，是面试入门的首选写法；
  5. 在此基础上，可通过双指针法将空间复杂度优化至O(1)，是面试的最优写法，需掌握其核心思路。
