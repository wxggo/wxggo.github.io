Leetcode-19:移除链表倒数第N个节点-双指针法

Created2026-02-03|Updated2026-02-03|Java学习笔记Leetcode Hot100
|Word Count:2.1k|Reading Time:7mins|Post Views:

1、题目

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

要求：尽可能只遍历链表一次。

示例

示例 1：
输入：head = [1,2,3,4,5], n = 2
输出：[1,2,3,5]
解释：链表倒数第2个节点是4，删除后链表变为[1,2,3,5]。

示例 2：
输入：head = [1], n = 1
输出：[]
解释：链表仅有一个节点，删除倒数第1个节点后为空。

示例 3：
输入：head = [1,2], n = 1
输出：[1]
解释：链表倒数第1个节点是2，删除后返回[1]。

提示

  • 链表中结点的数目为 sz，1 <= sz <= 30
  • 0 <= Node.val <= 100
  • 1 <= n <= sz

2、分析

2.1 问题核心

链表的特性是「单向遍历」，无法直接反向定位倒数第N个节点。常规思路是「两次遍历」：

  1. 第一次遍历统计链表长度 sz；
  2. 第二次遍历到 sz - n 位置，删除该节点。

但题目要求「尽可能只遍历一次」，双指针法（快慢指针） 是最优解：通过快慢指针的「步差」直接定位倒数第N个节点的「前驱节点」，仅需一次遍历即可完成删除，时间复杂度保持 O(n)，空间复杂度 O(1)。

2.2 核心设计点解析

（1）为什么引入dummy节点？

链表删除操作的痛点是「头节点无前置节点」：

  • 若要删除的是头节点（如示例2：链表[1]，n=1），直接操作head会导致空指针或返回错误；
  • dummy节点（哑节点）作为「虚拟头节点」，统一所有节点的删除逻辑（包括头节点），无需额外判断边界。

dummy节点的核心作用：

  • 让链表「所有节点（包括原头节点）都有前驱节点」；
  • 最终返回 dummy.next 即可得到删除后的链表头（无论原头节点是否被删除）。

（2）为什么是 while (right.next != null)？

双指针的核心是制造「n步的步差」，让慢指针停在「倒数第N个节点的前驱节点」：

  1. 先让快指针（right）先走n步，此时快慢指针间距为n；
  2. 再让快慢指针同步前进，直到快指针走到「最后一个节点」（right.next == null），此时慢指针（left）恰好停在「倒数第N个节点的前驱节点」。

若错误使用 while (right != null)：

  • 快指针会走到null，慢指针会停在「倒数第N个节点」，此时无法直接删除（需要前驱节点修改next指针）；
  • 极端情况（如删除头节点）会导致空指针异常。

2.3 算法流程

  1. 初始化dummy节点：dummy.next = head，快慢指针（left/right）均指向dummy；
  2. 快指针先走n步：制造n步的步差；
  3. 快慢指针同步前进：直到right.next == null（快指针到最后一个节点）；
  4. 删除目标节点：left.next = left.next.next（跳过倒数第N个节点）；
  5. 返回结果：dummy.next（删除后的链表头）。

3.2 代码逐行解释

（1）链表节点定义

  • 标准单链表节点定义：包含值val和指向下一节点的指针next。

（2）核心逻辑

  • 创建dummy哑节点，将其next指向原链表头节点；
  • 解决「头节点无前置节点」的删除边界问题。
  • 初始化快慢指针，均从dummy节点开始（保证步差计算的准确性）。
  • 快指针先走n步，此时快慢指针的间距为n，为后续定位倒数第N个节点做准备。
  • 快慢指针同步前进，直到快指针指向「最后一个节点」（right.next == null）；
  • 此时慢指针left恰好指向「倒数第N个节点的前驱节点」，是删除操作的关键位置。
  • 跳过倒数第N个节点，完成删除操作；
  • 由于dummy节点的存在，即使删除的是原头节点，该逻辑依然成立。
  • 返回删除后的链表头节点；
  • 若原头节点被删除，dummy.next会指向原第二个节点；若未删除，仍指向原头节点。

4、算法性能

  • 时间复杂度：O(n)，仅需一次遍历链表（快指针走n步 + 快慢指针同步走sz-n步，总计n + (sz-n) = sz步，sz为链表长度）；
  • 空间复杂度：O(1)，仅使用了dummy、left、right三个额外节点，无额外空间开销；
  • 核心优势：
    1. 一次遍历完成删除，效率优于两次遍历的暴力解法；
    2. dummy节点统一了边界处理，代码简洁且鲁棒性强；
    3. 双指针思路是链表问题的经典解法，可迁移至「找链表中点」「判断链表环」等场景。

5、测试示例

示例1：head = [1,2,3,4,5], n = 2

  1. dummy.next = [1,2,3,4,5]，left/right = dummy；
  2. 快指针走2步：right = 2（节点值）；
  3. 快慢指针同步前进，直到right.next == null：
    • right最终指向5（最后一个节点），left指向3；
  4. left.next = left.next.next → 3的next从4改为5；
  5. 返回dummy.next → [1,2,3,5]，符合预期。

示例2：head = [1], n = 1

  1. dummy.next = [1]，left/right = dummy；
  2. 快指针走1步：right = 1；
  3. 快慢指针同步前进：right.next == null，循环不执行；
  4. left.next = left.next.next → dummy的next从1改为null；
  5. 返回dummy.next → []，符合预期。

示例3：head = [1,2], n = 1

  1. dummy.next = [1,2]，left/right = dummy；
  2. 快指针走1步：right = 1；
  3. 快慢指针同步前进：right.next = 2 → 执行循环，left=1，right=2；
  4. right.next == null，循环终止，left.next = left.next.next → 1的next从2改为null；
  5. 返回dummy.next → [1]，符合预期。

6、扩展思考

（1）暴力解法对比（两次遍历）

暴力解法思路：

  1. 第一次遍历统计链表长度sz；
  2. 第二次遍历到sz - n - 1位置（倒数第N个节点的前驱），执行删除；
  3. 缺点：需要处理头节点删除的边界（如sz == n时直接返回head.next），且需两次遍历。

双指针法优势：一次遍历 + 无边界额外判断，代码更简洁高效。

（2）无dummy节点的写法（不推荐）

该写法需两次遍历，且边界判断增加代码复杂度，不如dummy节点方案优雅。

（3）适用场景扩展

双指针法是链表问题的核心技巧，可迁移至：

  • 寻找链表的中间节点（快慢指针，快指针走2步，慢指针走1步）；
  • 判断链表是否有环（快慢指针，快指针走2步，慢指针走1步，相遇则有环）；
  • 寻找链表的倒数第k个节点（与本文思路一致，仅不执行删除操作）。
