Leetcode-048:旋转图像（矩阵）-转置+翻转经典解法（原地修改版）

前言

旋转图像（矩阵顺时针旋转90度）是Leetcode数组/矩阵模块的经典高频题，也是面试中考察「原地算法」和「矩阵操作逻辑」的核心题目。本题的核心要求是不使用额外空间完成矩阵旋转，而「先转置矩阵、再翻转每一行」是最直观、最易理解的原地最优解法——该解法将复杂的90度旋转拆解为两个简单的矩阵基础操作，既符合直觉又能保证时间效率。

1、题目

2、分析

2.1 问题核心：顺时针旋转90度的拆解思路

顺时针旋转90度是一个复合操作，直接推导每个元素的旋转后坐标容易出错，而将旋转拆解为「矩阵转置 + 每行左右翻转」 是解决本题的核心思路——两个基础操作的组合，恰好等价于顺时针旋转90度，且均能在原地完成，满足题目空间要求。

2.2 核心概念与操作逻辑

为理解拆解思路，先明确两个基础操作的定义和旋转等价性：

  1. 矩阵转置：将矩阵的行和列互换（即 matrix[i][j] 与 matrix[j][i] 交换）；

2.3 核心难点：原地转置与翻转的循环边界

本题的核心难点不是「转置+翻转」的思路，而是如何在原地完成这两个操作，且不重复交换元素：

  • 转置的循环边界：若遍历所有 i 和 j 交换 matrix[i][j] 和 matrix[j][i]，会导致元素被重复交换（比如 i=0,j=1 交换后，i=1,j=0 又交换回来），因此需限制 j < i，只遍历矩阵的「下三角区域」（含对角线）；
  • 行翻转的循环边界：每行翻转只需遍历前半部分元素（j < n/2），与后半部分对称位置交换，避免重复操作。

3.2 代码核心点逐行解释

（1）矩阵维度获取

  • 核心逻辑：题目保证矩阵是 n×n 的方阵，因此只需获取行数（或列数）即可；
  • 作用：为后续循环边界提供基准值。

（2）原地矩阵转置（核心难点）

  • 循环边界 j < i：仅处理矩阵中「行号大于列号」的元素（下三角区域），比如：
    • i=1 时，j=0（交换 matrix[1][0] 和 matrix[0][1]）；
    • i=2 时，j=0、1（交换 matrix[2][0]&matrix[0][2]、matrix[2][1]&matrix[1][2]）；
  • 避免重复交换：若 j < n 会导致每个元素被交换两次（回到原位置），j < i 确保每个元素仅交换一次；
  • 原地实现：通过临时变量 temp 完成两个元素的交换，无额外空间开销。

（3）原地翻转每一行

  • 外层循环：遍历矩阵的每一行；
  • 内层循环边界 j < n/2：仅处理每行前半部分元素，与后半部分对称位置交换，比如：
    • n=3 时，j=0（交换第0列和第2列，中间列无需交换）；
    • n=4 时，j=0、1（交换第0列&第3列、第1列&第2列）；
  • 对称位置计算：matrix[i][n-1-j] 是第 i 行第 j 列的对称位置，比如 j=0 对应最后一列，j=1 对应倒数第二列。

4、算法性能分析

本解法是旋转矩阵的原地最优解，时间和空间复杂度均达到理论最优，兼顾效率和理解性。

4.1 时间复杂度：O(n²)

n 为矩阵的边长（n×n 矩阵），整个解法包含两次嵌套循环：

  1. 矩阵转置：遍历下三角区域，元素数量为 n(n-1)/2，时间复杂度 O(n²)；
  2. 行翻转：遍历每行前半部分，总元素数量为 n×(n/2)，时间复杂度 O(n²)；
  • 总时间复杂度为 O(n²) + O(n²) = O(n²)，完美适配题目 n≤20 的输入规模。

4.2 空间复杂度：O(1)

整个解法仅使用了临时变量 temp 作为交换媒介，未创建任何额外的数组/矩阵，所有操作均在原矩阵上完成，空间复杂度为常数级 O(1)，完全满足题目「原地修改」的要求。

4.3 核心优势

  1. 逻辑直观：将复杂的旋转拆解为两个基础矩阵操作，易理解、易推导，面试中不易出错；
  2. 原地最优：空间复杂度 O(1)，符合题目核心要求；
  3. 代码简洁：无冗余逻辑，核心操作仅两个循环，可读性高；
  4. 扩展性强：掌握该思路后，可快速推导逆时针旋转等变体问题的解法。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. n=1：转置循环 j < 0 无操作，行翻转循环 j < 0.5 无操作，矩阵保持不变，正确；
  2. n为偶数/奇数：
    • 偶数 n=4：行翻转 j < 2，恰好交换前两列与后两列，正确；
    • 奇数 n=3：行翻转 j < 1，中间列无需交换，正确；
  3. 负数元素：交换逻辑与元素值无关，负数也能正确旋转，正确。

6.3 与直接坐标映射解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
转置+翻转 O(n²) O(1) 逻辑直观、易书写 需两步操作 面试首选书写
坐标映射 O(n²) O(1) 一步到位 坐标公式易出错 追求极致简洁

坐标映射的核心公式（顺时针旋转90度）：原位置 (i,j) → 新位置 (j, n-1-i)，但需借助临时矩阵或分层次交换，逻辑复杂度高于转置+翻转。

总结

旋转图像（矩阵）的转置+翻转原地解法是本题的最优解，也是理解矩阵操作的核心思路，本文核心总结如下：

  1. 顺时针旋转90度的核心拆解思路：「矩阵转置 + 每行左右翻转」，两个基础操作的组合等价于顺时针旋转；
  2. 原地转置的关键：循环边界 j < i，仅遍历下三角区域，避免元素重复交换；
  3. 原地行翻转的关键：循环边界 j < n/2，仅遍历每行前半部分，避免元素重复交换；
  4. 本解法时间复杂度 O(n²)、空间复杂度 O(1)，是面试的首选写法；
  5. 基于该思路可快速推导逆时针旋转等变体问题，掌握矩阵操作的核心逻辑。
