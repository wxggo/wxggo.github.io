Leetcode-039:组合总和-深度优先搜索（DFS）回溯经典解法

Created2026-02-27|Updated2026-02-27|Java学习笔记Leetcode Hot100
|Word Count:3k|Reading Time:11mins|Post Views:

前言

组合总和问题是Leetcode回溯算法模块的经典入门题，也是面试高频考察的基础题之一，其核心考察对深度优先搜索（DFS） 和回溯思想的理解，尤其是“恢复现场”这一回溯算法的核心细节。

1、题目

给定一个无重复元素的整数数组 candidates 和一个目标整数 target，找出 candidates 中所有可以使数字和为 target 的组合。

candidates 中的数字可以无限制重复被选取，且解集不能包含重复的组合。

示例

示例 1：
输入：candidates = [2,3,6,7], target = 7
输出：[[2,2,3],[7]]
解释：2+2+3=7，7=7，这两组是所有和为7的合法组合，且无重复。

示例 2：
输入：candidates = [2,3,5], target = 8
输出：[[2,2,2,2],[2,3,3],[3,5]]
解释：2*4=8、2+3*2=8、3+5=8，三组均为合法组合。

示例 3：
输入：candidates = [2], target = 1
输出：[]
解释：2大于1，无法组成和为1的组合，返回空列表。

提示

  • 1 <= candidates.length <= 30
  • 2 <= candidates[i] <= 40
  • candidates 中的所有元素互不相同
  • 1 <= target <= 40

2、分析

2.1 问题核心：回溯算法的“选或不选”分支决策

组合总和问题的核心遵循回溯算法的“选或不选”核心思想：
对数组中的每个元素，我们有两个决策分支——「选当前元素」或「不选当前元素」，通过递归遍历所有分支，筛选出元素和等于target的组合，即为合法解。

由于题目允许元素无限制重复选取，「选当前元素」的分支中，下一次递归仍从当前元素的索引开始；「不选当前元素」的分支中，下一次递归需跳过当前元素（索引+1），这一规则既保证了元素可重复选，又避免了出现[2,3]和[3,2]这类重复组合。

2.2 核心难点：递归的“现场恢复”与结果集的正确存储

回溯算法的核心难点有两个，也是本题最易出错的点：

  1. 现场恢复：递归过程中，存储当前路径的path是引用类型（所有递归分支共享同一个path），若选完元素后不“撤销选择”，后续递归分支会使用被修改的path，导致路径错误；
  2. 结果集存储副本：若直接将path加入结果集，由于path是引用，后续修改会同步改变结果集中的内容，因此必须存储path的副本。

2.3 核心解题思路（DFS回溯）

基于上述分析，梳理出完整的解题步骤：

  1. 初始化：定义结果集ans（存储所有合法组合）、路径path（存储当前递归的元素路径）；
  2. 定义递归函数：函数参数包含「当前遍历索引i」「剩余目标值left（target - 已选元素和）」「候选数组candidates」「结果集ans」「当前路径path」；
  3. 递归终止条件：
    • 终止条件1：left == 0（已选元素和等于target）→ 找到合法组合，将path的副本加入ans，返回；
    • 终止条件2：i >= candidates.length（遍历完所有元素）或left < 0（已选元素和超过target）→ 无合法组合，直接返回；
  4. 分支决策：
    • 分支1（不选当前元素）：递归调用，索引+1，left不变；
    • 分支2（选当前元素）：将当前元素加入path → 递归调用（索引不变，left - 当前元素值）→ 递归结束后恢复现场（移除path最后一个元素）；
  5. 触发递归：从索引0、剩余目标值target开始递归，最终返回ans。

3.2 代码核心点逐行解释

（1）结果集与路径初始化

  • ans：全局结果集，存储所有和为target的合法组合；
  • path：全局路径变量，所有递归分支共享同一个path，这是“恢复现场”的前提。

（2）递归终止条件1（合法组合）

  • 核心逻辑：left == 0表示已选元素和等于target，找到合法组合；
  • 关键细节：new ArrayList<>(path)是创建path的副本并加入ans，若直接写ans.add(path)，后续path的修改会同步改变ans中的内容，导致结果错误。

（3）递归终止条件2（无效分支）

  • i >= candidates.length：遍历完所有元素，无更多可选元素；
  • left < 0：已选元素和超过target，后续无论选/不选都不可能得到合法组合；
  • 作用：提前终止无效递归分支，减少不必要的计算，提升效率。

（4）不选当前元素分支

  • 索引i+1：跳过当前元素，后续递归只能选后面的元素，避免重复组合；
  • left不变：未选当前元素，剩余目标值无变化。

（5）选当前元素分支 + 恢复现场（核心！）

  • path.add(candidates[i])：将当前元素加入路径，标记“选该元素”；
  • dfs(i, left - candidates[i], ...)：索引i不变（可重复选当前元素），剩余目标值减去当前元素值；
  • path.remove(path.size() - 1)：恢复现场，递归结束后移除最后一个元素，回到“未选该元素”的状态，保证后续分支的路径正确。

4、算法性能分析

本解法是组合总和问题的基础最优解，兼顾逻辑直观性和效率，适合面试入门书写。

4.1 时间复杂度：O(S)

S为所有合法组合的元素个数之和，这是组合类回溯问题的典型时间复杂度：

  • 最坏情况下，需要遍历所有可能的组合分支，时间复杂度由合法组合的数量决定；
  • 由于题目限制candidates[i] >= 2、target <= 40，实际递归次数可控，无超时风险。

4.2 空间复杂度：O(target/min(candidates))

空间开销主要来自两部分：

  1. 递归调用栈：递归深度由“能选取的最小元素”决定，最多递归target/min(candidates)层（如candidates=[2]、target=8时，递归深度为4）；
  2. 路径path ：path的最大长度与递归深度一致；
  • 总空间复杂度为O(target/min(candidates))，属于线性级别。

4.3 核心优势

  1. 逻辑直观：完全贴合回溯“选或不选”的核心思想，易理解、易推导，适合面试入门书写；
  2. 无冗余计算：通过终止条件提前剪枝无效分支，减少递归次数；
  3. 扩展性强：是理解“组合总和II（元素不可重复选）”“子集”“排列”等回溯问题的基础。

5、经典示例手动模拟（直观理解）

以示例1：candidates = [2,3,6,7], target = 7 为例，手动模拟核心递归过程，直观理解“选或不选”和“恢复现场”：

5.1 初始状态

递归入口：dfs(0, 7, [2,3,6,7], ans, [])

5.2 关键递归分支

  1. 选2分支：

    • path = [2]，递归dfs(0, 5, ...)（剩余目标值7-2=5）；
    • 再次选2：path = [2,2]，递归dfs(0, 3, ...)（剩余目标值5-2=3）；
    • 再次选2：path = [2,2,2]，递归dfs(0, 1, ...)（剩余目标值3-2=1）→ left=1 < 2，后续分支均无效，恢复现场→path = [2,2]；
    • 选3：path = [2,2,3]，递归dfs(1, 0, ...)（剩余目标值3-3=0）→ 触发终止条件1，ans添加[2,2,3]，恢复现场→path = [2,2]；

  2. 选7分支：

    • 跳过2、3、6，递归到dfs(3, 7, ...)；
    • 选7：path = [7]，递归dfs(3, 0, ...)→ 触发终止条件1，ans添加[7]，恢复现场→path = []；

5.3 最终结果

ans = [[2,2,3],[7]]，与示例结果一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 无合法组合（如candidates=[2], target=1）：递归分支均触发left<0或i越界，ans为空，正确；
  2. 单个元素组成目标值（如candidates=[7], target=7）：选7后left=0，加入结果集，正确；
  3. 重复选同一元素（如candidates=[2], target=8）：选4次2后left=0，加入[[2,2,2,2]]，正确。

6.2 面试高频追问1：如何优化？（剪枝）

原解法可通过排序+提前剪枝进一步提升效率：

  1. 先对candidates排序（如[2,3,6,7]）；
  2. 递归中若candidates[i] > left，由于数组已排序，后续元素均大于candidates[i]，可直接终止当前分支，无需递归；

总结

组合总和的DFS回溯解法是理解回溯算法的核心入门题，本文核心总结如下：

  1. 回溯算法的核心是「选或不选」的分支决策，本题通过“选则索引不变、不选则索引+1”的规则，实现元素可重复选且避免重复组合；
  2. “恢复现场”是回溯算法的核心细节：选元素后必须remove最后一个元素，保证后续分支的路径正确；
  3. 结果集必须存储path的副本，否则会因引用传递导致结果错误；
  4. 基础解法可通过“排序+剪枝”优化效率，是面试中加分的关键；
  5. 掌握本题后，可快速迁移至“组合总和II”“子集”“排列”等回溯类问题，核心逻辑一致。
