Leetcode-031:下一个排列-字典序排列的核心解法（原地修改版）

Created2026-02-27|Updated2026-02-27|Java学习笔记Leetcode Hot100
|Word Count:2.8k|Reading Time:10mins|Post Views:

前言

下一个排列问题是Leetcode数组模块的经典高频题，也是面试中考察「字典序理解」和「原地算法」的核心题型。本题要求在不使用额外空间的前提下，找到给定数字序列的“下一个排列”（字典序中比当前大的最小排列），若已是最大排列则返回最小排列。本文将从字典序的核心规则出发，拆解下一个排列的生成逻辑，详细推导“找、换、反”三步核心解法，结合代码逐行解析关键逻辑，帮助理解原地修改的最优实现。

1、题目

实现获取 下一个排列 的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列（即，组合出比当前排列大的最小整数）。
如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。
必须原地修改，只允许使用额外常数空间。

示例

示例 1：
输入：nums = [1,2,3]
输出：[1,3,2]
解释：字典序中，[1,2,3]的下一个更大排列是[1,3,2]。

示例 2：
输入：nums = [3,2,1]
输出：[1,2,3]
解释：[3,2,1]是最大的排列，无更大排列，返回最小排列[1,2,3]。

示例 3：
输入：nums = [1,1,5]
输出：[1,5,1]
解释：比[1,1,5]大的最小排列是[1,5,1]。

示例 4：
输入：nums = [1,3,2]
输出：[2,1,3]
解释：字典序中，[1,3,2]的下一个更大排列是[2,1,3]。

提示

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 100

2、分析

2.1 问题核心：字典序的下一个更大排列规则

字典序的核心是“找最小的增大值”：要生成比当前排列大的最小排列，需在尽可能靠右的位置进行最小幅度的增大，而非整体调整。这一规则决定了下一个排列的生成必须遵循“从右向左找突破口”的思路。

2.2 核心难点：如何找到“最小幅度增大”的位置？

直接暴力枚举所有排列并找下一个更大值的方法，时间复杂度为O(n!)，完全不适用（n=100时枚举量爆炸）。原地修改的最优思路是拆解为四步核心逻辑，通过“找升序对→找最小更大值→交换→反转右侧”，将时间复杂度优化至O(n)，且空间复杂度为O(1)。

2.3 核心解题思路（四步拆解）

下一个排列的生成可拆解为四个核心步骤，每一步都服务于“最小幅度增大”的目标：

  1. 找突破口：从右向左遍历，找到第一个满足nums[i] < nums[i+1]的位置i（记为nums1Index）。这一步的意义是：i右侧的所有元素都是降序排列（已是最大子排列），无法通过调整右侧增大整体值，因此i是唯一能“增大”的突破口；
  2. 找最小增大值：若找到nums1Index，从右向左遍历i右侧的元素，找到大于nums[i]的最小数字（记为nums[j]）。这一步保证了“最小幅度增大”——用右侧最小的更大值替换nums[i]，是增大整体值的最小选择；
  3. 交换值：交换nums[i]和nums[j]，完成“增大”的核心操作；
  4. 反转右侧：将i+1到数组末尾的元素反转。原因是：交换后i右侧仍为降序排列（最大子排列），反转后变为升序（最小子排列），保证整体是“比当前大的最小排列”；
  5. 边界处理：若未找到nums1Index（即整个数组是降序），直接反转整个数组，得到最小排列。

2.4 核心逻辑合理性验证

以示例nums = [1,3,2]为例验证逻辑：

  • 步骤1：从右向左找，i=0时nums[0]=1 < nums[1]=3，确定nums1Index=0；
  • 步骤2：找右侧大于1的最小数，即2（索引2）；
  • 步骤3：交换后数组变为[2,3,1]；
  • 步骤4：反转i+1=1到末尾的元素，得到[2,1,3]，与示例结果一致。

3.2 代码核心点逐行解释

（1）找突破口：定位nums1Index

  • 遍历起点：n-2，因为要比较i和i+1，避免数组越界；
  • 遍历方向：从右向左，保证找到的是最右侧的升序对左端点（满足“最小幅度增大”）；
  • 初始值-1：作为“未找到升序对”的标记（数组整体降序）。

（2）找最小增大值：定位nums2Index

  • 初始值nums2 = Integer.MAX_VALUE：保证第一次比较时能捕获到第一个大于nums[i]的数；
  • 核心逻辑：遍历i右侧所有元素，筛选出大于nums[i]的数，并记录其中的最小值及其索引；
  • 交换操作：完成“增大”的核心步骤，此时i右侧仍为降序（因为原右侧是降序，交换的是最小的更大值，不改变降序特性）。

（3）反转右侧：生成最小子排列

  • 边界适配：若nums1Index=-1（数组整体降序），则nums1Index+1=0，反转整个数组，得到最小排列；
  • 反转逻辑：将降序的子数组转为升序，保证i右侧是最小的子排列，最终整体是“比当前大的最小排列”；
  • 辅助函数reverse：通过双指针原地反转，时间复杂度O(k)（k为子数组长度），无额外空间开销。

（4）辅助函数：swap和reverse

  • swap：原地交换两个元素，空间复杂度O(1)；
  • reverse：双指针遍历交换，左指针右移、右指针左移，直到相遇，完成原地反转。

4、算法性能分析

本解法是下一个排列问题的最优解，时间复杂度线性级，空间复杂度常数级，完全满足题目“原地修改”的要求。

4.1 时间复杂度：O(n)

n为数组nums的长度，整个解法包含三次线性遍历（均为O(n)）：

  1. 找nums1Index：最多遍历n-1次，O(n)；
  2. 找nums2Index：最多遍历n-1次，O(n)；
  3. 反转数组：最多遍历n次，O(n)；
  • 总时间复杂度为O(n) + O(n) + O(n) = O(n)，适配题目n<=100的输入规模，且可拓展至更大数组。

4.2 空间复杂度：O(1)

所有操作均为原地修改：

  • 仅使用了nums1Index、nums2Index、nums2等常数个临时变量；
  • swap和reverse函数无额外空间开销；
  • 总空间复杂度为O(1)，完全符合题目“额外常数空间”的要求。

4.3 核心优势

  1. 原地修改：无额外数组开销，满足题目空间限制；
  2. 逻辑贴合规则：完全遵循字典序“最小幅度增大”的核心规则，易理解、易推导；
  3. 效率最优：线性时间复杂度，无冗余遍历；
  4. 边界兼容：天然处理“最大排列”的边界情况，无需额外判断。

5、经典示例手动模拟（直观理解）

以示例4：nums = [1,3,2]（n=3） 为例，手动模拟核心步骤：

5.1 步骤1：找nums1Index

  • 遍历起点i=1（n-2=1）：nums[1]=3 > nums[2]=2，不满足；
  • 遍历i=0：nums[0]=1 < nums[1]=3，满足，nums1Index=0；

5.2 步骤2：找nums2Index并交换

  • 遍历j=1：nums[0]=1 < nums[1]=3，nums2=3，nums2Index=1；
  • 遍历j=2：nums[0]=1 < nums[2]=2，nums2=min(3,2)=2，nums2Index=2；
  • 交换nums[0]和nums[2]，数组变为[2,3,1]；

5.3 步骤3：反转右侧

  • 反转区间nums1Index+1=1到2：
    • 交换nums[1]和nums[2]，数组变为[2,1,3]；
  • 最终结果与示例一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 数组长度=1：nums1Index=-1，反转区间0到0（无操作），数组不变，正确；
  2. 数组整体降序（如[3,2,1]）：nums1Index=-1，反转整个数组为[1,2,3]，正确；
  3. 数组有重复元素（如[1,1,5]）：找nums2Index时会选择最小的更大值，交换后反转右侧，得到[1,5,1]，正确；
  4. 数组已为升序（如[1,2,3]）：找到nums1Index=1，交换nums[1]和nums[2]，反转右侧（无操作），得到[1,3,2]，正确。

6.2 与全排列的关联

下一个排列是生成全排列的核心基础：

  • 从最小排列（升序）开始，反复调用nextPermutation，可生成所有字典序排列；
  • 相比递归生成全排列，该方法时间效率更高（O(n) per call），且空间开销更小。

6.3 面试高频追问：如何实现“上一个排列”？

若面试官追问“找字典序中比当前小的最大排列”，只需反转核心逻辑：

  1. 从右向左找第一个nums[i] > nums[i+1]的位置i；
  2. 从右向左找i右侧小于nums[i]的最大数j；
  3. 交换i和j；
  4. 反转i+1到末尾（此时反转降序为升序，保证是“比当前小的最大排列”）。

总结

下一个排列的原地修改解法是本题的最优解，核心是理解字典序“最小幅度增大”的规则，本文核心总结如下：

  1. 下一个排列的核心规则是“在最右侧找突破口，用最小幅度增大整体值”；
  2. 解法拆解为四步：找升序对左端点→找右侧最小更大值→交换→反转右侧，每一步都服务于“最小幅度增大”；
  3. 代码的核心是原地操作（swap、reverse），空间复杂度O(1)，时间复杂度O(n)；
  4. 边界情况（最大排列、重复元素、长度为1）无需额外处理，代码天然兼容；
  5. 掌握该解法后，可快速迁移至“上一个排列”等衍生问题，是面试的核心考点。
