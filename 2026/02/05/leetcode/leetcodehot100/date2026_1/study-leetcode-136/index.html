Leetcode-136:只出现一次的数字-异或最优解&哈希基础解

Created2026-02-05|Updated2026-02-05|Java学习笔记Leetcode Hot100
|Word Count:1.1k|Reading Time:4mins|Post Views:

前言

Leetcode136是位运算和哈希表的入门基础题，题目约束简单（除一个元素外，其余均出现两次），有两种典型解法：哈希集合是直观的基础解，适合算法入门；异或位运算是最优解，仅3行核心代码实现线性时间+常数空间，是面试首选。本文精简梳理两种解法的核心思路、代码实现和性能差异，快速掌握解题关键。

1、题目

给定非空整数数组，除某个元素只出现一次外，其余每个元素均出现两次，找出该唯一元素。

要求

  • 线性时间复杂度 O(n)；
  • 进阶：不使用额外辅助空间 O(1)。

示例

输入：nums = [4,1,2,1,2] → 输出：4
输入：nums = [2,2,1] → 输出：1
输入：nums = [1] → 输出：1

2、解法一：哈希集合（HashSet）——基础直观解

利用集合不存重复元素的特性，通过「添加-移除」筛选唯一元素，逻辑简单易理解。

2.1 核心思路

遍历数组，元素首次出现则加入集合，第二次出现则从集合移除；遍历结束后，集合中仅剩唯一出现的目标元素。

2.3 性能分析

  • 时间复杂度：O(n)，集合增删查平均时间均为O(1)；
  • 空间复杂度：O(n)，需额外存储临时元素，不满足进阶要求。

3、解法二：异或位运算——最优解（面试首选）

利用异或运算的核心性质，实现无额外空间的最优解，代码极简，是本题的核心解法。

3.1 核心前置知识

异或运算是一种二进制位运算，符号为^，运算规则为：相同为 0，不同为 1（如1^1=0，1^0=1，0^0=0）。针对本题，只需掌握其四个核心数学性质（无需关注底层二进制），这是解法的根本依据：
自反性：任何数与自身异或，结果为 0 → a ^ a = 0；
恒等性：任何数与 0 异或，结果为自身 → a ^ 0 = a；
交换律：异或运算的顺序不影响结果 → a ^ b = b ^ a；
结合律：异或运算的分组不影响结果 → (a ^ b) ^ c = a ^ (b ^ c)。

3.2 核心思路

将数组所有元素依次异或，重复出现的元素两两异或为0，最终0与唯一元素异或，结果即为目标元素。

3.3 完整代码

3.4 代码说明

  • 结果初始化为0是关键，确保首次异或从0开始，不破坏结果；
  • ans ^= num 是 ans = ans ^ num 的简写，遍历一次即可得到结果。

3.5 性能分析

  • 时间复杂度：O(n)，仅一次线性遍历，异或为常数时间操作；
  • 空间复杂度：O(1)，仅使用一个临时变量，满足题目进阶要求。

4、简单示例模拟

以nums = [4,1,2,1,2]为例，直观看异或执行过程：

  • 初始化：ans = 0
  • 0^4=4 → 4^1=5 →5^2=7 →7^1=6 →6^2=4
  • 最终结果：4（与分组异或 4^(1^1)^(2^2)=4^0^0=4 结果一致）

5、两种解法核心对比

对比维度 哈希集合解法 异或位运算解法
时间复杂度 O(n)（平均） O(n)（最优）
空间复杂度 O(n)（额外空间） O(1)（常数空间）
代码简洁度 中等 极简（3行核心）
题目要求适配 满足基础要求 满足所有要求（含进阶）
面试适配性 基础思路回答 首选最优解

总结

  1. 哈希集合是基础解，利用集合唯一性实现，逻辑直观但需额外空间；
  2. 异或位运算是最优解，依托自反性和恒等性，代码极简且无额外空间，是面试必写；
  3. 异或解法的关键前提是其余元素出现偶数次，此约束下异或为最优选择。
