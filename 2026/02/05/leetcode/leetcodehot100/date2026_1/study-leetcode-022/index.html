Leetcode-022:生成括号-DFS回溯经典解法（路径优化版）

Created2026-02-05|Updated2026-02-05|Java学习笔记Leetcode Hot100
|Word Count:3.2k|Reading Time:11mins|Post Views:

前言

生成括号问题是Leetcode回溯/DFS模块的经典入门题，也是面试高频考察的基础题之一，其核心考察对回溯算法、有效括号规则和剪枝优化的理解。本题的解法以DFS回溯为核心，其中「路径数组优化版」是性能最优的写法——通过字符数组存储回溯路径，避免字符串拼接的额外开销，同时通过精准剪枝剔除无效分支，大幅提升效率。

1、题目

数字 n 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。
数字 n 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。

核心定义：有效括号

有效括号需满足两个核心条件：

  1. 任意前缀中，左括号数量 ≥ 右括号数量；
  2. 最终左右括号数量相等（均为n）。

示例

示例 1：
输入：n = 3
输出：[“((()))”,”(()())”,”(())()”,”()(())”,”()()()”]

示例 2：
输入：n = 1
输出：[“()”]

示例 3：
输入：n = 2
输出：[“(())”,”()()”]

提示

  • 1 <= n <= 8（实际DFS剪枝后可轻松处理该规模，拓展至n=10也无压力）

2、分析

2.1 问题核心：有效括号的生成规则

生成有效括号的核心是过程合法而非仅结果合法：若在生成过程中某一步出现「右括号数量 > 左括号数量」，则后续无论如何补充，该分支都不可能是有效组合，可直接剪枝。

这是因为有效括号的本质是「匹配」：每一个右括号都必须有对应的左括号，因此生成时需满足：

  1. 左括号的数量不超过n（总对数限制）；
  2. 右括号的数量不超过当前已生成的左括号数量（匹配限制）。

2.2 核心解法：DFS回溯+路径优化

暴力解法的思路是生成所有2^(2n)种括号组合，再逐一验证有效性，但该方法时间复杂度极高（n=8时组合数达65536）。

DFS回溯+剪枝是最优思路：通过深度优先遍历构建括号组合，每一步仅选择合法的括号（左/右），无效分支直接剪枝；同时使用字符数组存储路径，避免字符串拼接的频繁内存操作，进一步优化性能。

2.3 核心概念定义（解法基础）

为后续推导清晰，先明确回溯过程中的核心变量定义：

  1. i：当前已生成的括号数量（路径长度），终止条件为i == 2n（生成完所有括号）；
  2. left：当前路径中左括号的数量，用于判断是否可添加左/右括号；
  3. path：字符数组，存储当前回溯路径（优化点：替代字符串拼接，降低时间/空间开销）；
  4. ans：结果列表，存储所有有效括号组合。

2.4 核心解题思路

  1. 初始化：创建结果列表ans，初始化长度为2n的字符数组path（路径容器）；
  2. DFS回溯：从i=0、left=0开始递归，每一步做两个判断：
    • 若左括号数量 < n，则添加左括号，递归至i+1、left+1；
    • 若右括号数量（i-left） < 左括号数量，则添加右括号，递归至i+1、left；
  3. 终止条件：当i == 2n时，将path转为字符串加入结果列表；
  4. 返回结果：递归结束后，ans即为所有有效组合。

3.2 代码核心点逐行解释

（1）路径优化：字符数组替代字符串

  • 核心逻辑：字符串是不可变对象，每次拼接（如path + "("）都会生成新字符串，带来O(n)的额外开销；字符数组是可变的，直接修改指定位置的字符，仅在最终生成结果时转为字符串，时间/空间效率大幅提升；
  • 作用：将路径构建的时间复杂度从O(n²)降至O(n)，是本题代码的核心优化点。

（2）DFS方法参数定义

  • i：当前已填充的括号数量（路径下标），范围0~2n；
  • left：当前路径中左括号的数量，用于快速判断右括号是否合法；
  • n：生成括号的总对数（固定值）；
  • path：当前回溯路径的字符数组，传递引用避免重复创建。

（3）递归终止条件

  • 核心逻辑：当路径长度达到2n时，说明已生成完整的括号组合（且过程合法，因剪枝条件），将字符数组转为字符串加入结果列表；
  • 注意点：new String(path)会复制数组内容，避免后续回溯修改已加入结果的字符串。

（4）剪枝条件1：添加左括号

  • 核心逻辑：左括号数量未超过总对数n时，可添加左括号；
  • 递归操作：路径下标i+1（填充下一个位置），左括号数left+1（更新计数），无回溯操作（因字符数组会被后续覆盖）。

（5）剪枝条件2：添加右括号

  • 核心逻辑：i-left是当前路径中右括号的数量，需满足「右括号数 < 左括号数」（保证匹配）；
  • 等价写法：if (right < left)（若显式定义right = i-left），原写法省略变量定义，更简洁；
  • 剪枝价值：直接剔除「右括号数 ≥ 左括号数」的无效分支，避免无意义的递归。

4、算法性能分析

本解法是生成括号问题的性能最优版，时间复杂度为卡特兰数级别，空间复杂度为线性级别，完全适配题目输入规模。

4.1 时间复杂度：O(4ⁿ/√n)

时间复杂度由卡特兰数决定（第n个卡特兰数为C(2n,n)/(n+1)），卡特兰数的渐近增长为O(4ⁿ/√n)，具体分析：

  • 每一步递归有2种选择（左/右括号），但剪枝条件剔除了大部分无效分支，实际递归次数远小于2^(2n)；
  • n=8时，卡特兰数为1430，远小于暴力解法的65536次递归，效率提升显著。

4.2 空间复杂度：O(n)

空间开销主要来自两部分：

  1. 递归调用栈：递归深度为2n，但因Java栈帧优化，实际空间复杂度为O(n)；
  2. 路径数组path：长度为2n，空间复杂度为O(n)；
  • 总空间复杂度为O(n) + O(n) = O(n)，无额外冗余开销。

4.3 核心优势

  1. 性能最优：字符数组路径优化避免字符串拼接，剪枝条件剔除无效分支；
  2. 逻辑简洁：仅两个核心剪枝条件，递归逻辑清晰，面试易书写；
  3. 无冗余操作：递归过程中仅修改数组元素，无重复创建对象；
  4. 易拓展：可直接适配n增大的场景（如n=10），无性能瓶颈。

5、经典示例手动模拟（直观理解）

以示例3：n=2为例，手动模拟DFS回溯过程，直观理解路径构建与剪枝逻辑：

5.1 初始化

  • path = new char[4]，ans = []，调用dfs(0,0,2,path)。

5.2 递归过程（核心步骤）

  1. i=0, left=0：

    • 左括号条件满足（0<2）：path[0]='('，递归dfs(1,1,2,path)；
    • 右括号条件不满足（0-0=0 ≥ 0）：跳过。

  2. i=1, left=1：

    • 左括号条件满足（1<2）：path[1]='('，递归dfs(2,2,2,path)；
    • 右括号条件满足（1-1=0 < 1）：path[1]=')'，递归dfs(2,1,2,path)。

  3. 分支1：i=2, left=2（path=[(,(,,]）：

    • 左括号条件不满足（2=2）：跳过；
    • 右括号条件满足（2-2=0 < 2）：path[2]=')'，递归dfs(3,2,2,path)。

  4. 分支1-1：i=3, left=2（path=[(,(,),_]）：

    • 左括号条件不满足：跳过；
    • 右括号条件满足（3-2=1 < 2）：path[3]=')'，递归dfs(4,2,2,path)。

  5. 分支1-1-1：i=4（终止条件）：

    • path=[(,(,),)] → 转为字符串"(())"，加入ans → ans=["(())"]。

  6. 分支2：i=2, left=1（path=[(,),,]）：

    • 左括号条件满足（1<2）：path[2]='('，递归dfs(3,2,2,path)；
    • 右括号条件不满足（2-1=1 ≥ 1）：跳过。

  7. 分支2-1：i=3, left=2（path=[(,),(,]）：

    • 左括号条件不满足：跳过；
    • 右括号条件满足（3-2=1 < 2）：path[3]=')'，递归dfs(4,2,2,path)。

  8. 分支2-1-1：i=4（终止条件）：

    • path=[(,),(),)] → 转为字符串"()()"，加入ans → ans=["(())","()()"]。

5.3 最终结果

ans=["(())","()()"]，与示例3结果一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理边界场景，代码天然兼容：

  1. n=1：仅生成"()"，递归仅2步，正确；
  2. n=0（非题目要求）：若兼容该场景，可在入口方法添加if (n==0) return ans;，正确；
  3. n=3：生成5种有效组合，与示例1一致，正确。

面试中可能拓展至更复杂的括号问题，核心思路仍基于本题的剪枝逻辑：

  1. 多种括号生成（如圆括号+方括号）：增加括号类型判断，保证每种括号的匹配规则；
  2. 生成第k个有效括号：结合卡特兰数的计数特性，直接定位第k个组合，避免生成所有组合。

总结

生成括号的DFS回溯路径优化解法是本题的性能最优解，也是理解回溯算法的经典案例，本文核心总结如下：

  1. 有效括号的核心生成规则：左括号数≤n，右括号数≤左括号数（过程合法比结果合法更重要）；
  2. 回溯解法的核心优化：用字符数组存储路径，避免字符串拼接的额外开销；
  3. 剪枝条件是效率关键：仅保留合法分支，剔除「右括号数≥左括号数」「左括号数>n」的无效递归；
  4. 时间复杂度为卡特兰数级别（O(4ⁿ/√n)），空间复杂度为O(n)，是面试首选书写版本；
  5. 在此基础上，可拓展至动态规划解法或更复杂的括号生成问题，核心剪枝逻辑通用。
