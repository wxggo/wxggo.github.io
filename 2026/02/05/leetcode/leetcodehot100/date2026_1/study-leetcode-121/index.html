Leetcode-121:买卖股票的最佳时机-一次遍历贪心最优解

Created2026-02-05|Updated2026-02-05|Java学习笔记Leetcode Hot100
|Word Count:1.5k|Reading Time:5mins|Post Views:

前言

Leetcode121是贪心算法的经典入门题，也是数组类高频基础题，题目要求仅一次买卖股票求最大利润，核心解法为一次遍历贪心策略。该解法无需复杂逻辑，仅通过两个变量维护关键状态，就能实现O(n)时间复杂度+O(1)空间复杂度的最优解，代码极简且易理解，是面试中本题的唯一首选写法。本文精简梳理题目要求、核心思路、代码实现及执行过程，快速掌握解题关键。

1、题目

给定数组 prices，其中 prices[i] 是某支股票第 i 天的价格。你只能选择某一天买入股票，并在未来某一不同日子卖出，计算能获取的最大利润。若无法获取利润，返回0。

示例

示例1：输入：prices = [7,1,5,3,6,4] → 输出：5
解释：第2天（价格1）买入，第5天（价格6）卖出，利润6-1=5。
示例2：输入：prices = [7,6,4,3,1] → 输出：0
解释：价格持续下跌，无利润可赚。
示例3：输入：prices = [1,2] → 输出：1

提示

  • 1 <= prices.length <= 10^5
  • 0 <= prices[i] <= 10^4

2、核心解法：一次遍历贪心算法（最优解）

本题核心逻辑是低买高卖，贪心策略体现在：遍历过程中实时维护历史最低买入价格和当前最大利润，一次线性遍历即可得到结果，无任何额外空间开销。

2.1 核心思路

  1. 初始化两个变量：min 记录遍历到当前的历史最低价格（初始为第一天价格，作为初始买入基准），ans 记录当前能获得的最大利润（初始为0，适配无利润返回0的要求）；
  2. 从第二天开始遍历价格数组（第一天仅能买入，无法卖出）：
    • 先计算当前价格卖出的利润（当前价格 - 历史最低价格），与当前最大利润比较，更新最大利润；
    • 再将当前价格与历史最低价格比较，更新历史最低价格；

      关键：先算利润再更新最低价格，避免出现「当天买当天卖」的无效操作，保证买入在卖出之前；

  3. 遍历结束后，ans 即为所求的最大利润。

2.2 完整代码

2.3 代码核心点说明

  1. 变量初始化：min 初始化为prices[0]，保证首次比较有合理基准；ans 初始化为0，无需额外判断，无利润时直接返回；
  2. 循环范围：i 从1开始，跳过无法卖出的第一天；
  3. 操作顺序：先更新利润，后更新低价是解题关键，若顺序颠倒，会导致用当天价格作为买入价计算当天利润，违反题目规则；
  4. 工具方法：使用Math.max和Math.min简洁实现最值更新，替代手动if判断，让代码更简洁高效。

2.4 性能分析

  • 时间复杂度：O(n)，仅对价格数组进行一次线性遍历，每个元素仅处理一次，完全适配题目10^5的数组长度要求，无性能瓶颈；
  • 空间复杂度：O(1)，仅使用length、min、ans三个临时变量，无任何数组、集合等辅助空间，是最优的空间复杂度。

3、经典示例手动模拟（直观理解）

以示例1：prices = [7,1,5,3,6,4] 为例，逐步模拟遍历过程，清晰看到min和ans的更新变化：

  • 初始化：length=6，min=7，ans=0；
  • i=1（价格1）：ans = max(1-7=-6, 0) → 0；min = min(7,1) → 1；
  • i=2（价格5）：ans = max(5-1=4, 0) → 4；min = min(1,5) → 1；
  • i=3（价格3）：ans = max(3-1=2, 4) → 4；min = min(1,3) → 1；
  • i=4（价格6）：ans = max(6-1=5, 4) → 5；min = min(1,6) → 1；
  • i=5（价格4）：ans = max(4-1=3, 5) → 5；min = min(1,4) → 1；
  • 遍历结束，返回ans=5，与示例结果一致。

特殊示例模拟（示例2：prices = [7,6,4,3,1]）：

  • 初始化：min=7，ans=0；
  • 遍历过程中，所有价格与min的差值均为负数，ans 始终保持0，min 持续更新为更低价格；
  • 最终返回0，符合题目要求。

4、边界情况天然适配

本解法无需额外添加判空或边界处理代码，天然兼容所有题目边界场景：

  1. 数组长度为1：如prices=[5]，循环不执行，直接返回ans=0，正确；
  2. 价格持续上涨：如prices=[1,2,3,4,5]，遍历后ans=4（5-1），正确；
  3. 价格包含0：如prices=[0,5,3,8]，ans=8（8-0），正确；
  4. 价格先跌后涨再跌：如prices=[5,3,8,2,9]，ans=7（9-2），正确。

5、总结

本题是贪心算法的入门经典，核心解法的关键总结如下：

  1. 核心思想：低买高卖，通过遍历实时维护历史最低价格和当前最大利润，以局部最优操作实现全局最优解；
  2. 代码关键：先计算利润再更新最低价格，避免当天买卖的无效操作，这是本题的解题细节；
  3. 性能最优：一次线性遍历实现O(n)时间复杂度，常数变量实现O(1)空间复杂度，适配大数据量要求；
  4. 拓展性：该贪心思路是股票系列问题的基础，后续可拓展至多次买卖、含冷冻期等复杂股票问题的解题框架。
