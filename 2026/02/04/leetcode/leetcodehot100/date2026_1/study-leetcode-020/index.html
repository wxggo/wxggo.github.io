Leetcode-020:有效的括号-栈经典解法（HashMap+Deque版）

Created2026-02-04|Updated2026-02-04|Java学习笔记Leetcode Hot100
|Word Count:2.5k|Reading Time:8mins|Post Views:

1、题目

给定一个只包括 '('，')'，'{'，'}'，'['，']' 的字符串 s ，判断字符串是否有效。

有效字符串需满足：

  1. 左括号必须用相同类型的右括号闭合。
  2. 左括号必须以正确的顺序闭合。
  3. 每个右括号都有一个对应的相同类型的左括号。

示例

示例 1：
输入：s = “()”
输出：true

示例 2：
输入：s = “()[]{}”
输出：true

示例 3：
输入：s = “(]”
输出：false

示例 4：
输入：s = “([)]”
输出：false

示例 5：
输入：s = “{[]}”
输出：true

提示

  • 1 <= s.length <= 10^4
  • s 仅由括号 ()[]{} 组成

2、分析

2.1 问题核心：括号匹配的“先进后出”特性

括号匹配的核心遵循先进后出（FILO） 规则：嵌套的括号必须“先开后闭”，例如{[]}中，先开{、再开[，则必须先闭]、再闭{。这一特性与栈的操作逻辑完全契合——栈的入栈（push）对应括号的“开启”，出栈（pop）对应括号的“闭合校验”。

2.2 核心难点：如何高效匹配括号类型？

暴力解法的思路是遍历字符串，逐个记录左括号的位置和类型，遇到右括号时反向查找最近的未匹配左括号，该方法的时间复杂度为O(n²)，且逻辑复杂易出错。

栈+哈希表是解决该难点的最优思路：

  1. 哈希表（HashMap）：提前存储“左括号→右括号”的映射关系，实现括号类型的O(1)快速匹配；
  2. 栈（Deque）：存储待匹配的右括号（左括号入栈时，直接压入其对应的右括号），遇到右括号时，只需弹出栈顶元素并校验是否相等，即可完成匹配判断。

2.3 核心解题思路

  1. 边界快速过滤：若字符串长度为奇数，必然无法完全匹配，直接返回false；
  2. 初始化映射与栈：构建左括号到右括号的HashMap，初始化空栈；
  3. 遍历字符串校验：
    • 遇到左括号：将其对应的右括号压入栈（待后续匹配）；
    • 遇到右括号：若栈为空（无待匹配左括号）或栈顶元素≠当前右括号（类型不匹配），返回false；否则弹出栈顶元素（完成一次匹配）；
  4. 最终校验：遍历结束后，栈必须为空（所有左括号均完成匹配，无残留），否则返回false。

2.4 核心边界条件（面试必关注）

  1. 字符串长度为奇数：直接无效；
  2. 右括号开头：栈为空时遇到右括号，直接无效；
  3. 左括号残留：遍历结束后栈非空（如((），直接无效；
  4. 括号类型不匹配：如(]，栈顶元素为)，与]不相等，直接无效。

3.2 代码核心点逐行解释

（1）奇数长度快速过滤

  • 核心逻辑：有效括号必须成对出现，长度为奇数时必然存在无法匹配的括号；
  • 作用：提前过滤无效输入，减少后续无意义计算，提升程序效率。

（2）HashMap映射初始化

  • 核心逻辑：将左括号作为key，对应右括号作为value，实现括号类型的O(1)快速查找；
  • 拓展性：若需支持更多类型括号（如<>），只需新增映射关系即可，代码扩展性强。

（3）字符数组转换与栈初始化

  • toCharArray()：将字符串转为字符数组，避免遍历中频繁调用charAt(i)（底层仍需遍历，转换后更高效）；
  • Deque替代Stack：Java中Stack是老旧类，Deque的push()/pop()方法完全兼容栈操作，且线程安全、性能更优。

（4）核心遍历校验逻辑

  • 左括号处理：不存储左括号本身，而是直接压入其对应的右括号，后续遇到右括号时可直接对比，减少一次映射查询；
  • 右括号处理：
    • 栈空判断：若栈为空，说明当前右括号无对应的左括号（如)()），直接无效；
    • 栈顶对比：弹出栈顶元素（待匹配的右括号），若与当前右括号不相等，说明类型不匹配（如(]），直接无效。

（5）最终栈空校验

  • 核心逻辑：遍历结束后，若栈非空，说明存在未匹配的左括号（如((），返回false；否则返回true；
  • 示例：输入(()，遍历结束后栈中剩余)，返回false。

4、算法性能分析

本解法的时间和空间复杂度均为线性级别，是有效的括号问题的最优解，兼顾效率和可读性。

4.1 时间复杂度：O(n)

n为字符串s的长度，整个解法仅包含一次线性遍历，且每次遍历的操作（HashMap查询、栈操作）均为O(1)：

  • 遍历字符串：O(n)；
  • 总时间复杂度为O(n)，完美适配题目10^4的输入规模。

4.2 空间复杂度：O(n)

空间开销主要来自两部分：

  1. 字符数组chars：与字符串等长，O(n)；
  2. 栈stack：最坏情况下（全为左括号），栈的大小为n/2，O(n)；
  3. HashMap：固定存储3组映射，为常数级O(1)，可忽略；
  • 总空间复杂度为O(n)。

4.3 核心优势

  1. 逻辑直观：完全贴合括号匹配的FILO特性，易理解、易推导，适合面试入门书写；
  2. 效率最优：线性时间复杂度，无超时风险；
  3. 扩展性强：新增括号类型只需修改HashMap，无需调整核心逻辑；
  4. 边界全覆盖：提前过滤奇数长度，遍历中校验栈空和类型，最终校验栈残留，覆盖所有无效场景。

5.1 示例5：s = “{[]}”（length=4，偶数）

遍历字符 类型 栈操作 栈状态 匹配结果

‘[‘ 左括号 push(‘]’) [ ‘}’, ‘]’ ] -
‘]’ 右括号 pop() → ‘]’，与’]’相等 [ ‘}’ ] 匹配成功
‘}’ 右括号 pop() → ‘}’，与’}’相等 [] 匹配成功
遍历结束，栈为空 → 返回true。

遍历字符 类型 栈操作 栈状态 匹配结果
‘(‘ 左括号 push(‘)’) [ ‘)’ ] -
‘[‘ 左括号 push(‘]’) [ ‘)’, ‘]’ ] -

遍历至第三个字符时直接返回false。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 空字符串：length=0（偶数），栈为空 → 返回true（符合题目隐含规则）；
  2. 奇数长度：如"(()"，边界判断直接返回false；
  3. 右括号开头：如)()，遍历第一个字符时栈为空 → 返回false；
  4. 全左括号：如"(((( "，遍历结束栈非空 → 返回false；
  5. 嵌套多层括号：如"{[()]}"，栈操作完美匹配嵌套顺序 → 返回true。

6.2 与纯栈解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
纯栈解法 O(n) O(n) 无需HashMap 匹配逻辑繁琐 仅支持少量括号
栈+HashMap O(n) O(n) 逻辑简洁、易扩展 需额外HashMap 面试首选、生产环境

6.3 面试高频追问：进阶优化与拓展

（1）空间优化：能否将空间复杂度降至O(1)？

答：无法实现。因为括号的嵌套结构必须依赖栈记录匹配顺序，最坏情况下（全左括号）必须存储n/2个待匹配括号，空间复杂度无法低于O(n)。

（2）支持更多括号类型（如<>）？

答：只需在HashMap中新增映射关系hashMap.put('<', '>')，核心逻辑无需修改，扩展性极强。

（3）如何统计无效括号的数量？

答：在遍历过程中记录：

  • 遇到右括号且栈空时，无效数+1；
  • 遍历结束后，栈的大小（未匹配左括号数）+ 无效数 = 总无效括号数。

6.4 进阶应用：表达式求值/括号最大深度

有效的括号是栈的基础应用，掌握后可快速迁移至更复杂的场景：

  1. 表达式求值（如3+2*(1+2)）：用栈存储操作数和运算符，括号的匹配逻辑与本题完全一致；
  2. 括号的最大深度（Leetcode-1614）：用栈的大小记录当前括号深度，入栈时+1，出栈时取最大值。
