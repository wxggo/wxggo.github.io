Leetcode-002:两数相加-链表版（迭代+递归双解法）

Created2026-02-04|Updated2026-02-04|Java学习笔记Leetcode Hot100
|Word Count:2.3k|Reading Time:8mins|Post Views:

前言

两数相加（链表版）是Leetcode链表模块的入门经典题，也是面试高频考察的基础题，核心考察对链表遍历和加法进位逻辑的理解。本题的核心特点是数字以逆序链表存储，恰好契合加法「从低位到高位」的计算顺序，解法分为「迭代（基础最优）」和「递归（简洁优雅）」两种：迭代法通过哑节点简化链表头处理，是工程中更常用的高效写法；递归法通过递归栈替代手动遍历，代码更简洁，适合理解递归思想。本文将详解两种解法的核心逻辑，拆解进位处理和边界场景的适配思路。

1、题目

给两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。请你将两个数相加，并以相同形式返回一个表示和的链表。

注意：可以假设除了数字0之外，这两个数都不会以0开头。

示例

示例 1：
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807（链表逆序存储，表头为个位）。

示例 2：
输入：l1 = [0], l2 = [0]
输出：[0]
解释：0 + 0 = 0。

示例 3：
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]
解释：9999999 + 9999 = 10009998，逆序存储后为[8,9,9,9,0,0,0,1]。

提示

  • 每个链表中的节点数在范围 [1, 100] 内
  • 0 <= Node.val <= 9
  • 题目数据保证列表表示的数字不含前导零

2、分析

2.1 核心原理

逆序链表的存储特性是解题关键：

  • 链表表头对应数字的个位，表尾对应最高位，遍历链表的顺序恰好是加法「个位→十位→百位」的计算顺序；
  • 加法进位规则：两个个位数相加（0-9）+ 进位（0或1），结果的个位为 和%10，新进位为 和/10（进位最大为1，因为9+9+1=19）。

2.2 核心难点

  1. 链表长度不一致：短链表遍历结束后，后续位需按「0」参与计算；
  2. 最后一位进位：两个链表遍历完毕后，若仍有进位（carry=1），需新增节点存储该进位；
  3. 递归终止条件：需同时判断两个链表遍历完毕且无进位，避免遗漏最后一位进位。

2.3 核心解题思路

迭代法（基础思路）

  1. 初始化「哑节点（dummy）」和「遍历指针（temp）」，哑节点用于简化链表头的返回逻辑；
  2. 初始化进位 carry=0，循环遍历两个链表（只要有一个未遍历完或有进位）；
  3. 计算当前位的和：累加两个链表当前节点值（空则为0）+ 进位；
  4. 创建新节点存储当前位结果（和%10），更新遍历指针和进位（和/10）；
  5. 循环结束后，返回哑节点的next（结果链表头）。

递归法（进阶思路）

  1. 递归函数参数：两个链表节点 + 进位；
  2. 递归终止条件：两个节点均为空且进位为0；
  3. 递归逻辑：计算当前位和 → 创建当前节点 → 递归处理下一位（传递更新后的链表节点和新进位）；
  4. 主函数调用递归函数，直接返回递归结果（无需哑节点）。

4、算法性能分析

4.1 时间复杂度

两种解法的时间复杂度均为 O(max(m, n)) （m、n为两个链表的长度）：

  • 迭代法：循环次数等于「较长链表长度 + 进位位（最多1次）」，为线性级别；
  • 递归法：递归深度等于「较长链表长度 + 进位位（最多1次）」，每个递归层操作均为O(1)。

4.2 空间复杂度

  • 迭代法：O(1) （除结果链表外，仅使用常数级临时变量）；
  • 递归法：O(max(m, n)) （递归栈的深度等于最长链表长度，无额外空间优化）。

4.3 解法对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
迭代 O(max(m,n)) O(1) 空间高效、易调试 代码稍繁琐 工程开发、高性能场景
递归 O(max(m,n)) O(max(m,n)) 代码简洁、逻辑直观 空间开销大 算法理解、短链表场景

5、拓展思考（面试高频考点）

5.1 边界场景天然适配

两种解法均无需额外处理以下边界场景：

  • 短链表补0：通过if (l1 != null)判断，空节点自动按0参与计算；
  • 全0输入：如l1=[0], l2=[0]，sum=0+0+0=0，直接返回[0]；
  • 超长进位：如示例3的7位+4位链表，递归/迭代会自动处理最后一位的进位1。

5.2 面试追问：正序链表相加

若题目改为「链表正序存储数字」（表头为最高位），核心思路是：

  1. 反转两个输入链表，转为逆序存储；
  2. 调用本文的相加逻辑；

6、总结

两数相加（链表版）的核心是「逆序遍历+进位处理」，本文核心总结如下：

  1. 逆序链表的存储特性契合加法计算顺序，是解题的天然优势；
  2. 迭代法通过「哑节点+循环遍历」实现，空间复杂度O(1)，是工程首选；
  3. 递归法通过「递归栈+终止条件」实现，代码简洁，适合理解递归思想；
  4. 进位处理规则（sum%10存值、sum/10进位）是两种解法的核心，需牢记；
  5. 面试中需掌握「正序链表相加」的拓展思路（反转链表+逆序相加+反转结果）。
