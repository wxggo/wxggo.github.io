Leetcode-017:电话号码的字母组合-DFS递归解法

Created2026-02-04|Updated2026-02-04|Java学习笔记Leetcode Hot100
|Word Count:3.6k|Reading Time:12mins|Post Views:

1、题目

给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。答案可以按 任意顺序 返回。
数字到字母的映射（与电话按键相同）如下：

示例

示例 1：
输入：digits = “”
输出：[]

示例 2：
输入：digits = “2”
输出：[“a”,”b”,”c”]

提示

  • 0 <= digits.length <= 4
  • digits[i] 是范围 ['2', '9'] 的数字。

2、分析

2.1 问题核心：多叉树的深度优先遍历

电话号码的字母组合本质是多层选择的组合问题：

  • 每一个数字对应一层选择（比如输入”23”，第一层选2对应的a/b/c，第二层选3对应的d/e/f）；
  • 所有组合就是从第一层到最后一层的「路径全排列」，对应多叉树的「根到叶子节点的所有路径」。

以输入”23”为例，多叉树结构如下：

所有根到叶子的路径（ad/ae/af/bd/be/bf/cd/ce/cf）就是最终结果，而递归（DFS）是遍历这种多叉树最直观、最高效的方式。

2.2 核心难点：如何设计递归？（为什么是本文代码的写法？）

递归解法的核心难点在于「路径的构建与复用」，常见的低效写法是每次递归传递字符串拼接的结果（如dfs(i, path + c)），但字符串是不可变对象，频繁拼接会产生大量临时对象，影响性能。

本文给出的代码是最优递归设计，核心优化点是「路径复用」：用固定长度的char[] path存储递归路径，通过「覆盖」而非「拼接」更新路径，仅在递归终止时将路径转为字符串存入结果。这种设计的底层逻辑可拆解为3个核心问题：

  1. 为什么path数组长度固定为digits.length()？
  2. 为什么递归层级i对应数字的索引？
  3. 为什么递归终止条件是i == digits.length()？
  4. 为什么遍历字母时直接path[i] = c覆盖即可？

这四个问题的答案，就是理解该递归设计的关键。

2.3 核心解题思路

  1. 映射关系预处理：用常量数组MAPPING存储数字到字母的映射，O(1)查询；
  2. 边界处理：输入为空时直接返回空列表，避免无效递归；
  3. 路径数组初始化：创建长度等于数字串长度的char[] path，用于复用存储递归路径；
  4. DFS递归遍历：
    • 递归参数：当前处理的数字索引i、结果列表ans、路径数组path、数字字符数组digits；
    • 终止条件：i == digits.length()（处理完所有数字，路径构建完成）；
    • 层级遍历：对当前数字对应的所有字母，更新路径数组的第i位，递归处理下一个数字；
  5. 返回结果：递归结束后，结果列表即为所有字母组合。

3.2 代码核心点逐行解释（重点：递归设计逻辑）

（1）映射表MAPPING的设计

  • 核心逻辑：数组索引与数字一一对应（索引2对应数字2，值为”abc”），索引0/1为空（因数字0/1无对应字母）；
  • 优势：MAPPING[digits[i] - '0']可O(1)获取当前数字的字母串，比HashMap映射更高效（数组访问比哈希表查询快）。

（2）路径数组path的初始化

  • 核心逻辑：n是数字串长度，每个组合的字符串长度必然等于n（比如”23”的组合都是2个字符），因此path长度固定为n；
  • 优势：复用同一个数组存储路径，避免频繁创建字符串（如path + c的拼接操作），降低内存开销。

（3）递归终止条件

  • 核心逻辑：i是当前处理的数字索引，i == digits.length()表示已经处理完所有数字（比如”23”的i从0→1→2，i=2时终止），此时path数组已存储完整的字母组合，转为字符串存入结果即可；
  • 合理性：递归的每一层对应一个数字的选择，i从0到n-1对应所有数字的选择完成，i=n时路径完整。

（4）递归层级与路径更新

  • 核心逻辑1：i是递归层级，对应数字串的索引，因此path[i]就是当前层级（当前数字）要选择的字母位置，直接覆盖即可（无需回溯！因为下一次循环会覆盖该位置的旧值）；
  • 核心逻辑2：dfs(i+1)表示处理下一个数字（进入多叉树的下一层），递归返回后，当前层的for循环会继续遍历下一个字母，自动覆盖path[i]的旧值，天然实现「回溯」；
  • 为什么无需手动回溯？比如处理”23”的a→d后，递归返回，for循环会处理a→e，此时path[1]会被e覆盖，无需手动重置为默认值——这是「路径复用+覆盖」的核心优势，代码更简洁。

4、递归设计的核心分析（为什么这么设计？）

4.1 为什么path长度固定为n？

  • 本质：每个字母组合的长度 = 数字串的长度（比如输入”23”，组合都是2个字符；输入”789”，组合都是3个字符）；
  • 设计优势：固定长度的数组可「精准覆盖」每一位的选择，避免动态扩容的开销，且数组的访问/修改是O(1)操作，比字符串拼接（O(k)，k为当前字符串长度）高效。

4.2 为什么递归层级i对应数字的索引？

  • 本质：递归的每一层对应一个数字的选择，i是当前处理的数字位置，天然与多叉树的「层级」一一对应；
  • 设计优势：通过i的递增，可有序遍历所有数字，避免重复/遗漏选择，逻辑直观且无冗余。

4.3 为什么终止条件是i == digits.length()？

  • 本质：i从0开始，i=0处理第一个数字，i=1处理第二个数字……i=n时所有数字处理完毕，路径构建完成；
  • 设计优势：终止条件与数字长度强关联，无需额外判断，逻辑简洁且无边界漏洞。

4.4 为什么直接path[i] = c覆盖即可？

  • 本质：递归的「回溯」是通过for循环的迭代实现的——当前字母处理完（递归返回），下一次循环会用新字母覆盖path[i]，旧值自动失效；
  • 设计优势：无需手动执行「回溯操作」（如path[i] = '\0'），减少代码冗余，且覆盖操作是O(1)，效率最高。

5、经典示例手动模拟（直观理解递归流程）

以输入digits = "23"为例，手动模拟递归执行流程，理解路径构建过程：

步骤1：初始化

  • n=2，path = new char[2]，调用dfs(0, ans, path, ['2','3'])。

步骤2：进入dfs(0, ...)

  • i=0 != 2，num=2，letters="abc"；
  • 遍历letters的第一个字符a：
    • path[0] = 'a'，调用dfs(1, ...)。

步骤3：进入dfs(1, ...)

  • i=1 != 2，num=3，letters="def"；
  • 遍历letters的第一个字符d：
    • path[1] = 'd'，调用dfs(2, ...)。

步骤4：进入dfs(2, ...)

  • i=2 == 2，终止条件触发，ans.add(new String(path)) → ans = ["ad"]，返回。

步骤5：回到dfs(1, ...)的for循环

  • 遍历letters的第二个字符e：
    • path[1] = 'e'，调用dfs(2, ...) → ans = ["ad","ae"]，返回；
  • 遍历letters的第三个字符f：
    • path[1] = 'f'，调用dfs(2, ...) → ans = ["ad","ae","af"]，返回。

步骤6：回到dfs(0, ...)的for循环

  • 遍历letters的第二个字符b：
    • path[0] = 'b'，调用dfs(1, ...) → 遍历d/e/f，添加”bd”/“be”/“bf”；
  • 遍历letters的第三个字符c：
    • path[0] = 'c'，调用dfs(1, ...) → 遍历d/e/f，添加”cd”/“ce”/“cf”。

最终结果

ans = ["ad","ae","af","bd","be","bf","cd","ce","cf"]，与示例一致。

6、算法性能分析

6.1 时间复杂度：O(3^m × 4^k)

  • m是对应3个字母的数字个数（2/3/4/5/6/8），k是对应4个字母的数字个数（7/9）；
  • 本质：每个数字的选择数相乘（比如”23”是3×3=9，”78”是4×3=12），这是组合问题的最优时间复杂度（必须遍历所有组合）。

6.2 空间复杂度：O(n)

  • 空间开销主要来自：
    1. 路径数组path：长度为n，O(n)；
    2. 递归调用栈：深度为n，O(n)；
  • 结果列表的空间属于输出开销，不计入算法空间复杂度。

6.3 核心优势

  1. 效率最优：路径复用避免字符串拼接，减少临时对象创建，执行效率比拼接版递归高30%+；
  2. 逻辑简洁：天然实现回溯，无需手动重置路径，代码行数少；
  3. 易拓展：可直接迁移至其他组合类递归问题（如子集、排列）。

7、拓展思考（面试高频考点）

7.1 边界情况天然适配

  • 输入为空：直接返回空列表，无需进入递归，正确；
  • 输入单个数字：递归层级从0→1，终止时添加所有单个字母，正确；
  • 输入含7/9（4个字母）：映射表已覆盖，递归正常遍历，正确。

7.2 与「字符串拼接版递归」的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
字符串拼接版 O(3^m×4^k) O(n + 3^m×4^k) 逻辑极简单 大量临时字符串 入门理解
路径复用版 O(3^m×4^k) O(n) 效率高、无冗余 需理解路径复用 面试最优书写

7.3 面试高频追问：迭代解法（BFS）

递归是DFS，迭代版可通过BFS实现（队列），核心思路是「层级构建」：

  • 对比：BFS的空间复杂度更高（队列存储中间结果），但可实现「按层输出」，适合需要按数字层级展示组合的场景。

7.4 递归回溯的通用思路

本题的递归设计是「组合类回溯」的通用模板，可总结为4步：

  1. 预处理：构建选择项的映射/列表；
  2. 路径初始化：用数组/列表复用存储路径；
  3. 递归函数：
    • 参数：当前层级、结果、路径、原始数据；
    • 终止条件：层级遍历完所有选择项；
    • 层级遍历：遍历当前层级的所有选择，更新路径，递归下一层；
  4. 返回结果。

该模板可直接用于Leetcode 39（组合总和）、46（全排列）、78（子集）等经典回溯题。

总结

电话号码的字母组合的路径复用版DFS递归解法是本题的最优解，也是理解回溯思想的关键，核心总结如下：

  1. 问题本质是多叉树的DFS遍历，递归层级对应数字索引，天然适配组合选择的逻辑；
  2. 路径复用是核心优化：固定长度的char[] path避免字符串拼接，覆盖操作天然实现回溯；
  3. 递归设计的3个核心：层级对应数字索引、终止条件为层级遍历完成、路径覆盖无需手动回溯；
  4. 该解法时间复杂度为组合问题的最优值，空间复杂度O(n)，是面试的首选书写方式；
  5. 掌握该递归模板后，可快速迁移至其他组合类回溯问题，形成通用解题思路。
