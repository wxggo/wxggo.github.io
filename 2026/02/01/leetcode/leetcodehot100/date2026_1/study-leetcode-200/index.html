Leetcode-200:实现岛屿数量统计-深度优先搜索（DFS）

Created2026-02-01|Updated2026-02-01|Java学习笔记Leetcode Hot100
|Word Count:2.2k|Reading Time:9mins|Post Views:

1、题目

给你一个由 '1'（陆地）和 '0'（水）组成的二维网格 grid，请你计算网格中岛屿的数量。

岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。此外，你可以假设该网格的四条边均被水包围。

输出：1
解释：整个网格中只有一个连通的陆地区域（岛屿）。

示例 2：
输入：

输出：3
解释：网格中有三个独立的连通陆地区域。

提示

  • m == grid.length
  • n == grid[i].length
  • 1 <= m, n <= 300
  • grid[i][j] 的值为 '0' 或 '1'

2、分析

2.1 问题核心

岛屿数量统计的本质是统计二维网格中「连通域」的数量（仅上下左右相邻的'1'构成一个连通域）。
常规思路是遍历每个网格节点，遇到未访问的'1'时，通过搜索（DFS/BFS）标记所有连通的'1'为「已访问」，每完成一次搜索则岛屿数+1。

深度优先搜索（DFS） 是最优解之一：基于递归的方式深度遍历连通区域，实现简单且空间开销可控，时间复杂度可达到O(mn)（m行n列，每个节点仅访问一次）。

2.2 核心思路

  • 遍历网格：逐行逐列检查每个格子，若当前格子是'1'，说明发现新岛屿，计数+1；
  • DFS标记已访问：从当前'1'出发，递归遍历上下左右四个方向的格子，将连通的'1'标记为非'1'（如'2'），避免重复计数；
  • 递归终止条件：越界（行/列超出网格范围）或当前格子非'1'（水/已访问的陆地）。

2.3 算法流程

  1. 初始化计数：result = 0，用于统计岛屿数量；
  2. 遍历网格：
    • 外层循环遍历每一行（i从0到grid.length-1）；
    • 内层循环遍历每一列（j从0到grid[0].length-1）；
    • 若grid[i][j] == '1'：启动DFS标记连通区域，result++；
  3. DFS递归处理：
    • 递归终止：i<0 || i>=grid.length || j<0 || j>=grid[0].length || grid[i][j]!='1'；
    • 标记当前格子为已访问（grid[i][j] = '2'）；
    • 递归遍历上（i-1,j）、右（i,j+1）、下（i+1,j）、左（i,j-1）四个方向；
  4. 返回结果：遍历完成后返回result。

3.2 代码逐行解释

（1）入口函数：numIslands

  • int result = 0：初始化岛屿计数器；
  • 双层循环：i遍历行、j遍历列，覆盖网格所有格子；
  • 核心判断：grid[i][j] == '1' 表示发现新的未访问岛屿，调用dfs标记连通区域后，计数器+1；
  • 最终返回result：即岛屿总数。

（2）DFS核心方法

  • 递归终止条件：
    • i < 0/i >= grid.length：行越界；
    • j < 0/j >= grid[0].length：列越界；
    • grid[i][j] != '1'：当前格子是水（'0'）或已访问的陆地（'2'）；
  • grid[i][j] = '2'：将当前陆地标记为已访问（原地修改，无需额外空间；若需保留原数组，可改用boolean[][] visited数组）；
  • 递归调用：依次遍历上、右、下、左四个方向，确保所有连通的陆地都被标记为已访问。

4、算法性能

  • 时间复杂度：O(mn)（m为行数，n为列数）。每个格子最多被访问一次（要么直接遍历，要么DFS标记），无重复计算；
  • 空间复杂度：最坏情况O(mn)（网格全为陆地，递归栈深度等于网格节点数）；平均情况O(log mn)（递归栈深度为岛屿的最大深度）；
  • 核心优势：
    1. 原地操作，无需额外存储（除递归栈外），空间利用率高；
    2. 递归实现简洁易懂，代码量少；
    3. 时间复杂度最优，适配题目中m,n<=300的规模。

5、测试示例

以示例2输入为例：

  1. 遍历i=0,j=0：grid[0][0]='1'，启动DFS，标记(0,0)、(0,1)、(1,0)、(1,1)为'2'，result=1；
  2. 遍历到i=2,j=2：grid[2][2]='1'，启动DFS，标记(2,2)为'2'，result=2；
  3. 遍历到i=3,j=3：grid[3][3]='1'，启动DFS，标记(3,3)、(3,4)为'2'，result=3；
  4. 遍历完成，返回result=3，与示例输出一致。

6.3 其他解法

  • 并查集（Union-Find）：将连通的陆地合并为同一集合，最终统计集合数量即为岛屿数，适合需要频繁合并/查询的场景；
  • 迭代版DFS：用栈替代递归，避免递归栈溢出问题，逻辑与递归版一致。

6.4 适用场景

DFS解法适合中小规模网格的岛屿统计，代码简洁易实现；BFS/并查集适合大规模网格或需避免递归的场景。该思路可扩展至「最大岛屿面积」「封闭岛屿数量」等同类二维网格连通域问题。
