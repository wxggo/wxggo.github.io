Leetcode Hot100 刷题顺序+题型分类总结（面试/复习专用）

Created2026-02-01|Updated2026-02-01|学习笔记面试学习
|Word Count:3.2k|Reading Time:10mins|Post Views:

前言

Leetcode Hot100 是大厂面试算法题的核心题库，覆盖了90%以上的高频考点。不合理的刷题顺序易导致“上手难、基础虚、越刷越慌”，而清晰的题型分类能让复习更有针对性。本文结合高频面试场景与学习规律，总结Hot100的最优刷题顺序和全量100题核心题型分类，适配面试复习阶段的知识点梳理与巩固。

一、Leetcode Hot100 合理刷题顺序（从易到难，贴合面试优先级）

刷题核心原则：先掌握基础数据结构，再攻克算法思想；先刷高频简单题，再啃进阶综合题，避免跳跃式刷题导致基础不牢。

1. 第一阶段：基础入门（搭建解题思维）

  • 核心模块：数组 → 字符串 → 链表
  • 刷题原因：三类题型是算法题的基础载体，代码实现简单、面试占比超30%，能快速建立“解题手感”，降低入门门槛。
  • 重点关注：
    • 数组：双指针（快慢/左右）、滑动窗口、二分查找（如两数之和、三数之和、最长无重复子串）；
    • 字符串：回文判断、字符串匹配（暴力/KMP）、拼接/分割技巧（如最长回文子串、有效的括号）；
    • 链表：反转、环检测、节点删除/合并（如反转链表、合并两个有序链表、环形链表）。
  • 刷题建议：每类刷5-8道Hot100题，重点理解“空间换时间”思路与“边界条件处理”（面试高频追问点）。

2. 第二阶段：数据结构进阶（夯实核心考点）

  • 核心模块：二叉树/二叉搜索树 → 栈/队列 → 哈希表
  • 刷题原因：二叉树是递归/遍历的核心载体（面试必考），栈/队列、哈希表是“时间复杂度优化”的关键工具，也是进阶题的基础。
  • 重点关注：
    • 二叉树：前/中/后序遍历（递归+迭代）、层序遍历、翻转二叉树、路径总和（如Leetcode 226翻转二叉树、104二叉树的最大深度）；
    • 栈/队列：单调栈、队列与栈互实现、滑动窗口最大值（依赖单调队列）；
    • 哈希表：HashMap/HashSet的应用、去重/映射场景、字母异位词分组（面试常追问哈希冲突处理）。
  • 刷题建议：二叉树优先掌握递归解法（面试先讲递归，再补充迭代优化），哈希表结合Java底层原理理解（如扩容机制、红黑树转换）。

3. 第三阶段：算法思想入门（掌握解题套路）

  • 核心模块：双指针 → 递归/回溯 → 贪心
  • 刷题原因：算法思想是“从基础题到进阶题”的桥梁，双指针优化时间复杂度，递归/回溯解决排列组合类问题，贪心是动态规划的简化版。
  • 重点关注：
    • 双指针：数组/链表/字符串的双指针应用（如盛最多水的容器、删除有序数组中的重复项）；
    • 递归/回溯：子集、组合、全排列（如Leetcode 78子集、46全排列），核心是“回溯剪枝”逻辑；
    • 贪心：区间调度、买卖股票的最佳时机，重点理解“局部最优推全局最优”的证明（面试高频追问）。
  • 刷题建议：回溯题先画递归树（面试写题前画树，体现思路完整性），贪心题总结“贪心策略”的适用场景。

4. 第四阶段：进阶算法（攻克面试压轴题）

  • 核心模块：动态规划 → 图论（DFS/BFS）→ 特殊技巧（并查集/单调栈）
  • 刷题原因：动态规划是大厂面试压轴题核心，图论覆盖“岛屿、最短路径”等高频场景，特殊技巧是难题优化的关键。
  • 重点关注：
    • 动态规划：子序列问题、背包问题、打家劫舍（核心是“状态定义+转移方程”）；
    • 图论：DFS/BFS（岛屿数量、单词接龙）、最短路径（Dijkstra）、拓扑排序；
    • 特殊技巧：并查集（朋友圈问题）、单调栈（接雨水、柱状图中最大的矩形）。
  • 刷题建议：动态规划先刷“单维度状态”题（如爬楼梯），再刷“二维状态”题（如最长公共子序列）；图论熟记DFS/BFS模板（面试可直接套用）。

5. 第五阶段：综合压轴（适配面试场景）

  • 核心模块：设计题 → 高频模拟题 → 数学类
  • 刷题原因：设计题考察代码设计能力，模拟题考察逻辑严谨性，数学类题覆盖基础数学思维。
  • 重点关注：
    • 设计题：LRU缓存、LFU缓存（重点是数据结构选型）；
    • 模拟题：字符串相乘、螺旋矩阵（考察边界处理能力）；
    • 数学类：阶乘后的零、x的平方根（考察公式推导）。
  • 刷题建议：设计题总结“缓存淘汰策略”“数据结构组合使用”（如LRU用哈希表+双向链表）。

基于你提供的分类框架，我进行了去重精简（保留Hot100核心100题）和面试频率分级，使用红色高亮标注了各个分类中的超高频必考题（S级），并优化了题型分布数量。

补充说明

跨题型题目归属原则：

  • 有效的括号：归入栈类（核心考点是栈匹配，非字符串处理）
  • 接雨水：归入数组类（核心解法为双指针/DP，单调栈为辅）
  • 买卖股票系列：归入动态规划类（DP为通用解法，贪心仅适用于无限交易）
  • LRU缓存：归入哈希表类（核心考点是哈希+双向链表设计）

面试超高频Top 20（必背清单）：
若时间有限，优先掌握以下标🔥的S级题目，覆盖90%面试场景：

1, 11, 15, 42, 53, 56, 3, 5, 19, 21, 25, 141, 206, 102, 104, 105, 226, 236, 20, 84, 146, 128, 46, 78, 70, 121, 198, 300, 322, 1143, 200, 207, 55, 208, 295

数量分布合理性：

  • 线性结构（数组+链表+字符串）：34题（基础，面试考察率40%）
  • 树（二叉树）：15题（核心，面试考察率25%）
  • 动态规划：18题（难点，面试考察率20%）
  • 其他（图+回溯+设计+栈+哈希+数学）：33题（综合，面试考察率15%）

三、面试复习阶段的刷题建议

1. 第一遍刷题：“懂解法”

  • 目标：理解核心思路，能写出AC代码，不追求最优解；
  • 方法：刷完一类题后总结“模板”（如二叉树遍历模板、回溯模板），形成笔记。

2. 第二遍刷题：“抠细节”

  • 目标：优化时间/空间复杂度，理解“最优解的底层逻辑”；
  • 方法：针对高频题思考“面试官追问的优化方案”（如递归转迭代、暴力转双指针）。

3. 第三遍刷题：“模拟面试”

  • 目标：限时做题（30分钟/题），口头讲解思路；
  • 方法：按“思路→核心逻辑→边界条件→时间复杂度”的顺序讲解，贴合面试答题逻辑。

四、总结

Leetcode Hot100的刷题核心是“先基础后进阶，先高频后冷门”，题型分类的本质是“按考点聚类，避免零散刷题”。面试复习阶段，不要只追求“刷完题”，更要注重“每道题吃透考点、能讲清思路、应对追问”。

建议每天保持1-2道题的刷题量，刷完一类题后做复盘（如二叉树刷完后总结所有遍历方式的代码模板），形成自己的知识体系。算法题面试考察的不仅是代码能力，更是“解题思路的清晰性”和“问题分析的逻辑性”——这也是按顺序、分题型刷题的核心目的。
