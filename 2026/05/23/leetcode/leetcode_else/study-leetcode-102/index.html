前言

Leetcode102 是二叉树中非常经典的基础题，主要考察 BFS 广度优先搜索 和 层序遍历。
题目要求按照从上到下、从左到右的顺序，一层一层遍历二叉树，并将每一层的节点值单独放入一个列表中。
这道题也是很多二叉树题目的基础，例如：

  • 二叉树的锯齿形层序遍历；
  • 二叉树的右视图；
  • 二叉树的最大深度；
  • 每一层的平均值。

本文使用 cur + nxt 的方式实现分层遍历，逻辑清晰，适合面试讲解和复习。

2、分析

2.1 核心思路

二叉树的层序遍历，本质是 BFS 广度优先搜索。

BFS 的特点是：

先访问离根节点近的节点，再访问更远的节点。

  2. 将当前节点的左孩子、右孩子加入 nxt；
  3. 当前层遍历结束后，将 vals 加入答案；
  4. 更新 cur = nxt，继续遍历下一层。

cur 表示当前正在遍历的这一层。

4.4 nxt 保存下一层

nxt 用来保存当前层所有节点的孩子节点。

这里必须先加入左孩子，再加入右孩子，因为题目要求每一层从左到右输出。

4.5 vals 保存当前层节点值

vals 保存当前层所有节点值。

4.6 更新当前层

当前层遍历结束后：

  • vals 是当前层结果；
  • nxt 是下一层节点。

所以先把 vals 加入答案，再让 cur 指向下一层。

写法 核心思想 特点

如果是普通层序遍历，cur + nxt 写法很直观；如果是图搜索、最短路径等问题，队列写法更通用。

8、面试常见问题

8.1 为什么层序遍历是 BFS？

因为它是按照距离根节点的远近访问节点。

先访问第 1 层，再访问第 2 层，再访问第 3 层，这正是 BFS 的典型特点。

8.2 DFS 能不能做？

可以。

DFS 也能完成层序遍历，只需要递归时记录当前深度 depth，然后把节点值加入对应层的列表。

但本题用 BFS 更自然。

8.3 为什么要先加入左孩子？

因为题目要求每一层从左到右输出。

如果先加入右孩子，结果顺序就会错误。

10、总结

Leetcode102 是二叉树 BFS 的基础题，核心是按层遍历。

本文代码使用 cur + nxt 实现层序遍历：

  1. cur 保存当前层节点；
  2. nxt 保存下一层节点；
  3. 遍历 cur 时收集节点值到 vals；
  4. 将左右孩子加入 nxt；
  5. 当前层结束后，ans.add(vals)；
  6. 更新 cur = nxt，进入下一层。

面试中需要重点掌握：

  • BFS 的基本思想；
  • 如何区分每一层；
  • 空树边界处理；
  • 左右孩子的加入顺序；
  • cur + nxt 写法和队列写法的区别；
  • 时间复杂度和空间复杂度分析。

只要能清楚解释“当前层”和“下一层”的关系，就说明已经理解了二叉树层序遍历的核心。
