2.1 问题核心

本题是经典的数组排序问题。常见做法包括：

  • 快速排序；
  • 归并排序；

本文重点总结 随机快速排序 + 三路划分 的写法。

这是三路划分最容易写错的地方。

4.4 小于 pivot 时为什么 cur 要移动？

因为 [less, cur - 1] 这一段本来就是等于 pivot 的区域。

当 nums[cur] < pivot 时，交换 cur 和 less，换到 cur 位置的元素一定是等于 pivot 的元素，已经处理过，因此 cur 可以右移。

7、面试常见问题

7.1 快速排序为什么平均是 O(n log n)？

因为每一层递归都要处理全部元素，总处理量是 O(n)。

如果每次划分比较均匀，递归层数大约是 log n。

7.2 快速排序什么时候会退化成 O(n²)？

当每次选到的 pivot 都是当前区间的最大值或最小值时，划分会极不均衡。

例如每次划分后，一边为空，另一边有 n - 1 个元素。

递归深度就会变成 n，总时间复杂度退化为：

7.3 为什么三路划分适合处理重复元素？

如果数组中有很多重复元素，普通两路快排仍然可能反复处理这些重复值。

三路划分会一次性把所有等于 pivot 的元素放到中间区域。

中间区域已经有序，不需要继续递归，因此可以减少大量无效递归。

7.4 快速排序是稳定排序吗？

不是。

因为快速排序中存在交换操作，可能改变相同元素之间的相对顺序。

如果题目要求稳定排序，可以考虑使用归并排序。

7.5 快速排序和归并排序怎么选择？

算法 时间复杂度 空间复杂度 稳定性 特点
快速排序 平均 O(n log n)，最坏 O(n²) 平均 O(log n) 不稳定 原地排序，常数较小，实际表现好
归并排序 稳定 O(n log n) O(n) 稳定 性能稳定，但需要额外数组
堆排序 O(n log n) O(1) 不稳定 空间优秀，但实际常数较大

9、扩展思考

9.1 可以进一步优化递归深度吗？

可以。

为了避免递归深度过深，可以每次优先递归较小的区间，然后用循环处理较大的区间。

这样可以将递归栈深度控制在 O(log n)。

10、总结

本题使用的是 随机快速排序 + 三路划分。

核心要点如下：

  • 随机选择 pivot，降低快排退化概率；
  • 使用三路划分，将数组分成 < pivot、== pivot、> pivot 三个区域；
  • nums[cur] < pivot 时，交换到小于区，less++，cur++；
  • nums[cur] > pivot 时，交换到大于区，bigger--，但 cur 不动；
  • nums[cur] == pivot 时，直接 cur++；
  • 循环条件必须是 cur <= bigger，避免漏处理最后一个元素；
  • 递归时只处理小于区和大于区，中间等于区不需要继续排序。

这道题不仅考察排序算法本身，也很适合用来复习：

  • 快速排序；
  • 随机化算法；
  • 三路划分；
  • 荷兰国旗问题；
  • 递归边界；
  • 时间复杂度和空间复杂度分析。

面试中如果遇到排序相关问题，能够清楚解释这份代码的分区逻辑、cur 是否移动的原因，以及随机 pivot 的意义，基本就能说明对快速排序掌握得比较扎实。
