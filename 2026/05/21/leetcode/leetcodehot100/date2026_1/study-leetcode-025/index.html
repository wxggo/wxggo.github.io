1、题目

给你链表的头节点 head，每 k 个节点一组进行翻转，请你返回修改后的链表。

k 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 k 的整数倍，那么最后剩余不足 k 个节点需要保持原有顺序。

你不能只是单纯改变节点内部的值，而是需要实际进行节点交换。

  • 1 <= k <= n <= 5000
  • 0 <= Node.val <= 1000

2、分析

2.1 问题核心

这道题的本质是链表局部反转 + 分组拼接。

普通链表反转只需要把整条链表反转一次，而本题要求每 k 个节点为一组进行反转，并且最后不足 k 个节点不能反转。因此核心难点不在于“怎么反转链表”，而在于：

  1. 如何准确找到每一组的起点和终点；
  2. 如何只反转当前这一组，不影响后面的链表；
  3. 如何把反转后的这一组重新接回原链表；
  4. 如何处理最后不足 k 个节点的边界情况。

本题代码采用的是面试中非常常用的虚拟头节点 + 分组断链 + 局部反转 + 重新拼接写法。

2.2 为什么需要虚拟头节点 dummy？

链表题中，只要涉及头节点可能变化，就可以优先考虑虚拟头节点。

例如 head = [1,2,3,4,5], k = 2，第一组反转后，原来的头节点 1 会变成第二个节点，新的头节点变成 2。如果没有 dummy，就需要单独处理第一组反转后的头节点更新问题。

3.2 代码简洁解释

  • dummy ：虚拟头节点，用于统一处理第一组反转后头节点变化的问题；
  • pre ：始终指向当前分组的前一个节点，方便把反转后的分组接回原链表；
  • end ：每轮向后走 k 步，判断当前剩余节点是否够一组；
  • start ：当前分组的第一个节点，反转后会变成这一组的尾节点；
  • next ：当前分组后面的第一个节点，用于反转后重新连接后续链表；
  • end.next = null ：临时断开当前分组，让 reverse(start) 只反转当前 k 个节点；
  • pre.next = reverse(start) ：将反转后的分组头节点接到上一段链表后面；
  • start.next = next ：将反转后的分组尾节点接回下一段链表；
  • pre = start; end = pre; ：移动到下一组反转前的位置。

5、算法性能

  • 时间复杂度：O(n)
    每个节点只会被遍历和反转有限次数，整体仍然是线性复杂度。
  • 空间复杂度：O(1)
    只使用了若干辅助指针，反转过程原地完成，没有使用额外数组或栈。
  • 是否改变节点值：否
    本题要求交换节点本身，而不是交换节点的 val，该解法符合题意。

8、扩展思考

8.1 和普通链表反转的区别

普通链表反转只需要关心三个指针：

而 K 个一组反转除了要会反转链表，还需要额外处理：

  1. 分组边界；
  2. 不足 k 个节点不反转；
  3. 反转后的前后拼接；
  4. 第一组反转导致头节点变化。

因此本题是链表反转类题目的进阶版本。

8.2 能不能用递归？

可以。递归思路通常是：

  1. 先判断从 head 开始是否有 k 个节点；
  2. 如果不足 k 个，直接返回 head；
  3. 如果够 k 个，反转前 k 个节点；
  4. 将反转后的尾节点接上递归处理后的下一段链表。

递归写法更短，但会使用递归调用栈，空间复杂度为 O(n/k)；而当前迭代写法是 O(1) 空间，更适合面试中强调原地操作的场景。

8.3 面试时怎么讲？

面试回答可以按照“四步法”表达：

  1. 先用 dummy 统一处理头节点变化；
  2. 用 end 每次向后走 k 步，判断当前是否够一组；
  3. 够一组就保存 start 和 next，然后断开当前分组并调用反转函数；
  4. 反转后把这一组接回原链表，再移动 pre/end 处理下一组。

这样讲既能体现思路完整，也能说明你对边界情况和链表指针操作比较熟悉。

30秒口述版:
K 个一组翻转链表的核心是“分组 + 局部反转 + 重新拼接”。我会先加一个 dummy 节点统一处理头节点变化，然后用 end 指针每次向后找 k 个节点。如果不足 k 个就不反转；如果够一组，就记录当前组的 start 和下一组的 next，把当前组断开后调用普通链表反转函数，最后用 pre.next 接上反转后的头节点，再用 start.next 接回后续链表。整个过程每个节点只处理常数次，所以时间复杂度是 O(n)，空间复杂度是 O(1)。

9、总结

K 个一组翻转链表 是链表专题中非常经典的综合题，考察的不只是反转链表本身，更重要的是对链表边界、分组控制和指针拼接的理解。

这道题可以总结为一句话：

先找够一组，再断开反转，最后重新拼接；不足一组，保持原样。

掌握这道题后，再看两两交换链表节点、反转链表 II、旋转链表等题目，会更容易理解它们背后的指针操作套路。
