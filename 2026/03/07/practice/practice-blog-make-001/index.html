Blog-个人博客搭建教程：Hexo+GitHub Pages一站式部署（超详细实操版）

Created2026-03-07|Updated2026-03-07|学习笔记博客搭建
|Word Count:3k|Reading Time:9mins|Post Views:

本文为Hexo+GitHub Pages个人博客搭建的全流程实操笔记，基于实际部署步骤整理，从前期资源准备到环境配置，再到本地调试和远程上线，全程详细拆解每一步操作与命令，同时解析博客核心文件结构和日常更新方法，零基础也能快速搭建专属的免费个人博客，适合开发者、学习者打造个人技术笔记/知识分享平台。

一、前言

Hexo是一款轻量级的静态博客框架，搭配GitHub Pages的免费静态页面托管服务，是零基础搭建个人博客的最优选择——无需服务器、无需域名费用，部署简单且维护成本低，同时支持自定义主题、Markdown格式创作，完美适配技术笔记、学习总结的创作与展示。

本文的搭建流程基于Hexo框架，从环境安装到博客上线全程实操，核心实现本地编写-本地预览-一键部署的博客创作流程，且配置完成后后续仅需修改Markdown文件即可更新博客内容，同时兼顾域名优化（一级域名）、SSH安全连接等细节，搭建完成后即可拥有如https://xxx.github.io/的专属个人博客。

二、前期准备：资源与账号配置

2.1 资源获取

解压博客项目压缩包，获取两个核心文件夹：

  • 博客项目文件夹：如jphulab（可自定义重命名为自己的博客名，如wxggo）；
  • 软件安装文件夹：software，内含Git和Node.js安装包，为博客搭建的基础环境。

2.2 GitHub账号注册

  1. 用QQ邮箱单独注册GitHub专属账号，用于博客网站的代码托管与部署；
  2. 核心域名优化：新建GitHub仓库时，仓库名必须与GitHub账号名完全一致，搭建完成后可获得一级域名（如https://wxggo.github.io/），否则会生成二级域名，影响访问与展示；
  3. 完成注册后记住GitHub账号名与密码，后续SSH配置和部署会用到。

三、基础环境安装：Git+Node.js

博客搭建的基础环境为Git（版本控制与远程连接）和Node.js（Hexo框架运行环境，内含npm包管理工具），安装步骤如下：

  1. 进入software文件夹，依次双击安装Git-2.45.1-64-bit和node-v20.13.1-x64；
  2. 安装建议：安装位置可自定义，其余所有选项均保持默认即可，无需额外配置；
  3. 安装完成后无需手动启动，后续通过Git Bash窗口调用相关命令。

  1. 进入解压后的博客项目文件夹（如jphulab），在文件夹空白处单击右键，选择Open Git Bash here，打开Git命令行窗口；

6.4 GitHub后台配置SSH公钥

  1. 打开GitHub官网并登录账号，点击右上角头像，选择Settings进入个人设置；
  1. 在左侧菜单栏找到SSH and GPG keys，点击右上角New SSH key；
  1. 配置SSH密钥信息：
    • Title：自定义名称（如blog_ssh），无特殊要求；
    • Key type：选择Authentication Key（默认）；
    • Key：粘贴上一步复制的SSH公钥字符串；
  2. 点击Add SSH key完成添加，若需要验证GitHub密码，输入后即可。

  • 若出现连接失败，检查公钥是否复制完整、Git用户信息是否与GitHub一致。

八、博客核心文件结构解析

用VSCode打开博客项目文件夹（如jphulab），核心文件/文件夹的功能与作用如下，日常博客维护仅需修改少数文件夹，无需改动底层配置：

8.1 核心配置文件：_config.yml

Hexo的主配置文件，包含站点标题、博客描述、主题、部署地址、插件等所有核心配置，可自定义修改博客名称、作者、主题样式等，部署前需确认deploy模块的GitHub地址配置正确。

8.2 内容核心文件夹：source

博客的所有源文件均存放在此，是日常更新博客的核心操作文件夹，关键子文件夹：

  • _posts：存放博客的Markdown文章，默认作为博客的首页，新建/修改MD文件即发布/更新博客文章；
  • zh-CN：存放博客的中文版界面文件，实现多语言适配；
  • images：存放博客所需的所有图片、视频等静态资源，建议按文章分类建子文件夹；
  • 自定义页面文件夹：如Publication/Projects，对应博客的独立页面，修改文件夹下的index.md即可更新页面内容。

8.3 主题文件夹：themes

存放Hexo的主题文件，包含主题的模板、样式表、脚本等，可在此更换/自定义博客主题（如Butterfly、Next），主题的个性化配置可在主题文件夹下的_config.yml中修改。

8.4 其他文件夹

  • node_modules：存放npm安装的所有依赖包，无需手动修改；
  • public：执行生成命令后自动生成的静态页面文件，部署时会同步到GitHub；
  • scaffolds：博客文章的模板文件，可自定义Markdown文章的默认格式。

  5. 访问博客：在浏览器中打开自己的GitHub Pages域名（https://你的GitHub账号名.github.io/），即可看到上线后的个人博客，刷新页面即可查看最新内容。

十一、额外配置：GitHub Pages域名与状态查看

若需确认博客的部署状态、修改域名或查看访问配置，可通过GitHub仓库操作：

  1. 登录GitHub账号，进入与账号名一致的博客仓库；
  2. 点击仓库顶部的Settings，在左侧菜单栏找到Pages；
  3. 在此页面可查看：
    • 博客的官方访问域名；
    • 部署状态（是否成功、最新部署时间）；
    • 自定义域名配置（可绑定自己的域名）；
    • 部署分支与文件夹（默认无需修改）。

十二、博客日常更新方法

博客搭建完成后，日常更新仅需3步，无需重复配置环境：

  1. 用VSCode打开博客项目文件夹，在source/_posts中新建/修改Markdown文章，或更新其他页面的index.md文件；
  2. 在Git Bash窗口执行清理-生成-部署命令，一键同步：
  3. 等待30秒后，刷新博客域名，即可看到更新后的内容。

十三、总结

本次Hexo+GitHub Pages个人博客的搭建核心，是通过Hexo实现本地静态博客的生成与管理，通过GitHub Pages实现免费的静态页面托管，通过SSH密钥实现本地与Git的安全免密连接，整个流程零基础可操作，且配置完成后后续维护成本极低。

搭建的关键细节总结：

  1. GitHub仓库名与账号名一致，可获得一级域名，提升博客展示效果；
  2. SSH密钥配置是部署的核心，需确保公钥复制完整、Git用户信息与GitHub一致；
  3. 日常更新仅需操作source文件夹，无需改动底层配置，执行hexo cl && g && d即可一键部署；
  4. 本地预览功能可快速验证修改效果，避免直接部署错误内容。

Hexo框架支持丰富的主题定制和插件扩展，后续可根据需求更换博客主题、添加评论功能、配置文章搜索等，打造个性化的个人技术博客/知识分享平台。
