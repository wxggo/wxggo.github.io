Leetcode-064:最小路径和-从递归到递推的动态规划解法（基础到优化版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100
|Word Count:3.1k|Reading Time:12mins|Post Views:

前言

最小路径和是Leetcode动态规划模块的经典高频题，也是面试中考察「递归思维→动态规划优化」的核心题目之一。本题的核心难点在于如何将暴力递归的指数级时间复杂度优化为线性级，而从「递归（自顶向下）」到「递推（自底向上）」的转化，是理解动态规划核心思想的关键路径。

1、题目

给定一个包含非负整数的 m x n 网格 grid ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

说明：每次只能向下或者向右移动一步。

示例 1：
输入：grid = [[1,3,1],[1,5,1],[4,2,1]]
输出：7
解释：因为路径 1→3→1→1→1 的总和最小。

示例 2：
输入：grid = [[1,2,3],[4,5,6]]
输出：12
解释：路径 1→2→3→6 的总和最小（或1→4→5→6，总和均为12）。

提示

  • m == grid.length
  • n == grid[i].length
  • 1 <= m, n <= 200
  • 0 <= grid[i][j] <= 100

2、分析

2.1 问题核心：状态转移与路径选择

最小路径和的本质是「每一步的局部最优选择，最终导向全局最优」：

  • 到达网格中任意位置 (i,j) 的最小路径和，仅依赖于「到达 (i,j-1) 的最小路径和」（从左来）和「到达 (i-1,j) 的最小路径和」（从上来）；
  • 核心逻辑：minPath(i,j) = min(minPath(i,j-1), minPath(i-1,j)) + grid[i][j]。

2.2 暴力递归：直观但低效的初始思路

2.2.1 递归核心思路

递归的本质是「自顶向下」的拆解：要计算到达右下角 (m-1,n-1) 的最小路径和，先计算到达其左侧 (m-1,n-2) 和上侧 (m-2,n-1) 的最小路径和，再取最小值加上当前网格值；以此类推，直到拆解到起点 (0,0)（基准条件）。

2.2.2 递归的时间瓶颈

暴力递归的时间复杂度为 O(2^(m+n))（指数级）：

  • 每个位置 (i,j) 都会被重复计算多次（比如网格中间的位置，会被多个上层路径重复递归调用）；
  • 面对 m、n 最大为 200 的网格，指数级复杂度会直接超时，无实际解题价值，但却是理解状态转移的基础。

2.3 递推优化：从递归到动态规划的核心转化

递推的本质是「自底向上」的填充：

  1. 定义 dp 数组：dp[i][j] 表示到达 (i,j) 的最小路径和；
  2. 初始化边界：
    • 第一行：只能从左来，dp[0][j] = dp[0][j-1] + grid[0][j]；
    • 第一列：只能从上来，dp[i][0] = dp[i-1][0] + grid[i][0]；
  3. 状态转移：填充 dp 数组，dp[i][j] = min(dp[i][j-1], dp[i-1][j]) + grid[i][j]；
  4. 结果输出：dp[m-1][n-1] 即为答案。

递归→递推的简化过程

维度 暴力递归 递推（动态规划）
思路方向 自顶向下（拆解问题） 自底向上（填充结果）
计算方式 重复计算子问题 每个子问题仅计算一次
时间复杂度 O(2^(m+n))（指数级） O(mn)（线性级）
空间复杂度 O(m+n)（递归栈深度） O(mn)（dp数组）

3、程序：从递归到递推的实现

3.1 暴力递归版（基础思路）

递归版核心点解释

  • 边界处理：i < 0 || j < 0 时返回 Integer.MAX_VALUE，表示该方向不可达，不会影响 Math.min 的结果；
  • 基准条件：到达起点 (0,0) 时，直接返回网格值（起点的路径和就是自身）；
  • 状态转移：严格遵循「左侧/上侧取最小 + 当前值」的核心逻辑，是递推版的思路基础。

3.2 递推版（动态规划优化）

递推版核心点逐行解释

（1）dp数组扩容：避免越界判断
  • 核心设计：将 dp 数组的行/列各扩容 1（变为 (m+1)x(n+1)），虚拟出「第0行」和「第0列」作为边界；
  • 价值：原网格的 (i,j) 对应 dp 数组的 (i+1,j+1)，无需额外判断 i==0 或 j==0 的边界情况，统一用 Math.min 处理。
（2）虚拟边界初始化
  • 逻辑：将虚拟的「第0列」全部设为极大值，表示从「上侧越界位置」到达当前位置不可行；
  • 适配性：和「第0行」的初始化逻辑一致，保证状态转移时的 Math.min 只会选择有效路径。
（3）基准条件与状态转移
  • 基准条件：起点 (0,0) 对应的 dp[1][1] 直接赋值为网格值；
  • 状态转移：完全继承递归的核心逻辑，但「左侧路径和」对应 f[i+1][j]，「上侧路径和」对应 f[i][j+1]，且每个子问题仅计算一次。
（4）结果返回
  • 逻辑：原网格的右下角 (m-1,n-1) 对应 dp 数组的 (m,n)，直接返回该值即可。

4、算法性能分析

4.1 递归版性能（仅作思路参考）

  • 时间复杂度：O(2^(m+n))
    每个位置会被递归调用多次，随着 m/n 增大，计算量呈指数级增长，m=n=200 时完全无法运行；
  • 空间复杂度：O(m+n)
    递归栈的深度等于网格的最大路径长度（m+n-1），无额外空间开销。

4.2 递推版性能（最优解）

  • 时间复杂度：O(mn)
    仅需两层循环遍历网格，每个位置仅计算一次，m=n=200 时总计算量为 40000，完全满足时间要求；
  • 空间复杂度：O(mn)
    主要开销为 dp 数组，可进一步优化为 O(n)（滚动数组），但 O(mn) 已足够简洁且易理解。

4.3 递推版核心优势

  1. 效率最优：线性时间复杂度，完全适配题目输入规模；
  2. 逻辑统一：通过扩容 dp 数组，避免了繁琐的边界判断，代码简洁性高；
  3. 思路可迁移：自底向上的递推思路是动态规划的核心，可直接迁移至「不同路径」「三角形最小路径和」等同类问题。

6、拓展思考（面试高频考点）

6.1 空间优化：滚动数组（O(n)空间）

递推版的 dp 数组可优化为一维数组（滚动数组），仅保留当前行的计算结果：

  • 核心逻辑：dp[j] 既代表「上一行的 j 位置值」（从上来），也代表「当前行的 j-1 位置值」（从左来），通过覆盖实现空间复用；
  • 空间复杂度：从 O(mn) 降至 O(n)，是面试中的进阶优化点。

6.2 边界情况适配

递推版天然兼容各类边界场景：

  1. 单行网格（m=1）：只能从左到右，dp 数组直接累加即可；
  2. 单列网格（n=1）：只能从上到下，dp 数组第一列累加即可；
  3. 单元素网格（m=1,n=1）：直接返回 grid[0][0]，基准条件已覆盖。

6.3 递归优化：记忆化搜索

在暴力递归的基础上，增加「记忆化数组」缓存已计算的结果，将时间复杂度优化至 O(mn)：

  • 核心价值：记忆化搜索是「递归→递推」的中间态，既保留递归的直观性，又解决了重复计算问题，是面试中展示思路完整性的重要点。

7、总结

最小路径和的解题核心是「状态转移的统一逻辑 + 从递归到递推的优化」，本文核心总结如下：

  1. 递归是思路基础：暴力DFS直观体现「状态转移」核心，但指数级复杂度无法落地；
  2. 递推是效率最优：自底向上填充dp数组，将时间复杂度优化至O(mn)，是基础最优解；
  3. 递归→递推的简化关键：
    • 递归：拆解问题，依赖上层子问题的结果；
    • 递推：填充结果，依赖下层子问题的结果；
    • 核心是「避免重复计算」，让每个子问题仅求解一次；
  4. 进阶优化：滚动数组可将空间复杂度降至O(n)，记忆化搜索是递归与递推的中间桥梁；
  5. 思路可迁移：该递推思路适用于所有「仅依赖相邻子问题」的动态规划题目，是掌握动态规划的核心基石。
