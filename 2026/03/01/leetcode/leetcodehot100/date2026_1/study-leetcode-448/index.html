Leetcode-448:找到所有数组中消失的数字-哈希集合法+原地标记最优解（空间优化版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100
|Word Count:2.9k|Reading Time:10mins|Post Views:

前言

找到所有数组中消失的数字是Leetcode数组模块的经典高频题，也是面试中考察「空间优化+原地算法」的核心题目之一。本题的核心难点在于如何在O(n)时间复杂度下，尽可能降低空间开销——基础哈希集合法易理解但空间复杂度为O(n)，而「原地标记法」可将额外空间优化至O(1)，是本题的最优解。

1、题目

给定一个范围在 1 ≤ a[i] ≤ n（n = 数组大小）的整型数组，数组中的元素一些出现了两次，另一些只出现一次。找到所有在 [1, n] 范围内但没有出现在数组中的数字。

示例

示例 1：
输入：nums = [4,3,2,7,8,2,3,1]
输出：[5,6]
解释：数字5、6在[1,8]范围内，但未出现在数组中。

示例 2：
输入：nums = [1,1]
输出：[2]
解释：数字2在[1,2]范围内，但未出现在数组中。

示例 3：
输入：nums = [2,2]
输出：[1]
解释：数字1在[1,2]范围内，但未出现在数组中。

提示

  • n == nums.length
  • 1 ≤ n ≤ 10^5
  • 1 ≤ nums[i] ≤ n

2、分析

2.1 问题核心：如何标记已出现的数字？

题目要求找到[1,n]中缺失的数字，核心逻辑是先标记所有已出现的数字，再筛选出未被标记的数字。标记的方式直接决定了算法的空间复杂度：

  • 基础思路：用额外空间（如哈希集合）存储已出现的数字，空间复杂度O(n)；
  • 最优思路：利用数组自身的下标作为“标记容器”，通过修改数组值的符号实现原地标记，额外空间复杂度O(1)。

2.2 核心难点：空间优化

暴力解法的思路是，遍历1到n，逐一检查每个数字是否在数组中，时间复杂度O(n²)，面对10^5的数组长度会超时；
哈希集合法将检查操作优化至O(1)，但需要O(n)的额外空间；
原地标记法是解决该难点的最优思路：

  1. 利用“数字x对应下标x-1”的映射关系，将出现过的数字对应下标位置的值标记为负数；
  2. 遍历结束后，仍为正数的位置对应的下标+1，即为缺失的数字；
    整体时间复杂度O(n)，额外空间复杂度O(1)（仅结果数组占用空间，不计入算法额外空间）。

2.3 两种核心解题思路

思路1：哈希集合法（基础解法）

  1. 初始化哈希集合，遍历数组将所有数字存入集合；
  2. 遍历1到n，检查每个数字是否在集合中，不在则加入结果列表；
  3. 返回结果列表。

思路2：原地标记法（最优解法）

  1. 原地标记：遍历数组，对每个数字x，找到其对应下标index = |x| - 1（取绝对值避免已标记的负数影响），若nums[index]为正，则将其标记为负（表示x已出现）；
  2. 筛选缺失数字：遍历数组，若nums[i]为正，说明i+1未被标记（即缺失），将i+1加入结果列表；
  3. 返回结果列表。

3、程序：两种解法完整实现

3.1 解法1：哈希集合法（基础易理解）

代码核心点逐行解释

  • 核心逻辑：利用哈希集合O(1)的查询特性，快速存储和检查数字是否出现；
  • 作用：将数组中所有已出现的数字“记录”下来，为后续筛选缺失数字做准备。
  • 遍历范围：题目要求找[1,n]内的缺失数字，因此遍历从1到n；
  • 筛选逻辑：不在哈希集合中的数字，即为未出现的缺失数字。

3.2 解法2：原地标记法（空间最优）

代码核心点逐行解释

  • 下标映射：数字x的取值范围是[1,n]，对应数组下标为x-1（数组下标从0开始）；
  • 取绝对值：若数字已被标记为负数（如之前遍历到重复数字时），需还原为原始值才能找到正确下标。
  • 重复标记防护：若数字重复出现，对应下标位置已被标记为负，无需再次修改，避免无效操作；
  • 标记逻辑：负数是“已出现”的标识，不影响后续遍历（后续会通过绝对值还原）。
  • 核心逻辑：未被标记为负的位置，说明对应的数字i+1从未出现在数组中，即缺失数字；
  • 下标还原：数组下标i对应数字i+1，需加1才能得到正确的缺失数字。

4、算法性能分析

4.1 哈希集合法

  • 时间复杂度：O(n)
    • 遍历数组存入哈希集合：O(n)；
    • 遍历1到n筛选缺失数字：O(n)；
    • 哈希集合的添加/查询操作均为O(1)，总耗时可忽略。
  • 空间复杂度：O(n)
    • 空间开销主要来自哈希集合，需存储数组中所有不重复的数字，最坏情况下（数组无重复）存储n个数字。
  • 适用场景：理解成本低，适合面试中快速写出基础解法，或对空间要求不高的场景。

4.2 原地标记法

  • 时间复杂度：O(n)
    • 第一次遍历标记数字：O(n)；
    • 第二次遍历筛选缺失数字：O(n)；
    • 无嵌套循环，总时间复杂度为O(n)。
  • 空间复杂度：O(1)
    • 仅使用常数个临时变量（index），数组修改为原地操作，无额外空间开销；
    • 结果列表是题目要求的输出，不计入算法额外空间。
  • 核心优势：
    1. 空间效率拉满：满足题目“不使用额外空间”的进阶要求；
    2. 时间效率最优：线性时间复杂度，适配10^5的输入规模；
    3. 原地操作：无需额外数据结构，代码执行效率更高。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

原地标记法无需额外处理各类边界场景，代码天然兼容：

  1. 数组长度为1：如nums=[1]，标记后数组为[-1]，遍历无正数，返回[]；nums=[2]（题目约束nums[i]≤n，此情况不存在）；
  2. 数组全重复：如nums=[1,1]，标记后数组为[-1,1]，遍历下标1为正，返回[2]；
  3. 数组无缺失：如nums=[1,2,3]，标记后数组为[-1,-2,-3]，返回[]。

6.2 面试高频追问：为什么要取绝对值？

若不使用绝对值，当遍历到已被标记为负数的数字时，会计算出负数下标（如数字-3对应下标-4），导致数组越界异常。取绝对值可还原数字的原始值，保证下标始终在[0,n-1]范围内。

6.3 面试高频追问：为什么要判断nums[index]>0？

若数字重复出现（如nums=[2,2]），第一次遍历数字2时，下标1的原值2会被标记为-2；第二次遍历数字2时，若不判断，会将-2再次取反为2，导致标记失效。该判断可避免重复标记，保证每个下标仅被标记一次。

6.4 进阶拓展：数组包含0/负数怎么办？

若题目拓展为数组包含0或负数，只需调整下标映射规则：

  1. 假设数字范围为[m,n]，先将所有数字偏移至[0, n-m]，再映射到数组下标；
  2. 或改用其他标记方式（如加n），避免符号冲突。

7、总结

找到所有数组中消失的数字的原地标记法是本题的最优解，也是理解“原地算法+空间优化”的核心案例，本文核心总结如下：

  1. 基础哈希集合法易理解，但空间复杂度O(n)，适合快速写出基础解法；
  2. 原地标记法的核心是“下标映射+符号标记”，将额外空间优化至O(1)，满足面试进阶要求；
  3. 核心细节需牢记：取绝对值避免下标越界，判断正数避免重复标记；
  4. 原地标记法是数组类空间优化问题的通用思路，可迁移至“找到重复数字”“数组元素计数”等同类问题；
  5. 两种解法的时间复杂度均为O(n)，但原地标记法的空间效率远高于哈希集合法，是面试中的首选写法。
