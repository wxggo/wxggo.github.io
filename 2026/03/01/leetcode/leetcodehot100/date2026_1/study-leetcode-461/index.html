Leetcode-0461:汉明距离-位运算经典解法（基础+简洁版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode 简单题
|Word Count:1.3k|Reading Time:4mins|Post Views:

前言

汉明距离是Leetcode位运算模块的经典简单题，也是理解「异或运算+二进制位统计」的入门题目。本题的核心逻辑极其简洁：先通过异或找到两个整数的不同二进制位，再统计这些不同位的数量，即可得到汉明距离。本文将讲解两种解法——简洁版（利用Java内置API）和基础版（手动统计二进制1的个数），兼顾代码简洁性和底层逻辑理解。

1、题目

两个整数之间的汉明距离指的是这两个数字对应二进制位不同的位置的数目。

给你两个整数 x 和 y，计算并返回它们之间的汉明距离。

示例

示例 1：
输入：x = 1, y = 4
输出：2
解释：
1 (0 0 0 1)
4 (0 1 0 0)
↑ ↑
上面的箭头指出了对应二进制位不同的位置，共2个。

示例 2：
输入：x = 3, y = 1
输出：1
解释：
3 (0 1 1)
1 (0 0 1)
↑
对应二进制位不同的位置共1个。

提示

  • 0 ≤ x, y ≤ 2^31 - 1

2、分析

2.1 问题核心：异或找不同位

位运算中，异或（^） 的特性是：两个二进制位相同则结果为0，不同则结果为1。因此：

  1. 对 x 和 y 执行异或操作（x ^ y），得到的结果中，所有值为1的二进制位，就是 x 和 y 不同的位；
  2. 统计异或结果中1的个数，即为汉明距离。

2.2 核心解题思路

  1. 计算 x ^ y，得到不同位的二进制标识；
  2. 统计该结果中二进制位为1的数量；
  3. 返回统计结果。

3、程序：两种位运算实现汉明距离

3.1 解法1：简洁版（利用Java内置API）

代码实现

代码解释

  • x ^ y：核心操作，快速定位所有不同的二进制位；
  • Integer.bitCount()：Java内置API，高效统计整数二进制表示中1的个数，底层实现经过优化，性能优于手动统计；
  • 优势：代码极简（仅1行核心逻辑），面试书写效率高，且性能有保障。

3.2 解法2：基础版（手动统计1的个数）

代码实现

代码解释

  • i = x ^ y：和简洁版一致，先获取不同位的标识；
  • while (i != 0)：循环处理所有二进制位，直到所有位都被检查完毕；
  • (i & 1) == 1：按位与操作，检查当前最低位是否为1（若为1则计数+1）；
  • i = i >> 1：右移一位，将次低位变为最低位，继续检查；
  • 优势：无需依赖内置API，完全手动实现核心逻辑，适合理解底层原理。

4、算法性能分析

4.1 时间复杂度

两种解法的时间复杂度均为O(1)：

  • 整数的二进制位数固定（Java中int为32位），因此统计1的个数的循环最多执行32次，属于常数级时间复杂度；
  • 无随输入规模变化的耗时操作，性能最优。

4.2 空间复杂度

两种解法的空间复杂度均为O(1)：

  • 仅使用了常数个临时变量（i、count），无额外空间开销。

4.3 解法对比

解法 优点 缺点 适用场景
简洁版 代码极简、性能优 依赖内置API，不体现底层逻辑 面试快速答题、实际开发
基础版 理解底层位运算逻辑 代码稍繁琐 学习位运算原理

4、经典示例手动模拟

以示例1：x=1，y=4 为例，手动模拟基础版执行过程：

  1. x ^ y = 1 ^ 4 = 5（二进制：0101）；
  2. 初始化 i=5，count=0；
  3. 第一次循环：i&1=1 → count=1，i=5>>1=2（二进制：0010）；
  4. 第二次循环：i&1=0 → count不变，i=2>>1=1（二进制：0001）；
  5. 第三次循环：i&1=1 → count=2，i=1>>1=0；
  6. 循环结束，返回count=2，与示例结果一致。

5、总结

汉明距离是位运算的入门级题目，核心逻辑可总结为：

  1. 异或找不同位：x ^ y 是定位不同二进制位的关键；
  2. 统计1的个数：可选择内置API（简洁）或手动循环（理解原理）；
  3. 性能最优：时间/空间复杂度均为O(1)，完全适配题目输入范围。

掌握本题的核心（异或+二进制位统计），可快速迁移至其他位运算题目（如统计二进制1的个数、找唯一数等），是理解位运算的基础。
