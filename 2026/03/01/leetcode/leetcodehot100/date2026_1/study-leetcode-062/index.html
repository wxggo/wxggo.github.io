Leetcode-062:不同路径-动态规划经典解法（初始化技巧版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100
|Word Count:2.7k|Reading Time:9mins|Post Views:

前言

不同路径是Leetcode动态规划模块的经典高频题，也是面试中考察「动态规划状态定义+状态转移」的核心题目之一。本题的核心难点在于如何定义状态并设置合理的初始条件，动态规划解法通过构建二维数组存储路径数、推导状态转移方程，实现了时间与空间复杂度的平衡，是本题的基础最优解。

1、题目

一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish”）。问总共有多少条不同的路径？

示例

示例 1：
输入：m = 3, n = 7
输出：28
解释：从左上角到右下角，仅能向右/向下移动，总共有28条不同路径。

示例 2：
输入：m = 3, n = 2
输出：3
解释：
路径1：右 → 下 → 下
路径2：下 → 右 → 下
路径3：下 → 下 → 右
总共有3条不同路径。

示例 3：
输入：m = 1, n = 1
输出：1
解释：仅一个格子，无需移动，路径数为1。

提示

  • 1 <= m, n <= 100
  • 题目数据保证答案小于等于 2 * 10^9

2、分析

2.1 问题核心：如何定义状态与转移方程？

不同路径的本质是「网格中每个位置的路径数 = 上方位置路径数 + 左方位置路径数」，因为只能从这两个方向到达当前位置。

比如网格中位置(i,j)（i行j列），到达它的路径数f[i][j] = f[i-1][j]（从上方来） + f[i][j-1]（从左方来）—— 这是状态转移的核心依据，也是动态规划解法的核心。

2.2 核心难点：如何简化边界初始化？

暴力解法的思路是递归枚举所有路径（时间复杂度O(2^(m+n))），面对m/n=100的规模会直接超时。

动态规划解法的核心是用空间换时间，同时优化初始条件：

  1. 边界特性：第一行所有位置只能从左方来，路径数恒为1；第一列所有位置只能从上方来，路径数恒为1；
  2. 简化初始化：原代码中通过设置f[0][1] = 1（或f[1][0] = 1），让f[1][1]（起点）的路径数自然为1，无需单独处理第一行/第一列的初始化，代码更简洁；
    整体时间复杂度优化至O(mn)（遍历m*n的网格），空间复杂度为O(mn)。

2.3 核心解题思路

  1. 初始化动态规划数组：创建(m+1)x(n+1)的二维数组f（多开一行一列简化边界处理）；
  2. 设置初始条件：令f[0][1] = 1，确保起点f[1][1]的路径数为1；
  3. 遍历填充数组：双重循环遍历网格，按状态转移方程计算每个位置的路径数；
  4. 返回结果：f[m][n]即为右下角的总路径数。

3.2 代码核心点逐行解释

（1）DP数组定义

  • 核心逻辑：数组维度设为(m+1)x(n+1)，而非mxn，目的是让行/列从1开始计数，避免处理i=0或j=0时的越界问题；
  • 作用：简化边界条件，让状态转移方程无需额外的if判断（如i>0、j>0）。

（2）初始条件设置

  • 核心逻辑：通过设置f[0][1] = 1，使得起点f[1][1] = f[0][1]（上方） + f[1][0]（左方，默认0） = 1，等价于手动设置起点路径数为1；
  • 优势：无需单独遍历第一行/第一列初始化路径数为1，一行代码完成所有边界的间接初始化，代码更简洁。

（3）状态转移方程

  • 变量对应：循环变量i/j从0开始，f[i+1][j+1]对应网格的(i+1)行(j+1)列；
  • 转移逻辑：f[i][j+1]是当前位置的上方（i行j+1列）路径数，f[i+1][j]是当前位置的左方（i+1行j列）路径数，两者之和即为当前位置的总路径数；
  • 合理性：完全贴合“仅能向下/向右移动”的规则，覆盖所有到达当前位置的可能路径。

（4）结果返回

  • 核心逻辑：f[m][n]对应网格的m行n列（右下角），其值即为从左上角到右下角的总路径数；
  • 简洁性：无需额外处理，直接返回目标位置的dp值即可。

4、算法性能分析

本解法是不同路径的基础最优解，兼顾效率和理解性，完全适配题目输入规模。

4.1 时间复杂度：O(mn)

  • m：网格的行数；
  • n：网格的列数；
  • 核心耗时：双重循环遍历m*n个网格位置，每个位置的计算仅需O(1)时间；
  • 适配性：面对m/n=100的网格规模，O(mn)的时间复杂度完全满足要求，无超时风险。

4.2 空间复杂度：O(mn)

  • 空间开销主要来自：(m+1)x(n+1)的二维DP数组；
  • 必要性：存储每个位置的路径数是动态规划的必然开销，该空间复杂度为基础解法的最优值（可进一步优化为O(n)，见拓展思考）。

4.3 核心优势

  1. 逻辑直观：完全贴合“仅能向下/向右移动”的规则，状态转移方程易理解、易推导；
  2. 代码简洁：核心逻辑仅10行左右，无冗余操作，面试书写效率高；
  3. 兼容性强：天然适配所有边界场景（单行/单列网格、1x1网格）；
  4. 扩展性强：是理解空间优化版、障碍版不同路径的基础，掌握后可快速迁移至进阶解法。

以示例2：m=3, n=2 为例，手动模拟代码执行过程：

5.1 初始化

DP数组f为4x3（m+1=4行，n+1=3列），所有值默认0，设置f[0][1] = 1。

5.2 遍历填充数组

外层循环i（0→1→2），内层循环j（0→1）：

i j f[i+1][j+1] = f[i][j+1] + f[i+1][j] 计算结果 数组状态（核心位置）
0 0 f[1][1] = f[0][1] + f[1][0] = 1 + 0 1 f[1][1]=1
0 1 f[1][2] = f[0][2] + f[1][1] = 0 + 1 1 f[1][2]=1
1 0 f[2][1] = f[1][1] + f[2][0] = 1 + 0 1 f[2][1]=1
1 1 f[2][2] = f[1][2] + f[2][1] = 1 + 1 2 f[2][2]=2
2 0 f[3][1] = f[2][1] + f[3][0] = 1 + 0 1 f[3][1]=1
2 1 f[3][2] = f[2][2] + f[3][1] = 2 + 1 3 f[3][2]=3

5.3 结果返回

f[3][2] = 3，与示例2的输出一致，验证了代码的正确性。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 单行网格：如m=1,n=5，遍历后f[1][5] = 1（仅能向右移动），正确；
  2. 单列网格：如m=5,n=1，遍历后f[5][1] = 1（仅能向下移动），正确；
  3. 1x1网格：m=1,n=1，f[1][1] = 1，正确。

6.2 面试高频追问：空间优化（一维DP）

基础解法的空间复杂度可从O(mn)优化至O(n)，核心思路是用一维数组复用空间：

核心思路

  1. 一维数组dp[n]，dp[j]表示当前行第j列的路径数；
  2. 初始时dp[0] = 1（第一列路径数恒为1）；
  3. 遍历每行时，dp[j] = dp[j]（上方路径数，即上一行的dp[j]） + dp[j-1]（左方路径数，即当前行的dp[j-1]）。

简洁代码实现

优化价值

  • 空间复杂度：从O(mn)降至O(n)，大幅减少空间开销；
  • 时间复杂度：仍为O(mn)，无性能损失；
  • 核心逻辑：利用一维数组的覆盖特性，复用空间存储上一行的结果，是动态规划空间优化的经典思路。

6.3 进阶问题：带障碍的不同路径

若题目拓展为「网格中存在障碍，障碍位置无法通过」，只需在状态转移时增加判断：

  1. 若当前位置是障碍，路径数设为0；
  2. 否则按原转移方程计算。
    该拓展是面试高频变种，掌握基础解法后可快速适配。

7、总结

不同路径的动态规划解法是本题的基础最优解，也是理解网格路径类动态规划问题的核心，本文核心总结如下：

  1. 路径计数的核心特征是「当前位置路径数=上方+左方路径数」，状态转移方程完全贴合移动规则；
  2. 动态规划的核心价值是用O(mn)的空间换O(mn)的时间，避免暴力递归的指数级复杂度；
  3. 核心代码需牢记：多维度数组简化边界+巧妙初始条件+状态转移方程；
  4. 面试中可通过「一维DP数组」将空间复杂度优化至O(n)，是进阶最优写法；
  5. 解法具备强扩展性，可适配单行/单列、带障碍等拓展场景。
