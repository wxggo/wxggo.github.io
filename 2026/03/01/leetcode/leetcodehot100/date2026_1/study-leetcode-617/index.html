Leetcode-617:合并二叉树-递归经典解法（基础最优版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100

前言

合并二叉树是Leetcode树模块的经典高频题，也是面试中考察「递归思想+树结构操作」的核心题目之一。本题的核心难点在于如何处理两个二叉树节点的存在性差异，并通过递归完成节点值合并与子树的逐层合并。其中「递归解法」是最直观、最易理解的基础最优解，该解法通过分解子问题（合并左子树、合并右子树），结合递归终止条件，实现二叉树的高效合并。

1、题目

给定两个二叉树，想象当你将其中一个覆盖到另一个上时，两个二叉树的一些节点便会重叠。你需要将它们合并为一个新的二叉树。合并的规则是：如果两个节点重叠，那么将它们的值相加作为合并后节点的新值；否则，不为 null 的节点将直接作为新二叉树的节点。

2、分析

2.1 问题核心：如何定义二叉树的合并规则？

合并二叉树的本质是「逐节点处理，按规则生成新节点」，核心规则可拆解为：

  1. 若节点A为空、节点B非空 → 合并结果为节点B；
  2. 若节点A非空、节点B为空 → 合并结果为节点A；
  3. 若节点A和B均非空 → 合并结果为“值为A.val+B.val，左子树为A左子树与B左子树合并结果，右子树为A右子树与B右子树合并结果”的节点。

这一规则天然适配「递归分解子问题」的思路——合并整棵树 = 合并当前节点 + 合并左子树 + 合并右子树。

2.2 核心难点：如何处理节点存在性差异？

暴力解法的思路是先遍历两棵树收集所有节点值，再重新构建合并后的树，但该方法需要额外的空间存储节点信息，且无法利用树的递归结构。

递归解法是解决该难点的最优思路：

  1. 以递归终止条件处理节点存在性差异（任一节点为空则直接返回另一节点）；
  2. 以递归调用处理子树合并（当前节点值相加后，递归合并左/右子树）；
  3. 整体时间复杂度优化至O(min(m,n))（m、n为两棵树的节点数，仅遍历存在的节点）。

2.3 核心解题思路

  1. 递归终止条件：若root1为空则返回root2，若root2为空则返回root1；
  2. 处理当前节点：若两个节点均非空，将root1的节点值更新为两节点值之和（原地合并，节省空间）；
  3. 递归处理子树：递归合并root1的左子树与root2的左子树，结果赋值给root1.left；递归合并root1的右子树与root2的右子树，结果赋值给root1.right；
  4. 返回结果：返回处理后的root1（合并后的根节点）。

3.2 代码核心点逐行解释

（1）递归终止条件

  • 核心逻辑：覆盖所有节点存在性差异的场景，避免空指针异常；
  • 价值：无需额外判断子节点是否为空，递归会自然处理所有边界情况（如单侧子树为空）。

（2）当前节点值合并

  • 原地合并优化：直接修改root1的节点值，而非新建TreeNode对象，节省O(min(m,n))的空间开销；
  • 逻辑合理性：两个节点均非空时，按规则合并值，是整棵树合并的基础。

（3）递归合并子树

  • 子问题分解：将“合并整棵树”拆解为“合并左子树”和“合并右子树”两个子问题，符合递归的核心思想；
  • 赋值逻辑：将子树合并的结果赋值给当前节点的左/右子节点，完成整棵树的逐层合并。

（4）返回合并结果

  • 核心逻辑：当前节点处理完成后，返回该节点作为上层递归的子树合并结果；
  • 最终结果：递归到根节点时，返回的root1即为合并后的完整二叉树根节点。

4、算法性能分析

本解法是合并二叉树的基础最优解，兼顾效率和理解性，完全适配题目输入规模。

4.1 时间复杂度：O(min(m, n))

  • m：第一棵树的节点数；
  • n：第二棵树的节点数；
  • 核心耗时：仅遍历两棵树中「重叠部分+非重叠的剩余部分」的节点，每个节点仅处理一次，总遍历次数为min(m,n)；
  • 适配性：面对2000个节点的输入规模，O(min(m,n))的时间复杂度无超时风险。

4.2 空间复杂度：O(min(m, n))

  • 空间开销主要来自：递归调用栈的深度（等于树的高度）；
  • 最坏情况：树为链式结构（如所有节点只有左子树），栈深度为min(m,n)；
  • 最优情况：树为完全二叉树，栈深度为log(min(m,n))；
  • 优化点：原地合并避免了新建节点的空间开销，是基础解法的最优空间复杂度。

4.3 核心优势

  1. 逻辑直观：完全贴合二叉树的递归定义，合并规则与递归分解思路高度契合；
  2. 代码简洁：核心逻辑仅10行左右，无冗余判断，面试书写效率高；
  3. 空间高效：原地合并策略节省额外节点空间，空间复杂度最优；
  4. 兼容性强：天然适配所有边界场景（空树、单侧子树为空、链式树）；
  5. 扩展性强：是理解迭代解法的基础，掌握后可快速迁移至非递归实现。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 空树合并：如root1为空、root2非空，直接返回root2；反之同理；
  2. 单节点树合并：如root1=[1]、root2=[2]，合并后为[3]；
  3. 单侧子树为空：如root1左子树为空、root2左子树非空，直接将root2左子树赋值给root1左子树；
  4. 链式树合并：如两棵树均为链式结构，递归会逐层处理每个节点，无异常。

7、总结

合并二叉树的递归解法是本题的基础最优解，也是理解树结构操作类问题的核心，本文核心总结如下：

  1. 二叉树合并的核心规则是“值相加、空节点替换”，递归终止条件完美覆盖节点存在性差异；
  2. 递归的核心价值是将“整棵树合并”拆解为“节点合并+子树合并”，符合树的递归定义；
  3. 核心代码需牢记：终止条件→合并当前节点值→递归合并子树→返回结果；
  4. 面试中可通过“迭代解法”展示对递归的深层理解，通过“非原地合并”适配不同题目要求；
  5. 解法具备强扩展性，可适配空树、链式树、非原地合并等拓展场景。
