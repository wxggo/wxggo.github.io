Leetcode-049:字母异位词分组-哈希表+排序经典解法（基础最优版）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100
|Word Count:2.9k|Reading Time:10mins|Post Views:

前言

字母异位词分组是Leetcode字符串模块的经典高频题，也是面试中考察「哈希表应用+字符串处理」的核心题目之一。本题的核心难点在于如何高效判断两个字符串是否为字母异位词，并完成分组存储。其中「哈希表+字符串排序」是最直观、最易理解的基础最优解，该解法通过将异位词排序后转化为相同的「特征键」，利用哈希表的O(1)查询特性完成分组。

1、题目

给定一个字符串数组，将字母异位词组合在一起。字母异位词指字母相同，但排列不同的字符串。

示例

示例 1：
输入：strs = [“eat”, “tea”, “tan”, “ate”, “nat”, “bat”]
输出：[[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]
解释：”eat/tea/ate”为一组（字母相同仅排列不同），”tan/nat”为一组，”bat”单独一组。

示例 2：
输入：strs = [“”]
输出：[[“”]]
解释：空字符串无字母，单独作为一组。

示例 3：
输入：strs = [“a”]
输出：[[“a”]]
解释：单个字符无排列差异，单独作为一组。

提示

  • 1 <= strs.length <= 10^4
  • 0 <= strs[i].length <= 100
  • strs[i] 仅包含小写字母

2、分析

2.1 问题核心：如何定义并识别字母异位词？

字母异位词的本质是「字符组成完全相同，仅排列顺序不同」，因此所有异位词经过相同规则的处理后，必然能得到一个完全相同的「特征标识」。

比如 “eat”、”tea”、”ate” 排序后都会得到 “aet”，这个统一的结果就是它们的「特征标识」—— 这是判断异位词的核心依据，也是分组的关键。

2.2 核心难点：如何高效分组存储异位词？

暴力解法的思路是，对每个字符串，逐一和已分组的字符串对比是否为异位词，若匹配则加入对应组，否则新建组。但该方法的时间复杂度为O(n²k)（n为字符串数量，k为字符串长度），面对10^4的数组长度会超时。

哈希表+排序是解决该难点的最优思路：

  1. 对每个字符串排序生成「特征键」；
  2. 以「特征键」为key、异位词列表为value构建哈希表；
  3. 遍历字符串时，根据「特征键」直接定位分组，O(1)完成添加操作；
    整体时间复杂度优化至O(nk logk)（排序单个字符串的时间为O(k logk)，n个字符串总排序时间为O(nk logk)）。

2.3 核心解题思路

  1. 初始化哈希表：创建Map<String, List<String>>，用于存储「特征键-异位词列表」的映射；
  2. 遍历处理每个字符串：
    • 将字符串转为字符数组并排序，生成「特征键」；
    • 若哈希表中存在该键，将当前字符串添加到对应列表；
    • 若不存在，新建列表并将当前字符串加入，再将键值对存入哈希表；
  3. 转换结果并返回：将哈希表中所有value（异位词列表）收集为List返回。

3.2 代码核心点逐行解释

（1）哈希表定义

  • 核心逻辑：key是排序后的「特征字符串」（如”aet”），value是该特征对应的所有异位词（如[“eat”,”tea”,”ate”]）；
  • 作用：利用哈希表O(1)的查询/插入特性，快速完成分组定位。

（2）生成特征键

  • 字符数组转换：字符串是不可变类型，转为字符数组才能排序；
  • 排序操作：Arrays.sort()对字符数组按字典序排序，保证所有异位词生成相同的key；
  • 核心价值：将「字符排列不同但组成相同」的字符串，转化为完全相同的key，实现异位词的统一标识。

（3）分组逻辑优化

  • 简化操作：原代码中「get列表→add元素→put回哈希表」可简化为直接m.get(key).add(s)（HashMap的get方法返回的是列表引用，修改引用即可）；
  • 逻辑合理性：存在key则复用已有列表，不存在则新建，保证每个key对应唯一的异位词列表。

（4）结果转换

  • 核心逻辑：HashMap的values()方法返回所有value的集合（Collection<List>），直接传入ArrayList构造器即可转为题目要求的List<List>；
  • 简洁性：无需手动遍历哈希表，一行代码完成结果转换。

4、算法性能分析

本解法是字母异位词分组的基础最优解，兼顾效率和理解性，完全适配题目输入规模。

4.1 时间复杂度：O(nk logk)

  • n：字符串数组的长度；
  • k：字符串的最大长度；
  • 核心耗时：对每个字符串排序的时间为O(k logk)，n个字符串总排序时间为O(nk logk)；哈希表的查询/插入操作均为O(1)，总耗时可忽略；
  • 适配性：面对10^4的字符串数量和100的字符串长度，O(nk logk)的时间复杂度完全满足要求，无超时风险。

4.2 空间复杂度：O(nk)

  • 空间开销主要来自：
    1. 哈希表存储的所有字符串（总长度为nk）；
    2. 排序时的字符数组（单次排序空间为O(k)，可复用，不计入总复杂度）；
  • 必要性：存储所有字符串是分组的必然开销，该空间复杂度为基础解法的最优值。

4.3 核心优势

  1. 逻辑直观：完全贴合异位词的定义，排序生成特征键的思路易理解、易推导；
  2. 代码简洁：核心逻辑仅10行左右，无冗余操作，面试书写效率高；
  3. 兼容性强：天然适配所有边界场景（空字符串、单字符、超长字符串）；
  4. 扩展性强：是理解计数优化版的基础，掌握后可快速迁移至更优解法。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 空字符串：如strs = [“”], 排序后key仍为””，哈希表存入[“”]，返回[[“]]，正确；
  2. 单字符字符串：如strs = [“a”], 排序后key为”a”，返回[[“a”]]，正确；
  3. 全相同字符串：如strs = [“aaa”,”aaa”], 排序后key均为”aaa”，返回[[“aaa”,”aaa”]]，正确；
  4. 全不同字符串：如strs = [“a”,”b”,”c”], 每个字符串生成唯一key，返回[[“a”],[“b”],[“c”]]，正确。

6.2 面试高频追问：空间/时间优化（计数法）

哈希表+排序解法的时间瓶颈在「字符串排序」（O(k logk)），面试中面试官大概率会追问如何将时间复杂度优化至O(nk) ，核心思路是「字符计数替代排序」：

核心思路

  1. 对每个字符串，统计每个小写字母的出现次数（如”eat”统计为a:1, e:1, t:1）；
  2. 将计数结果转为「特征键」（如”a1e1t1”）；
  3. 以计数生成的key为哈希表键，完成分组。

简洁代码实现

优化价值

  • 时间复杂度：统计字符次数为O(k)，生成key为O(26)（常数级），总时间复杂度降至O(nk)；
  • 空间复杂度：仍为O(nk)，但避免了排序的额外空间开销；
  • 核心API：computeIfAbsent是Java8+的HashMap便捷API，等价于「判断key是否存在，不存在则新建值，最终返回值」，简化代码书写。

6.3 进阶问题：区分大小写/包含特殊字符

若题目拓展为「字符串包含大写字母/数字/特殊字符」，只需调整「特征键生成规则」：

  1. 区分大小写：将计数数组长度扩展为52（26大写+26小写），或排序时保留大小写；
  2. 包含特殊字符：统计所有可能字符的ASCII码，或直接排序（兼容性更强）。

7、总结

字母异位词分组的哈希表+排序解法是本题的基础最优解，也是理解字符串分组类问题的核心，本文核心总结如下：

  1. 异位词的核心特征是「字符组成相同」，排序是生成统一特征键的最直观方式；
  2. 哈希表的核心价值是O(1)定位分组，将暴力的O(n²k)时间复杂度优化至O(nk logk)；
  3. 核心代码需牢记：排序生成key→哈希表分组→收集values返回；
  4. 面试中可通过「字符计数法」将时间复杂度优化至O(nk)，是进阶最优写法；
  5. 解法具备强扩展性，可适配空字符串、特殊字符、大小写区分等拓展场景。
