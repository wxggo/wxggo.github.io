Leetcode-283:比特位计数-动态规划+位运算最优解法（详解设计思路）

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode Hot100
|Word Count:2.5k|Reading Time:9mins|Post Views:

前言

比特位计数是Leetcode位运算模块的经典高频题，也是面试中考察「动态规划+位运算优化」的核心题目之一。本题的核心难点在于如何避免暴力统计每个数的二进制1的个数，而「动态规划+位运算」是时间/空间双最优解——通过位运算推导状态转移方程，将时间复杂度从O(nk)（k为二进制位数）优化至O(n)，空间复杂度优化至O(1)。

1、题目

给定一个非负整数 n，计算 0 到 n 之间每个数字的二进制表示中 1 的个数，并输出一个数组。

示例

示例 1：
输入：n = 2
输出：[0,1,1]
解释：
0 → 二进制 0 → 1的个数 0
1 → 二进制 1 → 1的个数 1
2 → 二进制 10 → 1的个数 1

示例 2：
输入：n = 5
输出：[0,1,1,2,1,2]
解释：
0 → 0 → 0
1 → 1 → 1
2 → 10 → 1
3 → 11 → 2
4 → 100 → 1
5 → 101 → 2

提示

  • 0 <= n <= 10^5

2、分析

2.1 问题核心：如何高效统计二进制中1的个数？

暴力解法的思路是，对每个数 i（0≤i≤n），逐位检查二进制位是否为1（如通过取模/除法或位运算），统计个数。但该方法的时间复杂度为O(nk)（k为i的二进制位数，最多17位），面对10^5的n虽能通过，但并非最优——我们可以利用「动态规划+位运算」复用已计算的结果，彻底规避重复统计。

2.2 核心思路：状态转移方程的设计逻辑

（1）二进制数的核心规律

任意整数 i 的二进制可拆分为「高位部分 + 最低位」：

  • 高位部分：i 右移 1 位（i >> 1），等价于 i 除以 2 取整（如 5→101 → 右移1位→10=2）；
  • 最低位：i 的二进制最后一位，可通过「i & 1」判断（1则为1，0则为0）。

例如：

  • i=5（101）：i>>1=2（10），i&1=1 → 1的个数 = 2的1的个数（1） + 1 = 2；
  • i=4（100）：i>>1=2（10），i&1=0 → 1的个数 = 2的1的个数（1） + 0 = 1；

由此可推导状态转移方程：
ans[i] = ans[i >> 1] + (i & 1)

（2）动态规划的核心价值

  • 状态定义：ans[i] 表示数字 i 的二进制中 1 的个数；
  • 初始状态：ans[0] = 0（0的二进制无1）；
  • 状态转移：利用已计算的 ans[i >> 1] 推导 ans[i]，无需重复统计；
  • 时间优化：从暴力的O(nk)降至O(n)，每个数仅需一次计算。

2.3 位运算的核心用法（本题关键）

本题用到的位运算是面试高频考点，先明确核心含义：

位运算 符号 含义 示例（十进制→二进制）
右移 >> 将二进制位整体右移，高位补0，等价于除以2取整 5(101) >> 1 = 2(10)
按位与 & 对应位均为1则结果为1，否则为0 5(101) & 1 = 1；4(100) & 1 = 0

这两个运算的组合是本题的「最优解钥匙」——右移获取高位的1的个数，按位与判断最低位是否为1，两者相加即得当前数的1的个数。

3.2 代码核心点逐行解释

（1）结果数组初始化

  • 核心逻辑：数组长度为n+1，对应0~n共n+1个数，ans[i]存储数字i的二进制1的个数；
  • 初始值：数组默认值为0，因此ans[0]无需手动赋值（0的二进制1的个数为0）。

（2）遍历与状态转移

  • 遍历范围：从1开始（ans[0]已确定），到n结束，覆盖所有需要统计的数；
  • 状态转移拆解：
    1. i >> 1：获取i的高位部分（去掉最低位），ans[i >> 1]是高位部分已统计的1的个数；
    2. i & 1：判断i的最低位是否为1（结果为1或0）；
    3. 相加：高位1的个数 + 最低位的1的个数 = i的总1的个数；
  • 示例验证（i=5）：
    i >> 1 = 2，ans[2]=1；i & 1 = 1；ans[5] = 1 + 1 = 2（符合预期）。

（3）结果返回

  • 核心逻辑：数组已存储0~n每个数的1的个数，直接返回即可，无需额外处理。

4、算法性能分析

本解法是比特位计数的最优解，时间/空间复杂度均达到理论下限，完全适配题目输入规模。

4.1 时间复杂度：O(n)

  • n：输入的非负整数；
  • 核心耗时：仅需一次遍历0~n，每个数的计算仅需2个位运算（O(1)），总时间复杂度为O(n)；
  • 适配性：面对10^5的n，O(n)的时间复杂度无任何超时风险，比暴力解法快一个数量级。

4.2 空间复杂度：O(1)

  • 结果数组ans的长度为n+1，是题目要求的输出，属于「必要开销」；
  • 除结果数组外，仅使用了常数级别的临时变量（i），因此额外空间复杂度为O(1)；
  • 最优性：无法进一步优化空间（必须存储结果），该复杂度为理论下限。

4.3 核心优势

  1. 极致高效：时间/空间双最优，是面试中「最优解」的标准答案；
  2. 逻辑简洁：核心代码仅5行，无冗余操作，面试书写效率高；
  3. 易理解：基于二进制数的拆分规律推导，符合直觉；
  4. 扩展性强：掌握该思路后，可快速迁移至其他位运算优化问题（如统计1的个数、二进制翻转等）。

6、拓展思考（面试高频考点）

6.1 位运算的常见用法（通用知识点）

本题仅用到右移（>>）和按位与（&），面试中还常考察以下位运算，需熟练掌握：

位运算 符号 核心用法 示例
左移 << 乘以2（低位补0），效率高于乘法 3(11) << 1 = 6(110)
按位或 | 将指定位设为1 3(11) | 4(100) = 7(111)
按位异或 ^ 相同位为0，不同位为1；0异或任何数=自身，数异或自身=0 3(11) ^ 5(101) = 6(110)；a ^ a = 0
按位取反 ~ 按位取反（0→1，1→0），注意Java中是补码运算 ~1 = -2（二进制补码层面）
清除最低位的1 i & (i-1) 消除i的二进制中最右侧的1，用于统计1的个数 6(110) & 5(101) = 4(100)

6.2 其他解法对比（理解最优解的价值）

除了动态规划+位运算，还有两种常见解法，可对比理解最优解的优势：

（1）暴力法（逐位统计）

  • 时间复杂度：O(nk)（k为二进制位数）；
  • 劣势：重复统计每个数的二进制位，效率低于最优解。

（2）利用内置函数（Java）

  • 优势：代码极简；
  • 劣势：面试中考察的是「实现逻辑」而非API调用，该解法无法体现位运算/动态规划能力。

6.3 边界情况天然适配

本解法无需额外处理边界场景，代码天然兼容：

  1. n=0：返回[0]，正确；
  2. n=1：返回[0,1]，正确；
  3. n=10^5：O(n)时间复杂度可快速处理，无性能问题。

7、总结

比特位计数的动态规划+位运算解法是本题的最优解，也是理解位运算优化的核心，本文核心总结如下：

  1. 核心规律：任意数的二进制可拆分为「高位+最低位」，高位的1的个数可复用已计算结果；
  2. 状态转移方程：ans[i] = ans[i >> 1] + (i & 1)，是位运算与动态规划结合的经典应用；
  3. 位运算价值：右移（>>）和按位与（&）是处理二进制位的基础操作，效率远高于算术运算；
  4. 性能最优：时间O(n)、额外空间O(1)，是面试中「最优解」的标准答案；
  5. 拓展性：掌握该思路可解决各类二进制位统计问题（如统计1的个数、判断2的幂等）。
