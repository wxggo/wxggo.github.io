Leetcode-543:二叉树的直径-深度优先搜索（DFS）经典解法

Created2026-03-01|Updated2026-03-01|Java学习笔记Leetcode 简单题
|Word Count:2.5k|Reading Time:9mins|Post Views:

前言

二叉树的直径是Leetcode树模块的经典简单题，也是理解「深度优先搜索（DFS）+ 二叉树深度计算」的核心题目。本题的核心陷阱在于「二叉树的直径不一定经过根节点」，因此需要遍历每个节点，计算其左右子树的深度和，最终取最大值。

1、题目

给定一棵二叉树，你需要计算它的直径长度。一棵二叉树的直径长度是任意两个结点路径长度中的最大值。这条路径可能穿过也可能不穿过根结点。

示例

示例 1：
输入：二叉树结构如下

输出：3
解释：该二叉树的直径是3，对应路径为 4→2→1→3 或 5→2→1→3，路径长度为3（路径上的边数）。

示例 2：
输入：root = [1,2]
输出：1
解释：路径为 1→2，长度为1。

提示

  • 树中结点数的范围是 [1, 10^4]
  • 结点值的范围是 [-100, 100]

2、分析

2.1 问题核心：直径的本质

二叉树的「直径」定义为：任意两个节点之间路径的边数最大值。
而任意两个节点的路径，必然以「某个节点」为中心，向左右子树延伸（即路径 = 左子树深度 + 右子树深度）。因此：

  1. 二叉树的直径 = 所有节点的「左子树深度 + 右子树深度」的最大值；
  2. 深度优先搜索（DFS）是遍历所有节点、计算子树深度的最优方式。

2.2 核心陷阱：直径不一定经过根节点

比如以下二叉树：

根节点1的左右深度和为 3（左子树深度3，右子树深度0），但节点4的左右深度和为2，节点2的左右深度和为3（左1+右2），而实际直径是节点2的左子树（3）→2→右子树（4→5/6），长度为3，若忽略非根节点的计算会得到错误结果。

2.3 核心解题思路

  1. 定义递归函数，计算每个节点的子树深度；
  2. 在递归过程中，对每个节点计算「左深度 + 右深度」，并更新全局最大直径；
  3. 递归结束后，全局最大直径即为答案。

  • 后序遍历逻辑：先计算左右子树深度，再处理当前节点（符合「先子后父」的深度计算逻辑）；
  • 直径计算：leftDepth + rightDepth 是当前节点作为路径中心的边数（比如左深度2、右深度1，路径边数为3）；
  • 深度返回：Math.max(left, right) + 1 是当前节点的子树深度（供父节点计算直径使用）。

4、算法性能分析

本解法是二叉树直径的最优解，时间/空间复杂度均达到理论下限，完全适配题目输入规模。

4.1 时间复杂度：O(n)

  • n：二叉树的节点数；
  • 核心逻辑：每个节点仅被递归访问1次，DFS遍历所有节点的时间为O(n)；
  • 适配性：面对10^4的节点数，O(n)的时间复杂度无超时风险，是二叉树遍历的最优时间复杂度。

4.2 空间复杂度：O(h)

  • h：二叉树的高度；
  • 空间开销：递归调用栈的深度（等于树的高度）；
  • 最坏情况：二叉树退化为链表（如只有左子树），h=n，空间复杂度为O(n)；
  • 最优情况：平衡二叉树，h=logn，空间复杂度为O(logn)；
  • 必要性：递归遍历二叉树的栈空间是必然开销，无优化空间（除非改用迭代版DFS）。

4.3 核心优势

  1. 逻辑高效：后序遍历天然适配「先算子树深度，再算当前节点直径」的需求；
  2. 代码简洁：核心递归逻辑仅10行左右，无冗余操作，面试书写效率高；
  3. 无边界漏洞：天然适配单节点树、链表树、平衡树等所有场景；
  4. 易拓展：可快速迁移至「二叉树最大路径和」等同类后序遍历问题。

  • 左/右子树均为空，leftDepth=0，rightDepth=0；
  • 计算直径：0+0=0 → maxDiameter 仍为0；
  • 返回深度：max(0,0)+1=1。

步骤2：访问节点5

  • 左/右子树均为空，leftDepth=0，rightDepth=0；

  • 返回深度：max(0,0)+1=1。

步骤3：访问节点2

  • 左子树深度（节点4）=1，右子树深度（节点5）=1；
  • 计算直径：1+1=2 → maxDiameter 更新为2；
  • 返回深度：max(1,1)+1=2。

步骤4：访问节点3

  • 左/右子树均为空，leftDepth=0，rightDepth=0；
  • 计算直径：0+0=0 → maxDiameter 仍为2；
  • 返回深度：max(0,0)+1=1。

步骤5：访问节点1

  • 左子树深度（节点2）=2，右子树深度（节点3）=1；
  • 计算直径：2+1=3 → maxDiameter 更新为3；
  • 返回深度：max(2,1)+1=3。

5.3 结果返回

maxDiameter=3，与示例输出一致。

6、拓展思考（面试高频考点）

6.1 边界情况适配性

本解法无需额外处理边界场景，天然兼容：

  1. 单节点树：root=[1]，maxDiameter=0（无路径），正确；
  2. 链表树：root=[1,2, null,3,null,4]，直径为3（4→3→2→1），代码计算节点2的直径=2（左1+右0）、节点3的直径=1（左0+右1）、节点1的直径=3（左3+右0），最终返回3，正确；
  3. 平衡树：完全二叉树，直径为树的高度-1，代码可正确统计。

6.2 迭代版DFS实现（避免递归栈溢出）

递归版在节点数极多（如10^4）时，可能触发栈溢出，可改用迭代版后序遍历：

  • 核心逻辑：用栈模拟递归，哈希表记录节点深度，避免栈溢出；
  • 适用场景：节点数极多的二叉树，性能与递归版持平。

6.3 面试高频追问：直径与路径的区别

  • 直径：路径的边数最大值；
  • 路径长度（节点数）：直径 + 1；
  • 若题目要求返回「路径的节点数最大值」，只需将最终结果+1即可。

7、总结

二叉树的直径是后序遍历+深度计算的经典应用，本文核心总结如下：

  1. 直径的本质：所有节点「左深度+右深度」的最大值，与根节点无关；
  2. 核心解法：DFS后序遍历，「求深度」与「统计直径」同步进行；
  3. 代码关键：全局变量记录最大直径，递归函数返回子树深度；
  4. 拓展优化：迭代版DFS可避免栈溢出，适配超大二叉树；
  5. 迁移价值：该思路可直接复用至「二叉树最大路径和」等后序遍历问题。

掌握本题的核心逻辑，能快速理解二叉树路径类问题的解题框架——「后序遍历统计子树信息，全局变量记录最优解」。
