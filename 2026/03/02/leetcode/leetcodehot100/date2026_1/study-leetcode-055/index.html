Leetcode-055:跳跃游戏-贪心算法经典解法（单次遍历最优版）

Created2026-03-02|Updated2026-03-02|Java学习笔记Leetcode Hot100
|Word Count:2.7k|Reading Time:9mins|Post Views:

前言

跳跃游戏问题是Leetcode数组模块结合贪心算法的经典代表题，也是面试中高频考察的基础贪心题之一。本题的核心考察对贪心策略的理解与应用——通过维护「当前能到达的最远位置」，将暴力枚举所有跳跃路径的思路优化为单次线性遍历，时间复杂度降至O(n)。

1、题目

给定一个非负整数数组 nums，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。

示例

示例 1：
输入：nums = [2,3,1,1,4]
输出：true
解释：可以先跳 1 步，从下标 0 到 1，然后再跳 3 步到达最后一个下标。

示例 2：
输入：nums = [3,2,1,0,4]
输出：false
解释：无论怎么跳跃，都会停在下标 4 前的位置（下标 3 只能跳 0 步），无法到达最后一个下标。

示例 3：
输入：nums = [0]
输出：true
解释：初始位置即为最后一个下标，无需跳跃。

示例 4：
输入：nums = [1,0,1,0]
输出：false
解释：下标 0 跳 1 步到下标 1，下标 1 只能跳 0 步，无法到达最后一个下标。

提示

  • 1 <= nums.length <= 10^4
  • 0 <= nums[i] <= 10^5

2、分析

2.1 问题核心：贪心策略的核心应用

跳跃游戏的核心无需枚举所有可能的跳跃路径（暴力思路），而是遵循贪心算法的局部最优推导全局最优：在遍历数组的过程中，始终维护「当前能到达的最远位置」，只要当前遍历的位置在该范围内，就更新最远位置；若遍历位置超出该范围，则说明无法到达后续位置，直接返回false。

这是因为：如果一个位置能被到达，那么该位置之前的所有位置都能被到达；而通过当前位置的跳跃能力，能扩展的最远位置是「当前下标 + 当前位置的跳跃长度」，只要这个扩展值更大，就更新最远位置——最终若最远位置能覆盖最后一个下标，说明可以到达。

2.2 核心难点：如何避免暴力枚举路径？

暴力解法的思路是，对每个位置，枚举所有可能的跳跃步数，递归/回溯判断是否能到达终点。但该方法的时间复杂度为O(2ⁿ)（最坏情况每个位置都有多种跳跃选择），面对10⁴的数组长度会直接超时。

贪心单次遍历是解决该难点的最优思路：通过一次遍历数组，仅维护一个「最远可达位置」变量，无需额外空间，遍历过程中仅做两次核心判断（当前位置是否可达、更新最远位置），将时间复杂度优化至O(n)，空间复杂度优化至O(1)。

2.3 核心概念定义（解法基础）

为后续推导清晰，先明确核心变量的严格定义：

  • 最远可达位置mx ：表示在遍历到当前位置i时，从起始位置出发能到达的最大下标（初始值为0，即起始位置）。

2.4 核心解题思路

  1. 初始化变量：定义mx = 0，表示初始时只能到达下标0；
  2. 单次遍历数组：对每个下标i：
    • 若i > mx：说明当前位置无法到达，直接返回false；
    • 否则：更新mx为max(mx, i + nums[i])（当前位置能扩展的最远位置）；
    • （可选优化）若mx >= nums.length - 1：提前返回true，无需继续遍历；
  3. 遍历完成返回true：说明所有位置都能到达，且最远位置覆盖了最后一个下标。

3.2 代码核心点逐行解释

（1）变量初始化

  • mx初始值为0：起始位置是下标0，初始只能到达该位置；
  • n存储数组长度：避免多次调用nums.length，提升代码效率。

（2）核心遍历判断

  • 核心逻辑：若当前遍历的下标i超过了「最远可达位置」，说明该位置无法到达，后续位置更无法到达，直接返回false；
  • 例如：nums = [3,2,1,0,4]，遍历到i=4时，mx的最大值是3（i=3时mx=3），i=4>3，返回false。

（3）更新最远可达位置

  • 核心逻辑：当前位置i能到达，那么从i出发能跳到的最远位置是i + nums[i]，若该值大于当前mx，则更新mx；
  • 例如：nums = [2,3,1,1,4]，i=0时，mx = max(0, 0+2)=2；i=1时，mx = max(2,1+3)=4（已覆盖最后一个下标4），可提前返回true。

（4）可选优化：提前终止

  • 优化逻辑：一旦最远可达位置覆盖最后一个下标，说明已能到达终点，无需继续遍历，减少无意义计算；
  • 作用：在数组较长且早期就能确定可达的场景下，显著提升效率（如示例1中i=1时即可提前返回）。

（5）遍历完成返回true

  • 逻辑：遍历完所有位置都未触发i > mx，说明所有位置都能到达，且mx必然覆盖最后一个下标（否则遍历中会提前返回）。

4、算法性能分析

本解法是跳跃游戏问题的最优解，时间和空间复杂度均达到理论最优，兼顾效率和简洁性。

4.1 时间复杂度：O(n)

n为数组nums的长度，整个解法仅包含一次线性遍历，遍历过程中每个操作（判断i>mx、更新mx、提前终止判断）均为O(1)，总时间复杂度为O(n)，完美适配题目10⁴的输入规模。

4.2 空间复杂度：O(1)

空间开销仅来自几个局部变量（mx、n、i），无额外数组或复杂数据结构，空间复杂度为O(1)，是极致的空间优化。

4.3 核心优势

  1. 逻辑简洁：贪心策略直击问题本质，无需复杂的递归/回溯，易理解、易书写；
  2. 效率最优：线性时间+常数空间，无超时风险，是面试最优写法；
  3. 扩展性强：该贪心思路可迁移至「跳跃游戏II（求最少跳跃次数）」等进阶问题；
  4. 鲁棒性高：天然适配各类边界场景，无需额外处理。

5、经典示例手动模拟（直观理解）

5.1 示例1：nums = [2,3,1,1,4]（返回true）

遍历过程：

  • i=0，mx=0：i<=mx → mx = max(0, 0+2)=2；mx=2 <4，继续；
  • i=1，mx=2：i<=mx → mx = max(2,1+3)=4；mx=4 >=4（最后一个下标），提前返回true。

5.2 示例2：nums = [3,2,1,0,4]（返回false）

遍历过程：

  • i=0，mx=0：i<=mx → mx = max(0,0+3)=3；mx=3 <4，继续；
  • i=1，mx=3：i<=mx → mx = max(3,1+2)=3；继续；
  • i=2，mx=3：i<=mx → mx = max(3,2+1)=3；继续；
  • i=3，mx=3：i<=mx → mx = max(3,3+0)=3；继续；
  • i=4，mx=3：i=4 > mx=3 → 返回false。

5.3 示例3：nums = [0]（返回true）

遍历过程：

  • i=0，mx=0：i<=mx → mx = max(0,0+0)=0；mx=0 >=0（最后一个下标），提前返回true。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 数组长度为1：无论nums[0]是否为0，遍历i=0时mx>=0，直接返回true，正确；
  2. 数组含0但可到达终点：如nums=[2,0,1,1,4]，i=1时mx=2（0+2），i=1<=2，更新mx=max(2,1+0)=2，后续正常遍历，最终返回true；
  3. 数组全0且长度>1：如nums=[0,0]，i=1时mx=0，i>mx，返回false，正确。

6.2 与暴力/动态规划解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
暴力回溯 O(2ⁿ) O(n) 思路直观 效率极低 极小数据量测试
动态规划 O(n²) O(n) 稳定性优于暴力 效率仍不足 中等数据量
贪心单次遍历 O(n) O(1) 时间空间最优 需理解贪心逻辑 面试最优书写

6.3 面试高频追问：跳跃游戏II（最少跳跃次数）

本题的进阶问题是「Leetcode45-跳跃游戏II」，要求计算到达最后一个下标的最少跳跃次数，核心思路仍基于贪心：

  • 维护「当前能到达的最远位置」curEnd和「下一步能到达的最远位置」nextMax；
  • 遍历数组时，更新nextMax；当遍历到curEnd时，说明需要跳一步，更新curEnd为nextMax，并累加跳跃次数；
  • 该解法仍为O(n)时间、O(1)空间，是面试的高频拓展考点。

总结

跳跃游戏的贪心算法单次遍历解法是本题的最优解，也是理解贪心思想的核心案例，本文核心总结如下：

  1. 解题核心是「局部最优推导全局最优」：维护「最远可达位置」，通过当前位置的跳跃能力扩展该位置；
  2. 核心判断逻辑：遍历位置若超出最远可达位置则返回false，否则更新最远位置；
  3. 解法的时间复杂度O(n)、空间复杂度O(1)，是面试的最优书写方式；
  4. 该贪心思路可迁移至跳跃游戏II等进阶问题，是数组贪心类题目的基础模板。
