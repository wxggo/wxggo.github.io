Leetcode-056:合并区间-排序+贪心经典解法（Java版）

发表于2026-03-02|更新于2026-03-02|Java学习笔记Leetcode Hot100
|字数总计:2.7k|阅读时长:9分钟|阅读量:

前言

合并区间问题是Leetcode数组/区间模块的经典高频题，也是面试中考察贪心算法和排序应用的典型场景。本题的核心难点在于如何高效处理重叠/相邻的区间，而排序+贪心是解决该问题的最优思路——通过对区间左端点排序，将无序的区间转化为有序序列，再用贪心策略逐个合并，既保证逻辑直观，又能将时间复杂度控制在最优级别。

1、题目

以数组 intervals 表示若干个区间的集合，其中单个区间为 intervals[i] = [starti, endi] 。请你合并所有重叠的区间，并返回 一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间 。

示例

示例 1：
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠，将它们合并为 [1,6]。

示例 2：
输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 可被视为重叠区间（相邻区间合并）。

示例 3：
输入：intervals = [[1,4],[2,3]]
输出：[[1,4]]
解释：区间 [2,3] 完全包含在 [1,4] 中，合并后仍为 [1,4]。

提示

  • 1 <= intervals.length <= 10^4
  • intervals[i].length == 2
  • 0 <= starti <= endi <= 10^4

2、分析

2.1 问题核心：如何处理无序的重叠区间？

合并区间的核心矛盾是区间的无序性：输入的区间可能以任意顺序排列（如左端点乱序），直接遍历无法高效判断重叠关系。例如输入[[2,6],[1,3]]，若不排序，直接遍历会误判为两个不重叠区间，导致合并失败。

2.2 核心思路：排序定序 + 贪心合并

解决该问题的最优策略分为两步，缺一不可：

  1. 排序定序：将所有区间按左端点升序排列，排序后所有区间的左端点呈递增趋势，只需关注当前区间与已合并最后一个区间的右端点即可判断是否重叠；
  2. 贪心合并：遍历排序后的区间，用列表存储合并结果：
    • 若列表为空，或当前区间左端点 > 列表最后一个区间的右端点（无重叠），直接将当前区间加入列表；
    • 若当前区间左端点 ≤ 列表最后一个区间的右端点（有重叠），更新列表最后一个区间的右端点为“两者右端点的较大值”（完成合并）。

贪心策略的合理性：排序后左端点递增，后续区间的左端点不会小于前面的区间，因此只需维护“已合并区间的最大右端点”，即可覆盖所有重叠情况。

2.3 核心解题步骤

  1. 排序预处理：对intervals数组按左端点升序排序，统一区间顺序；
  2. 初始化合并列表：用ArrayList存储合并后的区间（动态扩容，适配未知的合并结果长度）；
  3. 遍历合并：逐个遍历排序后的区间，按重叠规则判断是否合并；
  4. 结果转换：将ArrayList转换为二维数组返回。

（1）区间排序（核心预处理步骤）

  • 核心逻辑：通过Comparator自定义排序规则，按区间左端点（a[0]/b[0]）升序排列；
  • 排序意义：将无序区间转化为“左端点递增”的有序序列，后续只需比较右端点即可判断重叠，避免无序导致的漏判/误判；
  • 补充说明：若a[0] - b[0] > 0，a会排在b后面；若为负则反之，实现严格升序。

（2）初始化合并列表

  • 选择ArrayList的原因：合并结果的长度未知，ArrayList支持动态扩容，比固定长度数组更灵活；
  • 存储形式：每个元素是长度为2的int数组，对应一个合并后的区间。

（3）遍历判断与合并（贪心核心）

  • 空列表判断：首次遍历无合并区间，直接添加第一个区间；
  • 无重叠判断：当前区间左端点 > 最后一个合并区间的右端点，说明两个区间完全分离，直接添加；
  • 重叠合并：当前区间与最后一个合并区间重叠，需将最后一个区间的右端点更新为“两者右端点的最大值”（覆盖完全包含、部分重叠两种情况）。

（4）结果转换

  • 核心逻辑：将ArrayList转换为二维数组，适配题目返回值要求；
  • 参数意义：new int[merged.size()][] 指定转换后的数组长度为合并结果的数量，每行是长度为2的区间数组。

4、算法性能分析

本解法的时间复杂度由排序主导，空间复杂度为线性级别，是合并区间问题的最优解，兼顾效率和理解性。

4.1 时间复杂度：O(n log n)

n为intervals数组的长度：

  1. 排序操作：Arrays.sort的底层是双轴快速排序，时间复杂度为O(n log n)；
  2. 遍历合并：一次线性遍历，时间复杂度为O(n)；
  • 总时间复杂度为O(n log n) + O(n) = O(n log n)，完美适配题目10^4的输入规模。

4.2 空间复杂度：O(n)

空间开销主要分为两部分：

  1. 排序的系统栈空间：O(log n)（快速排序的递归栈）；
  2. 合并列表的存储空间：最坏情况下（无任何重叠），列表存储n个区间，空间复杂度为O(n)；
  • 总空间复杂度为O(n)（主导项）。

4.3 核心优势

  1. 逻辑直观：排序+贪心的思路贴合人类思维，易理解、易推导，面试中书写不易出错；
  2. 效率最优：排序是区间类问题的必要预处理，O(n log n)是该类问题的时间复杂度下限；
  3. 扩展性强：该思路可迁移至“区间插入”“区间交集”等同类问题。

5、经典示例手动模拟（直观理解）

以示例1：intervals = [[1,3],[2,6],[8,10],[15,18]] 为例，手动模拟核心步骤：

5.1 步骤1：排序（已按左端点升序，无需调整）

排序后区间仍为：[[1,3],[2,6],[8,10],[15,18]]

5.2 步骤2：遍历合并

  1. 遍历第一个区间[1,3]：merged为空，直接添加 → merged = [[1,3]]；
  2. 遍历第二个区间[2,6]：2 ≤ 3（有重叠），更新最后一个区间右端点为max(3,6)=6 → merged = [[1,6]]；
  3. 遍历第三个区间[8,10]：8 > 6（无重叠），直接添加 → merged = [[1,6],[8,10]]；
  4. 遍历第四个区间[15,18]：15 > 10（无重叠），直接添加 → merged = [[1,6],[8,10],[15,18]]。

5.3 步骤3：结果转换

将merged转换为二维数组，输出[[1,6],[8,10],[15,18]]，与示例结果一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 空数组：若intervals.length=0，排序无操作，merged为空，返回空数组，正确；
  2. 单个区间：直接添加到merged，返回原区间，正确；
  3. 完全重叠：如[[1,4],[2,3]]，合并后为[[1,4]]，正确；
  4. 相邻区间：如[[1,4],[4,5]]，合并后为[[1,5]]，正确；
  5. 乱序区间：如[[2,6],[1,3]]，排序后合并为[[1,6]]，正确。

6.2 面试高频追问：排序规则的拓展

（1）按右端点排序是否可行？

可行，但逻辑需调整：按右端点升序排序后，需从后往前遍历，或维护“已合并区间的最小左端点”，不如按左端点排序直观，因此面试中优先选择左端点排序。

（2）区间插入问题（Leetcode57）

若在合并后的区间中插入一个新区间，核心思路仍为：

  1. 将新区间加入原数组；
  2. 按左端点排序；
  3. 执行本文的贪心合并逻辑。

6.3 与暴力解法的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
暴力解法 O(n²) O(n) 无需排序 效率极低 小数据量测试
排序+贪心 O(n log n) O(n) 效率高、直观 需排序预处理 面试/工程实践

总结

合并区间的排序+贪心解法是本题的最优解，也是区间类问题的通用解题思路，核心总结如下：

  1. 合并区间的核心矛盾是区间无序性，排序（左端点升序）是解决该矛盾的关键预处理步骤；
  2. 贪心策略的核心是“维护已合并区间的最大右端点”，只需一次遍历即可完成合并，逻辑简洁；
  3. 核心判断规则：当前区间左端点 > 最后一个合并区间右端点 → 无重叠，否则合并（更新右端点为较大值）；
  4. 时间复杂度O(n log n)（排序主导），空间复杂度O(n)，是面试中首选的书写方式；
  5. 该思路可迁移至区间插入、区间交集等同类问题，掌握后可高效解决大部分区间类题目。
