Leetcode-034:在排序数组中查找元素的第一个和最后一个位置-二分查找（下界法+递归/迭代双实现）

Created2026-03-02|Updated2026-03-02|Java学习笔记Leetcode Hot100
|Word Count:3.4k|Reading Time:12mins|Post Views:

前言

在排序数组中查找元素的第一个和最后一个位置（Leetcode34）是二分查找的经典进阶题，也是面试中考察二分边界处理的高频题。本题的核心并非简单的“找到目标值”，而是“精准定位目标值的区间边界”——既要找到第一个出现的位置，也要找到最后一个出现的位置。

常规的二分查找只能判断元素是否存在，而本题的最优解是基于下界（lowerBound） 思想的二分查找：通过查找target的下界得到起始位置，查找target+1的下界再减1得到结束位置，将两次二分查找的结果组合即可得到答案。本文将详细推导该核心思路，并提供递归版和迭代版两种实现方式，拆解二分查找的边界条件、返回值等核心难点，帮助彻底掌握这类区间查找问题。

1、题目

给定一个按照升序排列的整数数组 nums，和一个目标值 target。找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 target，返回 [-1, -1]。

示例

示例 1：
输入：nums = [5,7,7,8,8,10], target = 8
输出：[3,4]
解释：8 在数组中第一次出现在索引3，最后一次出现在索引4。

示例 2：
输入：nums = [5,7,7,8,8,10], target = 6
输出：[-1,-1]
解释：数组中无6，返回[-1,-1]。

示例 3：
输入：nums = [], target = 0
输出：[-1,-1]
解释：数组为空，无目标值，返回[-1,-1]。

示例 4：
输入：nums = [2,2], target = 2
输出：[0,1]
解释：2 占据数组所有位置，起始为0，结束为1。

提示

  • 0 <= nums.length <= 10^5
  • -10^9 <= nums[i] <= 10^9
  • nums 是按非递减顺序排列的
  • -10^9 <= target <= 10^9

2、分析

2.1 问题核心：下界（lowerBound）的定义与应用

本题的核心是理解二分查找的下界（lowerBound） 概念：

  • 下界（lowerBound）：在有序数组中，lowerBound(nums, target) 返回第一个大于等于target的元素的索引（若所有元素都小于target，则返回数组长度）。

基于该定义，我们可以推导出区间边界的计算逻辑：

  1. 起始位置：target的下界即为第一个出现target的位置（若下界超出数组长度，或该位置元素不等于target，则说明target不存在）；
  2. 结束位置：target+1的下界减1，即为最后一个出现target的位置（因为target+1的下界是第一个大于target的元素位置，减1后就是最后一个target的位置）。

这一思路将“找区间边界”转化为“两次找下界”，避免了分别写“找左边界”和“找右边界”的重复逻辑，是代码最简洁、逻辑最统一的最优解法。

2.2 核心难点：二分查找的边界处理

二分查找的核心坑点集中在三个方面，也是本题的核心难点：

  1. 循环条件：是 left < right 还是 left <= right？
  2. 中间值判断：是 nums[mid] > target 还是 nums[mid] >= target？
  3. 返回值：为什么最终返回left而不是right？

这三个问题的答案直接决定二分查找的正确性，本文会在代码解析部分逐一拆解。

2.3 核心解题思路

  1. 查找target的下界：得到起始位置start；
  2. 合法性校验：若start超出数组长度，或nums[start] != target，说明target不存在，直接返回[-1,-1]；
  3. 查找target+1的下界：得到第一个大于target的位置，减1后即为结束位置end；
  4. 返回结果：组合start和end得到最终区间。

该思路的时间复杂度由两次二分查找决定，为O(logn)，远优于暴力遍历的O(n)，完美适配10^5的数组长度。

3、程序：递归版+迭代版双实现

3.1 实现一：递归版（quickSelect 实现下界查找）

递归版通过分治思想实现下界查找，代码简洁且符合二分查找的“分治本质”，适合理解递归逻辑，但需注意递归栈的空间开销。

代码核心点逐行解释

  • 递归的初始边界：left=0，right=n（而非n-1），因为下界可能是数组长度（所有元素都小于target）。
  • 合法性校验的两个条件：
    1. start == n：所有元素都小于target，无目标值；
    2. nums[start] != target：下界位置的元素不是target（说明数组中无target）。
  • 递归终止条件：left >= right时，left就是第一个>=target的位置（下界）；
  • 中间值判断：nums[pivot] < target时，说明当前位置及左侧都不可能是下界，递归右区间[pivot+1, right]；
  • 否则：当前位置可能是下界，递归左区间[left, pivot]（保留pivot，因为它可能是第一个>=target的位置）。

3.2 实现二：迭代版（lowerBound 标准实现）

迭代版是工程中更常用的写法，避免了递归的栈空间开销，也是面试中更推荐的写法，核心逻辑与递归版一致，但边界条件的处理更直观。

完整代码

代码核心点逐行解释

  • 循环条件用left <= right：确保最后一个元素（left=right时）也能被检查到，避免遗漏。若用left < right，可能会跳过最后一个需要判断的元素。
  • 核心判断逻辑：
    1. nums[mid] >= target：说明mid位置及左侧可能存在下界，将右边界移到mid-1（缩小左半区范围）；
    2. nums[mid] < target：说明下界一定在mid右侧，将左边界移到mid+1（缩小右半区范围）。
  • 为什么返回left？
    循环结束时，left > right，且left恰好是第一个>=target的位置：
    • 若所有元素都<target，循环结束后left = nums.length（符合下界定义）；
    • 若存在元素>=target，left会停在第一个满足条件的位置。

4、算法性能分析

4.1 时间复杂度：O(logn)

n为数组nums的长度，两次下界查找均为二分查找，每次二分查找的时间复杂度为O(logn)，总时间复杂度为O(logn) + O(logn) = O(logn)，远优于暴力遍历的O(n)，能高效处理10^5级别的数组。

4.2 空间复杂度

  • 递归版：O(logn)，递归调用栈的深度为二分查找的层数（即logn）；
  • 迭代版：O(1)，仅使用常数个临时变量（left、right、mid等），无额外空间开销。

4.3 核心优势

  1. 逻辑统一：通过“下界”思想将“找左右边界”转化为两次相同逻辑的二分查找，避免写两套不同的边界判断代码；
  2. 效率最优：对数级时间复杂度，无超时风险；
  3. 鲁棒性强：天然适配各种边界场景（空数组、目标值不存在、目标值全数组覆盖等）；
  4. 扩展性高：lowerBound是二分查找的通用模板，可迁移至所有“找第一个满足条件元素”的场景。

5、经典示例手动模拟（直观理解）

以示例1：nums = [5,7,7,8,8,10], target = 8 为例，手动模拟迭代版lowerBound的执行过程：

5.1 步骤1：查找target=8的下界（起始位置）

  • 初始：left=0，right=5
  • 第一次循环：mid=2 → nums[2]=7 < 8 → left=3
  • 第二次循环：mid=4 → nums[4]=8 ≥8 → right=3
  • 第三次循环：mid=3 → nums[3]=8 ≥8 → right=2
  • 循环结束（left=3 > right=2），返回left=3（起始位置正确）。

5.2 步骤2：查找target=9的下界（计算结束位置）

  • 初始：left=0，right=5
  • 第一次循环：mid=2 → nums[2]=7 <9 → left=3
  • 第二次循环：mid=4 → nums[4]=8 <9 → left=5
  • 第三次循环：mid=5 → nums[5]=10 ≥9 → right=4
  • 循环结束（left=5 > right=4），返回left=5；
  • 结束位置=5-1=4（与示例结果一致）。

5.3 步骤3：组合结果

起始位置3，结束位置4 → 返回[3,4]，符合预期。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

两种实现方式均无需额外处理边界场景，代码天然兼容：

  1. 空数组：nums.length=0，lowerBound返回0，合法性校验start==0 && nums[0]（数组越界）→ 返回[-1,-1]，正确；
  2. 目标值不存在：如nums=[5,7,7,8,8,10], target=6 → lowerBound(6)返回1，nums[1]=7≠6 → 返回[-1,-1]，正确；
  3. 目标值全数组覆盖：如nums=[2,2], target=2 → lowerBound(2)返回0，lowerBound(3)返回2 → 结束位置=1 → 返回[0,1]，正确；
  4. 目标值仅出现一次：如nums=[1,3,5], target=3 → lowerBound(3)返回1，lowerBound(4)返回2 → 结束位置=1 → 返回[1,1]，正确。

6.2 递归版 vs 迭代版对比

实现方式 时间复杂度 空间复杂度 优点 缺点 适用场景
递归版 O(logn) O(logn) 逻辑简洁、易理解 递归栈开销 算法学习、小数据量
迭代版 O(logn) O(1) 无栈开销、效率更高 边界条件稍多 工程实现、面试书写

6.3 二分查找的常见坑点总结

本题的迭代版集中体现了二分查找的核心坑点，面试中需牢记：

  1. mid计算：用left + (right-left)/2而非(left+right)/2，避免left+right超出int范围；
  2. 循环条件：找下界用left <= right，找普通值可灵活调整；
  3. 边界移动：nums[mid] >= target时right=mid-1，nums[mid] < target时left=mid+1，避免死循环；
  4. 返回值：下界查找最终返回left，而非right。

6.4 进阶拓展：通用区间查找

本题的下界思想可拓展至所有“有序数组的区间查找”场景：

  • 找最后一个<=target的元素：upperBound(target) -1；
  • 找第一个>target的元素：lowerBound(target+1)；
  • 统计target出现的次数：lowerBound(target+1) - lowerBound(target)。

总结

Leetcode34的核心解法是基于下界（lowerBound） 的二分查找，本文核心总结如下：

  1. 本题的核心思路：将“找左右边界”转化为“两次找下界”——起始位置=target的下界，结束位置=target+1的下界-1；
  2. 下界（lowerBound）的定义：有序数组中第一个>=target的元素索引，是解决区间查找的通用模板；
  3. 递归版实现简洁易理解，迭代版无栈开销更适合工程场景，二者核心逻辑一致；
  4. 二分查找的核心坑点：mid计算、循环条件、边界移动、返回值，需逐一理清避免出错；
  5. 该解法时间复杂度O(logn)，空间复杂度迭代版O(1)，是本题的最优解，也是面试的首选写法。

掌握下界查找的思想，不仅能解决本题，更能应对所有“有序数组的精准定位”问题，是二分查找进阶的关键。
