Leetcode-72:编辑距离-记忆化DFS经典解法（递归版）

Created2026-03-06|Updated2026-03-06|Java学习笔记Leetcode Hot100
|Word Count:3.4k|Reading Time:12mins|Post Views:

前言

编辑距离（Edit Distance）是Leetcode动态规划模块的经典难题，也是面试高频考察的压轴题之一，其核心考察对递归思维、重叠子问题和记忆化优化的理解。本题的解法有多种，其中记忆化DFS（深度优先搜索） 是最直观、最易理解的基础最优解，该解法通过递归拆解问题为子问题，再通过记忆化数组避免重复计算，将暴力递归的指数级时间复杂度优化为多项式级别。

1、题目

给你两个单词 text1 和 text2，计算出将 text1 转换成 text2 所使用的最少操作数。

你可以对一个单词进行如下三种操作：

  1. 插入一个字符；
  2. 删除一个字符；
  3. 替换一个字符。

示例

示例 1：
输入：text1 = “horse”, text2 = “ros”
输出：3
解释：
horse → rorse (将 ‘h’ 替换为 ‘r’)
rorse → rose (删除第二个 ‘r’)
rose → ros (删除 ‘e’)
共需3次操作。

示例 2：
输入：text1 = “intention”, text2 = “execution”
输出：5
解释：
intention → inention (删除 ‘t’)
inention → enention (将 ‘i’ 替换为 ‘e’)
enention → exention (将 ‘n’ 替换为 ‘x’)
exention → exection (将 ‘n’ 替换为 ‘c’)
exection → execution (插入 ‘u’)
共需5次操作。

示例 3：
输入：text1 = “”, text2 = “a”
输出：1
解释：text1为空，插入字符’a’即可，操作数为1。

提示

  • 1 <= text1.length, text2.length <= 500
  • text1 和 text2 由小写英文字母组成

2、分析

2.1 问题核心：递归拆解子问题

编辑距离的核心是最优子结构：将text1转换为text2的最少操作数，可拆解为更小的子问题——将text1的前i个字符转换为text2的前j个字符的最少操作数。

为清晰推导，先定义递归函数的核心含义（解法的基础）：

  • dfs(i, j)：表示将text1的前i个字符（即s[0..i]，s为text1的字符数组）转换成text2的前j个字符（即t[0..j]，t为text2的字符数组）所需的最少操作数。

基于该定义，我们可以通过递归逐步缩小问题规模，直到触达边界条件。

2.2 核心难点：重叠子问题导致暴力递归超时

暴力递归的思路是，直接按照递归定义拆解问题，但会出现大量重叠子问题（不同的递归路径会重复计算同一个dfs(i,j)）。例如计算dfs(3,2)和dfs(2,3)时，都会重复计算dfs(2,2)，且问题规模越大，重复计算的次数呈指数级增长（时间复杂度为O(3^(n+m))，n和m为两个字符串长度），面对500的长度会完全超时。

记忆化优化是解决该难点的核心思路：通过一个二维memo数组提前存储已经计算过的dfs(i,j)结果，后续递归遇到相同子问题时直接返回结果，无需重复计算，将时间复杂度优化至O(n*m)（n为text1长度，m为text2长度）。

2.3 递归逻辑拆解（核心）

（1）递归终止条件

当子问题缩小到其中一个字符串为空时，操作数为另一个字符串的长度：

  • 若i < 0（text1的前i个字符为空）：需要将text2的前j个字符全部插入到text1中，操作数为j + 1（j从0开始，j+1是字符个数）；
  • 若j < 0（text2的前j个字符为空）：需要将text1的前i个字符全部删除，操作数为i + 1。

（2）字符相等的情况

若s[i] == t[j]（当前字符匹配），则无需任何操作，直接递归处理前一个字符：
dfs(i, j) = dfs(i - 1, j - 1)

（3）字符不相等的情况

若s[i] != t[j]，需尝试三种操作，取操作数最少的一种，且每种操作后操作数+1：

  1. 删除操作：删除text1的第i个字符，递归处理dfs(i - 1, j)；
  2. 插入操作：在text1的第i个字符后插入t[j]（等价于匹配t[j]，递归处理dfs(i, j - 1)）；
  3. 替换操作：将text1的第i个字符替换为t[j]，递归处理dfs(i - 1, j - 1)。

因此，字符不相等时的递归公式为：
dfs(i, j) = 最小值(删除操作数, 插入操作数, 替换操作数) + 1

3.2 代码核心点逐行解释

（1）成员变量定义

  • s/t：将字符串转换为字符数组，避免递归中频繁调用charAt()，提升效率；
  • memo：二维记忆化数组，memo[i][j]存储dfs(i,j)的结果，初始值为-1表示未计算。

（2）memo数组初始化

  • 数组维度为n*m（n是text1长度，m是text2长度），覆盖所有子问题；
  • 初始值填充为-1，作为「未计算」的标记，后续计算后会更新为具体操作数。

（3）递归终止条件

  • i < 0：text1已无字符，要得到text2的前j个字符，需插入j+1个字符（例如j=0时插入1个字符）；
  • j < 0：text2已无字符，要清空text1的前i个字符，需删除i+1个字符（例如i=0时删除1个字符）；
  • 这是递归的「出口」，保证问题能缩小到可直接求解的规模。

（4）记忆化判断

  • 核心优化点：若memo[i][j] != -1，说明该子问题已计算过，直接返回结果，避免重复递归；
  • 例如计算dfs(2,1)时，若之前已计算过，无需再次拆解为dfs(1,1)、dfs(2,0)、dfs(1,0)，直接返回值即可。

（5）字符相等的递归逻辑

  • 当前字符匹配，无需任何操作，直接递归处理text1的前i-1个字符和text2的前j-1个字符；
  • 同时将结果存入memo[i][j]，后续遇到相同子问题可直接使用。

（6）字符不相等的递归逻辑

  • 依次计算删除、插入、替换三种操作的最少操作数，取最小值后+1（当前操作的代价）；
  • 结果存入memo[i][j]，完成该子问题的计算。

4、算法性能分析

本解法是编辑距离问题的基础最优解，兼顾理解性和效率，时间和空间复杂度均为多项式级别。

4.1 时间复杂度：O(n*m)

n为text1长度，m为text2长度：

  • 记忆化数组memo的维度为n*m，每个子问题dfs(i,j)只会被计算一次；
  • 每个子问题的计算仅包含常数次比较和递归调用（已计算的子问题直接返回）；
  • 总时间复杂度为O(n*m)，完美适配题目500的输入规模。

4.2 空间复杂度：O(n*m)

空间开销主要来自两部分：

  1. 记忆化数组memo：维度为n*m，空间开销O(n*m)；
  2. 递归调用栈：最坏情况下递归深度为n+m（如其中一个字符串为空），空间开销O(n+m)；
  • 总空间复杂度为O(n*m)（memo数组占主导）。

4.3 核心优势

  1. 逻辑直观：完全贴合编辑距离的递归定义，从「问题拆解」的角度思考，易理解、易推导；
  2. 效率最优：相比暴力递归的指数级时间复杂度，记忆化优化后可处理题目最大输入规模；
  3. 扩展性强：是理解迭代DP版的基础，掌握后可快速迁移至空间优化的DP解法。

5、经典示例手动模拟（直观理解）

以示例1：text1 = “horse”（s = [‘h’,’o’,’r’,’s’,’e’]），text2 = “ros”（t = [‘r’,’o’,’s’]） 为例，手动模拟核心递归步骤，直观理解记忆化的作用：

5.1 初始调用

minDistance调用dfs(4, 2)（i=4对应s[4]='e'，j=2对应t[2]='s'）。

5.2 关键子问题计算

  1. dfs(4,2)：s[4]='e' != t[2]='s'，需计算：

    • 删除：dfs(3,2)（s[3]=’s’ == t[2]=’s’ → dfs(2,1)）；
    • 插入：dfs(4,1)；
    • 替换：dfs(3,1)；
      取最小值+1，最终结果为3。

  2. dfs(2,1)：s[2]='r' != t[1]='o'，计算后结果为1，存入memo[2][1]，后续递归无需重复计算。

5.3 记忆化的作用

例如dfs(2,1)会被dfs(3,2)和dfs(2,2)多次调用，但仅第一次计算，后续直接返回memo[2][1]的值，避免了大量重复递归。

最终所有子问题计算完成后，dfs(4,2)返回3，与示例结果一致。

6、拓展思考（面试高频考点）

6.1 边界情况天然适配

本解法无需额外处理各类边界场景，代码天然兼容：

  1. 其中一个字符串为空：如text1=""，text2="a"，递归触发i<0条件，返回j+1=1，正确；
  2. 两个字符串完全相同：如text1="abc"，text2="abc"，所有字符匹配，递归返回0，正确；
  3. 字符串长度为1：如text1="a"，text2="b"，返回1（替换操作），正确。

6.2 与暴力递归/迭代DP的对比

解法 时间复杂度 空间复杂度 优点 缺点 适用场景
暴力递归 O(3^(n+m)) O(n+m) 无需额外空间 效率极低 小数据量测试
记忆化DFS O(n*m) O(n*m) 逻辑直观 递归栈开销 面试基础书写
迭代DP O(n*m) O(n*m) 无递归栈开销 逻辑稍抽象 面试常规书写
滚动数组DP O(n*m) O(min(n,m)) 空间最优 逻辑较复杂 面试最优书写

6.3 面试高频追问：迭代DP版（空间优化）

记忆化DFS的空间复杂度为O(n*m)，面试中面试官大概率会追问如何用迭代DP优化，核心思路是将递归的「自顶向下」改为「自底向上」，用二维DP数组替代递归和memo：

核心思路

  1. 定义dp[i][j]：将text1的前i个字符转换为text2的前j个字符的最少操作数；
  2. 初始化边界：dp[i][0] = i（删除i个字符），dp[0][j] = j（插入j个字符）；
  3. 状态转移：
    • 若s[i-1] == t[j-1]，dp[i][j] = dp[i-1][j-1]；
    • 否则，dp[i][j] = 最小值(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1。

总结

编辑距离的记忆化DFS解法是本题的基础最优解，也是理解编辑距离核心原理的关键，本文核心总结如下：

  1. 编辑距离的核心原理是最优子结构，可通过递归将大问题拆解为更小的子问题；
  2. 记忆化DFS的核心是通过memo数组存储已计算的子问题结果，避免重叠子问题的重复计算，将时间复杂度从指数级优化至O(n*m)；
  3. 递归终止条件需牢记：字符串为空时，操作数为另一个字符串的长度；
  4. 字符不相等时的三种操作（删除/插入/替换）是递归的核心，需理解每种操作对应的递归逻辑；
  5. 记忆化DFS的时间复杂度O(nm)、空间复杂度O(nm)，兼顾效率和理解性，是面试入门的首选写法；
  6. 在此基础上，可通过迭代DP将递归栈开销消除，甚至通过滚动数组将空间复杂度优化至O(min(n,m))，是面试的最优写法。
